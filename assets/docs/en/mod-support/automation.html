While they are generally less complex than a full vehicle, there are still a few issues Automation mods may suffer from that could get picked up on the repo.


Uploaded .car files

When producing a vehicle with automation, there are multiple ways it can be saved. For use on the BeamNG repository, we need the exported BeamNG version of your car only to be uploaded.

By default, the ingame export button should place the correctly packed zip file for the mod in your BeamNG mods folder, and this will be ready to be uploaded with no further modifications.


Incorrectly packed mods

If you have instead enabled the option to export to BeamNG as an unpacked mod, or otherwise have manually edited the files for the mod itself, you will need to repack the mod first so that it can be uploaded. This can be done by selecting the vehicles and art folders of your mod and compressing those to a zip file, which can then be uploaded to the repo.

More information on this process can be found at Correctly packing mods


Multiple vehicles inside one zip

If you would like to include multiple Automation cars inside one mod file then they will need to be packed as if it were a single mod still. Instead of just compressing the separate zips into one file, you will need to extract each mod individually and then merge the contents of their vehicles and art folders so that all the individual car folders fall under the same main folder.

Once this is done, you can select both the vehicles and art folders together and compress them to a single zip. This will create a packed file containing all of your cars in one file. More information on this process can be found at Correctly packing mods

If you get a message about duplicate files when trying to merge two vehicle folders together, do not worry. As these are Automation exports, the contents of these files will be the same and you only need the one copy available to the mod, so any duplicate warnings can safely be ignored.

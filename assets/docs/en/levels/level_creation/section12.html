Quick triage

When something breaks, check in this order:

  1. Console/log errors after level load.
  2. Missing files or wrong paths.
  3. Invalid JSON syntax.
  4. Missing materials or mapTo mismatches.
  5. Object placement/scale problems.
  6. Cache after large folder or material changes.

Fix the earliest error first. Later warnings are often caused by the first missing asset or invalid file.

12.1 Materials/textures

Orange/no-texture surfaces

Cause: bad path, missing texture, mapTo mismatch, or material file not loaded.

Fix:

  • Check the texture path in the material file.
  • Check that the material name and mapTo match the mesh material slot.
  • Verify the texture exists in the packaged level/mod.
  • Reload the level after material changes.

Surfaces look flat or faceted

Cause: normal map orientation, missing tangents, hard smoothing splits, or incorrect exported normals.

Fix:

  • Verify normal map orientation.
  • Export normals and tangents.
  • Check smoothing/edge split setup in the source mesh.

Reflections look overly strong

Cause: cubemap intensity or roughness values are too low for the surface.

Fix: reduce reflection/cubemap intensity, increase roughness, or use a less reflective material setup.

On High settings or higher reflections are set to use Level option by default

12.2 Geometry/roads

  • Symptom: flicker at intersections. Cause: coplanar overlaps. Fix: adjust offsets and decal layering.
  • Symptom: bumps/gaps on bridges. Cause: discontinuous MeshRoad collision. Fix: review spans and collision continuity.
  • Symptom: MeshRoad curves look angular. Cause: breakAngle is too high or nodes are poorly spaced. Fix: lower breakAngle or add support nodes.
  • Symptom: props are wrong scale. Cause: unapplied transforms or wrong export units. Fix: apply transforms and export in meters.

12.3 Vegetation/props

  • Symptom: trees floating or sunk. Cause: pivot/scale mismatch or terrain edits after placement. Fix: snap or re‑place; verify pivots.
  • Symptom: heavy FPS drops in forests. Cause: species/material count, density, or missing impostors/LODs. Fix: reduce variety/density; add LODs.
  • Symptom: repeated props are expensive. Cause: too many unique TSStatic objects or materials. Fix: use Forest, prefabs, material reuse, or simpler LODs where appropriate.

12.4 Terrain/water

  • Symptom: banding on slopes. Cause: 8‑bit sources or resampling artifacts. Fix: ensure 16‑bit heightmap; re‑export cleanly.
  • Symptom: water edges shimmer. Cause: alignment or overlap issues. Fix: adjust edges, ensure clean shoreline integration.
  • Symptom: terrain materials look wrong after import. Cause: layer maps/material texture set mismatch. Fix: check terrain material setup and reload.

12.5 Packaging/loading

  • Symptom: level fails on a clean profile. Cause: external dependencies, missing files. Fix: ensure self‑contained .zip; verify paths.
  • Symptom: changes not visible. Cause: stale cache. Fix: clear cache and relaunch.
  • Symptom: packaged level works only while the unpacked folder exists. Cause: package missing assets. Fix: move the working folder away and test only the .zip.
  • Symptom: level does not appear in selector. Cause: missing or invalid info.json / folder layout. Fix: verify metadata and package structure.

12.6 Logging

  • Use the console/log to identify missing asset warnings, path errors, or invalid JSON entries.
  • Search for your level name and the missing file name in the log.
  • If an object does not appear, check the items.level.json file that should contain it.

12.7 When to re‑author

  • If repeated fixes fail, reassess asset scale, collision complexity, or material count against the performance guidance.

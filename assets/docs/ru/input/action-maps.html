Обзор

Карты действий — это по сути группы привязок (bindings), объединённые под одним именем.

Их основное назначение — определять приоритеты своих привязок, а также позволять включать и отключать все их сразу.

Ей также можно назначить более высокий приоритет, чем обычным элементам управления вождением, — тогда нажатие стрелки вправо не будет пытаться повернуть транспортное средство, а вместо этого перемотает запись на 10 секунд вперёд.

Порядок выполнения карт действий

Когда генерируется событие , BeamNG.drive берёт первую карту действий и проверяет, содержит ли она соответствующие привязки для выполнения:

  • Если с данным событием не связано ни одной привязки , будет взята следующая карта действий по порядку и проверена снова (пока не закончатся все карты действий).
  • Если имеется какая-либо подходящая привязка , она/они будут выполнены, и после этого больше ни одна карта действий проверяться не будет*.

(* за исключением случаев, когда установлен флаг trap )

Иерархия

Карты действий могут быть активными или неактивными, и для каждого состояния существует отдельный список. Вы можете просмотреть оба списка в любое время, выполнив следующую команду в консоли GE-Lua:

dump(ActionMap:getList())

Существует второй список, содержащий все неактивные карты действий.

Неактивные карты никогда не принимаются во внимание, конечный пользователь не может использовать ни одну из их привязок, поэтому их порядок не имеет значения.

Пользовательские карты действий

Модификации могут добавлять дополнительные карты действий, просто указав новое имя поля actionMap в файлах действий формата .json.

Это имя в сочетании с суффиксом "ActionMap" будет использоваться в качестве идентификатора в дереве сцены. Например, использование 'actionMap':'FunStuff' приведёт к созданию карты действий, доступной через scenetree.findObject('FunStuffActionMap').

Все дополнительные карты действий по умолчанию будут неактивными .

Чтобы новые карты действий можно было использовать, их необходимо сначала активировать (а затем деактивировать, если/когда работа с ними будет завершена).

Активация и деактивация карт действий

Активация карты действий означает перемещение её из неактивного списка в активный.

Другими словами, карта действий помещается в активный список или извлекается из него.

Карты действий можно активировать в любое время, выполнив следующий код GE-Lua:

local actionMapName = "MyActionMap"
scenetree.findObject(actionMapName):push()

Чтобы впоследствии деактивировать её, используйте следующее:

local actionMapName = "MyActionMap"
scenetree.findObject(actionMapName):pop()

Это переместит карту действий обратно в неактивный список.

Карты действий активируются (помещаются) примерно сразу под GuiActionMap (см. Список активных карт действий ).

Таким образом, их приоритет ниже, чем у карт действий gui/menu/global, и выше, чем у ранее активированных карт действий.

На практике это означает, что модификация никогда не сможет переопределить, например, привязку ALT+F4, завершающую игру. Это сделано намеренно.

Включение и отключение карт действий

Это отображается как enabled = true или enabled = false при просмотре иерархии карт действий .

Это редко используемая возможность, задокументированная здесь для полноты описания. Она позволяет деактивировать карту действий без потери её позиции (приоритета) в Списке активных карт действий (не перемещая карту действий из активного в неактивный список).

Основное назначение — возможность отключать все привязки карты действий Menu, пока меню закрыты (и повторно включать их, если конечный пользователь начинает навигацию по меню), сохраняя при этом правильный приоритет её привязок.

Перехват обработанных событий

Этот булев флаг может быть установлен для каждой карты действий. Его значение по умолчанию — true.

Он отображается как trapHandledEvents = true или trapHandledEvents = false при просмотре иерархии карт действий .

Ещё одна редко используемая возможность, снова задокументированная здесь для полноты описания.

Если этот флаг равен false, поведение порядка выполнения карт действий изменяется: карты действий с более низким приоритетом будут проверяться даже после того, как привязка была найдена в текущей карте действий.

Например, если у первой половины стека карт действий этот флаг установлен в false, а у второй половины — в true, то будут обработаны все привязки из первой половины. После этого будет обработана первая карта действий с подходящей привязкой, а остальные карты действий не получат шанса на обработку (поскольку событие было только что перехвачено этой последней подходящей картой действий).

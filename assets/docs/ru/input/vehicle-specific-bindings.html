В качестве рабочего примера удобно использовать официальный транспорт cannon.

Обзор

Рекомендуем сначала прочитать Введение в систему ввода .

Начиная с версии 0.5, BeamNG.drive поддерживает транспортно-специфические привязки . Это специальные привязки, действующие только для определённого транспортного средства. Например, привязка для подъёма стрелы крана или привязка для выстрела шаром из пушки.

Транспортно-специфические привязки могут быть настроены конечными пользователями в меню Controls → Bindings, как и любая другая привязка (без редактирования каких-либо файлов).

Определение транспортно-специфических действий

Иными словами, файл может называться input_actions.json, но может также иметь суффикс, например: input_actions_my_custom_suffix.json

Формат файла действий

Указанные имена действий ("action_name_a" и т. д.) должны быть уникальны в пределах текущего транспортного средства. Если другой транспорт использует точно такое же имя действия, они не будут конфликтовать друг с другом, поскольку принадлежат разным транспортным средствам.

{
"shoot": {"order":1, "onChange":"electrics.values.shoot = VALUE", "title":"Shoot ball", "desc":"Shoots the cannon ball. Cannon will need to be reset to fire again" },
}

Моддинг транспортно-специфических действий

Если вы изменяете существующий транспорт, который уже содержит транспортно-специфические действия , то начиная с версии 0.7.0.0 вы можете добавлять дополнительные действия (вместо простой замены файла vehicles/my_vehicle/input_actions.json).

Для этого создайте новые файлы с именами input_actions*.json в той же директории. Например: input_actions_my_mod.json.

Все json-файлы, соответствующие этому шаблону имени, будут прочитаны, что позволяет добавлять любые необходимые новые действия в произвольном количестве файлов.

Определение привязок по умолчанию

Теперь, когда у нас есть транспортно-специфический файл input_actions.json, вы можете создавать новые привязки через стандартное меню Controls → Bindings. Это создаст файлы inputmap в директории settings/inputmaps/your_vehicle_name/.

Когда вы будете довольны набором привязок по умолчанию, переименуйте файлы с *.diff в *.json и перенесите их в vehicle/your_vehicle_name/inputmaps/.

Начиная с BeamNG.drive v0.25, у вас также есть возможность добавить произвольный суффикс к имени файла, чтобы избежать конфликтов с файлами inputmap, поставляемыми игрой. Например, если вы добавляете привязки для пикапа, рекомендуется выбрать уникальный суффикс, чтобы не конфликтовать с inputmap других модов или значениями по умолчанию. Например, вместо vehicles/pickup/inputmaps/keyboard.json следует использовать vehicles/pickup/inputmaps/keyboard_MyCoolMod.json. Суффикс может быть любым — главное, чтобы он был достаточно уникальным и не совпадал с уже используемыми другими модами.

Транспортно-специфические привязки автоматически появятся в правом верхнем углу окна после загрузки уровня. Таким образом, пользователи смогут легко узнать, как использовать транспортно-специфические привязки вашего мода.

После первого выпуска мода, например во второй версии, вы можете добавить новые действия и дополнительные привязки по умолчанию. BeamNG.drive обеспечит доступность этих новых привязок по умолчанию для всех пользователей, даже если они ранее настраивали привязки самостоятельно.

Важно: если вы не предоставите файлы привязок по умолчанию, пользователь не сможет воспользоваться транспортно-специфическим функционалом, пока самостоятельно не создаст необходимые привязки.

Пожалуйста, всегда предоставляйте привязки по умолчанию — пользователи будут вам благодарны. Ну или, по крайней мере, они не будут так сильно запутаны в том, как использовать ваш мод.

Реализация функциональности транспортного средства

Простой случай

Простейший случай — использование тастера (thruster) или гидро (hydro). Эти элементы легко реагируют на значения, передаваемые через электрическую шину транспортного средства, поэтому мы просто напишем код LUA, записывающий данные в эту шину.

Рассмотрим на примере: если у вас в input_actions.json есть следующее:

"onChange":"electrics.values.shoot = VALUE"

Тогда вы можете заставить гидро или тастер реагировать на это следующим образом:

  • Внутри секции Hydros файла .jbeam:
"inputSource":"shoot"
  • Или внутри секции Thrusters файла .jbeam:
"control":"shoot"

Смотрите транспортное средство cannon в качестве рабочего примера использования тастера.

Продвинутый случай

Вы можете получить полный контроль над происходящим, если готовы писать собственный код LUA.

Код LUA действий

Весь код lua в onChange/onUp/onDown может содержать специальные символы, которые автоматически заменяются системой привязок. Их можно использовать в любом порядке и любое количество раз. Это:

  • VALUE: заменяется на 1 в onDown, 0 в onUp, значения с плавающей точкой от 0 до 1 в onChange при isCentered равном false, и значения с плавающей точкой от -1 до 1 в onChange при isCentered равном true.
  • FILTERTYPE: соответствует параметру «Filter» в редакторе привязок и заменяется на 0 для фильтра Key, 1 для фильтра Pad и 2 для фильтра Direct.
  • PLAYER: если вы используете аргумент действия ctx, отличный от стандартного "vlua" (например "ts"), это может быть полезно для направления действий на транспортное средство определённого игрока в режиме multiseat. Если привязка была вызвана первым игроком, значение заменяется на 0, вторым игроком — на 1, третьим — на 2 и т. д.

Базовый пример:

"onChange":"input.event('throttle', VALUE, FILTERTYPE)"

Продвинутый вымышленный пример, лишённый практического смысла (он предназначен исключительно для демонстрации теоретических возможностей):

"onChange":"hydros.my_function(false, true, 1337, 'sample string',  VALUE,  -1 * VALUE / 5,  math.max(1, FILTERTYPE),  FILTERTYPE == 0)"
Пользовательские файлы LUA

Если ваши потребности достаточно сложны, вы можете включить пользовательские файлы .lua в мод вашего транспортного средства.

Эти файлы могут содержать функции, которые будут вызываться игровым движком в качестве обратных вызовов:

M.onInit()
M.onReset()
M.updateGFX(dt)

Они также могут экспортировать любые функции, которые вы хотите использовать в пользовательском коде input_actions.json. Например, файл с именем vehicles/my_vehicle/lua/custom_lua_file.lua:

M.enableWindshieldWipers(booleanValue)
M.setTurboLimit(psi)

Код действия затем может вызывать эти экспортированные функции, например:

onChange: "custom_lua_file.setTurboLimit(30 * VALUE)"

Рабочий пример

Включение рабочего примера потребовало бы прикрепления различных файлов, которые со временем могут устареть. Лучше воспользоваться уже существующим транспортным средством cannon, которое поставляется в комплекте с BeamNG.drive, тестируется с каждым релизом и достаточно простое, поэтому служит отличной отправной точкой.

Вы найдёте это транспортное средство в SteamApps\common\BeamNG.drive\content\vehicles\cannon.zip; распакуйте его во временную директорию и ознакомьтесь с различными файлами и директориями, упомянутыми в этом руководстве.

Обзор

Действие (Action) — это каждая из внутриигровых операций, которые могут выполняться после того, как игрок воздействовал на Элемент управления .

Например: поворот влево, смена камеры, открытие/закрытие меню.

Действия определяются в одном или нескольких .json-файлах.

Через моды можно добавлять новые дополнительные действия. Помните лишь, что если вы не предоставите вместе с ними привязки, пользователям вашего мода придётся вручную настраивать контроллеры.

Существует два основных вида действий: Обычные действия и Действия, специфичные для транспортного средства :

Обычные действия

Обычные действия — это те, которые могут выполняться независимо от того, какое транспортное средство используется в данный момент.

Они могут быть доступны даже в главном меню — до входа на уровень или создания какого-либо транспортного средства.

BeamNG.drive поставляется с большим количеством предопределённых обычных действий. Они определены в:

lua/ge/extensions/core/input/actions/*.json

Моды могут включать дополнительные обычные действия. Для этого необходимо предоставить один или несколько дополнительных json-файлов, следуя соглашению об именовании:

lua/ge/extensions/core/input/actions/*.json

Например:

lua/ge/extensions/core/input/actions/explosions_mod.json
lua/ge/extensions/core/input/actions/advanced_menus.json

Все действия, определённые там, будут доступны конечным пользователям в меню Options → Controls, в общем списке со стандартными действиями BeamNG.drive.

Хотя технически возможно заменить стандартный файл input_actions.json на изменённый, это означает, что автору мода придётся постоянно обновлять свой файл при каждом обновлении BeamNG.drive, добавляющем новые, изменяющем или удаляющем существующие действия. Поэтому делать это настоятельно не рекомендуется.

Действия, специфичные для транспортного средства

Действия, специфичные для транспортного средства, — это те, которые можно активировать только при управлении конкретным транспортным средством. При переключении на транспортное средство другого типа они будут заменены действиями нового транспортного средства.

Некоторые официальные транспортные средства BeamNG.drive поставляются с несколькими предопределёнными действиями, специфичными для них. Например, Small Cannon предоставляет возможность открыть огонь с помощью контроллера. Эти специфичные действия определены в:

vehicles/vehicle_directory_name/input_actions.json

Аналогично Обычным действиям , моды также могут включать дополнительные действия, специфичные для транспортного средства. Файлы мода должны соответствовать следующему соглашению об именовании:

vehicles/vehicle_directory_name/input_actions*.json

Например:

vehicles/hatch/input_actions_hydraulic_suspension.json
vehicles/pickup/input_actions_winch_cable.json

Все эти действия, как правило, можно найти в категории «Vehicle-specific» меню Options → Controls.

Также необходимо использовать уникальные имена файлов для действий, специфичных для транспортного средства, чтобы убедиться, что ваши действия не будут перезаписаны другими модами, использующими те же имена файлов.

Формат файлов действий

Файлы действий в основном соответствуют формату json . Они должны начинаться с { и заканчиваться }. Вот пример того, как должна выглядеть общая структура:

{
    "an_action"     : {"order":1,  "title":"Do Something", ...},
    "another_action": {"order":2,  "title":"Do Something Else", ...},
    "third_action"  : {"order":10, "title":"Do More Things", ...},
    //...
}

Каждая строка состоит из двух частей:

  • Первая строка на каждой позиции — это имя действия. Например, "an_action".
  • Остальная часть строки содержит параметры действия. Например, "order", "title", "ctx" и другие.

Имена действий

Имена действий должны быть уникальными, иначе они могут быть перезаписаны другими модами, использующими те же имена действий. В частности:

  • Имена обычных действий должны отличаться от всех остальных имён обычных действий.
  • Имена действий, специфичных для транспортного средства , должны быть уникальными в рамках текущего транспортного средства.

Самый простой способ гарантировать уникальность имён действий — использовать уникальный префикс. Например:

{
    "extra-menus-mod_an_action"     : { ... },
    "extra-menus-mod_another_action": { ... },
    "extra-menus-mod_third_action"  : { ... },
    //...
}

Описание

Указывает, какой исходный код будет выполнен при срабатывании действия. Контекст выполнения кода определяется параметром ctx текущего действия.

Должно быть определено хотя бы одно из них. Они срабатывают следующим образом:

  • onChange: выполняется при изменении положения контроллера. Поддерживает: клавиши/кнопки и оси
  • onDown: выполняется при нажатии клавиши/кнопки. Поддерживает: клавиши/кнопки
  • onUp: выполняется при отпускании клавиши/кнопки. Поддерживает: клавиши/кнопки
  • onRelative: выполняется при удержании правой кнопки мыши и перемещении мыши (не используйте, если не уверены в том, что делаете)

Написание кода действия

Поля onChange, onDown, onUp и onRelative содержат короткие фрагменты кода.

Это не отдельный язык сценариев. Это строки, содержащие код Lua, в которых система ввода перед выполнением кода заменяет несколько специальных слов-заполнителей.

Пример:

{
  "my_mod_toggle": {
    "cat": "vehicle",
    "order": 10,
    "ctx": "vlua",
    "onDown": "controller.getControllerSafe('myController').toggle()",
    "title": "My Mod Toggle",
    "desc": "Toggles a feature from my mod"
  }
}

Определение действия выполняет две задачи:

  1. Описывает, как действие отображается в меню Controls.
  2. Предоставляет код, который выполняется, когда привязка активирует действие.

Поля событий действия

Поле Когда выполняется Типичное применение
onChange При каждом изменении значения ввода. Работает с клавишами/кнопками и осями. Аналоговые элементы управления, удерживаемые кнопки, значения, которые должны следовать за вводом.
onDown При нажатии клавиши/кнопки. Переключатели, одноразовые команды, смена режима.
onUp При отпускании клавиши/кнопки. Снятие удерживаемого состояния, кратковременные элементы управления.
onRelative Для относительного ввода, например движения мыши, там где это поддерживается. Управление камерой/мышью.

Должно присутствовать хотя бы одно из этих полей.

Используйте onChange для осей и аналоговых элементов управления. Используйте onDown для простых переключателей кнопок.

Не полагайтесь на onDown или onUp для элементов управления, работающих только через оси. Оси должны использовать onChange.

Значения-заполнители

Перед выполнением кода действия система ввода заменяет специальные слова-заполнители.

Общие заполнители:

Заполнитель Значение
VALUE Текущее значение ввода. Обычно 1 при нажатии, 0 при отпускании, 0..1 для обычных аналоговых входов или -1..1, когда isCentered равен true.
FILTERTYPE Выбранный тип фильтра ввода из привязки. Используется функциями ввода транспортного средства.
PLAYER Индекс игрока для действий с поддержкой нескольких мест.
ANGLE Данные угла поворота руля, если они предоставлены привязкой/устройством.
LOCKTYPE Тип блокировки руля, если он предоставлен привязкой/устройством.
OSCLOCKHP Высокоточная метка времени, используемая некоторыми путями ввода.

Программная активация действий также может заменять:

Заполнитель Значение
VEHICLEID Идентификатор транспортного средства, переданный вспомогательной функции активации действия.

Пример:

{
  "set_table": {
    "order": 1,
    "ctx": "vlua",
    "onChange": "electrics.values.table = VALUE",
    "title": "Move Table",
    "desc": "Moves the table with an analog input"
  }
}

С центрированным элементом управления:

{
  "move_platform": {
    "order": 1,
    "ctx": "vlua",
    "isCentered": true,
    "onChange": "controller.getControllerSafe('platform').move(VALUE)",
    "title": "Move Platform",
    "desc": "Moves the platform in both directions"
  }
}

isCentered изменяет диапазон значений с 0..1 на -1..1.

Контекст выполнения

Поле ctx определяет, где будет выполняться код.

ctx Выполняется в Типичное применение
tlua Lua игрового движка. По умолчанию для обычных действий. Системы интерфейса/геймплея, глобальные команды, действия редактора/инструментов.
vlua Lua активного транспортного средства. По умолчанию для действий, специфичных для транспортного средства. Контроллеры транспортного средства, электрика, трансмиссия, гидравлика, расширения транспортного средства.
elua Асинхронный путь Lua игрового движка. Устаревшие/особые случаи. Предпочтительно использовать tlua для обычных действий GE Lua.
bvlua Все Lua-контексты транспортных средств. Широковещательная команда Lua транспортного средства для всех транспортных средств.

Устаревшие контексты:

ctx Статус
ui Устарел. Используйте вместо него tlua и guihooks.
ts Устарел. Используйте вместо него tlua.
slua Устаревший псевдоним, заменяемый на elua.

Если ctx не указан для обычного действия, система ввода по умолчанию использует tlua. Для действий, специфичных для транспортного средства, по умолчанию используется vlua.

Выбор контекста

Используйте vlua, когда действие управляет текущим транспортным средством:

"onChange": "input.event('throttle', VALUE, FILTERTYPE)",
"ctx": "vlua"

Используйте tlua, когда действие управляет игровыми системами, интерфейсом, инструментами редактора или глобальным геймплеем:

"onDown": "extensions.hook('onGameplayInteract')",
"ctx": "tlua"

Используйте bvlua только тогда, когда команду должны получить все транспортные средства:

"onDown": "electrics.set_warn_signal(true)",
"ctx": "bvlua"

Не вызывайте API, предназначенные только для транспортных средств, из tlua, и не вызывайте системы интерфейса игрового движка напрямую из vlua.

actionMap

actionMap управляет тем, в какую карту действий будет помещена привязка.

Поведение по умолчанию:

Условие Карта действий
cat равен menu MenuActionMap
ctx равен vlua VehicleCommonActionMap
действие, специфичное для транспортного средства VehicleSpecificActionMap
в остальных случаях NormalActionMap

Можно переопределить это с помощью:

"actionMap": "BigMap"

Окончательное имя объекта сцены — это имя карты действий плюс ActionMap, например:

BigMapActionMap

Особый случай:

"actionMap": "MenuIndependent"

Это создаёт независимую от меню карту действий для каждого действия, используя префикс MenuIndependent_.

Пример обычного действия

Это обычное действие выполняется в Lua игрового движка и переключает пользовательскую функцию интерфейса/инструмента:

{
  "my_mod_toggle_tool": {
    "cat": "menuExtra",
    "order": 900,
    "ctx": "tlua",
    "actionMap": "Normal",
    "onDown": "extensions.my_mod_tool.toggle()",
    "title": "Toggle My Tool",
    "desc": "Opens or closes my custom tool"
  }
}

Рекомендуемое расположение:

lua/ge/extensions/core/input/actions/my_mod_actions.json

Используйте уникальное имя файла и уникальные имена действий, чтобы избежать конфликтов с другими модами.

Пример действия, специфичного для транспортного средства

Это действие, специфичное для транспортного средства, записывает значение в таблицу electrics транспортного средства:

{
  "raise_arm": {
    "order": 1,
    "onChange": "electrics.values.arm = VALUE",
    "title": "Raise Arm",
    "desc": "Raises the vehicle arm"
  }
}

Рекомендуемое расположение:

vehicles/<vehicleName>/input_actions_my_mod.json

Действия, специфичные для транспортного средства, автоматически:

  • получают внутреннее пространство имён текущего транспортного средства
  • по умолчанию используют ctx = "vlua"
  • отображаются в категории элементов управления, специфичных для транспортного средства

Действия событий JBeam

Некоторые детали транспортного средства определяют данные триггеров/действий непосредственно в разделах events файлов JBeam.

Пример:

"events": [
  ["id", "title", "desc"],
  ["table_up", "ui.inputActions.tiltdeck.table_up.title", "ui.inputActions.tiltdeck.table_up.description",
    { "onChange": "electrics.values.table = VALUE", "order": 6 }
  ]
]

Код действия использует те же поля onChange, onDown, onUp и поведение заполнителей, что и input_actions.json.

Используйте этот подход, когда действие тесно связано с деталью или триггером. Используйте input_actions*.json, когда действие представляет собой обычную привязку на уровне транспортного средства.

Карты взаимодействий

Карты взаимодействий транспортного средства, использующие *.interaction.json, также используют тот же формат кода действий.

Они полезны для взаимодействий, таких как двери, защёлки, функции транспортного средства или специфичные для деталей элементы управления, которые должны отображаться в интерфейсе взаимодействия.

Примеры см. в документации по картам взаимодействий и документации по триггерам/взаимодействиям транспортного средства.

Лучшие практики

  • Используйте onDown для переключателей.
  • Используйте onChange для аналоговых значений и удерживаемых элементов управления.
  • Держите фрагменты кода действий короткими.
  • Помещайте сложную логику в модуль Lua или функцию контроллера, затем вызывайте эту функцию из действия.
  • Используйте ключи локализации для title и desc при выпуске контента для конечных пользователей.
  • Используйте уникальные имена действий и уникальные суффиксы файлов для модов.
  • По возможности предоставляйте привязки по умолчанию для действий, специфичных для транспортного средства.
  • Избегайте устаревших контекстов (ui, ts, slua).
  • Тестируйте действия с клавиатурой и хотя бы одним аналоговым устройством, если действие поддерживает аналоговый ввод.

Распространённые проблемы

Действие не отображается в Controls

Возможные причины:

  • JSON-файл находится не в той папке.
  • Синтаксическая ошибка в JSON.
  • Дублирующееся имя действия было перезаписано.
  • Неверная категория.
  • Действие, специфичное для транспортного средства, не активно для текущего транспортного средства.

Привязка отображается, но ничего не делает

Возможные причины:

  • У действия нет ни onChange, ни onDown, ни onUp, ни onRelative.
  • Использован неверный ctx.
  • Код вызывает модуль Lua, который не загружен в данном контексте.
  • Неверное имя контроллера, специфичного для транспортного средства.

Аналоговый элемент управления ведёт себя как кнопка

Возможные причины:

  • Действие использует onDown вместо onChange.
  • Отсутствует isCentered для центрированной оси.
  • Тип фильтра привязки не подходит.

Категории действий

Категория действия — обязательный параметр, влияющий только на внешнее отображение.

Он влияет на то, где действие может отображаться в различных внутриигровых меню (например, в иерархии действий меню Options → Controls).

Чтобы ознакомиться со списком доступных категорий, обратитесь к следующему файлу:

lua/ge/extensions/core/input/categories.lua

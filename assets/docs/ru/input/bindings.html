Обзор

Иногда также называется input mapping или inputmap.

Привязки определяют, какие элементы управления будут запускать то или иное действие .

Например: привязка газа связывает элемент управления ↑ с действием "accelerate".

Конечные пользователи могут настраивать привязки в меню Options → Controls.

  • Один элемент управления можно привязать к нескольким действиям (порядок карт действий определит, какое из них будет вызвано).
  • Одно действие может быть вызвано несколькими элементами управления.

Расположение файлов

Заводские привязки

Заводские привязки — это привязки, входящие в стандартную поставку игры.

При нажатии кнопки Options → Controls → RESET ALL они возвращаются к состоянию заводских привязок.

Они хранятся в папке установки игры в двух возможных местах:

  • Обычные привязки: settings/inputmaps/*.json
  • Привязки транспортных средств: vehicles/vehicle_directory_name/inputmaps/*.json

Примечание: некоторые из них могут быть упакованы внутри gameengine.zip для уменьшения объёма загрузки и занимаемого места.

Пользовательские привязки

Когда конечные пользователи настраивают свои привязки, эти различия сохраняются в пользовательской папке :

  • Обычные привязки: settings/inputmaps/*.diff
  • Привязки транспортных средств: settings/inputmaps/vehicle_directory_name/*.diff

Они содержат только то, что изменилось по сравнению со значениями по умолчанию (какие привязки были добавлены, удалены или заменены и т. д.). Если удалить все diff-файлы, игра вернётся к настройкам по умолчанию.

Имена файлов

Имена файлов привязок начинаются с идентификатора, за которым опционально следует суффикс, а затем расширение: identifierSuffix.extension.

  • Идентификатор: определяет, к какому контроллеру относится файл.
  • Суффикс: начиная с BeamNG.drive 0.25 суффикс можно включать или опускать, используя любой произвольный текст. Это позволяет модам предоставлять собственные привязки по умолчанию.
  • Расширение: как описано выше, может быть diff или json.

Например: keyboard.json, keyboardGiantExcavatorMod.json, c29b046d.diff и т. д.

Для поля идентификатора существует два варианта:

Идентификатор: тип устройства

Игра распознаёт любую из следующих строк:

  • 'mouse'
  • 'keyboard'
  • 'xidevice' — контроллер в режиме Xbox (например, геймпад Xbox или руль PC+Xbox, переведённый в режим Xbox вместо режима PC)
  • 'joystick' — джойстики (не Xbox)
  • 'wheel' — рули (не Xbox)
  • 'gamepad' — геймпады (не Xbox)
  • 'vinput' — виртуальный ввод; система для передачи ввода из произвольных источников, например из другой программы на компьютере или из наших приложений для Android/iOS для дистанционного управления

Идентификатор: USB Pidvid

Игра может распознавать USB-идентификаторы. Они выглядят как 8 шестнадцатеричных символов, например 1357acef. Они представляют 4 символа идентификатора производителя (vendor ID) + 4 символа идентификатора продукта (product ID).

Важно: как создателю мода, так и конечному пользователю не нужно выяснять имена pidvid вручную — это обрабатывается автоматически BeamNG.drive при создании или изменении привязок через меню Options → Controls.

Идентификатор продукта и идентификатор производителя — это коды, присвоенные конкретным производителям аппаратного обеспечения для продаваемых ими USB-устройств. Использование pidvid позволяет адаптировать привязки под конкретную модель устройства (например, «применять ко всем устройствам Logitech G27»), а не к обобщённому типу контроллера (например, «применять к любому wheel»).

Например, имя pidvid c29b046d соответствует устройству Logitech (vendor ID 046d) и их модели G27 (product ID c29b).

Мыши, клавиатуры и контроллеры Xbox не поддерживают идентификаторы pidvid в именах файлов привязок — только идентификаторы «тип устройства».

Формат файла

Важно: как создателю мода, так и конечному пользователю не нужно знать формат файла — не следует создавать эти файлы вручную; вместо этого используйте меню Options > Controls для определения заводских привязок вашего мода.

Файлы привязок в основном следуют формату json . Они должны начинаться с { и заканчиваться }. Ниже приведён пример того, как должна выглядеть общая структура:

keyboard.json:
{
  "bindings": [
    {"action": "menu_select", "control": "numpad9"},
    {"action": "menu_back", "control" : "numpad7"},
    // ...
  ]
}

Как видно, это простой список отношений действие ↔ элемент управления .

В некоторых случаях для каждой привязки содержится дополнительная информация (например, линейность привязки, параметры обратной силовой связи и т. д.). Мы не будем подробно останавливаться на этом.

Если файл является файлом .diff с настроенными пользовательскими привязками , он может содержать дополнительный список "removed", как в этом примере:

keyboard.diff:
{
  "bindings": [
    {"action": "menu_select", "control": "numpad3"},
    {"action": "accelerate", "control": "up", "filterType": "1"},
  ],
  "removed": [
    {"action": "menu_back", "control" : "numpad7"},
  ]
}

Раздел "bindings" внутри файла .diff может перезаписать заводскую привязку другим элементом управления (menu_select в примере выше) или другими/новыми параметрами (accelerate в примере выше).

Помимо "bindings" и "removed", в этих файлах могут присутствовать другие поля, такие как "guid", "name" и т. д. Вам не нужно беспокоиться об этом.

Карты взаимодействий определяют действия взаимодействия с транспортным средством в файлах с расширением:

*.interaction.json

Они используют тот же формат кодов действий, что и input_actions.json, но предназначены в первую очередь для интерактивных триггеров транспортного средства, особенно triggers2 и triggerEventLinks2.

Типичные сферы применения:

  • ручки дверей, капота, багажника, люка и задней двери
  • управление опорами прицепа и седельно-сцепным устройством
  • рычаги надстроек
  • управление оборудованием полицейских автомобилей
  • управление гидравликой
  • лебёдки, защёлки, замки и другие кликабельные элементы транспортного средства

Расположение файлов

Карты взаимодействий хранятся в папках транспортных средств:

vehicles/<vehicleName>/*.interaction.json

Примеры:

vehicles/cannon/cannon_default.interaction.json
vehicles/us_semi/us_semi_default.interaction.json
vehicles/common/common_default.interaction.json

Для одного транспортного средства может быть несколько файлов карт взаимодействий.

В общей папке транспортного средства можно определять совместно используемые действия, на которые могут ссылаться другие транспортные средства.

Базовая структура 

{
  "fileversion": 2,
  "actions": {
    "tailgate": {
      "order": 1005.1,
      "onDown": "controller.getControllerSafe('tailgateCoupler').toggleGroup()",
      "title": "vehicle.interaction.tailgate.toggle.title",
      "desc": "vehicle.interaction.tailgate.toggle.desc"
    }
  }
}

Объект actions содержит записи действий того же вида, что и в input_actions.json.

Поля верхнего уровня

Поле Тип Описание
fileversion integer Версия формата карты взаимодействий. В поставляемых файлах обычно используется значение 2.
actions object Словарь идентификаторов действий и их определений.

Поля действий

Каждая запись внутри actions является входным действием.

Общие поля:

Поле Тип Описание
order number Порядок отображения/сортировки.
onChange string Код, выполняемый при изменении входного значения.
onDown string Код, выполняемый при нажатии клавиши/кнопки/триггера.
onUp string Код, выполняемый при отпускании клавиши/кнопки/триггера.
title string Ключ перевода или краткий заголовок для отображения.
desc string Ключ перевода или краткое описание.
isCentered bool Использует входные значения в диапазоне -1..1 вместо 0..1.
icon string Необязательная подсказка UI/иконки, используемая в некоторых системах взаимодействия.

Полную модель кода действий см. в разделе Написание кода действий .

Код действий

Действия взаимодействия обычно выполняются в Lua транспортного средства.

Пример переключателя:

"door_FR": {
  "order": 1000.4,
  "onDown": "controller.getControllerSafe('door_FR_coupler').toggleGroup()",
  "title": "vehicle.interaction.doors.FR.toggle.title",
  "desc": "vehicle.interaction.doors.FR.toggle.desc"
}

Пример аналогового / удерживаемого действия:

"lift": {
  "order": 1.2,
  "onChange": "custom_input.liftBarrel(VALUE)",
  "title": "ui.inputActions.cannon.lift.title",
  "desc": "ui.inputActions.cannon.lift.description",
  "isCentered": true
}

VALUE заменяется текущим входным значением перед выполнением кода.

Используйте:

  • onDown для переключателей и одноразовых действий.
  • onChange для рычагов, ползунков, аналоговых входов и удерживаемых элементов управления.
  • onUp для поведения при отпускании.

Связь с triggers2

triggers2 определяет кликабельные триггерные области на транспортном средстве.

triggerEventLinks2 связывает эти триггерные области с входными действиями.

Пример:

"triggerEventLinks2": [
  ["triggerId:triggers2", "triggerInput", "inputAction"],
  ["door_R", "action0", "door_FR"]
]

Это означает:

Поле Значение
triggerId:triggers2 Идентификатор триггерной области.
triggerInput Какой триггерный ввод её активирует, например action0.
inputAction Идентификатор действия взаимодействия для выполнения.

Распространённые значения triggerInput:

Значение Значение
action0 Левая кнопка мыши / основное действие.
action1 Дополнительное действие, обычно Shift + левая кнопка мыши или правая кнопка мыши в зависимости от контекста.
action2 Средняя кнопка мыши / третичное действие.

inputAction обычно указывает на действие в файле *.interaction.json.

triggerEventLinks2 не ссылается на устаревшие events. Он ссылается на входные действия.

Общие действия

Общий файл взаимодействий:

vehicles/common/common_default.interaction.json

определяет множество совместно используемых действий, таких как действия для дверей, капота, багажника, защёлок, световых балок, сцепки, прицепа, гидравлики и контейнеров.

Детали транспортного средства могут ссылаться на эти действия из triggerEventLinks2.

Когда в ссылке на триггер используется префикс common:, действие разрешается из общей карты действий / набора общих действий, а не только из карты взаимодействий текущего транспортного средства.

Пример:

["door_R", "action0", "common:door_FR"]

Используйте общие действия, если нужное поведение уже реализовано и подходит для вашей детали.

actionsEnabled

Действия взаимодействия можно скрыть или отключить в интерфейсе до тех пор, пока деталь транспортного средства не объявит об их доступности.

Используйте секцию JBeam actionsEnabled:

"actionsEnabled": [
  ["id"],
  ["tiltUp"],
  ["tiltDown"],
  ["extend"],
  ["retract"]
]

Это полезно для транспортных средств с множеством опциональных деталей. Игроку должны отображаться только действия для установленных деталей.

Если действие отсутствует в actionsEnabled, его подсказка или легенда привязки может не отображаться, даже если действие существует.

Карта взаимодействий и input_actions.json

Файл Основное назначение
input_actions.json / input_actions*.json Общие привязки, специфичные для транспортного средства.
*.interaction.json Действия взаимодействия с транспортным средством, особенно кликабельные элементы управления triggers2.
JBeam events Устаревшие локальные определения действий детали, используемые со старыми triggers / triggerEventLinks.

Для новых кликабельных взаимодействий транспортного средства рекомендуется использовать:

triggers2 + triggerEventLinks2 + *.interaction.json

Используйте input_actions*.json для обычных элементов управления транспортным средством, не относящихся к кликабельной карте взаимодействий.

Устаревшие триггеры и события

Старые конфигурации триггеров используют:

triggers
triggerEventLinks
events

В этой системе events определяет код действия непосредственно внутри детали JBeam.

Более новые конфигурации triggers2 используют:

triggers2
triggerEventLinks2
*.interaction.json
actionsEnabled

Код действия переносится из JBeam events в карту взаимодействий.

Привязки по умолчанию

Действия взаимодействия могут иметь обычные привязки ввода.

Файлы привязок по умолчанию находятся в:

vehicles/<vehicleName>/inputmaps/

Пример:

vehicles/cannon/inputmaps/keyboard.json
vehicles/cannon/inputmaps/xidevice.json

Используйте меню «Управление» для создания привязок, затем при необходимости перемещайте/переименовывайте сгенерированные файлы .diff в папку inputmaps транспортного средства в качестве файлов .json по умолчанию.

Используйте уникальный суффикс для файлов привязок, предоставляемых модом:

vehicles/my_vehicle/inputmaps/keyboard_my_mod.json

Минимальный пример

Карта взаимодействий:

{
  "fileversion": 2,
  "actions": {
    "open_toolbox": {
      "order": 10,
      "onDown": "controller.getControllerSafe('toolboxLatch').toggleGroup()",
      "title": "vehicle.interaction.toolbox.open.title",
      "desc": "vehicle.interaction.toolbox.open.desc"
    }
  }
}

Ссылка в JBeam:

"actionsEnabled": [
  ["id"],
  ["open_toolbox"]
],

"triggerEventLinks2": [
  ["triggerId:triggers2", "triggerInput", "inputAction"],
  ["toolbox_handle", "action0", "open_toolbox"]
]

Триггер toolbox_handle вызывает действие open_toolbox из карты взаимодействий.

Примечания для инструментов

Для инструментов, генерирующих или изменяющих взаимодействия:

  • Сохраняйте fileversion.
  • Сохраняйте идентификаторы действий, на которые ссылаются triggerEventLinks2 и actionsEnabled.
  • Сохраняйте неизвестные поля действий.
  • Сохраняйте уникальность идентификаторов действий в пределах транспортного средства.
  • Не переносите код действий обратно в устаревшие events, если только намеренно не выполняется преобразование к старой системе триггеров.
  • Проверяйте каждую ссылку inputAction в triggerEventLinks2.
  • Проверяйте каждую запись actionsEnabled на соответствие доступным действиям.

Распространённые проблемы

Кликабельный триггер не реагирует

Возможные причины:

  • triggerEventLinks2.inputAction не совпадает с идентификатором действия.
  • Файл карты взаимодействий отсутствует или повреждён.
  • Код действия обращается к отсутствующему контроллеру.
  • Триггер использует неверный triggerInput.

Легенда привязок не отображает действие

Возможные причины:

  • Действие не указано в actionsEnabled.
  • Действие недоступно для текущей установленной детали.
  • Для действия не существует привязки.

Действие работает с клавиатуры, но не при нажатии на триггер

Возможные причины:

  • Привязка клавиатуры указывает на действие, а triggerEventLinks2 указывает на другой идентификатор.
  • Триггерная область расположена неверно.
  • Неправильно указан ввод действия триггера (action0, action1, action2).

Критерии проверки

  • JSON взаимодействий корректно разбирается.
  • Каждое действие triggerEventLinks2 успешно разрешается.
  • Каждое отображаемое взаимодействие указано в actionsEnabled.
  • Кликабельные триггеры активируют нужное действие.
  • Привязки клавиатуры/геймпада продолжают работать, если предоставлены стандартные inputmaps.
  • Отсутствующие опциональные детали не отображают нерелевантных действий.

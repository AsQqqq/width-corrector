Это руководство шаг за шагом объясняет, как обновить мод для использования новой системы окраски.

Пожалуйста, обратите особое внимание на синтаксис, в частности на использование скобок!

Library paints

Если ваш мод использует дубликаты определённых цветов окраски из ванильных автомобилей, рекомендуем проверить, есть ли эти краски в стандартной библиотеке красок , и заменить свои записи ссылками на краски из библиотеки. Это гарантирует, что при обновлении ванильной краски в будущем (из-за изменений графики или синтаксиса) изменение автоматически перенесётся в мод.

"paints": {
  "Night Blue": {
    "baseColor": [0.04,  0.06,  0.16,  1.0],
    "clearcoat": 0.8,
    "clearcoatRoughness": 0.07,
    "metallic": 0.8,
    "roughness": 0.5
  },
  (...)
},

До

"libraryPaints": [
  "Night Blue",
  (...)
],

Paint classes

Чтобы сократить количество доступных для выбора красок в меню выбора транспортного средства, некоторые краски, которые не были бы доступны с завода, можно перенести в класс Custom.

"paints": {
  "Night Blue": {
    "baseColor": [0.04,  0.06,  0.16,  1.0],
    "clearcoat": 0.8,
    "clearcoatRoughness": 0.07,
    "metallic": 0.8,
    "roughness": 0.5
  },
  (...)
},

До

"paints": {
  "Night Blue": {
    "class": "custom",
    "baseColor": [0.04,  0.06,  0.16,  1.0],
    "clearcoat": 0.8,
    "clearcoatRoughness": 0.07,
    "metallic": 0.8,
    "roughness": 0.5
  },
  (...)
},

После

Тот же подход можно применить к некоторым краскам из библиотеки .

"libraryPaints": [
  "matte_black",
  (...)
],

До

"libraryPaints": [
  { "id": "matte_black",    "class": "custom" },
  (...)
],

Multi-paint setups

В файлах конфигурации info.json, когда одновременно используется несколько цветов окраски (например, при двухцветных ливреях), следует заменить вхождения defaultPaintName1, defaultPaintName2 и defaultPaintName3 на наборы мульти-окраски . Это позволит корректно выбирать наборы окраски с подходящими цветовыми комбинациями.

Для заводских конфигураций добавьте наборы мульти-окраски с их красками в главный файл info.json транспортного средства:

"multiPaintSetups": [
  {
    "name": "Cream Two-Tone",
    "class": "factory",
    "paint1": "cream",
    "paint2": "light_brown",
    "paint3": "cream"
  },
  (...)
],

И сошлитесь на них в файлах info.json конфигураций:

"defaultPaintName1": "cream",
"defaultPaintName2": "light_brown",
"defaultPaintName3": "cream"

До

"defaultMultiPaintSetup":"Cream Two-Tone"

После

Для пользовательских конфигураций лучше объявлять набор мульти-окраски непосредственно внутри файла info.json данной конфигурации. Это позволит использовать его только в этой конкретной конфигурации.

"defaultPaintName1": "matte_dune",
"defaultPaintName2": "army_green",
"defaultPaintName3": "matte_black"

До

"defaultMultiPaintSetup":
{
  "name": "Two-Tone Adventure",
  "class": "custom",
  "paint1": "matte_dune",
  "paint2": "army_green",
  "paint3": "matte_black"
}

Система распределения красок используется для генерации реалистичного распределения цветов окраски среди транспортных средств в трафике. В будущем планируется также применять эту систему в различных системах случайного появления транспорта в карьерном режиме, например для соперников в гонках и инвентаря дилерских центров.

Implementation

В главный файл info.json транспортных средств, предназначенных для использования в трафике, мы добавляем несколько новых свойств:

"defaultRandomPaintProbability": 1,
"defaultRandomMultiPaintProbability": 1,
"randomPaintDistribution":{
  "white":5,
  "silver":4,
  "gray":1,
  "black":4,
  "Old White":4,
  "Pearl Black":4,
  "Old Black":4,
  "Metallic Blue":4,
  "Dark Blue":3,
  "Blue Silver":3,
  "Teal":3,
},
"randomMultiPaintDistribution":{
  "team_offroad_colors":10,
  "rennspecht_green_colors":1,
},

Это добавляет веса к функции случайного выбора краски, используемой для транспортных средств в трафике. defaultRandomPaintProbability — это вес по умолчанию, применяемый к каждой доступной краске. Затем в списке randomPaintDistribution для определённых красок их веса переопределяются, чтобы сделать их более или менее вероятными при появлении в трафике. То же самое относится к defaultRandomMultiPaintProbability и randomMultiPaintDistribution.

Отдельные конфигурации могут использовать те же свойства. Они переопределят значения, объявленные в главном файле info.json. Можно добавить новые веса красок и заменить отдельные существующие, при этом остальные веса будут унаследованы из главного файла info.json. Например, можно сделать отдельную конфигурацию в 10 раз более вероятной для появления в определённом цвете, переопределив вес этого цвета в её файле info.

Кроме того, файлы info конфигураций могут использовать новое свойство ignoreModelRandomPaintDistribution. Если оно установлено в true, это полностью заменит список randomPaintDistribution для данной конкретной конфигурации. Его можно использовать для ограничения определённых цветов на отдельных конфигурациях. Существует аналогичная опция ignoreModelRandomMultiPaintDistribution для multi-paint setup также.

Testing

Ручная проверка взвешенного распределения красок может быть затруднительной, поэтому для автоматизированного тестирования доступна консольная команда:

core_vehiclePaints.testRandomPaint('model', 'config', #samples)

Параметры — это внутреннее имя модели транспортного средства, название его конфигурации (можно установить в nil для тестирования всех конфигураций транспортного средства) и количество образцов для тестирования. Консоль эмулирует появление заданного количества транспортных средств/конфигураций (без их физического создания) и выводит список повторяющихся красок. Для каждой будет указано количество появлений, процент, тип (краска или multi-paint setup) и названия красок, использованных на всех трёх цветовых палитрах.

Пример 1: 1000 красок для модели simple_traffic core_vehiclePaints.testRandomPaint('simple_traffic', nil, 1000)

Пример 2: 1000 красок для модели simple_traffic с конфигурацией van_delivery core_vehiclePaints.testRandomPaint('simple_traffic', 'van_delivery', 1000)

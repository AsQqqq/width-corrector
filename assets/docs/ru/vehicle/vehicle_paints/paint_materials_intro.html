BeamNG использует систему материалов PBR (Physically-Based Rendering). Применительно к данной странице это означает, что автомобильная краска имеет несколько базовых свойств, которые определяют её внешний вид и реакцию на освещение, включая:

  • Цвет
  • Фактор металличности
  • Фактор шероховатости
  • Фактор лакового покрытия
  • Фактор шероховатости лакового покрытия

На этой странице вы найдёте обзор свойств PBR-материалов.

Определение материала, находящееся в файле materials.json, задаёт способ настройки каждого из этих свойств:

  • Оно может быть единым значением для всего материала. Например, цвет краски школьного автобуса можно задать жёлтым, и тогда пользователь не сможет его изменить. На практике такой подход для покрасочных материалов не применяется.
  • Оно может быть текстурой, наложенной на UV-развёртку транспортного средства, — по сути, определяющей, какие части 3D-модели обладают теми или иными свойствами краски. Например, краска школьного автобуса может иметь жёлтый цвет для большей части кузова и чёрные надписи и декали на отдельных участках. Этот подход используется, в частности, для нераскрашиваемых скинов.
  • Оно может быть полностью или частично раскрашиваемым — это означает, что часть или все параметры полностью или частично управляются одним или несколькими из трёх доступных цветовых палитр. Например, игрок может выбрать основной цвет краски школьного автобуса, тогда как надписи и декали всегда остаются чёрными. Это наиболее распространённый подход, применяемый в покрасочных материалах — как для базовой краски, так и для скинов. Именно этот тип материала рассматривается на данной странице.

Одна из цветовых палитр, отображаемых в игровом селекторе краски. Свойство «Chrominess» является устаревшей функцией и не работает с материалами на основе физически корректного рендеринга.

Раскрашиваемый покрасочный материал

В файле materials.json свойство instanceDiffuse определяет, является ли материал раскрашиваемым (true) или нет (false). Если оно установлено в true, color palette map будет определять, какие части UV-развёртки затрагиваются той или иной из трёх цветовых палитр. Используются полные значения цвета RGBA:

  • Красный означает, что будет использована первая цветовая палитра
  • Зелёный означает, что будет использована вторая цветовая палитра
  • Синий означает, что будет использована третья цветовая палитра
  • Прозрачный означает, что цветовая палитра применяться не будет (этот участок не раскрашиваем)

Смешение разных цветов позволяет создавать градиентные эффекты, образованные цветами краски, выбранными пользователем в разных палитрах. Пример карты цветовых палитр с градиентным эффектом.

Цвет, заданный пользователем в палитре, затем умножается на свойства материала. Например, значение ползунка металличности умножается на значение цвета в metallicMap и числовое значение в metallicFactor. На практике для красок транспортных средств мультипликаторные цвета устанавливаются в полный белый, а значения — в 1, чтобы избежать нежелательного смешения цветов.

Как правило, покрасочные материалы на транспортном средстве настраиваются следующим образом: для карты цветовых палитр используется полностью красная текстура, поверх которой накладывается opacity map. Белые участки карты непрозрачности определяют, какие части транспортного средства покрыты краской, делая их раскрашиваемыми с помощью первой цветовой палитры. Такая конфигурация упрощает добавление скинов на второй UV-слой. Подробнее об этом читайте на этой странице.

Свойства карты цветовых палитр

В файле materials.json есть несколько дополнительных свойств, влияющих на работу карты цветовых палитр:

  • colorPaletteMapUseUV: Если установлено значение 1, что обычно используется для покрасочных материалов, будет применяться второй UV-слой. Это упрощает размещение скинов.
  • paletteBaseColor: По умолчанию true; если установлено значение false, цветовые настройки палитры НЕ будут работать, и материал вместо этого будет использовать стандартные значения цвета, заданные для материала в карте базового цвета и/или факторе базового цвета, без умножения на значение из палитры. Это означает, что цветовая палитра будет влиять только на значения металличности, шероховатости, лакового покрытия и шероховатости лакового покрытия, но не на цвет краски.
  • Эквивалентные свойства существуют и для других значений цветовой палитры: paletteClearCoat, paletteClearCoatRoughness, paletteMetallic, paletteRoughness.

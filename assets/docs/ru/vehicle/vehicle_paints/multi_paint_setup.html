Схема мульти-покраски включает до 3 записей краски и имя набора. Каждая краска применяется к соответствующей цветовой палитре , что позволяет задать все доступные цвета краски при спауне транспортного средства.

Выбор схемы мульти-покраски при спауне транспортного средства.

Схемы мульти-покраски отображаются в отдельной вкладке меню покраски в конфигураторе транспортного средства. Схемы мульти-покраски в конфигураторе транспортного средства.

Определение схем мульти-покраски

Схемы мульти-покраски можно определить либо в библиотеке красок , либо в основном файле info.json транспортного средства. В обоих случаях определение выглядит одинаково:

"multiPaintSetups" : {
  "team_offroad_colors" : {
    "name": "Orange Race Colors",
    "paint1": "signal_orange",
    "paint2": "palladium_gray",
    "paint3": "black_diamond"
  },
  "rennspecht_green_colors" : {
    "name": "Rennspecht Green Colors",
    "paint1": "rennspecht_green",
    "paint2": "palladium_gray"
  },
  (...)
}

Ключи "paint1", "paint2" и "paint3" ссылаются на идентификаторы красок (Paint ID), которые будут применены для 1-й, 2-й и 3-й цветовой палитры соответственно. Если схема определена внутри файла библиотеки красок , краски также должны быть библиотечными. При определении схемы внутри файла info.json можно также использовать стандартные краски — в этом случае вместо идентификаторов используются имена красок. Если paint2 или paint3 не определены, для этой цветовой палитры будет использована paint1.

Файл info.json транспортного средства может использовать как собственные схемы мульти-покраски, так и общие из библиотеки красок.

"multiPaintSetups": [
  "rennspecht_green_colors",
  "rennspecht_black_colors",
  "rennspecht_blue_colors",
  {
    "name": "Battlehawk Purple",
    "paint1": "Extra Purple",
    "paint2": "Satin Black",
    "paint3": "Satin Gray"
  },
  {
    "name": "Battlehawk Green",
    "paint1": "Goblin Green",
    "paint2": "Satin Black",
    "paint3": "Satin Gray"
  },
],

В обоих случаях все цвета красок из библиотеки должны быть заранее объявлены в разделе libraryPaints. Их можно назначить с помощью пользовательского класса , чтобы не перегружать селектор красок.

Схема мульти-покраски может содержать только 1 краску и иметь то же имя, что и эта краска, маскируясь под обычный вариант окраски. Это удобно для добавления опции выбора однотонного цвета для транспортных средств, оснащённых двухтонным дизайном покраски по умолчанию.

Использование схем мульти-покраски по умолчанию

Как в основном файле info.json транспортного средства, так и в отдельном файле info_[config name].json конкретной конфигурации, схема мульти-покраски заменяет устаревшие значения "defaultPaintName1/2/3". Устаревший метод по-прежнему работает для обратной совместимости, однако defaultMultiPaintSetup имеет приоритет. Его можно использовать одним из следующих способов:

  • Указав имя схемы мульти-покраски, определённой в info.json или импортированной из библиотеки. Например: "defaultMultiPaintSetup":"Champagne". В этом случае при спауне транспортного средства через селектор эта схема мульти-покраски будет выделена.

Схема мульти-покраски по умолчанию для данной конфигурации выделена.

  • Определив собственную схему, специфичную для данной конфигурации. Например:
"defaultMultiPaintSetup":
{
  "name": "Turbo Race",
  "paint1": "Royal Blue",
  "paint2": "Solar Yellow",
  "paint3": "Solar Yellow"
}

В этом случае схема будет отображаться в списке мульти-покраски только при выборе данной конкретной конфигурации.

Схема мульти-покраски по умолчанию добавляется при выборе пользовательской конфигурации, которая её использует.

Моддерам рекомендуется полностью заменить использование синтаксиса defaultPaintName на схемы мульти-покраски.

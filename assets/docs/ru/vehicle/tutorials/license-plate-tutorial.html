Это руководство покажет вам, как создавать правильные номерные знаки для BeamNG.drive.

Требования

Для этого руководства вам потребуется лишь инструмент редактирования изображений и текстовый редактор. Мы будем использовать GIMP и Visual Studio Code в этом руководстве.

Вы можете воспользоваться этим файлом-шаблоном или создать всё с нуля. Шаблон содержит рабочий мод номерного знака, который можно изменять как угодно.

Если вы используете шаблон, просто поместите его в папку mods и распакуйте через менеджер модов внутри игры. Не забудьте переименовать файлы, как показано в разделе подготовки структуры папок этого руководства.

Введение

Для полноценного мода номерного знака необходимо иметь как минимум три файла — jbeam-файл, json-файл и файл текстуры.

Jbeam-файл определяет ваш новый номерной знак как дополнительную деталь, которую игрок сможет выбрать в меню выбора деталей. Он также содержит всю необходимую информацию о моде номерного знака и указывает путь к соответствующему json-файлу.

Json-файл указывает пути ко всем файлам текстур, которые вы будете использовать в своём моде номерного знака.

Файл текстуры будет служить фоном для вашего номерного знака.

Создание мода номерного знака

Подготовка структуры папок

Перейдите в пользовательскую папку , затем в /mods/unpacked и создайте несколько папок, соответствующих пути к файлам номерных знаков.

В итоге у вас должен получиться путь следующего вида:

/mods/unpacked/username_mod/vehicles/common/licenseplates/username_license_plate/

Части, выделенные жирным шрифтом, следует заменить на любые удобные для вас названия. Лучше избегать пробелов и специальных символов в именах файлов и путях — их можно заменить на - или _.

Убедитесь, что вы используете уникальные имена во избежание конфликтов с модами других пользователей или базовым контентом. Также следует избегать пробелов и специальных символов, так как они иногда могут вызывать проблемы.

Включение вашего имени пользователя в название папки — простой способ убедиться, что имя вашего мода будет уникальным.

username_mod соответствует имени zip-файла вашего мода и, следовательно, будет именем, отображаемым в менеджере модов внутри игры

Создание текстуры

Откройте GIMP или любой другой редактор изображений.

Сейчас мы создадим фон для номерного знака. Текстура фона должна соответствовать шаблонам: 512x256 пикселей для стандартного знака и 1024x196 пикселей для номерного знака формата 52-11.

Используйте этот файл-шаблон для создания собственного фона номерного знака. Не наносите номер на текстуру — он будет сгенерирован игрой и наложен поверх фоновой текстуры.

По завершении экспортируйте файл в формате .png в созданную ранее папку и дайте ему уникальное имя.

Если ваш номерной знак предусматривает новые текстуры для карт нормалей и карт отражений (specular), их можно пометить соответственно, добавив "_n" или "_s" в конце имени файла.

Обязательные изменения

Эти изменения необходимы для работы вашего мода.

  • name: Может быть любым.
  • gen: Определяет шаблон символов, которые будут сгенерированы, если пользователь не использует фиксированный текст. Подробнее см. этот раздел .
  • data section:
    • spriteImg / backgroundImg: Определяет, какая текстура будет использоваться для текста (spriteImg) и фона (backgroundImg). Замените это на путь к вашему собственному фоновому изображению.

Для пути важна часть внутри корневой папки мода. Например, в приведённом ниже случае путь будет выглядеть как "/vehicles/common/licenseplates/TutorialPlate/TestBackground.png" и "/vehicles/common/licenseplates/TutorialPlate/TestBackground_52-11.png"

Дополнительные свойства

Хотя прочие свойства изменять не обязательно, вы можете использовать их для добавления дополнительных элементов в свой мод.

  • data section:
    • formats: Игра поддерживает 2 формата номерных знаков по умолчанию. 30-15 — стандартный американский знак, а 52-11 — стандартный европейский. Рекомендуется использовать стандартные форматы для большинства применений, когда размеры достаточно близки. Если вам нужен только один из размеров, просто удалите ненужный раздел.
    • text: Определяет внешний вид текста. Измените значения x и y для позиционирования текста, скорректируйте коэффициент масштаба для изменения его размера, а RGB-значение параметра color — для изменения цвета текста. limit задаёт максимальное количество символов, отображаемых на номерном знаке.
    • generator: Вы можете задать внешний вид номерного знака, создав собственный html для него и указав ссылку здесь. Если эта строка отсутствует, будет использован html по умолчанию.
    • characterLayout: Ссылается на json-файл, определяющий шрифт, используемый для данного номерного знака. Подробнее см. этот раздел . Если эта строка отсутствует, будет использован html по умолчанию.
    • diffuse / bump / specular: Здесь необходимо указать пути к различным файлам текстур. Можно использовать до трёх типов карт: диффузную (diffuse), карту рельефа (bump) и/или карту отражений (specular).
    • fillStyle: Определяет цвет, который будет использоваться, если файл spriteImg / backgroundImg не задан. Может быть предопределённым HTML-цветом (например: black, white, ..) или значением RGB.
    • version: Это свойство предназначено только для внутреннего использования и не должно изменяться.
Если вы создали собственный мод карты, вы можете задать для неё собственный номерной знак по умолчанию. Для этого достаточно назвать папку с номерными знаками так же, как и карту, и переименовать один из json-файлов в licensePlate-default.json. Убедитесь, что пути в ваших jbeam-файлах соответствуют новым.

Создание jbeam-файла 

{
    "USERNAME_license_plate_2_1": {
    "information":{
        "authors":"BeamNG",
        "name":"License Plate Mod",
        "value":50
    },
    "slotType" : "licenseplate_design_2_1",
    "licenseplate_path" : "vehicles/common/licenseplates/user_license_plate/user_license_plate.json"
    }
}
  • Измените имя детали: Прежде всего измените имя детали во 2-й строке на уникальное.
  • Измените раздел информации:
    • authors: Введите имя автора
    • name: Введите название вашего номерного знака. Это имя будет отображаться в меню выбора деталей внутри игры.
  • slotType: Тип слота всегда должен быть "licenseplate_design_2_1"
  • licenseplate_path: Указывает путь к соответствующему json-файлу. Введите здесь путь к вашему json-файлу.

После выполнения всех вышеперечисленных шагов протестируйте мод в игре.

Откройте меню выбора деталей и проверьте, отображается ли ваш мод.

ГОТОВО! Если всё работает правильно, вы можете запаковать мод в zip-файл с помощью менеджера модов игры и загрузить его в репозиторий -> Uploading Mods

Дополнительные возможности


Изменённый стиль шрифта

Вы можете создать собственный файл platefont.json, если хотите использовать нестандартный шрифт. Просто скопируйте этот шаблон , переименуйте и измените его.

Наиболее важная часть файла — список chars. Здесь вы можете определить различные символы вашего шрифта и указать их расположение на текстуре.

Для каждого символа на знаке вам потребуется часть кода от { до }, — просто добавьте её на новой строке после },
"chars": {
  "count": "64",
  "char":
    {
      "id": "45",
      "x": "0",
      "y": "0",
      "width": "56",
      "height": "115",
      "xoffset": "0",
      "yoffset": "0",
      "xadvance": "56",
      "page": "0",
      "chnl": "15"
    },
 }
  • count: Указывает, сколько символов содержит ваш пользовательский шрифт.
  • id: Соответствует десятичному значению символа в таблице ASCII. Чтобы узнать значение каждого символа, воспользуйтесь этой ссылкой .
  • x / y: Определяет начальную точку символа.
  • width / height: Определяет размер символа в пикселях.

Добавление собственного формата знака

Рассмотрите возможность использования форматов, уже включённых в игру, если соотношение сторон близко к нужному.

Вам необходимо добавить новый меш, материал и файлы номерного знака.

Разместите их в папке следующим образом:

/vehicles/common/licenseplates/%NEW_FORMAT%/ или /vehicles/common/licenseplates/%NEW_MAP%/

Имя материала должно соответствовать соглашению об именовании: licenseplate- ширина в сантиметрах, дефис, высота в сантиметрах, необязательный код страны (одна или две буквы). Код страны следует использовать только в редких случаях, когда существует похожий формат, но требующий другого материала.

Также необходимо создать оформление по умолчанию, чтобы карты или стили, не определяющие ваш новый формат, всё равно отображали что-либо вместо ошибки no texture.

Создайте резервный файл здесь: /vehicles/common/licenseplates/default/licensePlate-default-%NEW_FORMAT%.json

Создание новых конфигураций

Самый простой способ создать новую конфигурацию транспортного средства — выбрать нужные детали в селекторе деталей, перейти в раздел «Save&Load» и сохранить свою конфигурацию.

Файл конфигурации будет сохранён в папке «vehicles» вместе с автоматически сгенерированной миниатюрой.

Чтобы включить эти конфигурации в свой мод, просто перетащите их в папку мода.

Формат файла конфигурации

Файл конфигурации содержит несколько параметров, в том числе:

  • format : Тип форматирования. Следует всегда оставлять без изменений.
  • licenseName : Номерной знак, используемый в конфигурации.
  • model : Модель транспортного средства, используемая в конфигурации. Следует всегда оставлять без изменений.
  • paint information : Цвет и другие настройки для каждого из 3 слотов покраски.
  • parts : Перечисляет все детали, используемые в конфигурации в каждом слоте. Если слот не указан, игра использует деталь по умолчанию для данного слота.
  • vars : Параметры каждой переменной транспортного средства. Если переменная не указана, игра использует значение по умолчанию для этой переменной.

Ручные изменения в файлах конфигурации

В большинстве конфигураций, которые будут включены в моды, обычно необходимо вручную удалить два раздела.

Первый — удалить раздел «paints». Этот раздел определяет, какой цвет использует конфигурация, однако он также жёстко задаёт цвет, с которым будет загружаться данная конфигурация, что означает: селектор цвета в меню выбора транспортного средства не будет работать корректно. Вместо этого цвета покраски конфигурации задаются в info-файле.

Второй — удалить раздел «licenseName». Опять же, для того чтобы в файле конфигурации использовался предпочтительный номерной знак игрока, а не жёстко заданное значение.

Создание основного info-файла

Если вы создаёте новый мод, вам потребуется info-файл для самого автомобиля, в котором будут определены некоторые свойства, отображаемые в меню выбора автомобиля: доступные варианты покраски, конфигурация по умолчанию и ряд других параметров.

В качестве отправной точки вы можете использовать этот файл .

Поля brand и name будут использоваться в качестве названия автомобиля в меню выбора транспортного средства. Большинство остальных элементов используются для фильтров.

Поле default_pc указывает на конфигурацию по умолчанию, а defaultPaintName — на покраску, которая будет использоваться, если в info-файле варианта не определена покраска.

Список покрасок определяет все цвета, которые будут доступны в меню выбора транспортного средства. Все параметры покраски задаются в диапазоне от 0 до 1. Исключение составляет alpha, однако она не оказывает никакого влияния на PBR-покраску.

Параметр baseColor задаётся в формате RGBA.

Генерация миниатюр

После того как вы создали все конфигурации, можно перейти к генерации миниатюр.

Миниатюры обычно генерируются на карте Showroom с помощью генератора миниатюр, который должен быть доступен при загрузке карты.

Чтобы сгенерировать миниатюры, выберите транспортное средство и нажмите зелёную кнопку воспроизведения. Миниатюры появятся в той же папке, что и файлы конфигурации.

Если миниатюры не отцентрированы, возможно, потребуется скорректировать Y-смещение внешней камеры транспортного средства.

Имейте в виду, что миниатюры, генерируемые по умолчанию, будут очень большими. Настоятельно рекомендуется изменить их размер и конвертировать в формат jpg, чтобы уменьшить объём мода.

Создание info-файлов

Каждая конфигурация также будет иметь собственный отдельный файл конфигурации с именем вида info_configName.json. В качестве отправной точки вы можете использовать этот файл .

Большинство параметров в этом файле служат лишь для справки и фильтрации. Наиболее важным является поле configuration — название конфигурации, отображаемое в меню выбора транспортного средства, а также покраска по умолчанию.

  • Acceleration stats : В info-файле содержится несколько показателей разгона — как в распространённых единицах км/ч (0–100, 0–200 и 100–200 км/ч), так и в милях в час (0–60, 0–120 и 60–120 mph).
  • Deceleration stats : Как и для разгона, предусмотрено несколько показателей торможения: тормозные пути с 100 до 0 км/ч в метрах и с 60 до 0 mph в футах.
  • BoundingBox : Вычисляется автоматической системой тестирования.
  • Braking G : Максимальная перегрузка при торможении
  • Config Type : Стандартные варианты: «Race», «Factory», «Police», «Custom», «Service», «Powerglow», «Rally»
  • Configuration : Название конфигурации, отображаемое в меню выбора транспортного средства.
  • Description : Описание конфигурации, отображаемое в меню выбора транспортного средства
  • Drivetrain : Тип привода транспортного средства. Обычно «FWD», «AWD», «RWD», «4WD» или «Other»
  • Fuel Type : Тип топлива транспортного средства. Обычно «Diesel», «Gasoline» или «Battery»
  • Induction Type : Тип наддува (если применяется). Обычно «NA», «SC», «Turbo» или любой из перечисленных с добавлением «+N2O»
  • Off-Road Score : Показатель проходимости. Вычисляется автоматической системой тестирования.
  • Performance Class : Вычисляется автоматической системой тестирования.
  • Population : Определяет частоту появления автомобиля в потоке трафика
  • Power : Пиковая мощность в киловаттах
  • PowerPeakRPM : Диапазон оборотов двигателя при пиковой мощности
  • Propulsion : Тип двигателя. Обычно «ICE» или «Electric»
  • Top Speed : Максимальная скорость в метрах в секунду
  • Torque : Пиковый крутящий момент в ньютон-метрах.
  • TorquePeakRPM : Диапазон оборотов двигателя при пиковом крутящем моменте
  • Transmission : Тип трансмиссии. Обычно «Manual», «Automatic», «Other», «DCT», «Sequential» или «CVT»
  • Value : Цена транспортного средства. В настоящее время используется только для сортировки конфигураций в селекторе деталей.
  • Weight : Масса транспортного средства в кг
  • Weight/Power : Соотношение массы к мощности
  • Years : Диапазон годов выпуска данной конкретной конфигурации
  • defaultPaintName1 : Название покраски, используемой по умолчанию для 1-го цветового слота. Должно совпадать с одной из покрасок, определённых в основном info-файле.
  • defaultPaintName2 : Название покраски, используемой по умолчанию для 2-го цветового слота. Должно совпадать с одной из покрасок, определённых в основном info-файле.
  • defaultPaintName3 : Название покраски, используемой по умолчанию для 3-го цветового слота. Должно совпадать с одной из покрасок, определённых в основном info-файле.

Автоматическое тестирование производительности

Хотя вы можете вручную заполнить различные значения производительности в info-файлах, существует способ заполнить большинство из них автоматически.

Для начала необходимо добавить эти файлы в папку игры BeamNG. В набор входят пакетный файл и файл параметров. Некоторые из этих файлов могут уже находиться в папке игры.

После того как вы поместили файлы в папку игры, измените название автомобиля в файле параметров на тот, который хотите протестировать, и запустите пакетный файл. Название автомобиля должно совпадать с названием папки транспортного средства.

Игра выполнит серию тестов для каждого варианта, включая тест проходимости, а также тесты разгона, максимальной скорости и торможения. Следует всегда держать окно игры активным, иначе результаты могут оказаться неточными.

После завершения тестов info-файлы можно найти в папке вашего транспортного средства. Сгенерированные info-файлы сохранят все ранее заданные параметры, за исключением измеренных показателей производительности.

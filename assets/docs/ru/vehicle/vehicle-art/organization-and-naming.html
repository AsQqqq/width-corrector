Исходные файлы

Для каждого из перечисленных элементов, как правило, должен быть собственный .blend и (при необходимости) собственный файл Substance Painter:

  • Каждая модель транспортного средства (например, D-Series, Vivace и т. д.)
  • Каждая уникальная модель колеса
  • Каждая уникальная шина

Организация файлов Blender

Финальный исходный файл .blend рекомендуется оформлять в соответствии со следующими рекомендациями, поскольку такая организация существенно упрощает и ускоряет работу с контентом (особенно в части экспорта):

Организация коллекций

Соглашения об именовании экспортируемых частей Blender:

  • Основные коллекции:

    • Source → Содержит исходные меши, которые не должны экспортироваться
    • Export → Содержит все меши, предназначенные для экспорта

  • Пример структуры:

    • Source
      • Любые исходные меши, не предназначенные для экспорта
    • Export
      • "dae file location"
        • Вариация кузова 1
          • Вариация кузова 2
          • Вариация двигателя 1
          • Вариация двигателя 2
      • "dae 2 file location"

  • Практический пример:

    • Source
      • Любые исходные меши, не предназначенные для экспорта
    • Export
      • \vehicles\common\beaconlights.dae
        • Beacon1
          • Beacon2
          • Beacon3
      • \vehicles\bluebuck\bluebuck_engine.dae
        • Base
          • Dragster
          • V8
          • Supercharger
            • Carburetor

**Эта страница изначально была создана в рамках внутренних художественных рекомендаций по разработке.

Хотя приведённые сведения по-прежнему полезны как лучшие практики организации, они могут быть не полностью применимы к вашему контенту и не обязательно должны строго соблюдаться при создании модов.

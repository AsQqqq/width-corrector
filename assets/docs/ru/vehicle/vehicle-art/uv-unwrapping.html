Оптимизация UV-карт и материалов

Это упростит работу: вы можете использовать столько материалов, сколько потребуется, поскольку Vulkan упаковывает текстуры при загрузке. Таким образом, ваши карты могут быть настолько маленькими, насколько вы хотите.

  • Оставляйте около 10% свободного пространства в каждой UV-карте для будущих изменений
  • Каждый острый рёбр должен также быть швом UV; фаскам нормалей нужно некоторое пространство
  • Между UV-островами необходимо соблюдать отступ в 8 пикселей
  • Плотность текстуры — 512 пикс./м в общем случае (если только не применяется техника тайловых текстур)
  • Убедитесь, что имена карт UV0 и UV1 используются последовательно
    • UV0 — для детализации текстуры и нормалей
    • UV1 — для дизайна ливреи

Упаковка UV

  • Текстуры должны разрабатываться для игрового разрешения 4098x4098 или меньше
  • Для достижения достаточно высокой плотности пикселей при наилучшем качестве требуется значительная оптимизация

Перекрытие UV

  • Не забывайте перекрывать UV у одинаковых объектов
  • Зеркально отражайте UV с осью симметрии (если только в текстурировании не будет очевидного отражения)
  • Рёбра одного и того же материала можно перекрывать
  • Шланги и провода следует перекрывать
  • Нижние части бамперов, крыльев и порогов можно разместить в общем UV-пространстве, если они не содержат уникальных деталей.
  • Грани на краях панелей на 2-й UV-карте могут быть спроецированы неортографически, если это необходимо для предотвращения сильного растяжения.
  • Альтернативные детали кузова должны иметь 2-ю UV-карту, спроецированную поверх друг друга, чтобы дизайн окраски выглядел одинаково на всех возможных версиях транспортного средства

Выравнивание UV-карт по прямоугольнику

  • Мы отдаём приоритет фаскам карты нормалей перед растяжением; карты, включающие фаску, должны быть строго вертикальными или горизонтальными
  • Пожалуйста, ознакомьтесь со справочными материалами, чтобы понять, когда растяжение слишком велико и допустимо использовать негоризонтальную или невертикальную UV-карту
  • Не забывайте минимизировать растяжение в пределах UV-карты, если фасок карты нормалей там нет

Не используйте UV с неправильной ориентацией и непрямыми границами!
Если граница не прямая, это вызовет нежелательные проблемы с антиалиасингом, как в данном примере:

UV1

(2-я UV-карта, для виниловых обёрток транспортного средства)

  • Эта карта должна быть удобной для восприятия, а UV-острова должны располагаться в соответствии с их реальным положением.
  • Эти карты не нужно выравнивать по прямоугольнику
  • Карты должны создаваться с минимальным растяжением
  • Большинство граней на этой UV-карте должны быть спроецированы ортографически — сверху, сбоку или спереди.
  • Неиспользуемые UV-карты должны иметь нулевую ширину и нулевую высоту, затем помещаться в координату 0,0.
  • Панели кузова не должны перекрываться, чтобы избежать зеркального отражения текста

Это делается для того, чтобы разработчики и модмейкеры могли легко создать ливрею для транспортного средства без зеркального текста или странного растяжения. В отличие от других карт, эта обычно создаётся в Photoshop, поэтому расположение UV-островов должно соответствовать их положению в 3D-пространстве.

Плотность пикселей

Интерьер

Плотность пикселей панелей кузова и внешних элементов должна быть близка к Vivace для достижения одинакового качества деталей карты нормалей.

Множители плотности пикселей:

  • Рулевое колесо / панели с текстом и иконками / кнопки / диффузоры = 150%
  • Приборная панель / панели перед полем зрения игрока = 100%
  • UV с низкой видимостью (пол, фоновые панели, кабели) = 60% с оптимизацией перекрытия
  • Скрытые UV = 30%

Экстерьер

  • Всегда видимые части, такие как панели кузова = 100%
  • Не всегда видимые части, такие как нижняя часть кузова = 75%
  • Грани, видимые только при отсоединении детали = 50%

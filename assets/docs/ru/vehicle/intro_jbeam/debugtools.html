Для помощи в проектировании Jbeam игра предоставляет несколько режимов отладки, которые позволяют получить подробную информацию о вашем Jbeam.

Консоль

Консоль открывается нажатием клавиши `. В ней отображается информация и ошибки, возникающие при загрузке автомобиля; она помогает разобраться, почему транспортное средство не загружается, отсутствуют текстуры и т. д.

Информация выделена цветом: ошибки — красным, предупреждения — жёлтым, сообщения — зелёным, отладочная информация — синим. Вы можете скрывать отдельные типы сообщений, нажимая на соответствующие кнопки в верхнем левом углу.

Текстовое поле внизу служит для ввода команд.

Замедленное воспроизведение

Инструмент замедленного воспроизведения, хотя и не является инструментом отладки в строгом смысле, может быть очень полезен при поиске проблем в jbeam. Он доступен в меню «Environment» либо по нажатию Alt + Up

Vehicle Config > Debug UI

Debug UI содержит множество способов отладки физических свойств JBeam транспортного средства. Этот интерфейс включает описанные ниже режимы отладки, а также функции спауна транспортных средств.

Debug UI включает селектор деталей транспортного средства, на которые распространяются инструменты отладки; он работает как список флажков — отмеченные детали будут затронуты, остальные — нет. Вы можете использовать значок галочки для быстрого выбора или снятия выбора со всех деталей. После изменения детали в собственном селекторе деталей транспортного средства или изменения видимости детали там изменения синхронизируются автоматически. Селектор также включает строку поиска.

Debug UI содержит следующие разделы (в числе прочих):

Визуализация нод

Режимы отладки нод по умолчанию доступны по нажатию Ctrl + M. В Debug UI можно настроить ширину и прозрачность подсветки. Доступны следующие режимы:

  • Simple — отображает ноды с цветовым кодированием на основе коллизий. Жёлтые ноды имеют как внутреннюю, так и внешнюю коллизию; светло-голубые ноды имеют внешнюю коллизию, но не имеют внутренней; фиолетовые ноды не имеют коллизии вовсе.
  • Highlighted Only — показывает подсвеченные ноды в цвете их подсветки.
  • Weights — отображает ноды в виде точек, размер которых зависит от веса ноды; цветовое кодирование аналогично режиму Simple.
  • Displacement — подсвечивает ноды, сместившиеся от своего начального положения при спауне вследствие упругой деформации. Чем больше смещение, тем выше непрозрачность цвета подсветки.
  • Velocities — показывает скорость ноды относительно опорных нод.
  • Forces — отображает вектор суммарных сил, приложенных к ноде. Очень полезен для поиска нестабильностей или нод, на которые воздействуют нежелательные коллизии треугольников.
  • Density — показывает, находится ли нода в данный момент в воздухе (зелёный) или в воде (красный).
  • Clusters — отображает каждый кластер нод отдельным цветом. Кластеры — это группы нод, соединённых балками. Всё транспортное средство изначально является единым кластером; отломившиеся части образуют собственные кластеры.
  • Main Cluster — показывает ноды основного кластера зелёным цветом, остальные — синим. Основной кластер — тот, за которым следует камера.

Помимо этих режимов отладки, в интерфейсе также есть флажок Show Highlighted. При его установке выбранный режим фактически совмещается с режимом Highlighted Only.

Текст нод

Режимы текстового отображения нод по умолчанию доступны по нажатию Ctrl + N. Доступны следующие режимы:

  • Name — показывает имя каждой ноды, заданное в jbeam. Если имя не задано (как у нод колёс), вместо него отображается порядковый номер.
  • Numbers — показывает порядковый номер каждой ноды. Используется преимущественно Lua-скриптами и физическим движком.
  • Name+Numbers — комбинация двух предыдущих режимов.
  • Weights — отображает имя и вес каждой ноды в кг. Также показывает суммарный вес всех нод в верхнем правом углу экрана.
  • Materials — отображает имя и назначенный материал каждой ноды.
  • Groups — для каждой ноды показывает её имя и группу, к которой она принадлежит. Для работы требуется установить флажок «Spawn in Debug Mode» в Debug UI.
  • Forces — отображает имя каждой ноды и значение вектора суммарных сил, приложенных к ней. Также показывает среднюю силу в верхнем правом углу экрана. Купоры отображаются красным шрифтом вместо чёрного и не учитываются при расчёте среднего значения.
  • Relative positions — отображает имя каждой ноды и её положение в локальных координатах транспортного средства.
  • World positions — отображает имя каждой ноды и её положение в мировых координатах.

В Debug UI можно включить флажок Show Wheels, который определяет, будет ли этот текст отображаться также для нод колёс. По умолчанию он отключён, поскольку эти данные загромождают вид и, как правило, не нужны в большинстве случаев.

Отладочный текст нод

Отображает дополнительный отладочный текст с информацией о некоторых дополнительных свойствах, заданных для отдельных нод. По умолчанию доступен по нажатию Ctrl + K. Доступны следующие режимы:

  • Latches — показывает звуковые события, связанные с защёлками транспортного средства, такие как событие открытия, закрытия и поломки, а также имя защёлки.
  • PropAnimation — показывает звуковые события, связанные с контроллерами анимации пропов, а также имя контроллера.
  • Soundscape — показывает звуковые события, связанные с различными звуками транспортного средства.
  • CombustionEngine — показывает звуковые события, связанные с двигателем и выхлопной системой, если они присутствуют в транспортном средстве. Отображает имя двигателя в метке, а также дополнительно в названии этого раздела, например «CombustionEngine mainEngine».
  • Powertrain — показывает звуковые события, связанные с различными устройствами трансмиссии транспортного средства.

Визуализация балок

Режимы отладки балок предлагают несколько представлений для отображения различных состояний балок. Они полезны при построении jbeam и поиске проблем, вызванных нестабильностью или утечкой свойств. Режимы отладки балок по умолчанию доступны по нажатию Ctrl + B. Некоторые режимы отладки балок имеют параметр диапазона значений для ограничения визуализируемых значений. Его можно задать через Vehicle Config > Debug с помощью ползунков «Range Min» и «Range Max», а также флажка «Show Infinity Values (FLT_MAX)».

Этот раздел содержит режимы визуализации балок на основе их текущего состояния в игре. Доступные режимы:

  • Simple — отображает балки зелёным цветом.
  • Type — рисует балки разными цветами в зависимости от их типа. Обычные балки — зелёные, опорные балки — фиолетовые, ограниченные балки — жёлтые, гидро-балки — тёмно-синие, балки с давлением — голубые, l-балки — серые, анизотропные балки — оранжевые. Эта цветовая легенда также доступна в игре.
  • With broken — то же, что режим «Type», но сломанные балки отображаются красным.
  • Broken only — отображает только сломанные балки. Полезен для исследования проблем, например поломки балок при спауне.
  • Support only — отображает только неповреждённые опорные балки. Полезен для настройки параметра beamLongBound у опорных балок.
  • Stress — показывает, насколько балки нагружены: красным — при сжатии, синим — при растяжении. Полезен для выявления проблем нестабильности и вибрации. Существует два режима stress: с ограничением диапазона значений через Debug UI и без него (последний называется «Stress (Old)»).
  • Displacement — показывает, насколько балки смещены вследствие упругой деформации относительно начальной длины. Синий означает, что балка растянута, красный — что укорочена. Debug UI позволяет ограничивать диапазон значений.
  • Deformation — показывает, насколько балки постоянно деформированы. Синий означает, что балка вытянута сверх начальной длины, красный — что укорочена. Debug UI позволяет ограничивать диапазон значений.
  • Break Groups — подсвечивает балки по breakGroups разными цветами. Полезен для выявления проблем с breakGroups, которые не работают должным образом или имеют утечки.
  • Deform Groups — подсвечивает балки по deformGroups разными цветами.
  • Bounded Beam Bounds — показывает только ограниченные балки.
    • Если балка не расширилась и не сжалась относительно начальной длины при спауне (с учётом предкомпрессии), она отображается полностью зелёной без выступа за ноды.
    • Если балка укорочена относительно начальной длины, текущая длина отображается зелёным, а с обеих сторон она дополняется до начальной длины тёмно-синим цветом. Если сработал shortBound балки, он показан голубым поверх синего, а светло-голубой дополнительно визуализирует переходную зону.
    • Если балка удлинена относительно начальной длины, зелёный цвет показывает исходную длину, а расширение заполняется красным. Если балка вышла за предел longBound, оставшееся расширение заполняется жёлтым, а оранжевый дополнительно визуализирует переходную зону.
  • Support Beam Bounds — то же, что описано выше, но для опорных балок. Переходные зоны не отображаются, поскольку имеют нулевую длину. Эти балки часто используют предкомпрессию меньше 1 в сочетании с высокими значениями longBound, поэтому у них нередко будут длинные жёлтые пределы longBound.
  • Frequency — подсвечивает балки, имеющие заданную частоту с заданной максимальной амплитудой.

Также существуют режимы визуализации балок, основанные на свойствах, заданных в Jbeam. В этих режимах отладки нижние значения диапазона отображаются белым, верхние — красным, а значения бесконечности (FLT_MAX) — фиолетовым. К этим режимам относятся:

  • Beam Damp Fast
  • Beam Damp Rebound
  • Beam Damp Rebound Fast
  • Beam Damp Velocity Split
  • Beam Deform
  • Beam Limit Damp
  • Beam Limit Damp Rebound
  • Beam Long Bound
  • Beam Precompression
  • Beam Precompression Range
  • Beam Precompression Time
  • Beam Short Bound
  • Beam Spring
  • Beam Strength
  • Bound Zone
  • Damp Cutoff Hz
  • Damp Expansion
  • Deform Limit
  • Deform Limit Expansion
  • Deformation Trigger Ratio
  • Long Bound Range
  • Precompression Range
  • Short Bound Range
  • Spring Expansion

Текст балок

Содержит дополнительную информацию по каждой балке:

  • IDs — показывает внутренний идентификатор балки.
  • Spawn Length — отображает исходную длину каждой балки в метрах.
  • Live Length — отображает текущую длину каждой балки в метрах.

Визуализация торсионных балок

Доступна только из Debug UI. Показывает торсионные балки транспортного средства. Полезна для отладки жёстких конструкций, таких как подрамники и подвеска. Доступные режимы:

  • Simple — для каждой торсионной балки показывает её ноды с цветовым кодированием от одного конца к другому в цветах: красный, оранжевый, жёлтый и зелёный соответственно. Также подсвечивает соединения между этими нодами отдельным цветом для каждой торсионной балки.
  • Without Broken — то же, что выше, но сломанные торсионные балки больше не подсвечиваются.
  • With Broken — то же, что выше, но сломанные торсионные балки выделяются более заметными цветами по сравнению с целыми.
  • Broken Only — то же, что выше, но подсвечиваются только сломанные торсионные балки.
  • Angle — непрозрачность торсионной балки визуализирует угол между её плечами. Минимальный и максимальный отображаемый диапазон можно задать в настройках.
  • Stress — непрозрачность торсионной балки визуализирует напряжение между её плечами. Минимальный и максимальный отображаемый диапазон можно задать в настройках.
  • Deformation — непрозрачность торсионной балки визуализирует деформацию между её плечами. Минимальный и максимальный отображаемый диапазон можно задать в настройках.
  • Damp — цвет соединений между нодами торсионной балки визуализирует её демпфирование так же, как при отладке демпфирования балок. Минимальный и максимальный отображаемый диапазон можно задать в настройках.
  • Deform — цвет соединений между нодами торсионной балки визуализирует её сопротивление деформации так же, как при отладке deform балок. Минимальный и максимальный отображаемый диапазон можно задать в настройках.
  • Spring — цвет соединений между нодами торсионной балки визуализирует её жёсткость так же, как при отладке spring балок. Минимальный и максимальный отображаемый диапазон можно задать в настройках.
  • Strength — цвет соединений между нодами торсионной балки визуализирует её сопротивление разрушению так же, как при отладке strength балок. Минимальный и максимальный отображаемый диапазон можно задать в настройках.

Визуализация рельсов и слайд-нод

Доступна только из Debug UI. Показывает рельсы и слайд-ноды транспортного средства. Полезна для отладки проблем с поломкой и деформацией балок при спауне из-за неправильно выровненных слайд-нод. Доступные режимы:

  • Simple — показывает каждый рельс и его слайд-ноды в собственном цвете. Слайд-ноды будут толще, чем ноды рельса.
  • Without Broken — то же, что выше, но сломанные рельсы и отсоединённые слайд-ноды больше не подсвечиваются.
  • With Broken — то же, что выше, но сломанные рельсы и отсоединённые слайд-ноды выделяются более заметными цветами по сравнению с целыми.
  • Broken Only — то же, что выше, но подсвечиваются только сломанные рельсы и отсоединённые слайд-ноды.

Визуализация треугольников коллизии

Режим отладки треугольников по умолчанию доступен по нажатию Ctrl + T. Полезен для отладки коллизии и аэродинамики. Доступные режимы:

  • Simple — показывает лицевую сторону треугольников зелёным, а тыльную — фиолетовым.
  • Type — то же, что выше, но лицевые стороны неколлайдируемых треугольников показаны жёлтым, а треугольников с давлением — синим.
  • Without Broken — то же, что выше, но сломанные треугольники больше не отображаются.
  • With Broken — то же, что выше, но сломанные треугольники отображаются красным с обеих сторон.
  • Broken Only — то же, что выше, но отображаются только сломанные треугольники.
  • Collidable Only — то же, что Without Broken, но также не показываются неколлайдируемые треугольники.

Прочее

  • Aerodynamics debug: Отладка аэродинамики доступна только через меню отладки, которое открывается после включения расширенных функций в настройках интерфейса. Она используется для отладки аэродинамики автомобиля и исследования таких проблем, как дисбаланс подъёмной силы и сопротивления. Имейте в виду, что хотя аэродинамические свойства задаются с помощью треугольников , силы рассчитываются в каждой ноде с учётом свойств, размера и ориентации прилегающих треугольников. Доступные параметры:
  1. Drag + Lift — показывает 3 вектора для каждой ноды. Жёлтый — нормальный вектор аэродинамической поверхности ноды, рассчитанный на основе размера и ориентации прилегающих треугольников. Красный — сопротивление, приложенное к данной ноде; этот вектор силы всегда направлен строго назад относительно воздушного потока. Синий — подъёмная/прижимная сила, приложенная к данной ноде; этот вектор силы всегда перпендикулярен сопротивлению.
  2. Angle of Attack — показывает угол между нормальным вектором ноды и воздушным потоком. Если нормальный вектор полностью совпадает с воздушным потоком, угол атаки составит 90 градусов и сила будет целиком сопротивлением. Если нормальный вектор перпендикулярен воздушному потоку, угол атаки составит 0 градусов и аэродинамические силы на ноде отсутствуют.
  3. Combined — этот режим объединяет информацию из двух предыдущих.

  • Center of Gravity: Режим отладки центра масс по умолчанию доступен по нажатию Ctrl + G. Красная точка показывает положение центра масс автомобиля. Синяя точка показывает положение центра давления — места, где усредняются все силы сопротивления и подъёмная сила. Два различных режима показывают центр масс соответственно с колёсами и без них.

  • Tire contact point debug: Показывает точку контакта с дорогой для каждого колеса транспортного средства.

  • Steering geometry debug (available since version 0.33): Бросает лучи из центров колёс в обоих направлениях. Точки пересечения этих лучей помогают визуализировать и отлаживать геометрию рулевого управления. Длина лучей регулируется.

  • Mesh visibility: Хотя это и не инструмент отладки сам по себе, регулятор видимости меша очень полезен для лучшего просмотра различных режимов отладки — он позволяет сделать меш частично или полностью прозрачным. Клавиши по умолчанию: Ctrl + NumpadPlus и Ctrl + NumpadMinus для повышения и понижения видимости меша соответственно.

UI Apps

BeamNG.drive включает ряд полезных отладочных UI-приложений, которые можно добавить в интерфейс. Их можно добавить на вкладке UI Apps в верхнем меню.

Advanced Wheels Debug

Это приложение показывает углы схождения и развала для каждого колеса транспортного средства, а также углы кастера и SAI для колёс с рулевой осью. Значения рассчитываются относительно refNodes транспортного средства и основаны на положении нод, заданных в секциях pressureWheels Jbeam. Может использоваться для настройки и отладки геометрии подвески, а также позволяет легко выявить проблемы выравнивания refNode.

Brake Thermal Debug

Позволяет выбрать 2 тормоза на транспортном средстве и отображает различные тепловые параметры. Может использоваться для отладки проблем с перегревом тормозов.

Engine Debug

Отображает некоторые сведения о двигателе, которые могут быть полезны для отладки потерь энергии из-за трения или температуры.

Engine Thermal Debug

Обширный динамический список различных тепловых свойств двигателя транспортного средства. Содержит информацию о повреждении двигателя, эффективности радиатора и охлаждающей жидкости, масляном голодании и многом другом. Позволяет точно определить причину перегрева и проблем с маслом, а также помогает при настройке системы охлаждения.

Node/Beam Info

Это приложение надёжно отображает статистику Jbeam транспортного средства: количество и процент деформированных или сломанных балок и торсионных балок, точный суммарный вес, количество треугольников и т. д. Может использоваться для оценки влияния Jbeam транспортного средства на производительность по сравнению с другими транспортными средствами. Также очень полезно для выявления проблем, которые иначе трудно заметить и которые приводят к неверному весу или деформированным балкам.

Powertrain Visualization

Визуализирует дерево трансмиссии транспортного средства со всеми его компонентами и проходящим через них крутящим моментом. Это можно изменить на мощность, нажав на текст «torque». Наведение курсора мыши на компонент отображает его имя и частоту вращения. Некоторые компоненты, такие как блокируемые оси колёс, отключаемые валы, блокируемые дифференциалы и раздаточные коробки, можно переключать, нажимая на них в приложении. Это приложение очень полезно для отладки таких проблем, как закручивание трансмиссии и неправильные конфигурации привода.

Shift Point Debug

Показывает анимированный график активных передач коробки передач в зависимости от оборотов двигателя и скорости колёс в км/ч, с отмеченными точками переключения и линией текущей передачи. Полезен для настройки точек переключения автоматических коробок передач.

Torque Curve

Практически незаменимый инструмент при настройке двигателей: это приложение показывает кривую крутящего момента в зависимости от оборотов (в Нм) и кривую мощности в зависимости от оборотов (в л.с.), отображая текущие и пиковые значения мощности и крутящего момента вместе с текущей точкой на кривой. Отдельные устройства принудительного наддува можно включать и отключать на графике; они будут показаны разными стилями линий.

Weight Distribution

Показывает нагрузку на каждое колесо транспортного средства несколькими способами одновременно: в виде фонового визуального выделения колёс с нагрузкой более 25%, в виде нагрузки по весу в кг, нагрузки по силе в Н, а также процента от общего веса на каждом колесе.

World Editor

World Editor, открываемый по умолчанию клавишей F11, содержит несколько полезных инструментов для отладки Jbeam. Их можно добавить через вкладку Window.

Engine Audio Debug

Находится на вкладке Audio. Этот инструмент позволяет в реальном времени изменять и тестировать конфигурационные значения звука двигателя и выхлопа. Полученные настройки Jbeam можно скопировать в буфер обмена и вставить в Jbeam-файл двигателя.

Suspension Audio Debug

Находится на вкладке Audio. Позволяет отлаживать параметры звука подвески, заданные в Jbeam.

Vehicle Manager

Находится на вкладке Experimental. При нажатии кнопки «Reload» создаёт следующие файлы в папке пользователя:

  • vehicleDebug_activeParts.json: необработанные данные Jbeam всех активных деталей транспортного средства в формате Json.
  • vehicleDebug_chosenParts.json: список всех установленных на транспортное средство деталей и их слотов, отформатированный аналогично конфигурационным файлам .pc.
  • vehicleDebug_config.json: конфигурационный файл текущего транспортного средства, аналогичный файлу .pc, но с большим объёмом данных.
  • vehicleDebug_data.json: все данные транспортного средства в том виде, в котором их видит игровой движок.
  • vehicleDebug_preTable.json: все секции Jbeam из всех деталей транспортного средства, объединённые в один Json-файл. Vehicle Manager позволяет отлаживать проблемы с вычислениями и конкатенацией строк в Jbeam, отображая, как эти данные выглядят на различных этапах их обработки.

Vehicle Editor

Vehicle Editor — это набор полезных инструментов Jbeam, доступных по нажатию Shift + F11 или через вкладки World Editor: Window > Experimental> Vehicle Editor. Он имеет две вкладки: Static Editor, ориентированный на редактирование файлов Jbeam, и Live Editor, ориентированный на отладку поведения Jbeam в реальном времени. Оба имеют настраиваемые макеты, которые можно сохранять, позволяя пользователю всегда держать под рукой любимые инструменты.

Static Editor Apps

  • Jbeam Modifier Leaking Visualizer: показывает, какие значения Jbeam, используемые в деталях, утекают в детали, загружаемые после них.
  • Jbeam Spellchecker: выполняет поиск опечаток в именах секций во всех файлах Jbeam и выводит их в консоль.
  • Jbeam Table Visualizer: отображает таблицы выбранных файлов Jbeam в интерфейсе.
  • Jbeam Variables Checker: выполняет поиск во всех файлах .pc переменных, выходящих за пределы диапазона, заданного в соответствующих файлах Jbeam, и выводит их в консоль. (Возможны ложные срабатывания, если одна и та же переменная определена в нескольких файлах с разными диапазонами)

Live Editor Apps

  • Vehicle Aero Debug: отображает различные значения, связанные с аэродинамикой; наиболее полезным является прижимная сила на осях.
  • Beam Visualization: то же, что в Debug UI.
  • Flexbody Debug: очень полезен при отладке проблем с шипами и растяжками — позволяет видеть, какими нодами интерполируется каждая вершина модели, и какие вершины интерполирует каждая из нод Jbeam. На вкладке «All Flexmeshes» есть инструменты, которые могут автоматически находить потенциально проблемные области, где вершинам не хватает близлежащих нод, что может вызывать проблемы с шипами.
  • General Data: отображает некоторые основные полезные данные в одном месте.
  • Jbeam Picker: позволяет выбирать отдельные ноды и балки транспортного средства курсором мыши и отображать некоторые статические и динамические данные о них.
  • Lights Debug: показывает направления, в которых светят все источники света транспортного средства.
  • Mirrors Debug: позволяет тестировать параметры Jbeam зеркал в реальном времени с помощью ползунков.
  • Node Triangle Self Collision Detector: подсвечивает ноды и треугольники, находящиеся в пределах заданного расстояния друг от друга, если они потенциально могут столкнуться. Также выводит данные об этих нодах и треугольниках в консоль. Диапазон по умолчанию — 0.025 м, что означает коллизию при спауне. Рекомендуется выдерживать расстояния не менее 0.035 м для предотвращения коллизий при изгибе кузова.
  • Node Visualization: то же, что в Debug UI.
  • Powertrain Inspector: показывает статические и динамические данные Jbeam выбранного устройства трансмиссии.
  • Prop Transformer: очень полезная утилита, значительно упрощающая позиционирование пропов на транспортном средстве. Чтобы выбрать проп, нажмите кнопку «Pick Prop». На транспортном средстве появятся точки — исходные ноды пропов; между разными пропами в одной исходной точке можно переключаться колесом мыши. Затем с помощью кнопок внизу можно активировать 3D-гизмо, позволяющее задавать положение и поворот пропов. Наконец, значения из текстовых полей можно скопировать в Jbeam вручную или автоматически с помощью кнопки, если используется baseTranslationGlobal.
  • Raw Vehicle Data: показывает все данные транспортного средства из игрового движка.

Пропы

В игре есть ряд спауниемых пропов, которые помогают при настройке параметров Jbeam. Чтобы видеть их, обязательно установите флажок Include Auxiliary Debug Content в селекторе транспортных средств.

Roof Crusher Tester

Этот проп используется для воспроизведения теста прочности крыши IIHS с целью достижения реалистичной деформации крыши на транспортных средствах.

Инструкция по использованию

  • Откройте JBeam-файл кузова тестируемого транспортного средства и добавьте тег {"fixed":true} к нодам пола.
  • Заспауньте Roof Crusher Tester на карте Grid, Small, Pure. Его можно найти ближе к концу списка при включённой опции «Include Auxiliary Debug Content». Заспауньте тестируемый автомобиль и переместите его в центр краша через World Editor. Используйте базовую модель автомобиля.
  • Нажмите на синий шар краша и дождитесь, пока он коснётся крыши транспортного средства. Если он не касается крыши должным образом, возможно, потребуется снять некоторые детали или воспользоваться красным и зелёным шарами для корректировки положения. Для низких транспортных средств также можно поднять автомобиль через World Editor.
  • Нажмите на синий шар ещё раз и дождитесь UI-сообщения с измеренным усилием. Значение также можно найти в консоли. Если зеркало транспортного средства помешало тесту (может случаться на модах, основанных на старых транспортных средствах), повторите тест с удалённым зеркалом.
  • Вычислите рейтинг прочности крыши транспортного средства, разделив результат теста на 9.81.
  • Вычислите ожидаемый рейтинг прочности крыши транспортного средства, умножив его вес на рейтинг теста прочности крыши IIHS. Используйте ближайший по характеристикам автомобиль, если не удаётся найти точную модель.
  • Автомобили до 2013 года обычно имеют рейтинг около 1.5; у более дешёвых он будет ниже — у очень дешёвых может опускаться до 1.00, у дорогих — достигать около 1.75. Большинство автомобилей 1980–2005 годов будут иметь рейтинг выше 1.5, а более поздние могут превышать 3. В 2013 году стандарты резко возросли — для таких автомобилей рейтинг составит около 4. С 2016 года рейтинг всегда не ниже 4, а для очень тяжёлых или дорогих автомобилей может превышать 5. SUV могут быть значительно прочнее или слабее обычных автомобилей в зависимости от конструкции кузова.
  • Корректируйте Jbeam до достижения разумного результата (примерно вдвое лучший рейтинг по сравнению с реальным — это нормально). Возможно, придётся переработать Jbeam для крыши и каждой стойки, чтобы у каждой были собственные риджифайеры, соединённые между собой легко деформируемыми балками. Возможно, потребуется протестировать, насколько стекло влияет на результаты, и соответственно ослабить его. Также может потребоваться настройка или удаление ломких балок кузова.
  • Не забудьте перенастроить прочность пола после этого, чтобы сохранить прочность транспортного средства при лобовом ударе.

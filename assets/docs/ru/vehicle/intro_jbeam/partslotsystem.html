Одной из возможностей, которая делает модификации автомобилей в BeamNG реальностью, является система компонентов и слотов.

Если вы когда-либо пользовались селектором деталей, вы знаете, что там существует множество компонентов — зачастую с несколькими вариантами — разбитых по уровням, причём доступность тех или иных вариантов детали зависит от того, что было выбрано на предыдущем уровне.

Как определяются слоты и компоненты

Всё, что вы видите в селекторе деталей, определяется в jbeam-файле каждого компонента. Вот небольшой пример:

"autobello_suspension_F": {             //Internal name of the component
    "information":{
        "authors":"BeamNG",             //Name in the part selectior
        "name":"Trailing Link Front Suspension",
        "value":400,
    },
    "slotType":"autobello_suspension_F",//Slot Type
    "slots":[                           //Slots
        ["type", "default", "description"],
        ["autobello_brake_F","autobello_brake_F", "Front Brakes"],
        ["wheel_F_4","steelwheel_11b_13x5_F", "Front Wheels"],
        ["autobello_spring_F","autobello_spring_F", "Front Springs"],
        ["autobello_steering","autobello_steering", "Steering"],
    ],
},

Каждая деталь имеет внутреннее имя, задаваемое самой первой строкой её jbeam-файла. Это имя используется игрой для обращения к данной детали, однако пользователю оно не отображается. Важно, чтобы это имя было уникальным — иначе возможны конфликты, при которых одни детали перезаписывают другие.

Раздел информации содержит основные сведения о компоненте, включая имя, которое будет отображаться в селекторе деталей.

Каждая деталь также имеет slotType , определяющий, в какое место транспортного средства она устанавливается.

Тип слота позволяет иметь несколько деталей с одним и тем же типом слота, выступающих в качестве альтернатив: все детали с одинаковым «типом слота» отображаются в селекторе деталей как различные варианты одной и той же детали.

Уникальный тип слота — main — обозначает базовый компонент транспортного средства. Он идентифицирует компонент как «корневую» деталь и загружается игрой первым. В каждом автомобиле может быть только одна деталь с таким типом слота.

Раздел slots определяет, какие компоненты являются «дочерними» по отношению к данному компоненту, на основе их slotType. Если продолжить пример с передней подвеской, здесь можно встретить такие элементы, как передние тормоза, колёса, амортизаторы, рулевое управление и т. д.

Конечным результатом является дерево выбора деталей, в котором деталь «main» находится на вершине, а ниже располагаются несколько уровней «дочерних» компонентов.

Как работают конфигурации

Наверняка вы замечали, что в игре у каждого автомобиля есть готовые конфигурации: несколько комплектаций, внедорожные и гоночные версии и т. д.

Принцип их работы довольно прост и основан на так называемых «файлах конфигурации». Ниже — небольшой пример того, что можно найти, открыв файл конфигурации.

{
"parts":{
  "coupe_enginemounts":"coupe_enginemounts_heavy",
  "coupe_oilpan_I4":"coupe_oilpan_I4_race",
  "coupe_engine_ecu":"coupe_engine_ecu_race",
  "coupe_engine_internals":"coupe_engine_internals_stage2",
  "coupe_taillight_L":"coupe_taillight_L",
  "paint_design":"coupe_skin_nomi",
  ...
}
"vars":{
  "$trackoffset_R":0.02,
  "$trackoffset_F":0.025,
  "$lsdpreload_R":125,
  ...
},

Файлы конфигурации представляют собой json-таблицу, содержащую список различных слотов и компонент, который должен использоваться в каждом слоте. Если слот отсутствует в файле конфигурации, будет использован компонент по умолчанию.

Файлы конфигурации также определяют значения, которые будут использоваться для различных переменных .

Помимо файлов конфигурации, нередко встречается и info-файл, описывающий различные сведения, которые отображаются для данного варианта в селекторе автомобилей, а также содержащий миниатюру.

Продвинутый пример: использование одного Jbeam-компонента для нескольких деталей

Грамотно применив некоторые продвинутые механизмы Jbeam — такие как переменные слотов и функции — можно создать 2 различных Jbeam-компонента, определённых одним и тем же кодом, и выбрать их одновременно. Для этого в родительских слотах необходима строковая переменная слота, например:

"slots": [
    ["type", "default", "description"],
    ["reusable_part_slot1", "", "Slot 1", {"nodeMove":{"x": 2,"y":0,"z":0}, "variables":{"$prefix":"a"}}],
    ["reusable_part_slot2", "", "Slot 2", {"nodeMove":{"x":-2,"y":0,"z":0}, "variables":{"$prefix":"b"}}],
],

Затем в самой переиспользуемой детали slotType должен быть таблицей, соответствующей обоим представленным типам, а каждая уникальная строка детали — например, имя группы и имя ноды — должна конкатенироваться с переменной-строкой, например:

"reusable_part": {
    "information":{
        "authors":"BeamNG",
        "name":"Reusable Part",
    },
    "slotType" : ["reusable_part_slot1", "reusable_part_slot2"],
    "nodes": [
        ["id", "posX", "posY", "posZ"],
        {"nodeWeight":1.0},
        {"collision":true},
        {"selfCollision":true},
        {"group":"$=$prefix..'_group'"},

        ["$=$prefix..'1'", 1,-1, 0],
        ["$=$prefix..'2'",-1,-1, 0],
        ["$=$prefix..'3'",-1, 1, 0],
        ["$=$prefix..'4'", 1, 1, 0],
        ["$=$prefix..'5'", 0, 0, 2],

        {"group":""},
    ],
    "beams": [
        ["id1:", "id2:"],
        {"beamPrecompression":1, "beamType":"|NORMAL", "beamLongBound":1, "beamShortBound":1},

        {"beamSpring":461000,"beamDamp":140},
        {"beamDeform":40000,"beamStrength":100000},

        ["$=$prefix..'1'", "$=$prefix..'2'"],
        ["$=$prefix..'1'", "$=$prefix..'3'"],
        ["$=$prefix..'1'", "$=$prefix..'4'"],
        ["$=$prefix..'2'", "$=$prefix..'3'"],
        ["$=$prefix..'2'", "$=$prefix..'4'"],
        ["$=$prefix..'3'", "$=$prefix..'4'"],

        ["$=$prefix..'1'", "$=$prefix..'5'"],
        ["$=$prefix..'2'", "$=$prefix..'5'"],
        ["$=$prefix..'3'", "$=$prefix..'5'"],
        ["$=$prefix..'4'", "$=$prefix..'5'"],
    ],
    "triangles": [
        ["id1:","id2:","id3:"],

        ["$=$prefix..'1'", "$=$prefix..'2'", "$=$prefix..'3'"],
        ["$=$prefix..'3'", "$=$prefix..'4'", "$=$prefix..'1'"],

        ["$=$prefix..'1'", "$=$prefix..'5'", "$=$prefix..'2'"],
        ["$=$prefix..'2'", "$=$prefix..'5'", "$=$prefix..'3'"],
        ["$=$prefix..'3'", "$=$prefix..'5'", "$=$prefix..'4'"],
        ["$=$prefix..'4'", "$=$prefix..'5'", "$=$prefix..'1'"],
    ],
},

Также можно определить дополнительные булевы переменные слотов и использовать их совместно с модификатором отключения для переключения различных строк Jbeam-файла в зависимости от slotType.

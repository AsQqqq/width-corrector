This feature is supported as of BeamNG.tech and .drive version 0.31 to latest.

Секция cargoStorage в Jbeam используется исключительно для игрового режима доставки в карьере. В настоящее время она реализована совместно с грузами типа «Cargo Load Box» для транспортных средств.

Любая деталь, используемая в режиме доставки в карьере, должна содержать "cargo_load_" в своём внутреннем имени в Jbeam, чтобы отображаться в правильной категории при покупке.

Обязательные аргументы

string
type
Идентификатор группы нод/балок, свойства которых будут изменяться при увеличении массы груза
string
type
Unnamed Container
default
Название контейнера, отображаемое в интерфейсе доставки
table
type
Типы груза, перевозимого в данном контейнере
Поддерживаемые в настоящее время типы: "parcel", "fluid", "dryBulk".
number
type
Объём груза, вмещаемого контейнером (л)

Для типа "parcel" значение должно быть степенью числа 2 из соображений баланса.

Часто используемые значения:

  • 1 для Cargo Load Box S
  • 4 для Cargo Load Box M
  • 16 для Cargo Load Box L
  • 128 для Cargo Load Box XL
  • 800 для Fluid Tank XL и Aggregate Box XL

Примеры

Посылочные ящики

"cargoStorage":[
    ["groupId","name","cargoTypes","capacity"],
    ["roofBarsBoxA","Roof Bars Box A",["parcel"], 16],
    ["roofBarsBoxB","Roof Bars Box B",["parcel"], 16],
],

Жидкостной бак

"cargoStorage":[
    ["groupId", "name","cargoTypes", "capacity"],
    ["RoofbarTank1", "Roffbar Tank",["fluid"], 800],
],

Ящик для сыпучих грузов

"cargoStorage":[
    ["groupId", "name","cargoTypes", "capacity"],
    ["roofbarAggregate1", "Roofbar Aggregate Box",["dryBulk"], 800],
],

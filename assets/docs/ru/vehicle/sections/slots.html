Секции slot и slotType используются для определения того, где компонент располагается в дереве деталей транспортного средства. Для получения дополнительной информации об основах системы деталей и компонентов вы можете прочитать введение в систему деталей/слотов .

Каждая деталь также имеет slotType, который определяет, где в транспортном средстве она устанавливается, позволяя нескольким деталям выступать в качестве альтернатив друг другу. Один уникальный тип слота — main, который относится к «корневому» компоненту транспортного средства.

Секция slots определяет, какие компоненты являются children данного компонента, основываясь на их slotType.

Вы можете использовать необязательные аргументы в секции Slots для передачи аргументов всем секциям выбранной детали (и всем их дочерним деталям, при условии что значение не изменяется ими). Это часто используется с аргументами nodeOffset и nodeMove, которые применяют смещения к нодам, флексбоди и пропсам, фактически перемещая деталь, установленную в целевой слот, на заданное расстояние.

Переменные слота

Начиная с обновления 0.32, вы можете передавать значения переменных в слот через секцию Slots. Этот механизм называется переменными слота (Slot Variables).

Значения будут переданы в деталь, установленную в слот, и все её дочерние детали. Они будут работать так же, как значения переменных, заданные в меню настройки, но не будут редактируемы пользователем. Строковые значения также могут передаваться и затем конкатенироваться с другими строками, например в качестве префиксов имён нод. Булевые значения тоже поддерживаются, что в сочетании с модификатором отключения (Disable Modifier) позволяет включать и отключать секции Jbeam в зависимости от слота, в котором находится деталь.

Это позволяет создавать компоненты, которые легче переиспользовать: вместо того чтобы делать несколько отдельных вариантов с небольшими изменениями, изменения можно контролировать через переменные.

Версия 0.38 расширяет концепцию переменных слота, добавляя специальные переменные пространства имён (Namespace Variables) . Это 2 специфические переменные с именами $prefix и $suffix, используемые для сокращения синтаксиса.

Часто передаваемые аргументы

Это аргументы секций Nodes, Flexbodies и Props, которые почти исключительно используются в качестве необязательных аргументов в секции Slots для передачи их значений в эти секции. Они влияют на данную деталь и все её дочерние детали до тех пор, пока значение не будет сброшено обратно до 0.

Ко всем именам аргументов ниже можно добавить суффикс из одной цифры (1–9), определяющий порядок, например:

"nodeRotate1": {"x": 0, "y": 0, "z": 45}
"nodeMove2": {"x": 0, "y": 0, "z": 0.2},
"nodeOffset3": {"x": 0.1, "y": 0, "z":0},
"nodeRotate4": {"x": 5, "y": 0, "z": 0}

Операции с меньшим порядковым номером выполняются первыми. Отсутствие номера эквивалентно порядку 0. Порядок преобразований нод с одинаковым суффиксом: nodeRotate, nodeOffset, nodeMove. При использовании одного и того же суффикса (или без суффикса) дважды — значение в дочернем слоте перезапишет значение из родительского слота.

Преобразования нод являются интринсическими, то есть трансляции и повороты применяются вокруг осей ранее преобразованной системы координат. Например, "nodeRotate1": {"x": 0, "y": 0, "z": 45} повернёт систему координат вокруг оси Z на 45 градусов. Последующий "nodeRotate2": {"x": 15, "y": 0, "z": 0} повернёт новую ориентированную систему координат вокруг её оси X на 15 градусов.

dictionary
type
{"x":0, "y":0, "z":0}
default
Смещает все ноды, флексбоди и пропсы по осям YZ, а по оси X — в зеркальном порядке (м)
В большинстве случаев взаимозаменяем с nodeMove. В основном необходим при работе с колёсами, где смещение по X равномерно отдаляет/приближает ноды, флексбоди и пропсы с каждой стороны от центральной линии. Для флексбоди центр определяется как нулевое значение по оси X в аргументе "pos", что является значением по умолчанию. Это означает, что nodeOffset не будет корректно работать с флексбоди, у которых не задан аргумент pos.
dictionary
type
{"x":0, "y":0, "z":0}
default
Смещает все ноды, флексбоди и пропсы по осям XYZ (м)
В большинстве случаев взаимозаменяем с nodeOffset. Этот модификатор перемещает ноды, флексбоди и пропсы влево или вправо по оси X без зеркального смещения.

dictionary
type
{"x":0, "y":0, "z":0, "px":0, "py":0, "pz":0}
default
Поворачивает все ноды, флексбоди и пропсы на x, y, z градусов по осям XYZ вокруг центральной точки px, py, pz

Если точка вращения не задана, поворот выполняется вокруг начала координат, например "nodeRotate":{"x":-45, "y":0, "z":0} будет работать так же, как "nodeRotate":{"x":-45, "y":0, "z":0, "px":0, "py":0, "pz":0}.

Во избежание проблем, связанных с шарнирным замком (gimbal lock) , при повороте более чем вокруг одной оси лучше указывать только один поворот оси в каждом экземпляре nodeRotate. Например, вместо {"nodeRotate":{"x":90, "y":45, "z":0}} лучше использовать: {"nodeRotate1":{"x":90, "y":0, "z":0}, "nodeRotate2":{"x":0, "y":45, "z":0}}.

Простой пример

Компонент с единственным слотом без модификаторов.

"autobello_spring_F": {
    "slotType":"autobello_spring_F",
    "slots":[
        ["type", "default", "description"],
        ["autobello_shock_F","autobello_shock_F", "Front Shocks"],
    ],
},

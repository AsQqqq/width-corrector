Откуда BeamNG получила своё название — балки (beams).

Это соединения пружина/демпфер между нодами, составляющие основную структуру любого транспортного средства в игре и обеспечивающие деформацию и разрушение там, где это необходимо.

Существует несколько типов балок для самых разных целей, которые подробно описаны в соответствующих примерах. Стандартные балки работают как простые пружины с демпфированием, способны деформироваться и ломаться при необходимости и применяются в большинстве случаев.

Все аргументы, применимые к стандартным балкам, можно использовать с каждым из специализированных типов балок.

Необязательные аргументы

Секция Beams содержит множество необязательных аргументов. Мы разобьём их на категории, расположенные от наиболее часто используемых к наименее часто используемым.

Основные

Это наиболее базовые аргументы, используемые повсеместно в Jbeam-файлах и необходимые для корректной симуляции. Простое объяснение смотрите на вводной странице.

string
type
NORMAL
default
Тип балки определяет её поведение при сжатии и растяжении.

Возможные значения:

  • NORMAL — Балки общего назначения, не обладают специальными свойствами, используются в большинстве конструкций.
  • SUPPORT — Второй по распространённости тип балок. Они сопротивляются силам только при сжатии, но не при растяжении. При предкомпрессии начинают сопротивляться силам, когда достигают предкомпрессированной длины. Могут автоматически ломаться при достижении заданной длины. Обычно используются при соединении нескольких деталей: они не служат крепёжными элементами, а выполняют роль ограничителей, не позволяя нодам одной детали проникать внутрь поверхности другой.
  • HYDRO — Определяются в отдельной секции, но наследуют все свойства обычных балок; гидравлические элементы могут изменять длину по команде — как правило, используются для рулевых реек. Также могут задавать угол блокировки рулевого колеса автомобиля.
  • ANISOTROPIC — Имеют отдельные значения beamSpring и beamDamp для растяжения и для сжатия, могут задавать переходную зону между двумя вариантами поведения, а также автоматически ломаться при превышении определённого соотношения длин. Используются для моделирования конструкций, которые жёстче при растяжении, чем при сжатии, — например, шин (где генерируются автоматически), тросов или мягких верхов кузова.
  • BOUNDED — Более сложные, чем анизотропные балки; могут иметь разные значения beamSpring и beamDamp не только для сжатия и растяжения, но и для малых и больших скоростей сжатия и растяжения. Диапазоны сжатия и растяжения задаются как соотношение или в метрах, переходная зона может быть изменена. Вычислительно затратны и должны применяться только в компонентах подвески — например, в амортизаторах и буферах отбоя.
  • LBEAM — Задаются тремя идентификаторами нод вместо двух; сопротивляются изменению угла между двумя балками, образованными этими нодами, а не расстоянию между двумя нодами. Обеспечивают высокую боковую жёсткость при малом сопротивлении сжатию. Используются в шинах (генерируются автоматически) и листовых рессорах.
  • PRESSURED — Специальные балки для моделирования пневматических цилиндров. Их силы обратно пропорциональны текущей длине. Можно задать начальное давление в Па или PSI, максимальное давление, при превышении которого балка ломается, площадь поперечного сечения цилиндра и коэффициент объёма. Используются в шинах (генерируются автоматически) и прогрессивных пневматических пружинах.
  • BROKEN — Сломанные балки перестают сопротивляться силам. Этот тип нельзя задать непосредственно в Jbeam.

Разрушаемые

Используются только для разрушаемых балок — тех, у которых beamStrength отличается от "FLT_MAX". Такие балки применяются для имитации пластика или соединения различных компонентов транспортного средства.

string/table
type
Балка автоматически ломается, когда ломается другая балка из той же breakGroup

Если аргумент является строкой, балка принадлежит одной указанной breakGroup. Это стандартный случай.

["a","b", {"breakGroup":"group_1"}],

Если аргумент является таблицей, балка принадлежит ко всем указанным breakGroup. Это означает, что она автоматически ломается при поломке любой балки в любой из этих групп, а также инициирует поломку группы сразу в обеих группах. На практике это используется крайне редко и не поддерживается режимами отладки визуализации балок.

["a","b", {"breakGroup":["group_1","group_2"]}],
number
type
Задаёт поведение breakGroup
Если задано значение 0, эта балка будет ломать другие балки в breakGroup. Если задано значение 1, эта балка НЕ будет ломать другие в breakGroup, но будет ломаться вместе с группой.
boolean
type
false
default
Отключает разрушение меша

Разрушение меша позволяет динамически удалять полигоны флексбоди-меша при поломке лежащей в его основе балки. Это позволяет избежать чрезмерного растягивания мешей, когда часть назначенной им jbeam-структуры разрушается.

Отключение разрушения меша полезно, когда разрушаемая балка выполняет лишь роль структурного элемента, например в пассажирских салонах.

boolean
type
false
default
Отключает разрушение треугольников

Разрушение треугольников позволяет динамически переводить треугольники транспортного средства в состояние без коллизии и аэродинамики при поломке лежащей в их основе балки. Это применяется, когда треугольник соединяет два компонента, которые могут разделиться, — например, кузов и раму, — а также для разрушаемого стекла, чтобы динамически образовывать в нём физическое отверстие.

Отключение разрушения треугольников полезно, когда разрушаемая балка выполняет лишь роль структурного элемента, например в пассажирских салонах.

boolean
type
false
default
Отключает ошибки при отсутствии одной из нод балки

Используется в случаях, когда одна из нод, составляющих балку, расположена в необязательном компоненте.

Также скрывает предупреждения о дублирующихся балках.

Ограничители деформации

Используются для ограничения деформации балки. DeformLimitExpansion является наиболее часто используемым из них и применяется в большинстве Jbeam-файлов.

number
type
Ограничивает деформацию балки при сжатии в долях от исходной длины
number
type
deformLimit
default
Ограничивает деформацию балки при растяжении в долях от исходной длины
Используется для предотвращения чрезмерного растяжения.
number
type
Ограничивает прирост beamDeform этим значением (Н)
beamDeform возрастает по мере деформации балки; это свойство ограничивает деформацию. Полезно для моделирования трения в определённых типах подвески.

Группы деформации

Группы деформации используются для инициирования действия при деформации балки. Чаще всего применяются только для визуального повреждения стёкол/фар и повреждения трансмиссии.

string/table
type
Указывает, к какой группе деформации принадлежит данная балка.

Используется для активации групп деформации флексбоди . Также применяется для моделирования повреждений некоторых компонентов трансмиссии.

Если аргумент является строкой, балка принадлежит одной указанной deformGroup. Это стандартный случай.

["a","b", {"deformGroup":"group_1"}],

Если аргумент является таблицей, балка принадлежит ко всем указанным deformGroup. Это означает, что она одновременно запускает несколько видов поведения при деформации. На практике используется крайне редко и не поддерживается режимами отладки визуализации балок.

["a","b", {"deformGroup":["group_1","group_2"]}],

Балка с deformGroup не будет считаться дублирующейся в консоли. Это обусловлено распространённым случаем использования хрупких дублирующихся балок для имитации разрушаемого стекла.

Если балка имеет одновременно breakGroup и deformGroup, её поломка генерирует частицы стекла или дерева в зависимости от nodeMaterial обеих её нод. Это используется для стёкол транспортных средств и деревянных реквизитов, например пианино.

number
type
Уровень деформации, при превышении которого активируется deformGroup.

Типичные значения — очень малые числа, меньше 0.1. Если балка сократилась или удлинилась более чем на это значение, deformGroup будет активирована.

Наличие этого свойства у балки создаёт триггер деформации, при котором при каждой деформации любой балки с этим свойством выполняется дополнительный код. Это может быть ресурсоёмко, поэтому использовать данный параметр следует только там, где это необходимо.

Отладка

Предназначены для кратковременных целей отладки и должны быть удалены из финальной версии конструкции.

dictionary
type
{"radius":0,"col":"#00000000"}
default
Выделяет балку с указанными радиусом и цветом при включённом режиме отладки визуализации балок.

Таблица состоит из 3 элементов, включая 2 обязательных и 1 необязательный: "radius" — число, задающее радиус выделения в метрах, а "col" — либо строка с шестнадцатеричным значением цвета, либо таблица со значениями RGB. Пример использования:

["a","b", {
    "highlight":{
        "radius":0.005,
        "col":"#ff0000ff"
    }
}],

Следующий вариант даёт тот же эффект:

["a","b", {
    "highlight":{
        "radius":0.005,
        "col":{
            "r":255,
            "g":0,
            "b":0,
            "a":255
        }
    }
}],

"length" — необязательный аргумент, который перезаписывает длину цилиндра, выделяющего балку, указанным значением в метрах. Цилиндр может быть длиннее или короче балки по умолчанию. Центр цилиндра всегда находится в центре балки. Этот аргумент доступен начиная с версии BeamNG 0.34 и выше.

["a","b", {
    "highlight":{
        "radius":0.005,
        "col":"#ff0000ff",
        "length":0.1
    }
}],

Props — это меши, которые не деформируются, но могут анимироваться. Они очень полезны для таких элементов, как рулевые колёса, рычаги переключения передач, стрелки приборов, педали и другие детали, которые лучше анимировать, чем симулировать. Их также можно использовать для создания прожекторов, связанных с системой glowMap/electrics, чтобы сделать работающие фары.

По умолчанию меши будут расположены так же, как в вашей программе моделирования, однако их необходимо повернуть в нужное положение с помощью аргумента базового вращения. Подробнее см. рекомендации по моделированию

Система осей пропа

Каждый проп использует собственную систему отсчёта, состоящую из 3 нод. Это позволяет пропу перемещаться вместе с этими нодами — например, при столкновениях или для таких применений, как подвижные фары.

Ноды, выбранные для аргументов idRef, idX и idY, должны образовывать прямоугольный треугольник — аналогично refnodes . Эти ноды образуют оси X и Y системы координат пропа. Ось Z создаётся на их основе.

Например, пропы приборной панели Pigeon используют ноды "dsh2", "dsh2r", "dsh3" в качестве ref-, x- и y-нод соответственно: dsh2 находится в центре панели, dsh2r — справа от неё, dsh3 — снизу.

Это означает, что положительные значения по оси Y перемещают пропы вниз, а по оси Z — назад.

Все перемещения при трансляции выполняются в этой системе координат, а не в системе координат транспортного средства, как у нод и других смещений. Вращение выполняется вокруг каждой оси системы координат меша, с центром вращения в начале координат пропа, как описано в рекомендациях по моделированию

Функции и ограничения

Движение пропа задаётся функцией, использующей одно из значений electrics транспортного средства, вместе со значениями вращения и трансляции.

Значения вращения и трансляции определяют, на сколько градусов повернётся проп и на сколько метров сместится при увеличении значения функции на единицу. Например, если датчик температуры имеет следующий параметр вращения, он будет поворачиваться на 1 градус вокруг оси x при каждом изменении температуры на один градус.

{"x":5, "y":0, "z":0}

Угол поворота можно рассчитать, разделив используемый диапазон значения electrics на угол между минимальным и максимальным положениями.

Значения min и max используются для ограничения движения пропа. Например, для имитации «упоров» на шкалах приборов многих автомобилей, не позволяющих стрелке выходить за определённый диапазон. Они задаются как значения функции, а не как угол или трансляция.

Значение offset работает аналогично min и max, добавляя статический сдвиг к показанию прибора. Это используется для компенсации нулевой точки, которая не совпадает с начальным положением стрелки.

Multiplier делает именно то, что написано, однако он редко используется — вместо этого изменяют значения вращения и трансляции.

Возьмём в качестве примера тахометр D-Series. Он имеет диапазон отклонения 180 градусов при диапазоне значений 6000 об/мин, что даёт функцию вращения: 180 / 6000 = 0,03 градуса/об/мин. Используем отрицательное значение, поскольку стрелка движется по часовой стрелке.

Чтобы шкала не выходила за 6000 об/мин, max устанавливается равным 6000, min — 0. Для датчика температуры min и max могут быть, например, 60 и 120. Для offset: в данном случае при offset равном 0 стрелка указывает на 4000, то есть offset должен быть -4000, чтобы стрелка указывала на правильное значение. Если стрелка не выровнена с плоскостью приборной панели, параметр baseRotation используется для поворота пропа и его оси вращения.

(...)"baseRotation",           "rotation",                 "translation",       "min", "max", "offset", "multiplier"
(...){"x":160 "y":180, "z":0}, {"x":0, "y":0, "z":-0.03}, {"x":0, "y":0, "z":0}, 0,     6000, -4000,     1

Фары

Фары в beamNG состоят из двух основных элементов. Блик (flare) — шар света у излучателя, представляющий саму лампочку.

Второй элемент — «cookie», представляющий собой свет, отражённый от поверхностей «лучом» фары.

Фары часто используются в сочетании с glow map и эмиссионными текстурами, чтобы вся фара выглядела светящейся.

Фары включаются, как только значение их функции становится больше 0, и могут быть связаны с группами деформации и поломки, чтобы отключаться при повреждении физического источника света.

Фары могут плавно переходить между включённым и выключенным состояниями с помощью линейной шкалы, если в качестве значения функции используются пользовательские значения, заданные в разделе electrics . Для изменения эффекта перехода предусмотрены специальные аргументы:

(...){"lightScaling":{"brightnessMinInput":0, "brightnessMaxInput":1, "flareScaleMinInput":0.6, "flareScaleMaxInput":1, "lightColorOffsetRed":0,"lightColorOffsetGreen":60, "lightColorOffsetBlue":80}}

Однако, как правило, их не требуется задавать, поскольку значения по умолчанию уже дают убедительный эффект.

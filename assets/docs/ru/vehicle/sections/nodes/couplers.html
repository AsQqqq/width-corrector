This feature is supported as of BeamNG.tech and .drive version 0.6.0 to latest.

Система соединителей (coupler system) — это функция объединения двух нод. Соединённые ноды будут находиться в заданном диапазоне расстояний друг от друга и реагировать на одинаковые силы. До соединения ноды будут притягиваться друг к другу, как только расстояние между ними достигнет заданного значения. Система может работать как между нодами одного транспортного средства, так и между нодами двух разных машин — автоматически или по нажатию клавиши. Примеры использования: жёсткое крепление деталей кузова, фаркопы для прицепов, защёлки дверей, капотов, крышек багажника и задних бортов.

Для работы системы необходима нода со свойством couplerTag и другая нода со свойством Tag того же значения. В данном разделе мы будем называть их «нодой-соединителем» (coupler node) и «нодой-тегом» (tag node) соответственно, чтобы легче понять, какие свойства относятся к какой ноде. При соединении автомобиля с прицепом конец фаркопа на машине должен быть нодой-соединителем, а конец гнезда фаркопа на прицепе — нодой-тегом. При соединении двух разных деталей одного транспортного средства порядок не важен.

Функциональность

Система начинает работать, когда нода-соединитель переходит в состояние Active. Активные ноды-соединители и соответствующие им ноды-теги начинают притягиваться друг к другу, после чего объединяются в одну; сила в обоих направлениях одинакова. Подключённая нода-соединитель более не считается Active. Сломанные (Broken) ноды-соединители не могут быть переведены в состояние Active никаким способом и не будут снова подключаться к нодам-тегам.

При соединении двух нод одного транспортного средства в момент спауна лучше, чтобы они имели одинаковые координаты. Одна нода-соединитель может быть соединена только с одной нодой-тегом одновременно, однако её можно отсоединить и затем подключить к той же или другой ноде-тегу. Аналогично, нода-тег может быть соединена только с одной нодой-соединителем, но после отключения может быть подключена к той же или другой ноде-соединителю.

Системой соединителей также можно управлять через Advanced Coupler Control — эта тема слишком сложна для данного раздела документации и здесь не рассматривается. Ещё одна система управления соединителями — Couplings, используемая на седельных тягачах и прицепах.

Для самой системы нет разницы (кроме незначительных отличий производительности, которые вас не должны беспокоить) — принадлежат ли ноды одному транспортному средству или разным. Тем не менее существуют различные подсистемы, которые работают только при соединении двух разных транспортных средств, например масштабирование камеры при подключении прицепа.

Система переключения соединителей (Coupler Toggle System)

Тесно связанная с основной система, позволяющая фактически прицепить прицеп к автомобилю, — это Coupler Toggle System. Она активируется нажатием назначенной клавиши (по умолчанию L) или через радиальное меню. При использовании система сначала проверяет, соединены ли какие-либо из относящихся к ней нод с другими нодами. Если да — она их отсоединяет; если нет — переводит те, что не находятся в состоянии Broken, в состояние Active и включает визуализацию нод-соединителей и нод-тегов с помощью цветных шаров; одинаковые теги и couplerTags отображаются одним цветом.

Нода является релевантной для Coupler Toggle System, если у неё есть свойство couplerTag, её состояние не равно Broken и у неё нет свойства couplerLock или couplerWeld. Это означает, что только ноды, называемые в данном разделе нодами-соединителями, могут быть релевантными для этой системы.

Вы можете активировать Coupler Toggle System с помощью триггера . Если вы хотите активировать её для всех релевантных соединителей, используйте "beamstate.toggleCouplers()" без аргументов в качестве функции. Однако вы также можете передать аргументы для конкретной функциональности, например:

"beamstate.toggleCouplers('pintle', true, false, true)",

Нода-соединитель

string
type
Тег ноды-тега, к которой будет подключаться данная нода-соединитель

Часто используемые теги для сцепки разных транспортных средств:

  • "tow_hitch" — для стандартных автомобильных фаркопов и прицепов
  • "gooseneck_hitch" — для фаркопов типа гусиная шея на пикапах и прицепах
  • "fifthwheel_v2" — для полуприцепов (устаревший тег для старых модов: "fifthwheel")
  • "pintle" — для крюковых сцепок (pintle hitch) и переходных тележек прицепа
  • Теги для сцепки морских контейнеров: "cp1_h" — нижний горизонтальный, "cp2_h" — верхний горизонтальный, "cp_v" — вертикальный

Не рекомендуется использовать эти теги для соединения деталей одного транспортного средства.

number
type
Расстояние (в метрах), на котором данная нода будет пытаться соединиться с другой нодой, находясь в состоянии Active
Чаще всего используется для соединения автомобилей и прицепов. Большие значения делают выравнивание автомобиля/прицепа менее требовательным. Это свойство не обязательно использовать, если вместо него применяется couplerStartRadius.
number
type
То же, что и выше; кроме того, использование этого свойства переводит соединитель в состояние Active после спауна

Это означает, что ноды, находящиеся в данном радиусе друг от друга, попытаются соединиться сразу после спауна транспортного средства, без необходимости их переключения. Если это свойство не используется, Active не будет состоянием по умолчанию, и ноды потребуют переключения для соединения.

Чаще всего используется для соединения деталей одного транспортного средства, а не прицепов. Рекомендуется задавать очень малое значение, как и расстояние между деталями при спауне, которое в идеале должно быть равно 0, чтобы сразу после спауна не происходило значительного смещения.

boolean

false

Это означает, что нода, не активная при спауне, может стать активной только через Lua-код, не связанный с Coupler Toggle System. Аналогично, нода, активная при спауне (использующая свойство couplerStartRadius), может отключиться только при переходе в состояние Broken или через Lua-код.

Это полезно для систем, где соединитель должен блокироваться и разблокироваться по запросу, но не должен быть связан с Coupler Toggle System, а использование advanced coupler control было бы избыточным. Хороший пример — блокируемый задний борт самосвала.

Простой пример

Соединение двух нод одного транспортного средства.

Нода-соединитель:

["hd4l4",  1.15, -1.26, 1.20, {"couplerTag":"hood_side_latch_L","breakGroup":"hood_side_latch_L","couplerStrength":12000,"couplerRadius":0.01,"couplerStartRadius":0.01,"couplerWeld":true}],

Нода-тег:

["sd1l",  0.85, -1.12, 1.41, {"tag":"hood_side_latch_L","breakGroup":"hood_side_latch_L"}],

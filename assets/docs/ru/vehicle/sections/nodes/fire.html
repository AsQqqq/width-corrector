This feature is supported as of BeamNG.tech and .drive version 0.5.0 to latest.

Симуляция огня — это опциональная функция нод, позволяющая некоторым нодам загораться.

Чтобы включить симуляцию огня для ноды, ей необходимо задать температуру вспышки, соответствующую температуре самовоспламенения.

Ноды могут передавать тепло соседним нодам через теплопроводность, если она включена, или через излучение, когда нода горит.

Ноды колёс также связаны с этой системой: огонь нагревает их, что приводит к перегреву тормозов и потенциально к взрыву шин.

Аргументы нод

number
type
Температура (°C), при которой нода загорается.
В документации это часто называется температурой самовоспламенения.
number
type
tEnv
default
Начальная температура (°C) ноды.
Использование значения "thermals" в качестве baseTemp привяжет температуру ноды к температуре двигателя.
number
type
flashPoint
default
Температура (°C), при которой нода начинает дымиться.
number
type
Скорость, с которой нода сжигает своё топливо.
Фактическая скорость горения также зависит от количества оставшейся энергии и текущей температуры ноды.
number
type
0
default
Максимальное расстояние (в метрах), на которое нода передаёт тепло другим нодам через теплопроводность.
number
type
0
default
Количество энергии от столкновений, передаваемой в нагрев ноды.
number
type
0
default
Количество энергии в ноде (Дж).
number
type
1
default
Удельная теплоёмкость ноды в Дж/кг/°C.
Вместе с массой ноды этот параметр влияет на скорость её нагрева.
number
type
Процент испарения, выше которого нода считается испарившейся.

«Испарившаяся» нода будет иметь более низкую температуру вспышки и значительно более высокую скорость горения до тех пор, пока снова не конденсируется.

Испарение происходит при разрушении балки-контейнера ноды.

string
type
Идентифицирует ноду как часть контейнера.
Это приведёт к тому, что ноды внутри контейнера будут «испаряться» на определённую величину.

Аргументы балок

string
type
Идентифицирует балку как часть контейнера.

Разрушение балки вызовет разрыв контейнера, что позволит нодам внутри него «испариться».

Также будут созданы частицы вытекающего топлива, а пользователь получит уведомление через интерфейс о том, что топливный бак транспортного средства повреждён.

boolean
type
false
default
Отключает сообщение интерфейса о разрыве топливного бака.

При использовании совместно со свойством containerBeam интерфейс не будет оповещён о разрыве топливного бака.

Можно использовать для канистр и подобных объектов, когда нужны частицы вытекающего топлива при разрушении вне топливного бака транспортного средства.

Простой пример

Секция нод ковра салона транспортного средства, способных загораться

{"chemEnergy":1000,"burnRate":0.52,"flashPoint":300,"specHeat":0.8,"smokePoint":150,"selfIgnitionCoef":false},
["f1r", -0.316, -1.08, 0.233, {"nodeWeight":7.0, "group":["bastion_body","bastion_floor"]}],
["f1l",  0.316, -1.08, 0.233, {"nodeWeight":7.0, "group":["bastion_body","bastion_floor"]}],
{"chemEnergy":false,"burnRate":false,"flashPoint":false,"specHeat":false,"smokePoint":false,"selfIgnitionCoef":false},

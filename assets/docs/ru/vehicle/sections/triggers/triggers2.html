This feature is supported as of BeamNG.tech and .drive version 0.31 to latest.

Triggers2 — это улучшенная версия раздела Triggers. В сочетании с разделами triggerEventLinks2 и actionsEnabled она обеспечивает более естественную связь между триггерами и входными действиями. Эти разделы позволяют показывать и скрывать определённые действия в пользовательском интерфейсе в зависимости от деталей, установленных на транспортном средстве.

Карты взаимодействий

Triggers2 требует карты действий или карты взаимодействий для работы. Карта взаимодействий может быть определена в файле с суффиксом *.interaction.json. Формат файла совпадает с форматом input_actions.json , и он может заменить этот файл. Связанные с ним карты ввода с клавиатуры и контроллера продолжат работать после смены имени. Для целей модификации может существовать несколько файлов с суффиксом *.interaction.json. Также имеется общая карта взаимодействий в папке общего транспортного средства, содержащая некоторые типовые взаимодействия, например защёлки дверей. Пример файла карты взаимодействий:

{
    "fileversion": 2,
    "actions": {
        //the content here is the same as in input_actions.json
        "sleeper_door_R":      {"order":   50.1, "onDown":"controller.getControllerSafe('sleeperDoorRCoupler').toggleGroup()", "title": "vehicle.interaction.sleeperDoorR.toggle.title", "desc": "vehicle.interaction.sleeperDoorR.toggle.desc" },, 
        "sleeper_door_L":      {"order":   50.2, "onDown":"controller.getControllerSafe('sleeperDoorLCoupler').toggleGroup()", "title": "vehicle.interaction.sleeperDoorL.toggle.title", "desc": "vehicle.interaction.sleeperDoorL.toggle.desc" },
    }
}

Полный формат файла *.interaction.json и рабочий процесс см. в разделе Карты взаимодействий .

ActionsEnabled

Действия в файлах *.interaction.json, аналогично пользовательским картам действий, по умолчанию будут отключены (это означает, что подсказка, показывающая назначенную клавишу в правом верхнем углу экрана, отображаться не будет, однако код будет выполняться при нажатии этой клавиши). Их можно включить в Jbeam для детали транспортного средства, которая их использует, с помощью раздела ActionsEnabled. Он принимает только один аргумент — ID действия для включения. Пример использования:

"actionsEnabled": [
    ["id"],
    ["door_FR"],
],

Это особенно удобно для транспортных средств с большим количеством возможных действий, из которых одновременно доступна лишь небольшая часть в зависимости от выбранных деталей, — например, для седельного тягача с несколькими вариантами комплектации и внутренними органами управления.

    ["door_R_int", "d4r","d5r","d1r", "box", {"x":0.1, "y":0.03, "z":0.1}, {"x":-0, "y":0, "z":-0}, {"x":0, "y":0, "z":0}, {"x":0, "y":0, "z":0}, {"x":0.15, "y":-0.2, "z":0.0175}],

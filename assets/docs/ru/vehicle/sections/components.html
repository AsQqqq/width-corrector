This feature is supported as of BeamNG.tech and .drive version 0.32 to latest.

Компоненты — это универсальные хранилища данных в формате «ключ-значение», которые расширяют или перезаписывают код Jbeam. Они используют синтаксис словаря Json. Основное назначение компонентов — обеспечить совместимость между различными деталями автомобиля, которые можно выбирать независимо друг от друга.

"components": {
    "wheels": {
        "FL": {
            "diameter": 0.56,
            "something": 3,
        },
    },
    "cameras": {
        "MyDefault":{
            "distance":4,
            "distanceMin":3,
            "offset":{"x":0.0, "y":0.0, "z":0.59},
            "fov":65,
        }
    }
}

Использование

Компоненты используются в Jbeam посредством оператора $, внутри выражений — аналогично переменным. Оператор указывает на хранилище данных, которое будет использоваться в качестве значения.

"cameraExternal":"$= $components.cameras.MyDefault",

Расширение и переопределение

Деталь может расширять или перезаписывать данные компонента:

/// the main part
    "components": {
        "wheels": {
            "FL": {
                "diameter": 0.56,
                "something": 3,
            },
        },
    }

/// the sub part
    "components": {
        "wheels": {
            "RR": {
                "diameter": 0.1,
                "something": 1,
            },
            "FL": {
                "diameter": 2, // overwrites
                "somethingElse": 3,
            },
        }
    }

Область применения

Компоненты работают не только с табличными данными, но и с модификаторами:

"components": {
    "collisionEnabled": true,
},
"nodes" :[
   ["id", "posX", "posY", "posZ"],
   {"collision" :"$= $components.collisionEnabled"},
],

Их можно использовать в качестве строк таблицы:

"components": {
    ["node3", -0.90, -0.93, 0.23],
},
"nodes" :[
   ["id", "posX", "posY", "posZ"],
   ["node1", -0.90, -0.93, 0.23],
   ["node2", -0.33, -0.95, 0.23],
   "$= $components.node3",
   ["node4", 0.90, -0.93, 0.23],
   {"group":""},
],

Их также можно использовать внутри другого компонента, являющегося строкой таблицы:

"components": {
    "posZ": 0.23,
    ["node3", -0.90, -0.93, "$= $components.posZ"],
},
"nodes" :[
   ["id", "posX", "posY", "posZ"],
   ["node1", -0.90, -0.93, 0.23],
   ["node2", -0.33, -0.95, 0.23],
   "$= $components.node3",
   ["node4", 0.90, -0.93, 0.23],
   {"group":""},
],

Выражения

Компоненты работают и в функциях, и в выражениях. Синтаксис их использования там: $components.componentName:

"components": {
    "posZ": 0.03,
},
"nodes" :[
   ["id", "posX", "posY", "posZ"],
   ["node1", -0.90, -0.93, 0.23],
   ["node2", -0.33, -0.95, 0.23],
   ["node3", -0.90, -0.93, "$= $components.posZ + 0.20"],
   ["node4", 0.90, -0.93, 0.23],
   {"group":""},
],

Также можно обращаться к вложенным компонентам с помощью $components.parentComponent.childComponent:

"components": {
    "pos":{
        "X":-0.45,
        "Y":-0.93,
        "Z": 0.03,
    },
},
"nodes" :[
   ["id", "posX", "posY", "posZ"],
   ["node1", -0.90, -0.93, 0.23],
   ["node2", -0.33, -0.95, 0.23],
   ["node3", "$= $components.pos.X * 2", "$= $components.pos.Y", "$= $components.pos.Z + 0.20"],
   ["node4", 0.90, -0.93, 0.23],
   {"group":""},
],

Для вложенных компонентов можно применять операции vec3, quat и concat, например:

"$=concat($components.wheels.F, ',')"

Секция electrics

Секция компонентов может хранить внутри себя секцию electrics , которая будет обработана в штатном режиме. Это удобно для глубокого слияния таблиц, например при добавлении нового пользовательского значения или сглаживателя в другой части автомобиля.

"components": {
        "electrics": {
            "customValues": [
                ["electricsName", "electricsFunction"],
                ["police_flasher_filament","pwm(sign(electrics.lightbar), 0.5, 0.5) * electrics.electricalLoadCoef"],
            ]
            "smoothers": [
                ["electricsName", "smootherType", "params"],
                ["police_flasher_filament","temporalNonLinear", [4,4]],
            ]
        }
    }

Примеры

Несколько примеров того, как компоненты используются в стандартных автомобилях игры.

Базовые примеры

Когда деталь крепится к нодам другой детали, не связанной с ней напрямую, и имена нод могут различаться, имена нод для каждого варианта детали объявляются как компоненты и именно к ним обращается рассматриваемая деталь. Например, кузов-самосвал для седельного тягача крепится к обеим половинам рамы — передней и задней, — которые можно менять независимо. Поэтому для каждого варианта передней части рамы мы задаём имена граничных нод рамы, к которым будет крепиться кузов, в виде компонентов:

"components": {
    "frontFrameEndNodes": {
        "lastTopRight":     "ff5r"
        "lastTopLeft":      "ff5l"
        "lastBottomRight":  "ff15r"
        "lastBottomLeft":   "ff15l"
        "2ndLastTopRight":    "ff4r"
        "2ndLastTopLeft":     "ff4l"
        "2ndLastBottomRight": "ff14r"
        "2ndLastBottomLeft":  "ff14l"
    },
},

А в Jbeam-файле кузова-самосвала крепление выполняется через обращение к именам, хранящимся в компонентах, а не напрямую к именам нод:

"beams": [
    ["id1:", "id2:"],
    ["dbph1r", "$=$components.frontFrameEndNodes.lastTopRight"],
    ["dbph1r", "$=$components.frontFrameEndNodes.lastBottomRight"],
    ["dbph1l", "$=$components.frontFrameEndNodes.lastTopLeft"],
    ["dbph1l", "$=$components.frontFrameEndNodes.lastBottomLeft"],
],

Компоненты также можно использовать для смещения положения детали в зависимости от другой детали. В этом примере положение шины седельного тягача зависит от выбранного варианта колеса, поскольку разные варианты имеют различную глубину обода.

Продвинутый пример

Это наиболее сложный сценарий использования. Компоненты необходимы, когда визуальный меш, применяемый для детали, зависит от нескольких различных деталей автомобиля. Наглядный пример — карданный вал седельного тягача. Его длина и количество секций зависят от длины передней и задней частей рамы, положения переднего заднего ведущего моста и положения коробки передач, на которое может влиять размер двигателя. В этом случае сама деталь карданного вала определяет множество вариантов, а модификатор Disable выбирает варианты, которые не будут использоваться, на основе компонентов, определённых в соответствующих деталях. Имена самих мешей также могут быть переменными и храниться в компонентах — аналогично первому примеру.

Каждый содержит секцию Props, похожую на эту (упрощённую для наглядности), где вариант карданного вала выбирается в зависимости от компонентов, присутствующих в выбранных деталях:

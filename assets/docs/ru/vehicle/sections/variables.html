Переменные используются для всех настроек, к которым вы хотите предоставить пользователю доступ в меню «Тюнинг».

Как правило, сюда входят такие параметры, как объём топлива, множители тормозного усилия и силовой обратной связи, а также настройки развала-схождения колёс и подвески.

Использование переменных

Переменные можно использовать как самостоятельно, так и в составе функции. Их можно применять практически в любом месте jbeam, где в качестве значения используется число.

При самостоятельном использовании переменной просто подставьте её полное имя вместо числа, убедившись, что оно заключено в кавычки, как показано в примере ниже.

"beamLimitDampRebound":"$desert_damp_rebound_IFS",

При использовании переменной внутри функции она точно так же заменяет число, которое обычно стоит на этом месте. Функция должна начинаться с $=. Вся функция должна быть заключена в кавычки. Смотрите пример ниже — случай использования функции для изменения координаты Y ноды.

["fh2r", -0.65, "$=$caster_F*0.00175-1.2", 0.32],

Очень важно использовать синтаксис "$=Function", иначе парсер не вычислит функцию корректно.

Обязательные аргументы

string
type
Внутреннее имя переменной, которое будет использоваться при обращении к ней в остальной части вашего jbeam.
string
type
Тип переменной.
Единственный поддерживаемый тип — "range".
string
type
Единицы измерения, которые будут отображаться в меню тюнинга.
Если оставить это поле пустым, ползунок будет отображаться от 0% до 100%, а минимальное и максимальное значения не будут показаны в меню тюнинга.
string
type
Категория, под которой переменная будет отображаться в меню тюнинга.
Это может быть любое название на ваш выбор. При загрузке автомобиля игра сгруппирует вместе все переменные с одинаковой категорией.
number
type
Значение переменной по умолчанию.
number
type
Минимальное значение переменной.
number
type
Максимальное значение переменной.
number
type
Название переменной, которое будет отображаться на экране тюнинга.
string
type
Описание, которое будет показано при наведении курсора мыши на переменную.

Необязательные аргументы

string
type
Подкатегория, под которой переменная будет отображаться в меню тюнинга.

Это может быть любое название на ваш выбор. При загрузке автомобиля игра сгруппирует вместе все переменные с одинаковой категорией и подкатегорией.

Часто используется для разделения переменных передней и задней части.

number
type
Размер шага ползунка.
Если аргумент Unit оставить пустым, шаг будет равен 1% и изменить его здесь не получится.
number
type
Изменяет видимый минимум на ползунке.
Изменяет минимальное значение, отображаемое на ползунке. Фактическое минимальное значение, используемое внутри jbeam, по-прежнему будет определяться параметром "min".
number
type
Изменяет видимый максимум на ползунке.
Изменяет максимальное значение, отображаемое на ползунке. Фактическое максимальное значение, используемое внутри jbeam, по-прежнему будет определяться параметром "max".
boolean
type
false
default
Скрывает переменную в разделе «Тюнинг».
Полезно для передачи значения, которое будет использоваться в функции в другой части транспортного средства. Такое значение может быть переменной с одинаковыми значениями "default", "min" и "max".

Простой пример 

"variables": [
    ["name", "type", "unit", "category", "default", "min", "max", "title", "description"],
    ["$springheight_R", "range", "+m", "Suspension",  0, -0.04, 0.06, "Spring Height", "Raise or lower the suspension height"]
],

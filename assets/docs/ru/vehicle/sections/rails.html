Слайд-ноды — это узлы, перемещающиеся вдоль заранее заданных рельсов. Типичные применения включают рулевые рейки и некоторые элементы подвески, например стойки МакФерсона.

Slidenodes

Слайд-ноды требуют наличия уже созданных ноды и рельса, поскольку ссылаются на оба. Обратите внимание, что нода должна быть точно выровнена по рельсу, иначе при спавне, когда нода будет выравниваться, могут возникнуть проблемы.

"slidenodes": [ 
     ["id:", "railName", "attached", "fixToRail", "tolerance", "spring", "strength", "capStrength"],
 
     ["nodename", "Rail1", true, true, 0, 10000000, 100000000, 345435],
],

Rails

Рельсы — это пути для слайд-нод. Они состоят из одной или нескольких секций, определяемых с помощью нод.

Рельс может состоять из нескольких сегментов, и эти сегменты не обязаны быть прямолинейными.

"rails": {
      "Rail1":{"links:":["node1", "node2", "node3", "node4"], "broken:":[], "looped":false, "capped":true}
},

Обязательные аргументы

string
type
Имя рельса
Это первый аргумент, стоящий перед фигурными скобками. Совпадает с railName в разделе slidenodes.

table
type
Список нод, образующих рельс. Список должен содержать не менее двух нод.
"links:":["node1", "node2"],

"links:":["node1", "node2", "node3", "node4"],
table
type
[]
default
Какие ноды из таблицы links следует считать повреждёнными.
Устаревший аргумент, больше не используется.
boolean
type
false
default
Если ваш рельс представляет собой замкнутый контур, установите значение true, и нода сможет продолжать движение по кругу без остановки.
boolean
type
false
default
Наличие у рельса заглушек — ограничителей на его концах, не позволяющих ноде соскользнуть.
Заглушки — это начальная и конечная ноды рельса. Если capped=false, слайд-нода выскользнет из рельса, достигнув конца, и отсоединится. Если capped=true, нода остановится там, удерживаемая силой capStrength.

Rails2

Начиная с версии 0.38, раздел rails2 аналогичен разделу rails с теми же аргументами, но отформатирован иначе — в соответствии с остальными разделами Jbeam.

//these two are exactly the same
"rails": {
      "Rail1":{"links:":["node1","node2","node3"], "broken:":[], "looped":false, "capped":true},
      "Rail2":{"links:":["node4","node5"],         "broken:":[], "looped":false, "capped":true},
},
"rails2":[
    ["id",    "links:",                  "broken:","looped","capped"],
    ["Rail1", ["node1","node2","node3"], [],        false,   true   ],
    ["Rail2", ["node4","node5"],         [],        false,   true   ],
],

Преимущество этого синтаксиса перед стандартным rails состоит в возможности задавать идентификатор рельса динамически с помощью функций , так что одну деталь можно многократно использовать с разными именами рельсов (ранее это было возможно для всех разделов Jbeam, кроме rails).

Простой пример

Типичный пример использования рельса и слайд-ноды — рулевая рейка.

"rails": {
    "steeringrack":{
        "links:":["fh6r", "fh6l"], "broken:":[], "looped":false, "capped":true,
    },
},
"slidenodes": [
    ["id:", "railName", "attached", "fixToRail", "tolerance", "spring", "strength", "capStrength"],
    ["st1r", "steeringrack", true, true, 0.0, 18001000, "FLT_MAX", "FLT_MAX"],
    ["st1l", "steeringrack", true, true, 0.0, 18001000, "FLT_MAX", "FLT_MAX"],
],

Расширенный пример

Использование rails2 с функциями для динамических имён рельсов.

"rails2": [
    ["id", "links:", "broken:","looped","capped"],
    ["$= $prefix .. 'r1'", ["1r", "5r"], [], false, true],
    ["$= $prefix .. 'r2'", ["3r", "7r"], [], false, true],
    ["$= $prefix .. 'l1'", ["1l", "5l"], [], false, true],
    ["$= $prefix .. 'l2'", ["3l", "7l"], [], false, true],
],
"slidenodes": [
    ["id:", "railName","attached", "fixToRail","tolerance","spring", "strength", "capStrength"],
    ["0r", "$= $prefix .. 'r1'", true, true, 0.0, 60000, 400000, 400000],
    ["0r", "$= $prefix .. 'r2'", true, true, 0.0, 60000, 400000, 400000],
    ["0l", "$= $prefix .. 'l1'", true, true, 0.0, 60000, 400000, 400000],
    ["0l", "$= $prefix .. 'l2'", true, true, 0.0, 60000, 400000, 400000],

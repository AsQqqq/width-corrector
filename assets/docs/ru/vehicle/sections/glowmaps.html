Карты свечения (glowMaps) позволяют переключаться между несколькими материалами на основе числового значения.

Наиболее распространённый вариант применения карт свечения — освещение транспортного средства, откуда и происходит название. Сюда входят фары, задние фонари, подсветка приборной панели и т. д. В подобных случаях они используются для замены световых материалов на их эмиссивные варианты. Подробнее о световых материалах см. на этой странице. Однако карты свечения можно использовать в любом случае, когда требуется динамическая смена материала.

Синтаксис

В отличие от большинства других секций Jbeam, секция glowMap представляет собой не таблицу, а словарь. Ключ — это имя связанного материала, заданное в программе моделирования, а значение — ещё один словарь. Внутри него может содержаться несколько различных словарей, которые будут описаны ниже, а также 3 значения, именуемых «слотами материала» с ключами "off", "on" и "on_intense". Их значениями являются различные материалы, которые деталь будет использовать в зависимости от результата вычисления simpleFunction или advancedFunction.

"glowMap":{
  "taillights":{
    "simpleFunction":{"lowhighbeam":0.49,"brake":0.49}, "off":"lights", "on":"lights_on", "on_intense":"lights_on_intense"
  },
}

Хотя glowMaps поддерживают применение материала с тем же именем, что и их собственный ключ, рекомендуется избегать этого во избежание возможного нежелательного поведения из-за изменений в игре. Материал, заданный в качестве ключа glowMap, не требуется добавлять в файлы materials.json транспортного средства, так как он генерируется автоматически для каждого ТС — однако для материалов, расположенных в папке "common", это по-прежнему необходимо, так как они не привязаны к одному ТС, а используются всеми ТС совместно. Для этой цели применяются фиктивные материалы .

Доступные слоты материала

Каждая карта свечения предоставляет возможность использования 3 различных переключаемых материалов, обозначаемых как "off", "on" и "on_intense". Переключение между режимами происходит путём суммирования нескольких входных значений с применением множителей. Если суммарное значение меньше 0.0001, glowMap будет в состоянии "off"; от 0.0001 до 0.5 — "on"; выше — "on_intense". Результат функции может быть больше 1.

Состояние "on_intense" учитывается только в том случае, если оно явно задано. В противном случае будет использоваться состояние "on".

Вычисление значения Glowmap

Типичная секция glowMap транспортного средства выглядит следующим образом:

"glowMap":{
  //exterior lights
  "signal_L": {"simpleFunction":{"signal_L":0.49},             "off":"light","on":"light_on","on_intense":"light_on_intense"},
  "signal_R": {"simpleFunction":{"signal_R":0.49},             "off":"light","on":"light_on","on_intense":"light_on_intense"},
  "taillight":{"simpleFunction":{"lowhighbeam":0.49,"brake":1},"off":"light","on":"light_on","on_intense":"light_on_intense"},
  "chmsl":    {"simpleFunction":{"brake":100},                 "off":"light","on":"light_on","on_intense":"light_on_intense"},
  "foglight": {"simpleFunction":{"fog":1},                     "off":"light","on":"light_on","on_intense":"light_on_intense"},
  "headlight":{"simpleFunction":{"lowbeam":0.49,"highbeam":1}, "off":"light","on":"light_on","on_intense":"light_on_intense"},
  "reverse":  {"simpleFunction":{"reverse":0.49},              "off":"light","on":"light_on","on_intense":"light_on_intense"},
  //gauge faces and needles
  "gauges": {"simpleFunction":{"lowhighbeam":0.6}, "off":"gauges","on":"gauges_on"},
  //gauge lights
  "signal_L":    {"simpleFunction":"signal_L",     "off":"invis", "on":"decals_gauges"},
  "signal_R":    {"simpleFunction":"signal_R",     "off":"invis", "on":"decals_gauges"},
  "checkengine": {"simpleFunction":"checkengine",  "off":"invis", "on":"decals_gauges"},
  "hazard":      {"simpleFunction":"hazard",       "off":"invis", "on":"decals_gauges"},
  "battery":     {"simpleFunction":"battery",      "off":"invis", "on":"decals_gauges"},
  "highbeam":    {"simpleFunction":"highbeam",     "off":"invis", "on":"decals_gauges"},
  "parkingbrake":{"simpleFunction":"parkingbrake", "off":"invis", "on":"decals_gauges"},
  "lowfuel":     {"simpleFunction":"lowfuel",      "off":"invis", "on":"decals_gauges"},
  "abs":         {"simpleFunction":"abs",          "off":"invis", "on":"decals_gauges"},
}

Вся функциональность glowMap зависит от значений electrics Vehicle Lua. Список доступных значений по умолчанию см. здесь . Следует отметить, что хотя большинство значений всегда будут равны 0 или 1, некоторые из них могут возвращать промежуточные числа. Например, "brake" равно значению ввода торможения, то есть может быть в любой точке между 0 и 1. Имейте это в виду при определении веса различных функций, чтобы убедиться, что они работают как ожидается.

Значение может вычисляться тремя разными способами на основе этих значений:

  • Непосредственно взято из одного значения electrics — через бинарный вариант simpleFunction
  • Суммировано из нескольких значений electrics с множителями — через сложный вариант simpleFunction
  • Вычислено из фрагмента кода Lua, заданного внутри advancedFunction

Бинарная simpleFunction

Если аргумент "simpleFunction" задан и является строкой, ожидается, что это имя значения electrics. Это устанавливает значение glowmap равным точному значению этого electrics без каких-либо изменений или вычислений. Например, данная glowMap будет использовать материал "off", когда тормоз не задействован. Как только тормоз активирован, используется состояние "on".

"glowMap":{
  "brakelights":{"simpleFunction":"brake", "off":"lights", "on":"lights_on"},
}

Сложная simpleFunction

Аргумент "simpleFunction" также может быть словарём, содержащим имена значений electrics, каждому из которых назначен множитель, добавляемый к значению glowmap. Поддерживаются отрицательные модификаторы, и суммарный результат может быть больше 1. Это наиболее распространённый метод вычисления значения glowmap. Например, данная glowmap будет использовать материал "off", когда огни выключены и тормоз не задействован. Как только одно из двух активировано, используется состояние "on". Когда оба активированы, используется состояние "on_intense". Если это состояние не задано, вместо него будет использоваться состояние "on".

"glowMap":{
  "taillights":{
    "simpleFunction":{"lowhighbeam":0.49,"brake":0.49}, "off":"lights", "on":"lights_on", "on_intense":"lights_on_intense"
  },
}

В этом конкретном случае следует отметить, что lowhighbeam всегда будет равен 0 или 1, тогда как brake равно значению ввода торможения, то есть может быть в любой точке между 0 и 1.

Расширенный пример

Данная simpleFunction создаёт комбинированное значение glowmap для ДХО/указателя поворота.

Когда двигатель запущен, "running" устанавливается в true. Это прибавляет 0.49 к значению glowmap и переключает состояние в "on".
Когда активирован правый указатель поворота, "signal_right_input" устанавливается в true и из значения glowmap вычитается 1, что заставляет использовать материал "off" несмотря на то, что "running" равен true.
"signal_R" активируется и деактивируется по таймеру для достижения эффекта мигания. Когда он активен, к значению glowMap прибавляется 2, что даёт значение выше 0.5.

"glowMap":{
  "running_signal_R":{
    "simpleFunction": {
      "signal_right_input": -1,
      "running": 0.49,
      "signal_R": 2
    },
    "off":"lights", "on":"lights_on", "on_intense":"lights_on_intense"
  },
}

advancedFunction

advancedFunction поддерживает использование непосредственного фрагмента кода Lua для вычисления значения. Это даёт прямой контроль над glowmap без необходимости использования внешнего модуля Lua. Данная возможность используется нечасто; простейший вариант применения — дисплей передачи. Этот метод требует словаря со следующими ключами:

table/dictionary
type
Список всех значений electrics, используемых в следующей функции Lua

Синтаксис таблицы может использоваться при наличии одного значения:

"triggers":["gear"],

Синтаксис словаря используется при наличии нескольких значений. Каждое из них должно иметь уникальный ключ в пределах секции advancedFunction, однако имена ключей не имеют значения, так как при обработке они игнорируются.

"triggers":{"1":"gear","2":"gearIndex"},

При наличии одного значения ключ можно также опустить, задав его пустой строкой:

"triggers":{"":"gear"},
string
type
Код Lua для выполнения с использованием заданных значений

Этот код заключён в оператор return и должен всегда возвращать число. Возможно, потребуется использовать тернарные операторы Lua для преобразования булевых значений.

Ознакомьтесь со страницей синтаксиса для получения дополнительной информации о допустимых операторах и встроенных функциях .

Например, данная advancedFunction переключается в "on", когда выбрана передача парковки.

"glowMap":{
  "citybus_transmission_DSP_P": {
    "advancedFunction":{
      "triggers":{"":"gear"}, "cmd":"gear=='P' and 1 or 0"
    },
    "off":"invis""on":"citybus_transmission_DSP"
  },
}

А эта переключается в "on", когда коробка передач находится на 2-й передаче.

"glowMap":{
  "citybus_transmission_DSP_2": {
    "advancedFunction":{
      "triggers":{"":"gearIndex"}, "cmd":"gearIndex==2 and 1 or 0"
    },
    "off":"invis", "on":"citybus_transmission_DSP",
  },
}
Расширенный пример

Данная advancedFunction воспроизводит поведение комбинированной simpleFunction ДХО/указателя поворота, представленной в расширенном примере выше.

"glowMap":{
  "running_signal_R":{
    "advancedFunction":{
      "triggers": {
        "1": "signal_right_input",
        "2": "engineRunning",
        "3": "signal_R"
      },
      "cmd": "engineRunning*0.49 + signal_R*2 - signal_right_input"
    }
  }
}

Переходы материалов

Для реалистичной имитации галогенных фар и фар с герметичным стеклом предусмотрена возможность задавать переходы между состояниями световых приборов.

Это достигается путём изменения emissiveFactor светящихся световых материалов через сглаживатель (smoother) . В секции glowMap это реализуется путём использования специальных значений electrics , заданных в секции electrics , и указания порогового значения, при котором emissiveFactor материала "on" достигает максимального значения — это задаётся в аргументе "materialEmissiveScaling".

"glowMap":{
  "sealedbeam_auxlight": {
    "simpleFunction":{
      "foglight_filament":0.49
    }, 
    "off":"sealedbeam", "on":"sealedbeam_on", "materialEmissiveScaling":{"on_max":0.49}
  },
  "sealedbeam_auxlightglass": {
    "simpleFunction":{
      "foglight_filament":0.49
    }, 
    "off":"sealedbeam_glass", "on":"sealedbeam_glass_on", "materialEmissiveScaling":{"on_max":0.49}
  },
}

Несколько входных значений также поддерживаются так же, как они используются со стандартными функциями electrics. Благодаря динамическому emissiveFactor материала состояние "on_intense" световых приборов здесь не требуется — "on" будет использоваться как максимальное значение, а промежуточным состоянием будет переход между "off" и "on".

"glowMap":{
  "sealedbeam_headlight": {
    "simpleFunction":{
      "lowbeam_filament":0.49,"highbeam_filament":1
    }, 
    "off":"sealedbeam", "on":"sealedbeam_on_intense", "materialEmissiveScaling":{"on_max":1}
  },
  "sealedbeam_headlightglass": {
    "simpleFunction":{
      "lowbeam_filament":0.49,"highbeam_filament":1
    }, 
    "off":"sealedbeam_glass", "on":"sealedbeam_glass_on_intense", "materialEmissiveScaling":{"on_max":1}
  },
}

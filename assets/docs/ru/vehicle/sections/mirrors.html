This feature is supported as of BeamNG.tech and .drive version 0.31 to latest.

Mirrors — это секция в Jbeam, которая добавляет дополнительные камеры и отображает их изображение на определённых поверхностях меша, создавая зеркальные отражения. Поддерживаются как плоские зеркала, так и выпуклые (при соответствующем моделировании), вплоть до определённого ограничения поля зрения, которое будет изменено в будущем.

Зеркала не включены по умолчанию, так как нагружают производительность. Для их работы пользователю необходимо активировать опцию «Детализированные зеркала» в настройках графики. Включение этой опции открывает дополнительные параметры качества изображения. Если опция отключена, зеркала заменяются отражающими поверхностями, которые не отражают сам автомобиль.

Пользователь может настраивать зеркала в меню тюнинга, однако они должны быть корректно откалиброваны для положения сиденья по умолчанию изначально.

Обязательные аргументы

string
type
Меш, на котором будет отрисовываться изображение с камеры зеркала

Изображение зеркала будет формироваться на части этого меша, которой назначен материал «mirror». Как правило, это отражающий материал с изогнутыми нормалями по краям.

Начиная с версии 0.32, можно использовать имена материалов, начинающиеся с mirror_, что позволяет применить пользовательскую карту нормалей.

string
type
ID узла, используемого в качестве исходного положения камеры зеркала
string
type
ID первого дополнительного узла, используемого в качестве ориентира для начального поворота камеры
string
type
ID второго дополнительного узла, используемого в качестве ориентира для начального поворота камеры

Необязательные аргументы

dictionary
type
{"x":0, "y":0, "z":0}
default
Смещение позиции камеры от idRef в абсолютных координатах (м)
Визуализировать эту позицию можно с помощью отладочного приложения зеркал во вкладке Live Editor редактора транспортного средства (Shift+F11).
dictionary
type
{"x":0, "y":0, "z":0}
default
Поворот камеры в абсолютных координатах (градусы)
Используется система внутренних углов Эйлера +Y +Z +X.
dictionary
type
{"x":0, "y":0, "z":0}
default
Поворот изображения зеркала относительно поворота меша в абсолютных координатах (градусы)

Используется система внутренних углов Эйлера +Y +Z +X.

Устарело начиная с версии 0.32, будет удалено.

Не задавайте этот параметр — он будет перезаписан при изменении значений пользователем.

Необходимо исправить нормаль на меше или использовать пользовательскую карту нормалей с собственным материалом, имя которого начинается с mirror_*.

string
type
the mesh name
default
Название зеркала на экране настройки интерфейса
Новое в версии 0.32
number
type
0
default
Позиция по строке на экране настройки интерфейса

Новое в версии 0.32

0 — самое верхнее значение, 1 — строка ниже.

Если выше нет ни одного элемента, зеркало будет отображаться вверху, даже при значении row равном 1

string
type
mid
default
Позиция по столбцу на экране настройки интерфейса

Новое в версии 0.32

Если не задано, значение будет определено автоматически по имени меша. Если имя меша оканчивается на _L или содержит _L_, будет использован вариант left. Аналогичная логика применяется для right. По умолчанию используется mid, если ни одно из условий не выполнено.

Возможные значения: left, mid, right

string
type
mirrorInteriorMiddle
default
Название иконки на экране настройки интерфейса

Новое в версии 0.32

Если оставить пустым, применяется та же логика, что и для UiColumn.

Возможные значения:

  • mirrorLeftDefault
  • mirrorInteriorMiddle
  • mirrorRightDefault
  • mirrorLeftBig
  • mirrorLeftBigBottomWideAngle
  • mirrorLeftBigTop
  • mirrorLeftBonnet
  • mirrorRightBig
  • mirrorRightBigBottomWideAngle
  • mirrorRightBigTop
  • mirrorRightBonnet
  • mirrorRoundWideAngle
  • mirrorTopWideAngle

Стандартное зеркало автомобиля без смещения поворота

    ["mesh", "idRef:", "id1:", "id2:"],
    ["car_mirror_L","mi4l","mi3l","mi1l",{"refBaseTranslation":{"x":-0.110,"y":0.017,"z":0.156},"baseRotationGlobal":{"x":0,"y":0,"z":-10}}],
],

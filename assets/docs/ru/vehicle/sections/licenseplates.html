В jbeam существует 3 различных компонента номерных знаков. Свойство license plate_path задаёт путь к Json-файлу, используемому для генерации текстур номерного знака. Свойство license plateFormat определяет, какой из форматов из этого Json-файла будет применяться. Сама модель номерного знака, на которую накладываются текстуры, добавляется в раздел Flexbodies.

Глобальные модификаторы

Они присутствуют рядом с таблицами и словарями в корневой части Jbeam.

string
type
vehicles/common/licenseplates/default/licensePlate-default.json
default
Путь к Json-файлу с настройками для генерации номерного знака.

Это свойство присутствует в общих деталях в слоте License Plate Design (внутреннее имя: licenseplate_design_2_1).

Свойство работает только в том случае, если slotType детали содержит 'licenseplate_design' в названии.

string
type
30-15
default
Выбирает, какой из форматов Json-файла будет использоваться.

Значения, используемые в официальных номерных знаках — 30-15 и 52-11; они представляют собой пропорции размеров номерного знака в игре для американских и европейских знаков соответственно. Вы можете добавить собственный формат — для этого его необходимо описать в Json.

Это свойство присутствует в частях номерных знаков отдельных транспортных средств. Поскольку 30-15 является форматом по умолчанию, он обычно не задаётся для вариантов с американскими знаками, и только для европейских вариантов устанавливается значение 52-11. В отдельных случаях из-за приоритета загрузки деталей действует исключение, при котором формат 30-15 необходимо задавать вручную.

Формат по умолчанию 30-15 будет применяться только в том случае, если slotType детали содержит '_licenseplate' в названии.

Модели номерных знаков

3D-модели номерных знаков находятся в общей папке и могут использоваться на любом транспортном средстве. В файле "empty.dae" в корне общей папки присутствует 1 вариант американского формата номерного знака с именем "licenseplate", а также несколько вариантов европейских знаков в файле "licenseplate-52-11.dae" в подпапке driver_training. Варианты отличаются кривизной — каждый согнут по разному радиусу (чем меньше радиус, тем сильнее изогнут знак), что позволяет устанавливать их на различные бамперы. Доступны следующие варианты:

  • "licenseplate-52-11" - прямой
  • "licenseplate-52-11-r4" - изогнутый с радиусом 4 м
  • "licenseplate-52-11-r2" - изогнутый с радиусом 2 м
  • "licenseplate-52-11-r1_5" - изогнутый с радиусом 1,5 м
  • "licenseplate-52-11-r1" - изогнутый с радиусом 1 м
  • "licenseplate-52-11-r0_5" - изогнутый с радиусом 0,5 м
  • "licenseplate-52-11-r0_3" - изогнутый с радиусом 0,3 м
  • "licenseplate-52-11-b45" - согнутый посередине примерно под углом 90 градусов

Если ни один из них не подходит к вашему кастомному бамперу, вы можете создать собственный вариант, отредактировав один из существующих. Пока материал и UV-развёртка остаются неизменными, он будет работать так же, как оригинальные знаки.

Создание Jbeam-файла номерного знака для транспортного средства может показаться сложным при таком объёме информации, однако на практике всё очень просто. Всё, что нужно сделать — задать свойству licenseplateFormat правильное значение, а затем добавить меш номерного знака в группу, к которой относится бампер (или другая деталь, к которой крепится знак), и позиционировать его с помощью модификаторов "pos" флексбоди. Помните, что задние номерные знаки также необходимо повернуть на 180 градусов по оси Z с помощью модификатора "rot". Также имейте в виду, что позиции европейских номерных знаков могут отличаться от американских.

Скины — это специальные детали, которые при установке заменяют определённые материалы другими. Они используются преимущественно для ливрей транспортных средств и прочих вариантов окраски, а также для выбора цвета и оформления салона.

Скины имеют более низкий приоритет, чем glowMaps , поэтому для замены материала с помощью скина необходимо заменить все 3 выходных материала, определённых в glowMap. Они также имеют более низкий приоритет, чем deformGroups , поэтому повреждённый вариант материала тоже должен быть заменён.

Скины определяются как глобальные модификаторы в корневой детали Jbeam, аналогично свойствам номерного знака . Таким образом, они видны каждому компоненту транспортного средства.

Синтаксис материалов

Заменяющие материалы скинов определяются в файлах materials.json так же, как обычные материалы. Как правило, они выносятся в отдельный файл skin.materials.json для наглядности. Принцип работы скина задаётся в разделе "mapTo" материала (который обычно совпадает с именем для наглядности). Он состоит из 3 частей, разделённых точками:

"mapTo": "pickup.skin.ambulance",

Первая часть — значение mapTo материала, который будет заменён. У большинства транспортных средств материал, содержащий внешнюю окраску, имеет то же значение mapTo, что и название автомобиля.

Вторая часть — слот материала. Слоты материалов позволяют разным деталям заменять разные материалы. Слот материала определяется параметром slotType того слота, в котором находится деталь, вызывающая скин. Слот материала совпадает со slotType, если в названии slotType присутствует "skin_". Если slotType равен "paint_design", слот материала будет просто "skin".

Слот материала можно переопределить в Jbeam с помощью свойства "skinType" следующим образом:

{
"nine_skin_taxi": {
    "information":{
        "authors":"BeamNG",
        "name":"Taxi Cab",
        "value":0,
    },
    "slotType" : "skin_nine_sedan",
    "skinType" : "skin_body",
    "skinName" : "taxi",
    },
}

Это полезно, когда slotType является таблицей и поэтому не может использоваться как слот материала:

{
"nine_skin_twotone_custom": {
    "information":{
        "authors":"BeamNG",
        "name":"Two Tone Custom Paint",
        "value":5000
    },
    "slotType" : ["skin_nine_roadster","skin_nine_coupe","skin_nine_sedan","skin_nine_van","skin_nine_panelvan","skin_nine_pickup"],
    "skinType" : "skin_body",
    "skinName" : "twotone_custom"
},
}

Третья часть — название скина. Оно задаётся глобальными модификаторами, описанными в разделе выше. Они взаимозаменяемы, однако для ясности лучше использовать globalSkin для скинов, изменяющих основную внешнюю окраску, и skinName для всех остальных.

Остальной синтаксис аналогичен обычному материалу. Подробнее см. справочную страницу системы скинов и примеры материалов .

Динамические текстуры

Динамические текстуры — это вспомогательный скин, присутствующий на каждом транспортном средстве, который позволяет красить его с помощью редактора ливрей в режиме карьеры. Синтаксис Jbeam должен выглядеть следующим образом:

{
"moonhawk_skin_dynamicTextures": {
    "information":{
        "authors":"BeamNG",
        "name":"Dynamic Textures",
        "isAuxiliary": true,
        "value":1350
    },
    "slotType" : "paint_design",
    "dynDecalMaterials": ["moonhawk", "moonhawk_nosecone_alt"],
    "globalSkin" : "dynamicTextures"
  },
}

В него входит необязательный раздел Jbeam dynDecalMaterials — таблица с материалами транспортного средства, на которые будет воздействовать система. Если этот раздел отсутствует, система воздействует на материал с таким же именем, что и у транспортного средства.

Обратите внимание, что dynamicDecals работают только со slotType "paint_design" и несовместимы с модификатором skinType.

Простой пример

Стандартная ливрея транспортного средства.

Jbeam:

"pickup_skin_ambulance": {
    "information":{
        "authors":"BeamNG",
        "name":"Ambulance Livery",
        "value":5000,
    },
    "slotType" : "paint_design",
    "globalSkin" : "ambulance",
},

Начало файла skin.materials.json:

"pickup.skin.ambulance": {
  "name": "pickup.skin.ambulance",
  "mapTo": "pickup.skin.ambulance",
  "class": "Material",
  /* ... */
},

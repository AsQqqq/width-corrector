1Основы soft-body

BeamNG работает на физике мягких тел (soft-body). Машина - это не жёсткий объект, а живая структура из узлов и балок, которая гнётся, деформируется и ломается под реальными силами.

  • Узел (node) - точечная масса. Несёт вес и отвечает за большинство столкновений. Узлы свободно двигаются в пространстве.
  • Балка (beam) - пружинный соединитель ровно между двумя узлами. Балки не имеют массы; они задают жёсткость (beamSpring), демпфирование (beamDamp), насколько гнутся до необратимой деформации (beamDeform) и когда рвутся (beamStrength).

Целая машина разбита на множество деталей (parts) (кузов, подвеска, двигатель, панели, колёса...), каждая - в своём файле JBeam. Детали соединяются через систему слотов - именно она делает машину конфигурируемой и тюнингуемой.

Ключевая мысль: простой ящик из балок складывается, потому что балки могут вращаться в узлах. Жёсткость даёт диагональная раскосина - триангулируйте конструкцию.

2Система координат

BeamNG использует систему Z-вверх в единицах СИ (метры, килограммы). Оси машины не совпадают с привычным соглашением ISO/робототехники, и на этом спотыкается почти каждый новичок:

ОсьНаправление
+XВлево (значит вправо - это -X)
+YНазад (значит вперёд - это -Y)
+ZВверх

Координаты узлов в JBeam - это метры относительно машины. Увеличение X двигает узел влево, увеличение Y - назад, увеличение Z - вверх. Каркас машины привязан к refNodes (обычно рядом с центром кузова, внизу).

Мировое пространство (привязано к карте) и пространство машины (относительно неё) - разные системы; игровые/сенсорные системы принимают обе, поэтому следите, в какой вы находитесь.

3Папки и файлы

Распакованные моды лежат в папке игры mods/unpacked/. Добавьте к имени мода свой ник, чтобы оно было уникальным. Файлы машины лежат в vehicles/<имя>/:

%localappdata%\BeamNG.drive\<версия>\mods\unpacked\
└─ username_mymod/
   └─ vehicles/
      └─ username_mymod/
         ├─ mymod.jbeam          # корневая деталь (slotType "main")
         ├─ mymod_engine.jbeam   # другие детали...
         ├─ mymod.dae            # 3D-модель (меш)
         ├─ main.materials.json  # материалы / текстуры
         ├─ info.json            # карточка машины в списке
         └─ mymod.pc             # сохранённая конфигурация
ФайлЧто это
*.jbeamСтруктура и физика детали: узлы, балки, слоты, привязка меша. Ядро мода.
*.pcСохранённая конфигурация: какая деталь стоит в каждом слоте, плюс покраска и переменные. Создаётся в игре через Save&Load.
info.jsonКарточка машины в списке выбора: бренд, имя, конфигурация по умолчанию, покраски.
info_<config>.jsonДанные по конфигурации: имя и характеристики (мощность, вес, привод) для фильтра.
*.dae3D-модель (Collada), обычно из Blender. Привязывается к узлам через flexbodies.
*.materials.jsonОписания материалов, ссылающиеся на файлы текстур.
*.dds / *.pngТекстуры, используемые материалами.

4Синтаксис JBeam

JBeam - это смягчённый JSON. Это JSON, но дружелюбнее: разрешены комментарии // и /* */, а запятые между элементами необязательны. Регистр важен. Файл - это плоский словарь, ключи которого - имена деталей:

{
  "part_name": {
    "slotType": "main",   // корневую деталь спавнит игра
    "information": { "name": "My Part", "authors": "You" },
    "nodes": [ ... ],
    "beams": [ ... ]
  }
}

Таблицы (как в электронной таблице)

Большинство секций - таблицы: первая строка задаёт имена столбцов, следующие строки - данные.

"nodes": [
  ["id", "posX", "posY", "posZ"],   // строка заголовка
  ["n1",  0.5, -1.2, 0.3],
  ["n2",  0.7, -1.5, 0.2]
]

Имя столбца, оканчивающееся на : - это ссылка на секцию: оно ссылается на id из другой секции (например, балки ссылаются на id узлов):

"beams": [
  ["id1:", "id2:"],   // ":" значит "это id узлов"
  ["n1",   "n2"]
]

Модификаторы (объекты свойств)

Объект {...}, помещённый среди строк, задаёт свойства для строк после себя, пока вы их не сбросите. Так вы избегаете повторения значений в каждой строке:

"nodes": [
  ["id", "posX", "posY", "posZ"],
  {"nodeWeight": 25, "group": "body"},   // действует ниже
  ["n1", 0.0, -1.5, 0.0],
  ["n2", 0.37, -0.98, 0.22, {"nodeWeight": 3}],  // только для этой строки
  {"group": ""}                             // сброс группы
]

Словари (обычные объекты ключ/значение) используются для разовых данных, например настроек камеры или двигателя, вместо табличной формы.

5Детали и система слотов

Каждая деталь объявляет свой slotType. Родительская деталь задаёт слоты, которые принимают дочерние детали по совпадению slotType, так что слот может занять любая подходящая деталь. Ровно одна деталь имеет slotType = "main": это корень, загружаемый первым.

main (корневая деталь)
├─ слот "mymod_engine"    → любая деталь с этим slotType
├─ слот "mymod_suspension_F" → варианты передней подвески
└─ слот "mymod_wheels_F"     → варианты передних колёс

Секция slots перечисляет для каждого слота его slotType, деталь по умолчанию и описание для интерфейса. Именно это дерево показывают меню деталей и тюнинга, и именно его записывает файл .pc.

6Справочник секций

Секции, которые встречаются внутри детали чаще всего:

СекцияНазначение
informationОтображаемое имя, авторы, цена детали.
slotType / slotsКуда встаёт эта деталь и какие дочерние детали принимает.
nodesТочечные массы: позиции, вес, столкновения и группировка.
beamsСоединения между узлами: жёсткость, демпфирование, деформация, прочность.
trianglesПоверхности из трёх узлов для аэродинамики и столкновений.
flexbodiesПривязка видимого меша к узлам, чтобы модель деформировалась вместе с каркасом.
hydrosБалки с изменяемой по команде длиной (руль, актуаторы).
rails / slidenodesУзлы, скользящие по рельсе (рулевые рейки, жёсткость).
torsionbarsСопротивление вращению между рычагами (стабилизаторы).
refNodesОпорные узлы, задающие ориентацию машины и привязку камеры.
camerasПозиции камер снаружи и в салоне.
propsНедеформируемые анимированные меши (приборы, педали, стрелки).
powertrain / engineТрансмиссия: кривая момента двигателя, коробка, дифференциалы.
variablesНастраиваемые значения в интерфейсе (например, жёсткость пружин).
controller / electricsLua-контроллеры и шина электрики (свет, сигналы).

7Модели и материалы

Видимая машина - это 3D-меш (обычно .dae из Blender). Сам по себе он не физический - он привязывается к структуре узлов через секцию flexbodies, чтобы гнуться и ломаться вместе с балками под ним.

  • Важен origin. Экспортируйте меш с origin, выровненным по машине, иначе модель окажется смещена относительно узлов.
  • Достаточно узлов. Flexbody нужно минимум 3 неколлинеарных узла, охватывающих меш, иначе будут ошибки "VY node" и плохая деформация.
  • Материалы. *.materials.json связывает имена материалов из меша с файлами текстур (.dds/.png) и настройками шейдера.
Полностью: гайд по моделированию машин

8Конфигурации и info-файлы

Файл .pc - это сохранённый набор машины: выбранная деталь для каждого слота, цвета покраски и значения переменных. Создаётся в игре через Save&Load, кладётся в папку мода как готовый пресет.

  • info.json - главная карточка машины: бренд, имя, доступные покраски и default_pc (какая конфигурация грузится по умолчанию).
  • info_<config>.json - по одному на конфигурацию: её отображаемое имя плюс характеристики (мощность, момент, вес, привод, разгон, максималка) для фильтра и информационной панели.

9Частые ошибки

  • Пропущенные кавычки вокруг строк (особенно id узлов) ломают разбор - используйте редактор с подсветкой.
  • Непарные скобки. Консоль игры укажет файл и примерное место несоответствия.
  • Одинаковые имена деталей - вторая тихо перезапишет первую. Держите имена уникальными.
  • Смещение flexbody - неверный origin в Blender ставит меш не туда.
  • Нет ссылки на меш - опечатка в имени flexbody или отсутствие .dae не даёт модели загрузиться.
  • Мало узлов у flexbody - нужно 3+ неколлинеарных узла, охватывающих меш.
  • Нет раскосин - деталь без диагональных балок складывается на спавне.
  • Разбаланс параметров балок - огромные spring/damp относительно веса узла дают вибрацию и нестабильность.
  • Узлы в геометрии на спавне выталкивают панели; сдвиньте узлы или настройте self-collision.
  • Ошибки в консоли (дубли балок, битые ссылки) бьют по производительности - проверяйте консоль перед релизом.

10Советы и практики

  • Отступы пробелами и выравнивание id/координат в столбцы - так вертикальные/пакетные правки безопасны и быстры.
  • Ограничивайте find/replace выделенными строками, чтобы случайно не задеть модификаторы JBeam в другом месте.
  • Отлаживайте визуализацией балок в режиме Deformation, чтобы найти перегруженные балки; отключите гравитацию и трансмиссию, чтобы изолировать проблемы конструкции.
  • Следите за стоимостью столкновений - держите структуры с наземным контактом разреженными, используйте NONCOLLIDABLE-треугольники в плотных скоплениях узлов и держите минимум ~3 см между узлами и треугольниками.
  • Ломайте рельсы и торсионы через маленькие фиктивные балки между их узлами.
  • Начинайте с малого. Простая деталь с малым числом подвижных частей быстрее всего учит процессу.

11Инструменты и процесс

  • Blender - бесплатное 3D-моделирование для меша и расстановки узлов.
  • VS Code + официальное расширение JBeam - редактирование с валидацией и подсветкой.
  • Редактор изображений (GIMP / Paint.NET) для текстур.
  • Консоль игры и debug-виды - основная обратная связь по ошибкам и напряжениям конструкции.

Сам WidthCorrector тут помогает: пакетно менять вес узлов и параметры балок по всей папке, смотреть 3D-каркас узлов/балок, править кривые двигателя и редактировать файлы во встроенном редакторе кода.

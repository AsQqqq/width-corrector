Введение

Пользовательский интерфейс и приложения BeamNG.drive написаны с использованием фреймворка AngularJS (1.5.8).

Некоторое знакомство с фреймворком будет полезным, хотя и не обязательным. В данном контексте приложения представляют собой простые директивы в модуле beamng.apps.

Каждое приложение состоит из 3 важных частей:

  1. Файл app.js со всем кодом, используемым приложением
  2. Файл app.json, содержащий информацию о приложении
  3. Файл изображения app.png, отображаемый в селекторе приложений

Файл app.js

Директива приложения BeamNG в общем случае имеет следующую структуру:

angular.module('beamng.apps')
.directive('myApp', ['StreamsManager', function (StreamsManager) {
  return {
    template:  '[Some HTML Content]',
    replace: true,
    restrict: 'EA',
    link: function (scope, element, attrs) {
      // An optional list of streams that will be used in the app
      var streamsList = [/* streams here */];

      // Make the needed streams available.
      StreamsManager.add(streamsList);

      // Make sure we clean up after closing the app.
      scope.$on('$destroy', function () {
        StreamsManager.remove(streamsList);
      });

      scope.$on('streamsUpdate', function (event, streams) {
        /* Some code that uses the streams' values */
      });
    }
  };
}]);

Приведённый выше пример — это приложение, использующее потоки данных транспортных средств (что является наиболее распространённым случаем). Однако так бывает не всегда: приложения могут использоваться для самых разных целей. Ещё раз отметим, что код должен быть включён в функцию link файла app.js file, например:

angular.module('beamng.apps')
.directive('myApp', ['StreamsManager', function (StreamsManager) {
  return {
    template:  '<button ng-click="hello()">Click Me</button>',
    replace: true,
    restrict: 'EA',
    link: function (scope, element, attrs) {
      scope.hello = function () {
        // do something here.
      };
    }
  };
}])

Иногда приложениям необходимо хранить некоторые данные. В этом случае в папку приложения можно добавить дополнительный файл settings.json с этими данными. Чтобы его использовать, директива должна быть немного изменена, как показано в этом примере:

angular.module('beamng.apps')
.directive('myApp', ['StreamsManager', function (StreamsManager) {
  return {
    template:  '[Some HTML Content]',
    replace: true,
    restrict: 'EA',
    // [1] we "require" the bngApp parent controller
    require: '^bngApp',
    // [2] the controller is available as 4th argument of the link function
    link: function (scope, element, attrs, ctrl) {
      var streamsList = ['sensors'];

      StreamsManager.add(streamsList);

      // [3] Use a variable to keep the settings
      var appSettings = null;

      // When DOM is ready and controllers are set up, get the stored settings.
      element.ready(function () {
        // [4] Call the getSettings() function of the controller
        ctrl.getSettings()
          .then(function (settings) {
            appSettings = settings;
          })
      });

      scope.$on('$destroy', function () {
        StreamsManager.remove(streamsList);
        // [5] Optionally save the (possibly modified) app settings when done
        ctrl.saveSettings(appSettings);
      });

      scope.$on('streamsUpdate', function (event, streams) {
        /* Some code that uses the streams' values */
      });
    }
  };
}])

Файл app.json

Этот файл очень прост и содержит лишь информацию о приложении, необходимую для его загрузки. Тем не менее следует быть очень внимательным с названиями полей, поскольку опечатки или пропущенные поля могут повлиять на корректную работу приложения. Содержимое файла имеет следующий вид:

{
  "name" : "My App",
  "author": "Me",
  "version": "0.1",
  "description": "Just a tutorial app",
  "directive": "myApp",
  "domElement": "<my-app></my-app>",
  "css": { "width": "150px", "height": "150px", "top": "200px", "left": "200px" },
  "preserveAspectRatio": true
}

Обязательные поля:

Поле
Описание
Поле
name
Описание
Отображаемое название приложения
Поле
author
Описание
Авторы приложения
Поле
version
Описание
Номер версии приложения
Поле
description
Описание
Краткое описание приложения
Поле
directive
Описание
Название директивы (то же, что и в файле app.js)
Поле
domElement
Описание
DOM-элемент, который непосредственно содержит приложение. Значение domElement определяется из имени директивы и представляет собой простое преобразование из camelCase в lisp-case: в нашем примере это будет myApp → <my-app></my-app>
Поле
css
Описание
CSS-атрибуты по умолчанию, применяемые при первом запуске приложения. Это значения ширины и высоты, а также свойства top/bottom и left/right, которые также определяют угол экрана, к которому будет привязано приложение

Файл app.png

Это изображение, отображаемое в окне выбора приложений. Рекомендуемый размер: 250x120px.

Использование приложения

Чтобы сделать ваше приложение видимым в игре, переместите папку с файлами приложения в директорию UI пользовательской папки (<Userfolder>\ui\modules\apps).

После этого приложение появится в списке выбора. Возможно, потребуется закрыть и снова открыть список.

На этой странице описывается, как добавить топливные заправочные станции и электрические зарядные станции на уровень.

Настройка топливной станции состоит из двух основных частей:

  • Запись объекта в файле facilities.facilities.json уровня.
  • Пара объектов уровня для каждой колонки:
    • BeamNGGameplayArea
    • BeamNGPointOfInterest

Запись объекта определяет данные станции: название, изображение предпросмотра, поддерживаемые типы топлива и ссылки на объекты колонок. Объекты уровня задают место, где игрок может взаимодействовать со станцией.

Расположение файла

Топливные станции определяются в файле объектов уровня:

levels/<levelName>/facilities/facilities.facilities.json

Пример:

levels/italy/facilities/facilities.facilities.json

Изображения предпросмотра также можно размещать в той же папке:

levels/<levelName>/facilities/<previewImage>.jpg

Базовая структура

Топливные станции перечислены в массиве gasStations.

Пример:

{
  "gasStations": [
    {
      "id": "fuel_example_station",
      "name": "Example Fuel Station",
      "description": "A fuel station near the main road.",
      "preview": "fuel_example_station.jpg",
      "energyTypes": ["gasoline", "diesel", "unknown"],
      "pumps": [
        ["fuel_example_station_pump_area_1", "fuel_example_station_pump_icon_1"],
        ["fuel_example_station_pump_area_2", "fuel_example_station_pump_icon_2"]
      ]
    }
  ]
}

Файл facilities.facilities.json может содержать и другие типы объектов. В таком случае gasStations является лишь одним из разделов файла.

Поля записи заправочной станции

Поле Тип Описание
id String Уникальный идентификатор станции.
name String Отображаемое название или ключ локализации.
description String Отображаемое описание или ключ локализации.
preview String Имя файла изображения предпросмотра. Обычно хранится в папке facilities уровня.
energyTypes Array Типы топлива/энергии, поддерживаемые данной станцией.
pumps Array Список пар объектов колонок. Каждая пара содержит имя объекта зоны и имя объекта иконки.
prices Object Необязательные данные о ценах на топливо. Используются табло цен и системой карьеры.

Ссылки на колонки

Каждая запись колонки представляет собой массив из двух имён объектов:

["<areaObjectName>", "<iconObjectName>"]

Пример:

[
  "fuel_example_station_pump_area_1",
  "fuel_example_station_pump_icon_1"
]

Первый объект должен быть BeamNGGameplayArea.

Второй объект должен быть BeamNGPointOfInterest.

Порядок важен:

[BeamNGGameplayArea name, BeamNGPointOfInterest name]

Типы энергии

Массив energyTypes определяет, какие типы накопителей энергии транспортного средства станция может дозаправлять или перезаряжать.

Распространённые значения:

Тип энергии Описание
gasoline Бензиновое топливо.
diesel Дизельное топливо.
electricEnergy Энергия аккумулятора электромобиля.
kerosine Реактивное/авиационное топливо.
n2o Закись азота.
any Позволяет станции заправлять любым типом энергии.
unknown Резервное значение для типов топлива, не перечисленных явно.
Используйте точный тип энергии, соответствующий накопителю энергии транспортного средства. Лучшим справочником для нестандартных типов топлива служит официальный контент уровней и транспортных средств.

Типичные наборы типов энергии

Обычная топливная станция

Используйте для большинства обычных заправочных станций:

"energyTypes": ["gasoline", "diesel", "unknown"]

Поддерживает бензин, дизельное топливо и неизвестные/резервные типы топлива.

Электрическая зарядная станция

Используйте для зарядных станций электромобилей:

"energyTypes": ["electricEnergy"]

Универсальная станция

Используйте только в том случае, если станция должна поддерживать все типы энергии:

"energyTypes": ["any"]

Настройка игровых объектов

Для каждой колонки создайте:

BeamNGGameplayArea
BeamNGPointOfInterest

BeamNGGameplayArea определяет зону взаимодействия.

BeamNGPointOfInterest определяет место отображения иконки колонки в игровом мире.

Размещение BeamNGGameplayArea

Разместите BeamNGGameplayArea там, где транспортное средство должно останавливаться для дозаправки.

Рекомендации по размещению:

  • Расположите зону на земле рядом с топливной колонкой или вокруг неё.
  • Сделайте её достаточно широкой, чтобы транспортное средство могло в неё въехать.
  • Сделайте её достаточно большой для транспортных средств разных размеров.
  • Сделайте её достаточно маленькой, чтобы транспортные средства не активировали её с соседних полос или дорог.
  • По возможности выровняйте её по заправочному боксу.

Игрок может взаимодействовать с топливной станцией, когда его транспортное средство перекрывает эту зону.

Если зона взаимодействия слишком мала или расположена слишком далеко от колонки, игроки могут не иметь возможности надёжно дозаправиться.

Размещение BeamNGPointOfInterest

Разместите BeamNGPointOfInterest на колонке или над ней.

Рекомендации по размещению:

  • Поместите его над топливной колонкой или зарядным устройством.
  • Держите его визуально связанным с колонкой.
  • Используйте одну точку интереса на каждую зону колонки.
  • Дайте ему уникальное имя.
  • Сошлитесь на него в соответствующей паре колонок в файле facilities.facilities.json.

Положение точки интереса используется для иконки в игровом мире и маркера на миникарте.

Именование объектов

Используйте понятные и стабильные имена объектов.

Пример схемы именования:

fuel_<stationName>_pump_area_1
fuel_<stationName>_pump_icon_1
fuel_<stationName>_pump_area_2
fuel_<stationName>_pump_icon_2

Пример:

fuel_apex_mountainside_pump_area_1
fuel_apex_mountainside_pump_icon_1

Затем укажите их в файле объектов:

"pumps": [
  ["fuel_apex_mountainside_pump_area_1", "fuel_apex_mountainside_pump_icon_1"],
  ["fuel_apex_mountainside_pump_area_2", "fuel_apex_mountainside_pump_icon_2"]
]
Имена в файле JSON должны точно совпадать с именами объектов уровня.

Полный базовый пример 

{
  "gasStations": [
    {
      "id": "fuel_apex_mountainside",
      "name": "Apex Mountainside Fuel",
      "description": "Fuel station near the mountainside road.",
      "preview": "fuel_apex_mountainside.jpg",
      "energyTypes": ["gasoline", "diesel", "unknown"],
      "pumps": [
        ["fuel_apex_mountainside_pump_area_1", "fuel_apex_mountainside_pump_icon_1"],
        ["fuel_apex_mountainside_pump_area_2", "fuel_apex_mountainside_pump_icon_2"]
      ]
    }
  ]
}

Необходимые объекты уровня:

BeamNGGameplayArea:     fuel_apex_mountainside_pump_area_1
BeamNGPointOfInterest:  fuel_apex_mountainside_pump_icon_1

BeamNGGameplayArea:     fuel_apex_mountainside_pump_area_2
BeamNGPointOfInterest:  fuel_apex_mountainside_pump_icon_2

Необязательное изображение предпросмотра:

levels/<levelName>/facilities/fuel_apex_mountainside.jpg

Пример электрической зарядной станции 

{
  "gasStations": [
    {
      "id": "charge_downtown_01",
      "name": "Downtown Charging Station",
      "description": "Electric vehicle charging station.",
      "preview": "charge_downtown_01.jpg",
      "energyTypes": ["electricEnergy"],
      "pumps": [
        ["charge_downtown_01_area_1", "charge_downtown_01_icon_1"],
        ["charge_downtown_01_area_2", "charge_downtown_01_icon_2"]
      ]
    }
  ]
}

Станции с electricEnergy используют иконки карты и взаимодействия в стиле зарядных устройств.

Изображение предпросмотра

Поле preview указывает на изображение, используемое в интерфейсе карты и объектов.

Пример:

"preview": "fuel_apex_mountainside.jpg"

Рекомендуемое расположение:

levels/<levelName>/facilities/fuel_apex_mountainside.jpg

Рекомендации:

  • Используйте чёткое изображение станции.
  • Следите за разумным размером файла.
  • Используйте описательное имя файла.
  • Убедитесь, что имя файла в JSON совпадает с реальным файлом.

Локализация

Поля name и description могут содержать обычный текст или ключи локализации.

Пример с обычным текстом:

"name": "Apex Mountainside Fuel",
"description": "Fuel station near the mountainside road."

Пример с ключом локализации:

"name": "levels.italy.gasStationPoints.fuel_apex_mountainside",
"description": "levels.italy.gasStationPoints.fuel_apex_mountainside.description"

Для публичных модов обычный текст вполне допустим. Для контента с поддержкой нескольких языков используйте ключи локализации и предоставьте переводы.

Необязательные цены на топливо

Топливные станции могут опционально определять данные о ценах с помощью объекта prices.

Пример:

{
  "id": "fuel_example_station",
  "name": "Example Fuel Station",
  "description": "A station with fuel price data.",
  "preview": "fuel_example_station.jpg",
  "energyTypes": ["gasoline", "diesel", "unknown"],
  "pumps": [
    ["fuel_example_station_pump_area_1", "fuel_example_station_pump_icon_1"]
  ],
  "prices": {
    "gasoline": {
      "priceBaseline": 1.65
    },
    "diesel": {
      "priceBaseline": 1.72
    }
  }
}

Объект prices индексируется по типу топлива.

Поля цен

Поле Тип Описание
priceBaseline Number Базовая цена для данного типа топлива.
priceRandomnessGain Number Необязательная величина случайного отклонения.
priceRandomnessBias Number Необязательное смещение случайного отклонения.
displayObjects Array Необязательные имена объектов, используемых для отображения цифр на физическом табло цен.
disabled Boolean Если true, табло цен показывает отключённые заглушки.
us_9_10_tax Boolean Необязательный вспомогательный параметр отображения для американских ценников на топливо в формате 9/10.
Поддержка цен на топливо полезна прежде всего для систем карьеры/свободного режима и физических ценовых табло. Базовой топливной станции раздел prices не нужен.

Необязательные физические табло цен

Станция может управлять физическими объектами ценовых табло с помощью displayObjects.

Табло настраивается как список позиций цифр. Каждая позиция цифры содержит одно или несколько имён объектов.

Пример:

"prices": {
  "gasoline": {
    "priceBaseline": 1.659,
    "displayObjects": [
      ["fuel_price_gasoline_digit_1"],
      ["fuel_price_gasoline_digit_2"],
      ["fuel_price_gasoline_digit_3"],
      ["fuel_price_gasoline_digit_4"]
    ],
    "us_9_10_tax": true
  }
}

Цена форматируется до трёх знаков после запятой и затем отображается в виде отдельных цифр.

Например:

1.659 -> 1659

Каждый объект табло заменяется во время выполнения соответствующим мешем цифры.

Если задан параметр disabled, табло показывает заглушки -:

"prices": {
  "diesel": {
    "disabled": true,
    "displayObjects": [
      ["fuel_price_diesel_digit_1"],
      ["fuel_price_diesel_digit_2"],
      ["fuel_price_diesel_digit_3"],
      ["fuel_price_diesel_digit_4"]
    ]
  }
}
Настройка физического табло цен является необязательной и более сложной. Для большинства станций параметр displayObjects можно опустить.

Поведение во время выполнения

Топливные станции загружаются из файла объектов уровня.

При наличии они отображаются как точки интереса на карте и в виде маркеров колонок в игровом мире.

Игрок может взаимодействовать со станцией, когда:

  • Транспортное средство перекрывает одну из зон колонки BeamNGGameplayArea станции.
  • Станция поддерживает тип энергии транспортного средства.
  • Топливные станции включены для текущего игрового режима.

В режиме свободной езды поддерживаемые баки заправляются напрямую.

В режиме карьеры дозаправка может обрабатываться системой топливных транзакций карьеры.

Поведение совместимости топлива

Когда игрок взаимодействует со станцией, игра проверяет типы накопителей энергии транспортного средства.

Поведение:

  • Если станция поддерживает тип топлива транспортного средства, этот бак может быть заправлен.
  • Если станция имеет тип any, могут быть заправлены все типы топлива.
  • Если у транспортного средства есть неподдерживаемый тип топлива, этот бак не заправляется.
  • Накопители типа air игнорируются.
  • unknown может использоваться как резервная категория для нераспознанных типов топлива.

Если у транспортного средства несколько накопителей энергии, станция может частично заправить только поддерживаемые.

Поведение карты и маркеров

Маркеры топливных станций формируются на основе записи объекта.

Положение маркера определяется по зонам колонок. Радиус рассчитывается исходя из расположения колонок.

Иконка в игровом мире отображается рядом с объектом BeamNGPointOfInterest каждой колонки.

Тип иконки маркера зависит от типов энергии:

Тип станции Поведение иконки
Топливная станция Иконка топливной колонки.
Электрическая зарядная станция Иконка зарядного устройства.

Станция считается электрической, если её energyTypes включает:

"electricEnergy"

Рекомендации

  • Добавляйте одну запись gasStations на каждую станцию.
  • Используйте уникальный и стабильный id.
  • Используйте понятные имена объектов для зон и иконок колонок.
  • Используйте один BeamNGGameplayArea и один BeamNGPointOfInterest на каждую колонку.
  • Размещайте зону взаимодействия там, где должно останавливаться транспортное средство игрока.
  • Делайте зону взаимодействия достаточно большой для разных транспортных средств.
  • Размещайте точку интереса над колонкой или на ней.
  • Добавьте изображение предпросмотра для интерфейса карты.
  • Используйте ["gasoline", "diesel", "unknown"] для обычных топливных станций.
  • Используйте ["electricEnergy"] для зарядных станций.
  • Не ссылайтесь в списке pumps на несуществующие объекты.
  • Сохраняйте имена колонок неизменными после выпуска, чтобы не нарушить ссылки.
  • Тестируйте с несколькими типами и размерами транспортных средств.

Распространённые проблемы

Станция не отображается

Возможные причины:

  • Станция не указана в gasStations.
  • Файл facilities.facilities.json находится в неправильной папке.
  • Файл JSON содержит ошибки.
  • Топливные станции отключены в текущем режиме/настройках.
  • У станции нет допустимых пар объектов колонок.

Подсказка взаимодействия не появляется

Возможные причины:

  • Транспортное средство не перекрывает BeamNGGameplayArea.
  • Зона взаимодействия слишком мала.
  • Имя объекта зоны взаимодействия не совпадает с записью в JSON.
  • Станция не поддерживает тип топлива транспортного средства.
  • Пара колонок ссылается на несуществующие объекты.

Иконка отображается в неправильном месте

Возможные причины:

  • BeamNGPointOfInterest расположен неправильно.
  • В паре колонок указан неверный объект иконки.
  • Порядок элементов в паре колонок перепутан.

Правильный порядок:

["area_object_name", "icon_object_name"]

Транспортное средство не заправляется

Возможные причины:

  • Транспортное средство использует тип энергии, не указанный в energyTypes.
  • Станция не включает unknown или any.
  • У транспортного средства несколько баков, и только часть из них поддерживается.
  • Топливные транзакции в режиме карьеры недоступны или заблокированы.

Изображение предпросмотра отсутствует

Возможные причины:

  • Неверное имя файла в поле preview.
  • Изображение не включено в уровень или мод.
  • Изображение находится не в ожидаемой папке.

Итог

Чтобы добавить топливную станцию:

  1. Создайте запись в разделе gasStations в файле:

    levels/<levelName>/facilities/facilities.facilities.json

  2. Задайте станции уникальный id.

  3. Укажите name, description и preview.

  4. Задайте поддерживаемые energyTypes.

  5. Для каждой колонки создайте:

    • BeamNGGameplayArea
    • BeamNGPointOfInterest

  6. Добавьте каждую пару колонок в массив pumps:

    ["pump_area_name", "pump_icon_name"]

  7. При необходимости добавьте изображение предпросмотра и данные о ценах.

  8. Протестируйте станцию в игре с совместимыми транспортными средствами.

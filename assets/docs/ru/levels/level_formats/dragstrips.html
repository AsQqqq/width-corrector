Эта страница описывает локальные данные трасс для дрэг-гонок на уровне.

Трассы для дрэга задают этапы дрэг-гонок. Типичная настройка трассы обычно включает параметры, геометрию полосы, путевые точки полос, определения таймеров, необязательные префабы и интеграцию с объектами/POI.

Расположение файлов

Данные трассы для дрэга хранятся в:

levels/<levelName>/dragstrips/

Типичные файлы:

<stripName>.dragSettings.json
<stripName>_main.strip.json
dragStripData.dragData.json
<stripName>.facility.json

Современный игровой процесс обнаруживает трассы для дрэга на уровне путём сканирования файлов *.dragSettings.json.

Поведение при обнаружении

Система дрэг-гонок сканирует папку dragstrips/ текущего уровня в поисках:

*.dragSettings.json

Каждый файл настроек считается точкой входа. Остальные файлы, например *.strip.json или файлы объектов, полезны только при ссылке на них из файла настроек, миссии, записи объекта или в рамках рабочего процесса совместимости.

Создание файлов трассы с помощью инструментов редактора

По возможности используйте редактор дрэг-гонок для создания новых данных трассы. Он позволяет создавать и сохранять полосы, данные полос, настройки дрэга, зоны и специфические файлы объектов.

Рекомендуемый рабочий процесс:

  1. Откройте уровень в World Editor.
  2. Откройте редактор дрэг-гонок.
  3. Сначала создайте макет трассы.
  4. Определите полосы, границы, путевые точки старта, постановки на старт и финишной линии.
  5. Сохраните трассу как *.strip.json.
  6. Создайте настройки дрэга, ссылающиеся на сохранённую трассу.
  7. Настройте таймеры, фазы гонки, контекст и необязательные префабы.
  8. Сохраните настройки как *.dragSettings.json.
  9. Добавьте или подключите данные объекта/POI, если трасса должна отображаться в свободном режиме или интерфейсе карьеры.
  10. Проверьте постановку на старт, обратный отсчёт, завершение гонки, сброс и отображение на большой карте.

Создайте трассу перед файлом настроек. Файл настроек является точкой входа, но для полноценной работы ему нужен корректный макет трассы.

Ручное создание файлов трассы

Ручное создание возможно, однако трансформы полос и путевые точки удобнее расставлять в редакторе. При ручном создании придерживайтесь следующего порядка:

  1. Создайте папку dragstrips/.
  2. Создайте макет *.strip.json с границами полос и путевыми точками.
  3. Создайте файл *.dragSettings.json, ссылающийся на трассу.
  4. При необходимости добавьте *.facility.json или интеграцию с объектом.
  5. Добавляйте необязательные префабы только после того, как гонка заработает.

Минимальная современная конфигурация:

levels/<levelName>/dragstrips/example_main.strip.json
levels/<levelName>/dragstrips/example.dragSettings.json

Файл настроек должен указывать на файл трассы:

{
  "id": "example",
  "stripId": "/levels/example/dragstrips/example_main.strip.json"
}

Используйте абсолютные игровые пути, начинающиеся с /levels/, для ссылок, которые могут загружаться из миссий или других систем.

*.dragSettings.json structure

Минимальный файл настроек:

{
  "id": "exampleStrip",
  "context": "freeroam",
  "dragType": "headsUpRace",
  "stripId": "/levels/example/dragstrips/exampleStrip_main.strip.json",
  "canBeReseted": true,
  "canBeTeleported": true,
  "timers": [
    {
      "id": "time_1_4",
      "label": "1/4 mile",
      "shortLabel": "1/4",
      "type": "distanceTimer",
      "distance": 402.336,
      "important": true
    }
  ],
  "phases": [
    { "name": "stage", "dependency": true, "startedOffset": 0 },
    { "name": "countdown", "dependency": true, "startedOffset": 0 },
    { "name": "race", "dependency": false, "startedOffset": 0 },
    { "name": "stop", "dependency": false, "startedOffset": 0 }
  ],
  "prefabs": {
    "christmasTree": { "isUsed": false },
    "displaySign": { "isUsed": false },
    "paths": { "isUsed": false },
    "decorations": { "isUsed": false }
  }
}

Поля настроек

Поле Обязательно Примечания
id Да Стабильный идентификатор настроек дрэга.
stripId Да Путь к файлу *.strip.json.
context Рекомендуется Обычно freeroam для локальных трасс уровня.
dragType Рекомендуется Тип гонки, например headsUpRace.
canBeReseted Необязательно Разрешено ли поведение сброса.
canBeTeleported Необязательно Разрешено ли поведение телепортации.
timers Необязательно При отсутствии или некорректных данных могут использоваться значения по умолчанию.
phases Необязательно Последовательность фаз гонки.
prefabs Необязательно Переключатели отображения/префабов.

Записи таймеров поддерживают типы distanceTimer и velocity. Расстояния указываются в метрах.

Поля трассы

Поле Обязательно Примечания
id Да Стабильный идентификатор трассы.
name Рекомендуется Имя для отображения и отладки.
description Необязательно Описание.
lanes Да Одно или несколько определений полос.
lanes[].id Рекомендуется Стабильный идентификатор полосы.
lanes[].laneOrder Рекомендуется Визуальный/игровой порядок полос.
lanes[].boundary Рекомендуется Область полосы. По возможности используйте границы, созданные в редакторе.
lanes[].waypoints Да Путевые точки старта, постановки, финишной линии и прочие необязательные.
waypoints[].type Да Например spawn, stage или endLine.
waypoints[].transform Да Позиция, поворот и масштаб.
waypoints[].waypoint Необязательно Игровое поведение путевой точки, например режим скорости.

Трансформы в новых файлах трасс используют объектный стиль для векторов/кватернионов:

{
  "position": { "x": 0, "y": 0, "z": 0 },
  "rotation": { "x": 0, "y": 0, "z": 0, "w": 1 },
  "scale": { "x": 1, "y": 1, "z": 1 }
}

Некоторый старый контент может использовать компактные имена pos, rot и scl. Для новых создаваемых файлов трасс предпочтительнее использовать поля объектного стиля.

Роли файлов

Файл Назначение
*.dragSettings.json Точка входа для трассы дрэга. Определяет идентификатор, контекст, тип дрэга, таймеры, фазы, использование префабов и ссылку на файл трассы.
*.strip.json Данные макета трассы: полосы, границы полос, трансформы путевых точек, зоны и метаданные полос.
dragStripData.dragData.json Старые/общие данные дрэг-гонок, используемые некоторым поставляемым контентом и рабочими процессами совместимости.
*.facility.json Специфические данные объекта для трассы дрэга.

Связь настроек и трассы

Файл настроек дрэга должен ссылаться на файл макета трассы:

{
  "id": "exampleStrip",
  "stripId": "/levels/example/dragstrips/exampleStrip_main.strip.json"
}

После того как миссии, POI или данные карьеры/свободного режима ссылаются на трассу, идентификаторы настроек, трассы и имена файлов должны оставаться неизменными.

Файл настроек описывает поведение активности. Файл трассы описывает расположение полос, границ и путевых точек. При редактировании разделяйте эти обязанности:

  • Изменяйте таймеры, фазы, контекст или тип гонки в файле настроек.
  • Изменяйте геометрию полос, порядок полос и трансформы путевых точек в файле трассы.
  • Изменяйте интеграцию с большой картой/карьерой/свободным режимом в данных объекта или POI.

При создании настроек с нуля начните с простого:

  1. Одна ссылка на трассу.
  2. Один тип дрэга.
  3. Базовые фазы.
  4. Один или два важных таймера.
  5. Без необязательных префабов.

После того как гонка корректно запускается и завершается, добавьте больше таймеров, дисплеев и визуального оформления.

Данные полос и путевых точек

Файл трассы описывает каждую полосу. Полосы обычно содержат:

  • Ограничивающий объём.
  • Порядок полос и отображаемые имена.
  • Путевые точки: старт, постановка и финишная линия.
  • Необязательные данные зоны.

Эти путевые точки являются игровыми позициями, а не обычными узлами графа навигации по дороге. Следите за их выравниванием с видимой трассой и поддерживающими префабами.

Важные типы путевых точек обычно включают:

Путевая точка Назначение
Spawn Место размещения участников гонки перед постановкой на старт.
Stage Место выстраивания участников перед обратным отсчётом.
End line Место оценки результатов времени/скорости.

Порядок полос должен соответствовать тому, что видит игрок. Неверный порядок полос может сделать метки интерфейса, постановку на старт или расстановку соперников запутанными даже при технически корректном старте гонки.

Для двухполосных трасс проверяйте каждую полосу независимо. Распространённая ошибка при создании — исправить полосу игрока, оставив при этом неправильно выровненными путевые точки старта, постановки или финишной линии полосы соперника.

Таймеры и фазы

Настройки дрэга могут определять таймеры и фазы гонки: постановка на старт, обратный отсчёт, гонка и остановка. Имена таймеров и фаз используются игровым процессом дрэг-гонок и интерфейсом, поэтому по возможности редактируйте их через редактор дрэг-гонок.

Типичные данные таймеров включают контрольные точки затраченного времени и измерения скорости в трап-зоне. Используйте краткие метки, поскольку они могут отображаться в интерфейсе результатов гонки.

Префабы и визуальное оформление

Настройки дрэга могут включать вспомогательные префабы, например:

  • Огни рождественской ёлки (стартовый светофор).
  • Информационные табло.
  • Направляющие пути.
  • Декоративные элементы.

Включайте только те префабы, которые существуют и выровнены с трассой. Отсутствующие или неправильно расположенные декоративные префабы могут создать впечатление неработающей гонки при корректной логике.

К размещению префабов следует относиться как к визуальному оформлению, а не как к основной логике гонки. Если новая трасса не работает, отключите необязательные префабы при отладке настроек и данных полос.

Связь с объектами и миссиями

Трассы для дрэга могут отображаться как объекты свободного режима/карьеры, а также использоваться в миссиях дрэг-типа. Миссия может ссылаться на файл настроек дрэга, тогда как интерфейс свободного режима/карьеры может использовать связанные данные объекта или POI.

Общее поведение объектов описано в разделе Facilities .

Миссии дрэг-типа могут напрямую указывать на файл настроек. Если трасса работает в свободном режиме, но не в миссии, сначала проверьте ссылку в миссии, затем файл настроек и файл трассы.

Устаревшие данные и совместимость

Некоторый поставляемый контент включает файл dragStripData.dragData.json. Он может содержать старые или общие данные трасс для дрэга и может пересекаться с новыми файлами настроек/трасс. Для нового или активно поддерживаемого контента предпочтительнее использовать точку входа *.dragSettings.json в сочетании с макетом *.strip.json.

Не удаляйте файлы совместимости из поставляемого контента без проверки всех потребителей, которые могут их использовать.

Устранение неполадок

Трасса не отображается

Проверьте:

  • Файл настроек оканчивается на .dragSettings.json.
  • Файл находится в папке levels/<levelName>/dragstrips/.
  • Функция дрэга включена в текущем режиме.
  • Файл настроек корректно разбирается.

Гонка начинается в неверном месте

Проверьте:

  • stripId указывает на нужный файл *.strip.json.
  • Путевые точки старта и постановки в полосе расставлены правильно.
  • Порядок полос соответствует видимой трассе.
  • У игрока и соперника достаточно места для старта.

Неверное время или результаты

Проверьте:

  • Путевые точки финишной линии установлены на предполагаемом финише.
  • Расстояния таймеров соответствуют предполагаемой длине гонки.
  • Фазы гонки соответствуют выбранному типу дрэга.
  • Метки дисплея соответствуют назначению таймера.

Метка свободного режима отсутствует

Проверьте:

  • Данные объекта или POI корректно ссылаются на трассу для дрэга.
  • Идентификатор объекта и идентификатор настроек дрэга стабильны.
  • Дрэг-гонки включены на большой карте или в режиме свободного режима.

Контрольный список проверки

  • *.dragSettings.json загружается и ссылается на существующий файл трассы.
  • Идентификаторы трассы и имена файлов стабильны.
  • Границы полос охватывают предполагаемые полосы.
  • Путевые точки старта, постановки и финишной линии расставлены и ориентированы правильно.
  • Таймеры и фазы соответствуют предполагаемому типу гонки.
  • Префабы, на которые ссылаются настройки, существуют при их включении.
  • Данные объекта или POI корректно разрешаются, если трасса должна отображаться в интерфейсе свободного режима/карьеры.
  • Миссии дрэга, ссылающиеся на трассу, могут корректно запускаться и сбрасываться.
  • Файлы совместимости сохраняются при наличии старых потребителей, которые могут их использовать.

В этом документе описываются JSON-файлы автобусных маршрутов BeamNG, используемые уровнями.

Файлы автобусных маршрутов задают маршруты, варианты маршрутов, порядок остановок, вспомогательные данные навигации, изображения предварительного просмотра маршрута и данные спауна автобусов.

Расположение файлов

Файлы автобусных маршрутов обычно хранятся по пути:

levels/<levelName>/buslines/*.buslines.json

Примеры:

levels/west_coast_usa/buslines/wca.buslines.json
levels/italy/buslines/italy.buslines.json
levels/GridMap/buslines/gridmap.buslines.json

Базовая структура 

{
  "version": 1,
  "routes": [
    {
      "routeID": "1",
      "variance": "a",
      "name": "levels.example.buslines.routes.1a",
      "direction": "Downtown",
      "routeColor": "#3b9c00",
      "vehicle": {
        "model": "citybus",
        "config": "base"
      },
      "tasklist": [
        "bus_stop_01",
        "bus_stop_02",
        "bus_stop_03"
      ],
      "spawnLocation": {
        "pos": { "x": 0, "y": 0, "z": 0 },
        "rotAngAxisF": { "x": 0, "y": 0, "z": 1, "w": 90 }
      }
    }
  ]
}

Поля верхнего уровня

Поле Тип Описание
version integer Версия формата файла автобусного маршрута.
routes array Список определений маршрутов.

Поля маршрута

Поле Тип Описание
routeID string Номер/идентификатор маршрута. Варианты могут иметь одинаковый идентификатор маршрута.
variance string Вариант маршрута, например "a" или "b".
name string Ключ локализации или отображаемое название маршрута.
direction string Ключ локализации или отображаемый текст направления/конечного пункта маршрута.
routeColor string Цвет в формате hex, используемый для отображения маршрута в интерфейсе.
vehicle object Необязательный объект с моделью/конфигурацией транспортного средства.
tasklist array[string] Упорядоченный список идентификаторов остановок/задач автобусного маршрута.
spawnLocation object Необязательные начальная позиция и ориентация спауна автобуса.
requiredWaypoints object Обязательные контрольные точки для отдельных участков маршрута.
navhelp object Дополнительные навигационные подсказки (путевые точки) между остановками.
previews array[string] Имена изображений предварительного просмотра для интерфейса маршрута.

tasklist

tasklist задаёт упорядоченный список остановок/задач для маршрута.

"tasklist": [
  "bus_stop_01",
  "bus_stop_02",
  "bus_stop_03"
]

Идентификаторы в этом списке должны совпадать с определениями остановок/задач, используемыми игровой системой автобусных маршрутов.

Во многих стандартных уровнях записи busline сопоставлены с объектами остановок BeamNGTrigger и навигационными данными.

navhelp

navhelp предоставляет дополнительные навигационные подсказки для участков маршрута.

"navhelp": {
  "bus_stop_02": ["wp_main_01", "Bridge_A", "Bridge_B"]
}

Значения обычно ссылаются на имена навигационных нод из BeamNGWaypoint, нод, созданных DecalRoad, или сегментов map.json.

requiredWaypoints

requiredWaypoints позволяет указать обязательные для прохождения точки маршрута.

"requiredWaypoints": {
  "bus_stop_03": {
    "wp": ["checkpoint_01"],
    "failmsg": "You got too far-off the route"
  }
}

Это полезно в случаях, когда маршрут должен проходить по конкретной дороге или соединительному участку.

spawnLocation

spawnLocation задаёт точку старта автобуса.

"spawnLocation": {
  "pos": { "x": -337.572, "y": 605.564, "z": 75.0979 },
  "rotAngAxisF": { "x": 0, "y": 0, "z": 1, "w": 135.337 }
}

rotAngAxisF.w — угол поворота в градусах вокруг заданной оси.

Замечания для инструментов редактирования

Для инструментов, редактирующих автобусные маршруты:

  • Сохраняйте routeID и variance неизменными для существующих маршрутов.
  • Проверяйте каждый идентификатор остановки в tasklist.
  • Проверяйте имена путевых точек, используемых в navhelp и requiredWaypoints.
  • Сохраняйте ключи локализации в полях name и direction, если маршрут локализован.
  • Следите за тем, чтобы пути и имена изображений предварительного просмотра соответствовали интерфейсу маршрута.

Распространённые проблемы

  • Маршрут не работает, потому что идентификатор остановки в tasklist не существует.
  • Навигация ведёт автобус в неверном направлении из-за того, что navhelp ссылается на отсутствующие или неправильные имена путевых точек.
  • В интерфейсе маршрута отсутствует текст, потому что поле name или direction указывает на несуществующий ключ локализации.
  • Автобус спаунится с неверной ориентацией из-за неправильного значения rotAngAxisF.

Критерии проверки

  • JSON-файл автобусного маршрута корректно разбирается.
  • Каждый маршрут имеет стабильные значения routeID, variance, name и tasklist.
  • Идентификаторы остановок и имена путевых точек разрешаются корректно.
  • Точка спауна безопасна для указанного транспортного средства.
  • Маршрут можно проехать от начала до конца в игре.

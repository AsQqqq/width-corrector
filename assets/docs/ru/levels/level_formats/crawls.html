На этой странице описаны локальные данные crawl-маршрутов уровня.

Данные crawl-маршрутов задают внедорожный геймплей троп через сопутствующие файлы для маршрутов, путей, границ и стартовых позиций. Некоторые crawl-миссии могут также содержать локальные crawl-файлы под gameplay/missions/.

Расположение файлов

Данные crawl-маршрутов уровня хранятся в:

levels/<levelName>/crawls/

Типичные имена файлов:

<crawlName>.trail.json
<crawlName>.path.json
<crawlName>.boundary.json
<crawlName>.startingPosition.json
presets.json

Система сохранения/загрузки crawl-данных распознаёт файлы по суффиксу. Соблюдайте суффиксы в точности:

.trail.json
.path.json
.boundary.json
.startingPosition.json

Создание crawl-файлов с помощью инструментов редактора

По возможности используйте редактор Crawl Data Editor для создания нового crawl-контента. Он записывает сопутствующие файлы в формате, ожидаемом системой crawl.

Рекомендуемый рабочий процесс:

  1. Откройте уровень в World Editor.
  2. Откройте Crawl Data Editor.
  3. Создайте или отредактируйте путь, расставив узлы маршрута вдоль предполагаемой тропы.
  4. Добавьте флажки контрольных точек восстановления там, где игрок должен иметь возможность восстановиться.
  5. Создайте границу, охватывающую предполагаемый маршрут.
  6. Создайте стартовую позицию и проверьте положение её иконки.
  7. Сохраните файлы пути, границы, стартовой позиции и тропы в levels/<levelName>/crawls/.
  8. Протестируйте crawl с нуля и из контрольной точки восстановления.

Если вы конвертируете старые данные гонок/путей, сначала воспользуйтесь инструментами конвертации, а затем вручную проверьте созданные файлы.

Ручное написание crawl-файлов

При ручном создании лучше всего создавать сопутствующие файлы в следующем порядке:

  1. *.path.json
  2. *.boundary.json
  3. *.startingPosition.json
  4. *.trail.json

Файл тропы должен быть последним, поскольку он ссылается на остальные файлы.

Минимальная структура папки:

levels/<levelName>/crawls/example.path.json
levels/<levelName>/crawls/example.boundary.json
levels/<levelName>/crawls/example.startingPosition.json
levels/<levelName>/crawls/example.trail.json

По возможности используйте совпадающие базовые имена. Совпадение имён не является строго обязательным, но значительно упрощает создание и отладку.

Роли файлов

Файл Назначение
*.trail.json Высокоуровневый дескриптор тропы. Содержит ссылки на путь, границу, стартовую позицию, превью, правила и дополнительные префабы.
*.path.json Упорядоченные узлы маршрута, данные контрольных точек, радиусы, повороты и флаги узлов.
*.boundary.json Данные полигона/объёма игровой области.
*.startingPosition.json Трансформ спауна/восстановления и позиция иконки.
presets.json Необязательные пресеты редактора для создания crawl-маршрутов.

Ссылки тропы

Файл тропы связывает сопутствующие файлы вместе с помощью таких полей, как:

{
  "pathId": "/levels/example/crawls/example.path.json",
  "boundaryId": "/levels/example/crawls/example.boundary.json",
  "startingPositionId": "/levels/example/crawls/example.startingPosition.json"
}

Используйте пути относительно корня уровня и синхронизируйте связанные файлы при переименовании crawl-маршрута.

Файлы тропы являются высокоуровневыми crawl-файлами. Обычно они не содержат полный путь или границу непосредственно в себе — они указывают на сопутствующие файлы. Это позволяет повторно использовать или заменять данные пути, границы или стартовой позиции, однако битые ссылки могут сделать в остальном корректный crawl невидимым или неработоспособным.

При ручном написании тропы начните только с обязательных ссылок и добавляйте необязательные метаданные позже:

{
  "name": "trail.example.title",
  "pathId": "/levels/example/crawls/example.path.json",
  "boundaryId": "/levels/example/crawls/example.boundary.json",
  "startingPositionId": "/levels/example/crawls/example.startingPosition.json",
  "pathReversed": false
}

Добавляйте превью, миниатюру, правила и префабы после того, как тропа успешно загрузится.

*.trail.json fields

Поле Обязательное Примечания
name Рекомендуется Отображаемое имя или ключ перевода.
description Необязательное Описание или ключ перевода.
pathId Да Путь к *.path.json.
boundaryId Да Путь к *.boundary.json.
startingPositionId Да Путь к *.startingPosition.json.
startingPositionIdReversed Необязательное Стартовая позиция для движения в обратном направлении.
pathReversed Необязательное Должен ли путь читаться в обратном порядке.
thumbnail Необязательное Путь к миниатюре для интерфейса.
preview Необязательное Путь к превью для интерфейса.
rules Необязательное Настройки правил crawl-маршрута.
prefabs Необязательное Ссылки на вспомогательные префабы.
metadata Необязательное Информация о создании/изменении/версии.
isFromMission Необязательное Помечает данные тропы, принадлежащие миссии.

Файлы пути

Файлы пути хранят упорядоченные узлы маршрута. Узлы могут включать:

  • Отображаемое/отладочное имя.
  • Позицию.
  • Поворот.
  • Радиус.
  • Флаги, такие как данные контрольных точек восстановления или боковые отступы.

Путь должен следовать предполагаемому направлению движения. Если тропа поддерживает движение в обратном направлении, предпочтительнее использовать явные настройки реверса, а не дублировать практически идентичные пути, если только обратный маршрут не требует других контрольных точек.

При создании путей вручную:

  • Сохраняйте порядок узлов от старта до финиша.
  • Используйте читаемые имена узлов.
  • Используйте увеличенные радиусы для сложных внедорожных участков.
  • Добавляйте флаги контрольных точек восстановления там, где неудача была бы особенно неприятной.
  • Тестируйте с самыми медленными и крупными транспортными средствами, ожидаемыми для crawl-маршрута.

Минимальный *.path.json:

{
  "name": "example_path",
  "description": "Example crawl path",
  "nodes": [
    {
      "name": "Start",
      "position": [0, 0, 0],
      "rotation": [0, 0, 0, 1],
      "radius": 4,
      "flags": {}
    },
    {
      "name": "Checkpoint 1",
      "position": [20, 0, 2],
      "rotation": [0, 0, 0, 1],
      "radius": 5,
      "flags": {
        "isRecoveryCheckpoint": true,
        "recovery": 1
      }
    }
  ]
}

*.path.json fields

Поле Обязательное Примечания
name Рекомендуется Внутреннее/отображаемое имя.
description Необязательное Описание для редактора/отладки.
nodes Да Упорядоченный массив узлов пути.
nodes[].name Рекомендуется Полезно для отладки и конвертации.
nodes[].position Да Мировая позиция в виде [x, y, z].
nodes[].rotation Необязательное Массив кватерниона.
nodes[].radius Рекомендуется Радиус контрольной точки/маршрута.
nodes[].flags Необязательное Флаги восстановления, отступа, нормали или другие игровые флаги.

Файлы границы

Файлы границы определяют игровую область crawl-маршрута. Делайте границу достаточно широкой, чтобы прощать обычные ошибки при вождении, но не настолько большой, чтобы игроки могли обойти задуманное испытание.

Если граница пересекает маршрут, игроки могут неожиданно терпеть неудачу или сбрасываться. Если она слишком велика, crawl может потерять своё игровое определение.

По возможности создавайте границы в редакторе. Ручное редактирование границ подвержено ошибкам, поскольку порядок и расположение вершин имеют значение.

Минимальная форма *.boundary.json:

{
  "name": "example_boundary",
  "vertices": [
    [0, -10, 0],
    [100, -10, 0],
    [100, 10, 0],
    [0, 10, 0]
  ],
  "bot": {
    "active": false,
    "normal": [0, 0, -1],
    "pos": [0, 0, -10]
  },
  "top": {
    "active": false,
    "normal": [0, 0, 1],
    "pos": [0, 0, 10]
  },
  "customFields": {
    "names": {},
    "tags": {},
    "types": {},
    "values": {}
  }
}

Формат границы разделяется с сериализацией зон сайтов. По возможности создавайте её через инструменты Sites или Crawl.

Файлы стартовой позиции

Файлы стартовой позиции определяют место старта или восстановления игрока. Обычно они включают:

  • Позицию трансформа.
  • Поворот трансформа.
  • Радиус.
  • Позицию иконки для отображения в интерфейсе или на карте.

Размещайте старты на устойчивой, достаточно ровной поверхности и тестируйте их с классами транспортных средств, ожидаемых для crawl-маршрута.

При ручном написании помните, что iconPosition предназначен для отображения в интерфейсе/на карте и не обязательно совпадает с фактическим трансформом спауна.

Минимальный *.startingPosition.json:

{
  "name": "Example Start",
  "description": "Start for the example crawl",
  "transform": {
    "position": [0, 0, 1],
    "rotation": [0, 0, 0, 1],
    "radius": 5
  },
  "iconPosition": [0, 0, 3],
  "metadata": {
    "created": "2026-06-08",
    "modified": "2026-06-08"
  }
}

*.startingPosition.json fields

Поле Обязательное Примечания
name Рекомендуется Отображаемое/отладочное имя.
description Необязательное Описание.
transform.position Да Фактическая позиция спауна/восстановления.
transform.rotation Да Ориентация в виде кватерниона.
transform.radius Рекомендуется Радиус спауна/восстановления.
iconPosition Рекомендуется Позиция иконки в интерфейсе/на карте.
metadata Необязательное Данные о создании/изменении.

Локальные crawl-данные миссии

Некоторый crawl-геймплей находится в папках современных миссий:

gameplay/missions/<levelName>/crawl/<missionName>/

Локальные crawl-данные миссии могут использовать те же концепции, что и локальные crawl-данные уровня. Проверяйте папку миссии, прежде чем считать, что все crawl-файлы находятся в levels/<levelName>/crawls/.

Примечания по конвертации

Некоторые crawl-файлы конвертируются из старых данных race.race.json. Конвертированные файлы часто содержат описания вроде "Converted from race.race.json". После конвертации считайте созданные файлы пути, границы, тропы и стартовой позиции активными crawl-данными.

При конвертации вручную проверяйте результат. Имена узлов пути, контрольные точки восстановления, боковые отступы и границы в конвертированных файлах могут быть технически корректными, но всё равно требовать игровой настройки.

Пресеты

presets.json — это данные редактора/инструментария. Он может хранить многократно используемые пресеты для создания crawl-маршрутов и должен храниться вместе с уровнем только в том случае, если эти пресеты будут полезны при дальнейшем редактировании.

Не используйте пресеты как игровые данные. Активный crawl должен быть представлен файлами тропы, пути, границы и стартовой позиции.

Устранение неполадок

Тропа не отображается

Проверьте:

  • Суффикс файла — .trail.json.
  • Тропа ссылается на существующие файлы pathId, boundaryId и startingPositionId.
  • Указанные пути являются путями относительно корня уровня и содержат правильное имя уровня.
  • Функция или миссия, которая должна отображать тропу, включена.

Игрок появляется в неправильном месте

Проверьте:

  • Трансформ стартовой позиции находится на предполагаемой поверхности земли.
  • Поворот направлен к первому участку маршрута.
  • Радиус подходит для размера транспортного средства.
  • Позиция иконки не перепутана с фактическим трансформом спауна.

Crawl завершается неудачей неожиданно

Проверьте:

  • Граница охватывает предполагаемую траекторию движения.
  • Флаги контрольных точек восстановления расставлены там, где необходимо.
  • Радиусы узлов пути достаточно велики для маршрута.
  • Локальные файлы миссии не переопределяют и не заменяют локальные данные уровня.

Контрольный список проверки

  • *.trail.json ссылается на корректные файлы пути, границы и стартовой позиции.
  • *.path.json содержит упорядоченные узлы с разумными радиусами и поворотами.
  • *.boundary.json охватывает предполагаемый маршрут и не пересекает нормальное движение.
  • *.startingPosition.json указывает на безопасную позицию спауна/восстановления.
  • Пути к превью и миниатюре существуют, если они используются.
  • Локальные crawl-файлы миссии хранятся вместе с миссией, которой они принадлежат.
  • Конвертированные файлы проверены и настроены после конвертации.

В этом документе описывается файл signals.json, используемый уровнями BeamNG для управления дорожными сигналами.

signals.json хранит данные о дорожных сигналах: знаки «Стоп», светофоры, контроллеры сигналов и временны́е последовательности фаз. Файл используется системами дорожного ИИ, навигации, визуализации сигналов и игрового процесса.

Расположение файла

Файл дорожных сигналов по умолчанию находится в корне уровня:

levels/<levelName>/signals.json

Пример:

levels/west_coast_usa/signals.json

Если файл существует, он автоматически загружается при запуске уровня.

Связанные файлы

Файл Назначение
signals.json Экземпляры сигналов, контроллеры и последовательности уровня.
signalControllerDefinitions.json Необязательные пользовательские определения состояний и типов контроллеров для конкретного уровня.
settings/trafficSignals.json Глобальные определения состояний и типов контроллеров по умолчанию.
map.json / navgraph Используется для привязки сигналов к узлам графа дорог.

Обзор

Дорожные сигналы делятся на три основных типа данных:

Тип Описание
Экземпляр (Instance) Размещённая в мире точка сигнала. Содержит позицию, направление, контроллер и необязательную последовательность.
Контроллер (Controller) Определяет тип сигнала и набор доступных состояний.
Последовательность (Sequence) Определяет временны́е фазы для одного или нескольких контроллеров, как правило для светофорных перекрёстков.

Данные хранятся в трёх массивах верхнего уровня:

{
  "instances": [],
  "controllers": [],
  "sequences": {}
}

или в современных сохранённых файлах:

{
  "instances": [],
  "controllers": [],
  "sequences": []
}
В некоторых старых файлах sequences может быть представлен как пустой объект ({}). В новых файлах следует использовать массив ([]).

Минимальный пример знака «Стоп»

Простую настройку знака «Стоп» можно реализовать с одним контроллером и множеством экземпляров.

{
  "controllers": [
    {
      "id": 2,
      "name": "stop",
      "type": "signStop",
      "isSimple": true,
      "states": [
        {
          "state": "basicStop"
        }
      ]
    }
  ],
  "instances": [
    {
      "id": 1,
      "name": "stop 1",
      "controllerId": 2,
      "sequenceId": 0,
      "pos": [734.165, 755.369, 177.949],
      "dir": [-0.925, 0.378, -0.005],
      "group": "group_1_3",
      "startDisabled": false
    }
  ],
  "sequences": []
}

Создаётся один экземпляр знака «Стоп», использующий общий контроллер stop.

Идентификаторы и ссылки

Сигналы используют числовые идентификаторы для связи элементов между собой.

Поле Описание
instance.id Уникальный идентификатор экземпляра сигнала.
controller.id Уникальный идентификатор контроллера.
sequence.id Уникальный идентификатор последовательности.
instance.controllerId Ссылка на идентификатор контроллера.
instance.sequenceId Ссылка на идентификатор последовательности. 0 означает отсутствие последовательности / базовое поведение.

Идентификаторы должны быть уникальными среди всех экземпляров, контроллеров и последовательностей.

Не используйте один и тот же числовой идентификатор для нескольких элементов. Дублирующиеся идентификаторы удаляются или игнорируются при загрузке данных сигналов.

Экземпляры сигналов

Экземпляры сигналов представляют размещённые в мире сигналы.

Пример:

{
  "id": 1,
  "name": "stop 1",
  "controllerId": 2,
  "sequenceId": 0,
  "pos": [734.165, 755.369, 177.949],
  "dir": [-0.925, 0.378, -0.005],
  "group": "group_1_3",
  "startDisabled": false
}

Поля экземпляра

Поле Тип Описание
id number Уникальный идентификатор экземпляра сигнала.
name string Имя экземпляра сигнала. Также используется для связи с объектами сигналов в мире.
pos array[3] Позиция точки сигнала в мире.
dir array[3] Направление, в котором обращён / действует сигнал.
group string Необязательное имя перекрёстка / группы.
controllerId number Контроллер, используемый этим экземпляром. Обязателен для работающих сигналов.
sequenceId number Последовательность, используемая этим экземпляром. 0 означает отсутствие последовательности / простой сигнал.
useCurrentLane bool Необязательное/экспериментальное поведение для конкретной полосы. В настоящее время не используется.
startDisabled bool Если true, сигнал запускается в неактивном/выключенном состоянии.

Позиция и направление

pos определяет, где размещается логическая точка сигнала.

"pos": [734.165, 755.369, 177.949]

dir определяет направление, в котором действует сигнал.

"dir": [-0.925, 0.378, -0.005]

Для знака «Стоп» или светофора dir как правило должен указывать в направлении движения транспортных средств, подъезжающих к сигналу или проезжающих мимо него.

Система сигналов использует pos и dir для:

  • Поиска ближайшего/наиболее подходящего участка дороги
  • Определения того, проехало ли транспортное средство мимо сигнала
  • Привязки состояния сигнала к рёбрам навигационного графа
  • Отправки команд сигнала ИИ транспортного средства

Группы сигналов

Необязательное поле group объединяет связанные сигналы, как правило на одном перекрёстке.

Пример:

"group": "group_8_9_10_11"

Группы — это преимущественно метаданные редактора / организационная информация. Редактор может автоматически группировать экземпляры, находящиеся рядом, использующие одну последовательность и обращённые к одной и той же зоне перекрёстка.

Контроллеры сигналов

Контроллеры определяют тип поведения сигнала для экземпляра.

Пример контроллера знака «Стоп»:

{
  "id": 2,
  "name": "stop",
  "type": "signStop",
  "isSimple": true,
  "states": [
    {
      "state": "basicStop"
    }
  ]
}

Поля контроллера

Поле Тип Описание
id number Уникальный идентификатор контроллера.
name string Имя контроллера.
type string Ключ типа контроллера из определений контроллеров.
isSimple bool Если true, контроллеру не требуется логика временно́й последовательности.
defaultIndex number Необязательный индекс состояния по умолчанию.
states array Упорядоченный список состояний контроллера.

Состояния контроллера

Каждый контроллер содержит упорядоченный список состояний:

"states": [
  {
    "state": "greenTrafficLight",
    "duration": 12
  },
  {
    "state": "yellowTrafficLight",
    "duration": 4
  },
  {
    "state": "redTrafficLight",
    "duration": 1
  }
]

Поля состояния контроллера

Поле Тип Описание
state string Ключ состояния из определений контроллеров.
duration number Длительность данного состояния в секундах. Необязательно для простых контроллеров.

Отрицательная длительность интерпретируется как бесконечная в логике редактора/среды выполнения.

Простые контроллеры

Простым контроллерам не требуется последовательность.

Пример:

{
  "id": 2,
  "name": "stop",
  "type": "signStop",
  "isSimple": true,
  "states": [
    {
      "state": "basicStop"
    }
  ]
}

Экземпляры сигналов, использующие простые контроллеры, могут применять:

"sequenceId": 0

Это типично для:

  • Знаков «Стоп»
  • Знаков «Уступи дорогу»
  • Статических дорожных знаков
  • Простых сигналов, работающих постоянно

Последовательности

Последовательности определяют временны́е фазы сигналов.

Они используются преимущественно для светофоров на перекрёстках.

Пример:

{
  "id": 10,
  "name": "intersection_sequence",
  "startTime": 0,
  "startDisabled": false,
  "ignoreTimer": false,
  "phases": [
    {
      "controllerIds": [3]
    },
    {
      "controllerIds": [4]
    }
  ]
}

Поля последовательности

Поле Тип Описание
id number Уникальный идентификатор последовательности.
name string Имя последовательности.
phases array Упорядоченный список фаз.
startTime number Начальное смещение в секундах. Может быть отрицательным для пропуска вперёд.
startDisabled bool Если true, последовательность запускается в отключённом/выключенном состоянии.
ignoreTimer bool Если true, последовательность не продвигается вместе с основным таймером.

Фазы

Фаза определяет, какие контроллеры работают одновременно.

{
  "controllerIds": [3, 4]
}

Каждый контроллер может использоваться в последовательности только один раз.

Последовательность может содержать несколько фаз:

"phases": [
  { "controllerIds": [3] },
  { "controllerIds": [4] }
]

Длительность фазы вычисляется по наибольшей длительности контроллера в этой фазе.

Расширенная настройка времени начала фаз

Фаза может дополнительно определять startTime:

{
  "startTime": 5,
  "controllerIds": [3]
}

Это позволяет фазам перекрываться или начинаться с явными смещениями внутри временно́й шкалы последовательности.

Если startTime не задан, фазы выполняются последовательно одна за другой.

Определения контроллеров

Имена type и state контроллера разрешаются из данных определений контроллеров.

Определения по умолчанию загружаются из:

settings/trafficSignals.json

Уровень может переопределить или дополнить их с помощью:

levels/<levelName>/signalControllerDefinitions.json

При загрузке уровня система сигналов:

  1. Загружает определения по умолчанию.
  2. Ищет signalControllerDefinitions.json в папке уровня.
  3. Объединяет пользовательские states и types с определениями по умолчанию.
  4. Загружает signals.json.

signalControllerDefinitions.json

Этот необязательный файл определяет пользовательские состояния сигналов и типы контроллеров для уровня.

Упрощённый пример:

{
  "states": {
    "customFlashingYellow": {
      "name": "Custom Flashing Yellow",
      "action": "slow",
      "duration": 3,
      "lights": ["yellow"],
      "flashingLights": [
        ["yellow"],
        ["black"]
      ],
      "flashingInterval": 0.5
    }
  },
  "types": {
    "customWarningLight": {
      "name": "Custom Warning Light",
      "states": ["customFlashingYellow"],
      "defaultIndex": 1,
      "isSimple": true
    }
  }
}

Поля определения состояния

Поле Тип Описание
name string Отображаемое имя состояния.
action string Действие ИИ/навигации, связанное с этим состоянием.
duration number Длительность по умолчанию в секундах.
lights array[string] Видимые цвета огней для этого состояния.
flashingLights array[array[string]] Необязательная последовательность массивов цветов огней для мигающего поведения.
flashingInterval number Интервал между сменами цвета мигающих огней.

Поля определения типа

Поле Тип Описание
name string Отображаемое имя типа контроллера.
states array[string] Упорядоченный список ключей состояний, используемых данным типом.
defaultIndex number Индекс состояния по умолчанию.
isSimple bool Определяет, можно ли использовать данный тип без последовательности.

Действия сигналов

Состояния сигналов могут определять action. Это действие передаётся ИИ транспортного средства через навигационные данные сигналов.

Известные названия действий:

Действие Значение
alert Поведение предупреждения/осторожности.
stop Поведение остановки.
briefStop Поведение кратковременной остановки.
slow Поведение снижения скорости/осторожности.
none Никаких специальных действий.

Внутри системы эти действия преобразуются в числовые значения и передаются в Lua транспортного средства.

Цвета огней

Состояния сигналов определяют видимые цвета огней с помощью строк.

Распространённые цвета:

red
amber
yellow
green
cyan
blue
white
black

black означает выключено.

Пример:

"lights": ["red", "black", "black"]

Объекты дорожных сигналов используют поля цветов экземпляра:

instanceColor
instanceColor1
instanceColor2
instanceColor3

Система сигналов включает и выключает эти значения в зависимости от текущего состояния.

Привязка объектов сигналов

Экземпляр сигнала — это логические данные. Видимые 3D-объекты светофора/знака представляют собой отдельные объекты сцены.

Видимые объекты сигналов, как правило, являются объектами TSStatic с динамическим полем:

signalInstance

Значение должно совпадать с именем экземпляра сигнала.

Пример:

signalInstance = "traffic light 1"

При смене состояния сигнала обновляются все связанные объекты с данным значением signalInstance.

Если экземпляр сигнала не имеет связанных объектов, он всё равно может влиять на ИИ/навигацию, однако визуальный меш светофора меняться не будет.

Обработка во время выполнения

При запуске уровня система сигналов:

  1. Загружает определения контроллеров.
  2. Загружает signals.json.
  3. Создаёт экземпляры, контроллеры и последовательности.
  4. Ожидает загрузки навигационного графа.
  5. Находит наиболее подходящий участок дороги для каждого экземпляра сигнала.
  6. Строит таблицу соответствия узлов карты и сигналов.
  7. Активирует последовательности и таймеры.
  8. Отправляет обновления сигналов ИИ транспортного средства.

Система сигналов зависит от навигационного графа. При перезагрузке навгрaфа дорожные сигналы финализируются повторно.

Сигналы узлов карты

Во время выполнения сигналы привязываются к рёбрам графа дорог.

Для каждого экземпляра сигнала система определяет наиболее подходящий участок дороги и сохраняет такие данные, как:

  • Имя экземпляра
  • Позиция
  • Состояние
  • Действие
  • Необязательное использование полосы

Эти данные передаются в Lua транспортного средства, чтобы ИИ-автомобили могли реагировать на знаки «Стоп», светофоры и другие сигналы.

Работа в редакторе

Используйте:

World Editor → Gameplay → Traffic Signals Editor

Редактор предоставляет вкладки для работы с:

  • Сигналами
  • Контроллерами
  • Последовательностями
  • Симуляцией

Редактор позволяет:

  • Создавать экземпляры сигналов
  • Перемещать и поворачивать экземпляры сигналов
  • Связывать видимые объекты сигналов
  • Создавать/редактировать контроллеры
  • Создавать/редактировать временны́е последовательности
  • Симулировать синхронизацию сигналов
  • Проверять ошибки сигналов
  • Сохранять signals.json
  • Редактировать пользовательские определения контроллеров

Базовый рабочий процесс

  1. Откройте редактор дорожных сигналов.
  2. Создайте или загрузите signals.json.
  3. Создайте контроллеры или используйте существующие типы контроллеров.
  4. Разместите экземпляры сигналов в мире.
  5. Назначьте каждому сигналу контроллер.
  6. Назначьте последовательность, если это тактируемый светофор.
  7. Свяжите видимые объекты сигналов TSStatic.
  8. Запустите симуляцию для проверки состояний.
  9. Сохраните файл.

Рекомендации

  • По возможности используйте редактор дорожных сигналов вместо ручного редактирования.
  • Обеспечивайте уникальность идентификаторов.
  • Обеспечивайте уникальность имён.
  • Используйте простые контроллеры для знаков «Стоп» и статических знаков.
  • Используйте последовательности для тактируемых светофоров.
  • Убедитесь, что экземпляры сигналов расположены рядом с действительными дорогами.
  • Направьте dir вдоль направления подъезда/проезда транспортных средств.
  • Связывайте видимые объекты сигналов с помощью signalInstance.
  • Используйте группы для перекрёстков.
  • Перед сохранением проверьте работу в режиме симуляции.
  • Выполняйте проверку через Tools → Check Signal Errors.

Распространённые проблемы

Сигнал не влияет на ИИ

Возможные причины:

  • Сигнал расположен слишком далеко от действительного участка дороги.
  • Поле controllerId отсутствует или содержит недопустимое значение.
  • Поле sequenceId содержит недопустимое значение.
  • Навигационный граф не был загружен/перезагружен.
  • Направление сигнала задано неверно.

Меш светофора не меняет огни

Возможные причины:

  • Отсутствует связанный объект TSStatic.
  • У объекта нет корректного динамического поля signalInstance.
  • Значение signalInstance не совпадает с именем сигнала.
  • Меш/материал не использует ожидаемые поля цветов экземпляра.

Сигнал отображается как ошибка в отладчике

Возможные причины:

  • Отсутствует контроллер.
  • Отсутствует последовательность.
  • Не удалось найти ближайший узел дороги.
  • Дублирующееся имя элемента.
  • Дублирующийся идентификатор.

Последовательность не продвигается

Возможные причины:

  • ignoreTimer имеет значение true.
  • Последовательность отключена.
  • Длительности контроллеров отсутствуют или бесконечны.
  • Длительность последовательности равна нулю.

Знак «Стоп» требует последовательность

Контроллеры знака «Стоп» / простые контроллеры сигналов обычно не требуют последовательности. Используйте:

"sequenceId": 0

Пользовательский тип сигнала не найден

Проверьте:

signalControllerDefinitions.json

и убедитесь, что type контроллера существует.

Итог

signals.json определяет логику дорожных сигналов уровня.

Файл содержит:

  • instances — размещённые точки сигналов
  • controllers — списки поведения/состояний сигналов
  • sequences — логика временны́х фаз

Система привязывает экземпляры сигналов к ближайшим рёбрам навигационного графа, обновляет видимые объекты сигналов через signalInstance и передаёт действия сигналов ИИ транспортного средства.

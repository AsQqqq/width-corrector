В этом документе описываются данные Forest Brush, используемые редактором леса World Editor Forest Editor.

Forest Brush — это группы правил размещения, существующие только в редакторе и предназначенные для рисования лесных объектов на уровне. Кисть может содержать один или несколько объектов ForestBrushElement. Каждый элемент ссылается на тип ForestItemData и определяет, как именно данный объект должен размещаться случайным образом.

Forest Brush не хранит непосредственно размещённые экземпляры леса. Это инструменты для создания данных о размещении леса.

Обзор

Forest Brush — это контейнер из одного или нескольких элементов кисти.

Каждый ForestBrushElement управляет:

  • Каким типом лесного объекта можно рисовать
  • Вероятностью выбора данного элемента
  • Диапазоном случайного поворота
  • Диапазоном случайного масштаба
  • Ограничениями по уклону
  • Ограничениями по высоте
  • Настройкой погружения/глубины

Это позволяет одним мазком кисти нарисовать естественно выглядящую смесь деревьев, камней, кустарников, пучков травы и других лесных объектов.

Примеры применения:

  • Кисти смешанного леса
  • Кисти случайного рассыпания камней
  • Кисти кустарника/травяного подлеска
  • Растительность, привязанная к высоте
  • Скальные камни, ограниченные по уклону
  • Случайная плотность леса и вариации масштаба

Типы объектов

Объект Назначение
ForestBrush Контейнер для одного или нескольких объектов ForestBrushElement.
ForestBrushElement Определяет настройки размещения для одного типа лесного объекта.
ForestItemData Тип лесного объекта, на который ссылается элемент кисти.

ForestBrush

ForestBrush — это класс-контейнер для объектов ForestBrushElement.

Он используется только в редакторе и хранится внутри специальной группы:

ForestBrushGroup

Если ForestBrushGroup не существует, движок создаёт её автоматически.

ForestBrush принимает только дочерние объекты типа ForestBrushElement. Объекты других типов игнорируются при добавлении в кисть.

ForestBrushElement

ForestBrushElement представляет один возможный тип лесного объекта в кисти.

Например, кисть «смешанный сосновый лес» может содержать:

  • Элемент маленькой сосны
  • Элемент средней сосны
  • Элемент большой сосны
  • Элемент кустарника
  • Элемент камня

Каждый элемент имеет собственные настройки вероятности, масштаба, уклона и высоты.

Поля

Поле Тип Значение по умолчанию Описание
forestItemData ForestItemData null Тип лесного объекта, который размещает данный элемент.
probability number 1 Относительная вероятность выбора данного элемента во время рисования.
rotationRange number 360 Максимальный диапазон случайного поворота в градусах.
scaleMin number 1 Минимальный случайный масштаб объекта.
scaleMax number 1 Максимальный случайный масштаб объекта.
scaleExponent number 1 Показатель степени смещения при выборе случайного масштаба.
sinkMin number 0 Минимальная вариация величины погружения.
sinkMax number 0 Максимальная вариация величины погружения.
sinkRadius number 1 Радиус, используемый для расчёта погружения в terrain, особенно на склонах.
slopeMin number 0 Минимальный уклон поверхности в градусах, при котором может быть размещён данный объект.
slopeMax number 90 Максимальный уклон поверхности в градусах, при котором может быть размещён данный объект.
elevationMin number -10000 Минимальная высота в мировых координатах, на которой может быть размещён данный объект.
elevationMax number 10000 Максимальная высота в мировых координатах, на которой может быть размещён данный объект.

Вероятность

probability управляет тем, как часто элемент выбирается относительно других элементов в той же кисти.

Само по себе это не абсолютный процент. Вместо этого шанс рассчитывается как вероятность данного элемента, делённая на сумму всех вероятностей в кисти.

Пример:

Элемент Вероятность Приблизительный шанс
Маленькая сосна 3 50%
Большая сосна 2 33%
Камень 1 17%

Суммарная вероятность:

3 + 2 + 1 = 6

Таким образом:

Pine small = 3 / 6 = 50%
Pine large = 2 / 6 = 33%
Rock       = 1 / 6 = 17%

Используйте более высокую вероятность для распространённых объектов и более низкую для редких акцентных объектов.

Диапазон поворота

rotationRange управляет случайным поворотом по рысканью (yaw).

Пример:

"rotationRange": 360

Это обеспечивает полный случайный поворот и полезно для деревьев, камней, кустарников и большинства природных объектов.

Меньшие значения ограничивают вариацию поворота:

"rotationRange": 30

Это может быть полезно для объектов, которым требуется более единообразная ориентация.

Диапазон масштаба

scaleMin и scaleMax определяют случайный масштаб объекта.

Пример:

"scaleMin": 0.8,
"scaleMax": 1.3

Это создаёт вариативность, чтобы повторяющиеся ресурсы не выглядели одинаково.

Типичное применение:

Тип ресурса Рекомендуемый диапазон масштаба
Деревья 0.8 - 1.4
Кустарники 0.7 - 1.5
Камни 0.6 - 1.8
Пучки травы 0.8 - 1.2

Показатель масштаба

scaleExponent смещает распределение случайного масштаба между scaleMin и scaleMax.

Значение по умолчанию:

"scaleExponent": 1

Значение 1 даёт нормальное линейное распределение.

Более высокие или более низкие значения могут смещать результат в сторону меньших или больших экземпляров в зависимости от распределения случайных величин, используемого редактором.

Используйте этот параметр, когда хотите, чтобы большинство объектов располагалось ближе к одному краю диапазона масштабов, а не были равномерно распределены.

Настройки погружения

Настройки погружения помогают естественно вписать объекты в terrain.

Поле Описание
sinkMin Минимальная величина погружения.
sinkMax Максимальная величина погружения.
sinkRadius Радиус, используемый для расчёта погружения у основания объекта.

Это полезно для:

  • Деревьев на склонах
  • Камней, частично утопленных в terrain
  • Кустарников, которые не должны парить в воздухе
  • Крупных объектов с неровными основаниями

Пример:

"sinkMin": 0.05,
"sinkMax": 0.2,
"sinkRadius": 1.5

Это позволяет размещать объект немного ниже поверхности terrain, чтобы избежать видимых зазоров.

Используйте значения погружения осторожно. Слишком большое погружение может скрыть корни, основания или нижнюю геометрию объекта.

Ограничения по уклону

slopeMin и slopeMax ограничивают размещение на основе уклона поверхности.

Пример для обычных деревьев:

"slopeMin": 0,
"slopeMax": 35

Пример для скальных камней:

"slopeMin": 25,
"slopeMax": 80

Это помогает контролировать, где появляются объекты:

  • Деревья на более ровном terrain
  • Камни на крутых склонах
  • Трава на пологих холмах
  • Отсутствие крупной растительности на скалах

Ограничения по высоте

elevationMin и elevationMax ограничивают размещение на основе высоты в мировых координатах.

Пример:

"elevationMin": 200,
"elevationMax": 800

Это полезно для управления биомами:

  • Низинная растительность
  • Горные деревья
  • Альпийские камни
  • Прибрежные растения
  • Растительность у снеговой линии

Пример настройки кисти

Кисть смешанного леса может содержать три элемента:

{
  "class": "ForestBrush",
  "name": "mixed_pine_forest"
}

Примеры элементов:

{
  "class": "ForestBrushElement",
  "forestItemData": "pine_small",
  "probability": 4,
  "rotationRange": 360,
  "scaleMin": 0.8,
  "scaleMax": 1.2,
  "slopeMin": 0,
  "slopeMax": 35,
  "elevationMin": -10000,
  "elevationMax": 10000
}

{
  "class": "ForestBrushElement",
  "forestItemData": "pine_large",
  "probability": 2,
  "rotationRange": 360,
  "scaleMin": 0.9,
  "scaleMax": 1.4,
  "slopeMin": 0,
  "slopeMax": 30
}

{
  "class": "ForestBrushElement",
  "forestItemData": "rock_medium",
  "probability": 1,
  "rotationRange": 360,
  "scaleMin": 0.6,
  "scaleMax": 1.8,
  "sinkMin": 0.05,
  "sinkMax": 0.2,
  "sinkRadius": 1.2,
  "slopeMin": 10,
  "slopeMax": 75
}

В данной кисти:

  • Маленькие сосны встречаются часто
  • Большие сосны встречаются реже
  • Камни появляются время от времени
  • Деревья избегают крутых склонов
  • Камни могут появляться на более крутом terrain

Связь с ForestItemData

forestItemData должен ссылаться на действительное определение лесного объекта.

Данные лесного объекта обычно определяются в:

levels/<levelName>/art/forest/managedItemData.json

Пример данных объекта:

{
  "pine_small": {
    "class": "ForestItemData",
    "internalName": "pine_small",
    "shapeFile": "/levels/example/art/shapes/trees/pine_small.dae"
  }
}

Ссылка в элементе кисти:

"forestItemData": "pine_small"

Если указанные данные объекта не существуют, элемент кисти не сможет разместить этот объект.

Рекомендации

  • Используйте несколько элементов кисти для естественной вариативности.
  • Используйте вероятность для управления соотношением распространённых и редких объектов.
  • Используйте диапазоны масштаба для избежания повторяемости.
  • Используйте ограничения по уклону, чтобы деревья не появлялись на скалах.
  • Используйте ограничения по высоте для управления биомами.
  • Используйте небольшие значения погружения для камней и деревьев на склонах.
  • Задавайте windScale в ForestItemData, а не в кисти, для управления поведением ветра.
  • Держите элементы кисти сфокусированными; создавайте отдельные кисти для разных биомов или групп ресурсов.

Распространённые проблемы

Кисть не размещает объект

Возможные причины:

  • forestItemData отсутствует или недействителен
  • Указанный ForestItemData не существует
  • Уклон выходит за пределы slopeMin / slopeMax
  • Высота выходит за пределы elevationMin / elevationMax
  • Вероятность слишком низкая по сравнению с другими элементами

Объекты парят над terrain

Немного увеличьте значения погружения или проверьте начало координат/точку опоры ресурса.

Объекты слишком глубоко утоплены

Уменьшите sinkMin, sinkMax или sinkRadius.

Слишком много одинаково выглядящих объектов

Увеличьте вариацию поворота, расширьте диапазон масштаба или добавьте больше элементов кисти.

Объекты появляются на скалах

Уменьшите slopeMax.

Камни не появляются на склонах

Увеличьте slopeMax или поднимите slopeMin в зависимости от желаемого диапазона размещения.

Итог

Forest Brush — это инструменты размещения, существующие только в редакторе и используемые в Forest Editor.

Кисть содержит один или несколько объектов ForestBrushElement. Каждый элемент ссылается на тип ForestItemData и определяет, как данный объект должен размещаться случайным образом с использованием правил вероятности, поворота, масштаба, погружения, уклона и высоты.

Они используются для эффективного рисования естественно выглядящих лесов, камней, кустарников и другой повторяющейся растительности на уровне.

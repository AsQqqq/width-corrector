Этот документ описывает файлы террейна BeamNG и сопутствующие файлы метаданных.

Данные террейна хранятся преимущественно в бинарном файле .ter. При сохранении террейна движок также записывает сопутствующий файл .terrain.json, описывающий данные террейна и используемые материалы.

Террейн в уровне представлен объектом сцены TerrainBlock. Объект TerrainBlock ссылается на файл .ter через поле terrainFile.

На этой странице описаны файлы террейна и данные материалов террейна. Описание объекта сцены, размещающего террейн в уровне, см. в разделе TerrainBlock .

Роли файлов

Файл Назначение
.ter Бинарный файл террейна, содержащий данные карты высот, данные карты слоёв и имена материалов террейна.
.terrain.json Файл метаданных, описывающий файл .ter. Полезен для инструментов и отладки.
.terrainheightmap.png Необязательная/экспортированная ссылка на изображение карты высот, записываемая при сохранении террейна.
items.level.json Содержит объект сцены TerrainBlock, ссылающийся на файл .ter.
*.materials.json Содержит определения TerrainMaterial, на которые ссылается файл .ter.

Типичное расположение файлов

Файлы террейна обычно хранятся внутри папки уровня.

levels/<levelName>/

Пример:

levels/example/terrain.ter
levels/example/terrain.terrain.json
levels/example/terrain.terrainheightmap.png

В некоторых старых уровнях или инструментах файлы террейна могут располагаться непосредственно в папке уровня, однако рекомендуется использовать отдельную папку для террейна.

Сам объект террейна хранится в данных сцены уровня, например:

levels/example/main/items.level.json

Объект TerrainBlock

Террейн размещается в уровне с помощью объекта TerrainBlock.

Пример:

{
  "class": "TerrainBlock",
  "name": "theTerrain",
  "terrainFile": "/levels/example/art/terrains/terrain.ter",
  "squareSize": 1,
  "maxHeight": 2048,
  "baseTexSize": 1024,
  "lightMapSize": 256,
  "screenError": 16,
  "castShadows": true,
  "position": [-1024, -1024, 0],
  "rotationMatrix": [1, 0, 0, 0, 1, 0, 0, 0, 1],
  "scale": [1, 1, 1]
}

Важные поля TerrainBlock

Поле Тип Описание
terrainFile string Путь к файлу данных террейна .ter.
minimapImage string Необязательное изображение миникарты для данного блока террейна.
materialTextureSet string Необязательный TerrainMaterialTextureSet, используемый новыми путями рендеринга террейна.
castShadows bool Определяет, отбрасывает ли террейн тени.
squareSize number Расстояние между образцами карты высот в метрах.
maxHeight number Максимальная высота террейна в метрах.
baseTexSize integer Разрешение генерируемой базовой текстуры террейна.
lightMapSize integer Размер текстуры лайтмапа террейна.
screenError integer Настройка LOD/ошибки экрана террейна.
Масштабирование объекта террейна отключено. Для управления размерами террейна используйте squareSize и maxHeight вместо масштаба объекта.

Размеры террейна

Файл .ter хранит квадратную сетку:

size x size

Ширина террейна в мировом пространстве приблизительно равна:

world size = size x squareSize

Пример:

size = 2048
squareSize = 1
world size = 2048 m x 2048 m

Если squareSize равен 2, та же сетка террейна охватывает:

4096 m x 4096 m

Масштаб высоты

Высоты террейна хранятся как беззнаковые 16-битные значения (u16).

Объект TerrainBlock преобразует сохранённые значения высоты в метры по формуле:

heightScale = maxHeight / 65536
heightMeters = storedHeight x heightScale

Пример с:

"maxHeight": 2048

Масштаб высоты равен:

2048 / 65536 = 0.03125 m

Таким образом, одна единица сохранённой высоты составляет приблизительно 0.03125 м.

Затем к этой высоте добавляется позиция объекта террейна.

Бинарный формат .ter

Файл .ter — основной бинарный файл данных террейна. Он хранит данные высот террейна, индексы слоёв материалов и список имён материалов террейна.

Файл читается последовательно от начала до конца.

Текущая структура сохранения

Текущий путь TerrainFile::save() записывает следующую бинарную структуру:

Шаг Поле Тип Размер Описание
1 version u8 1 байт Версия файла террейна.
2 size u32 4 байта Ширина/высота квадратной сетки террейна.
3 heightMap u16[size x size] size x size x 2 байта Образцы высоты.
4 layerMap u8[size x size] size x size байт Индекс материала террейна для каждого образца.
5 materialCount u32 4 байта Количество имён материалов террейна.
6 materialNames string array переменный Внутренние имена материалов террейна.

Псевдокод текущего сохранения:

write((u8)FILE_VERSION);
write(mSize);

write(mHeightMap.byteSize(), mHeightMap.address());
write(mLayerMap.byteSize(), mLayerMap.address());

write((u32)mMaterials.size());

for each material:
    write(material.internalName);
Текущая реализация предполагает данные в порядке little-endian.

Стратегия чтения

Для чтения текущего файла .ter:

  1. Прочитать version как u8.
  2. Проверить, что version <= FILE_VERSION.
  3. Прочитать size как u32.
  4. Вычислить sampleCount = size x size.
  5. Прочитать sampleCount значений u16 как карту высот.
  6. Прочитать sampleCount значений u8 как карту слоёв.
  7. Прочитать materialCount как u32.
  8. Прочитать materialCount строк.
  9. Разрешить имена материалов в определения TerrainMaterial.
  10. Построить сетку/квадродерево террейна для рендеринга и коллизий.

Упрощённый псевдокод:

u8 version;
read(version);

u32 size;
read(size);

u32 sampleCount = size * size;

u16 heightMap[sampleCount];
read(heightMap, sampleCount * sizeof(u16));

u8 layerMap[sampleCount];
read(layerMap, sampleCount * sizeof(u8));

u32 materialCount;
read(materialCount);

for i in materialCount:
    string materialName = readString();

Обработка версий

Первый байт — это версия файла террейна.

При загрузке:

  • Если version > FILE_VERSION, загрузка завершается с ошибкой.
  • Если version >= 7, используется современный путь загрузки.
  • Если version < 7, используется устаревший путь загрузки.

Для обычных инструментов и нового контента ориентируйтесь на описанную выше текущую структуру сохранения.

Загрузчик содержит пути совместимости для старых версий террейна. Они предназначены главным образом для загрузки старого контента и не должны использоваться как образец для новых файлов.

Кодирование карты высот

Карта высот хранится как:

u16 heightMap[size x size]

Каждый образец — беззнаковое 16-битное целое число.

Преобразование в метры:

heightMeters = storedHeight x (maxHeight / 65536)

Пример:

storedHeight = 32768
maxHeight = 2048

heightMeters = 32768 x (2048 / 65536)
heightMeters = 1024 m

Кодирование карты слоёв

Карта слоёв хранится как:

u8 layerMap[size x size]

Каждое значение — индекс в списке имён материалов террейна.

Пример:

0 = first terrain material
1 = second terrain material
2 = third terrain material

Значение 255 (U8_MAX) является специальным:

255 = empty terrain / hole

Пустой террейн используется системой террейна для пропуска рендеринга и коллизий в этих областях.

Имена материалов

После карты высот и карты слоёв файл хранит имена материалов террейна:

u32 materialCount
string materialNames[materialCount]

Эти имена разрешаются при загрузке с помощью:

TerrainMaterial::findOrCreate(name)

Если допустимые материалы не найдены, террейн переключается на предупреждающий материал.

Ограничение количества материалов

Карта слоёв террейна использует индексы u8, причём значение 255 зарезервировано для пустого террейна. Загрузчик поддерживает до 254 записей материалов террейна.

Материалы сверх этого лимита игнорируются.

В интересах производительности и удобства сопровождения следует держать количество материалов террейна значительно ниже технического предела.

Оценка размера файла

Для текущего файла террейна, без учёта строк имён материалов:

1 byte                   version
4 bytes                  size
size x size x 2 bytes    heightMap
size x size x 1 byte     layerMap
4 bytes                  materialCount
variable                 material names

Для 2048 x 2048:

heightMap = 2048 x 2048 x 2 = 8 MiB
layerMap  = 2048 x 2048 x 1 = 4 MiB

Итого данных карты:

~12 MiB

плюс небольшой заголовок и строки имён материалов.

Метаданные .terrain.json

При сохранении террейна движок записывает сопутствующий файл .terrain.json.

Пример:

{
  "version": 8,
  "datafile": "/levels/example/art/terrains/terrain.ter",
  "heightmapImage": "/levels/example/art/terrains/terrain.terrainheightmap.png",
  "size": 2048,
  "binaryFormat": "version(char), size(unsigned int), heightMap(heightMapSize * heightMapItemSize), layerMap(layerMapSize * layerMapItemSize), layerTextureMap(layerMapSize * layerMapItemSize), materialNames",
  "heightMapSize": 4194304,
  "heightMapItemSize": 2,
  "layerMapSize": 4194304,
  "layerMapItemSize": 1,
  "materials": [
    "grass",
    "rock",
    "asphalt"
  ]
}

Поля метаданных

Поле Тип Описание
version number Версия файла террейна.
datafile string Путь к бинарному файлу .ter.
heightmapImage string Путь к связанному/экспортированному изображению карты высот.
size number Размер сетки террейна.
binaryFormat string Человекочитаемое описание бинарной структуры террейна.
heightMapSize number Количество образцов карты высот.
heightMapItemSize number Размер каждого образца карты высот в байтах.
layerMapSize number Количество образцов карты слоёв.
layerMapItemSize number Размер каждого образца карты слоёв в байтах.
materials array[string] Внутренние имена материалов террейна.
Файл .terrain.json содержит описательные метаданные для инструментов и отладки. Движок загружает фактические данные террейна из файла .ter, на который ссылается объект TerrainBlock.
Строка binaryFormat может упоминать данные совместимости, используемые в старых версиях террейна. Для вновь сохранённых террейнов используйте текущую структуру сохранения, описанную в этом документе.

Материалы террейна

Файл .ter хранит только имена материалов террейна. Сами определения материалов — это отдельные объекты TerrainMaterial, хранящиеся в файле JSON с материалами.

Материалы террейна существуют в двух версиях: классический путь v1 и современный PBR-процесс v1.5. В разделах ниже описана версия v1.5 — как она включается, как работает и какие поля предоставляет.

Включение материалов террейна v1.5

Материалы террейна существуют в двух версиях:

Версия Когда используется Модель текстур
v1 (классическая) TerrainBlock.materialTextureSet не задан. Одиночные diffuseMap, normalMap, detailMap, macroMap.
v1.5 (PBR) TerrainBlock.materialTextureSet ссылается на TerrainMaterialTextureSet. baseColor, normal, roughness, ao, height, каждый с текстурами base/macro/detail.

Весь террейн переключается между версиями на основе настроек TerrainBlock. Флага версии для отдельного материала не существует: назначение materialTextureSet объекту TerrainBlock активирует v1.5 для каждого материала, нанесённого на данный террейн.

Обновление с v1 до v1.5

В World Editor в разделе Terrain Material Library доступно действие Upgrade Terrain Materials. Оно:

  • Создаёт объект TerrainMaterialTextureSet (сохраняется в art/terrains/main.materials.json).
  • Назначает его полю TerrainBlock.materialTextureSet.
  • Очищает устаревшие поля v1 (diffuseMap, normalMap, detailMap, macroMap) из существующих материалов террейна.
Обновление необратимо. Классические текстурные назначения v1 не переносятся в слоты v1.5, поэтому каждый материал террейна необходимо переработать с новыми текстурами base/macro/detail после обновления. После этого сохраните уровень, чтобы зафиксировать изменение TerrainBlock.

Файл материалов террейна

Материалы террейна v1.5 и объект TerrainMaterialTextureSet хранятся в:

levels/<levelName>/art/terrains/main.materials.json

В старых уровнях материалы террейна могут всё ещё храниться в art/terrains/materials.json. Это устаревшая структура; редактор предлагает действие Upgrade Terrain Material file format, которое перемещает их в main.materials.json.

Принцип работы материалов террейна v1.5

Материал террейна v1.5 формируется из пяти групп текстур, и каждая группа сэмплируется на трёх масштабах, которые объединяются в итоговую поверхность. Понимание этой модели — ключ к созданию качественного террейна.

Кратко (для первого материала): создайте текстуры baseColor, normal, roughness, ao и height, а для каждой из них подготовьте версии base (общий вид), macro (вариации среднего диапазона) и detail (крупный план). Назначьте их в Terrain Material Library World Editor, установите groundmodelName (например, GRASS) и оставьте значения дистанции/силы на типичных значениях по умолчанию, показанных ниже. В подразделах ниже объясняется тонкая настройка.

Пять групп текстур

Группа Что контролирует Цветовое пространство Как создавать
Base color Альбедо (цвет поверхности). sRGB Нарисуйте реальный цвет поверхности (зелёная трава, серая скала, …).
Normal Рельеф поверхности / неровности (карта нормалей в касательном пространстве). Linear Карта нормалей. Хранятся только красный/зелёный каналы; синий восстанавливается шейдером.
Roughness Глянцевость. Тёмный = глянцевый/мокрый, светлый = шероховатый/матовый. Linear (оттенки серого) Карта шероховатости в оттенках серого.
Ambient occlusion Самозатенение в углублениях при рассеянном освещении. Linear (оттенки серого) Карта AO в оттенках серого; белый = нет окклюзии.
Height Попиксельная высота, используемая для смешивания слоёв (см. ниже). Linear (оттенки серого) Карта высот в оттенках серого; белый = выступ, чёрный = углубление.

Масштабы base, macro и detail

Внутри одной группы текстуры base, macro и detail — это три тайловых слоя с разными масштабами, которые сэмплируются и объединяются попиксельно. Проще всего представить их как низко-, средне- и высокочастотные версии одной и той же поверхности:

Масштаб Типичный размер маппинга (*TexSize) Повторяемость Когда сэмплируется Роль
Base Большой — часто на весь террейн (≈ 128–2048 м) Низкая (большой тайл, иногда уникальный) Всегда, на любом расстоянии. Основа. Задаёт общий крупномасштабный вид и фактическое значение группы.
Macro Средний (≈ 30–80 м) Средняя В пределах macroDistances, не дальше 1000 м. Вариации среднего масштаба, разбивающие монотонность base на средней/большой дистанции.
Detail Малый (≈ 2–8 м) Высокая (мелкий тайл) В пределах detailDistances, не дальше 250 м. Мелкие, чёткие детали крупного плана.

Base — основной слой и единственный, задающий абсолютное значение (остальные его модифицируют). В официальных уровнях base — как правило, одна большая текстура, замапленная на весь террейн — зачастую одна и та же гарнитура t_terrain_base_*, общая для всех материалов (например, West Coast USA маппит её на 2048 м для всех материалов). Поскольку один тайл охватывает весь террейн, видимого повторения нет, но вблизи текстура мягкая/низкого разрешения — именно это исправляет слой detail.

Macro — второй слой с меньшим размером маппинга (обычно 30–80 м). Его задача — скрыть повторяемость base: крупные, мягкие пятна цвета или яркости, заметные на склоне холма, но не на отдельном тайле. Наиболее полезен на средней и большой дистанции; может затухать с помощью macroDistances.

Detail — третий слой с очень малым размером маппинга (обычно 2–8 м), поэтому часто повторяется и остаётся чётким прямо под камерой или транспортным средством (отдельные травинки, камушки, мелкие нормальные неровности). Поскольку высокочастотная тайловость становится одновременно заметной и дорогостоящей на расстоянии, detail затухает по detailDistances и никогда не сэмплируется дальше 250 м.

Что означает размер маппинга

*BaseTexSize, *MacroTexSize и *DetailTexSize — это мировые расстояния в метрах для одного тайла текстуры, а не её разрешение в пикселях.

  • Бо́льший размер → крупнее тайл → меньше повторов → широкий вид, мало видимой тайловости, но размытее вблизи.
  • Меньший размер → меньше тайл → больше повторов → чёткость вблизи, но тайловость становится заметной на расстоянии.

В официальном контенте base обычно равен размеру всего террейна, macro — около 30–80 м, а detail — около 2–8 м. (Если поле размера не задано, движок использует значения по умолчанию: 256 / 60 / 2 м для base / macro / detail.)

Зачем три масштаба

Одна тайловая текстура не может хорошо выглядеть на всех расстояниях: сделать тайл крупным — будет размытой вблизи; сделать мелким — тайловость станет заметной по всему террейну. Разделение поверхности на base + macro + detail позволяет террейну одновременно оставаться чётким там, где камера близко (detail), разнообразным на среднем расстоянии (macro) и свободным от очевидного повторения вдали (base).

Вкратце, по дистанции:

far   : base (+ macro)         -> broad, no obvious tiling
mid   : base + macro           -> base broken up by variation
near  : base + macro + detail  -> full crispness

Как использовать

  • Все масштабы обязательны: каждая группа должна иметь назначенные текстуры base, macro и detail. Редактор материалов террейна сообщает о пустом слоте как об ошибке валидации и не сохранит материал, пока у всех пяти групп не будут назначены все три текстуры.
  • Создавайте base color base как реальный цвет поверхности; создавайте текстуры macro/detail вокруг среднего серого, чтобы они добавляли только вариации (см. Как масштабы объединяются ).
  • Чтобы фактически «отключить» слой macro или detail в группе, всё равно назначьте текстуру, но сделайте её нейтральной — плоский средний серый для цвета/данных или плоская нормаль — чтобы она ничего не добавляла.
  • Используйте detail для добавления чёткости крупного плана и держите его дистанции короткими для повышения производительности.
  • Используйте macro для устранения видимой тайловости на больших открытых пространствах; мягкие, низкоконтрастные текстуры macro обычно выглядят лучше всего.

Как масштабы объединяются

Для цветовых и данных-групп (base color, roughness, AO, height) базовая текстура задаёт значение, а macro/detail добавляются как знаковые наложения вокруг среднего серого:

final = base + (macro  - 0.5) * 2 * macroStrength
             + (detail - 0.5) * 2 * detailStrength

По этой причине текстуры macro и detail должны создаваться вокруг среднего серого (0.5): серый = без изменений, более светлые области осветляют базу, более тёмные — затемняют. Для группы normal нормали base/macro/detail смешиваются, а не суммируются.

Смешивание слоёв на основе высоты

Это наиболее важное поведение, которое нужно понять. Когда два материала террейна перекрываются (там, где один нанесён поверх другого), движок не просто выполняет их перекрёстное затухание. Он сравнивает текстуру height каждого слоя попиксельно и пропускает тот, у которого высота больше.

Это создаёт естественные переходы — например, гравий проступает там, где он находится «выше» песка, вместо мягкого размытого шва.

Для использования этого: создавайте карту высот так, чтобы части, которые должны проявляться первыми при смешивании (камни, рельефный рисунок, камни), были светлее, а углублённые части (швы в кладке, карманы с песком) — темнее.

Настройка дистанции, силы и затухания

Эти поля управляют затуханием macro и detail с расстоянием камеры — они важны как для внешнего вида, так и для производительности.

  • macroDistances / detailDistances — [startFadeIn, near, far, endFadeOut] в метрах (по возрастанию). Вклад плавно включается на промежутке startFadeIn → near, остаётся полным между near и far, затем плавно выключается на промежутке far → endFadeOut.
  • *MacroStrength / *DetailStrength — [near, far] интенсивность наложения (0–1), позволяющая делать слой сильным вблизи и слабее вдали.
  • macroDistAtten / detailDistAtten — [near, far] (0–1), определяющие, насколько вклад спадает на краях затухания (1 = полностью затухает до нуля, 0 = остаётся).
Независимо от этих кривых, рендерер жёстко останавливает сэмплирование detail дальше 250 м и macro дальше 1000 м. Установка endFadeOut выше этого значения их не расширит — официальный контент часто оставляет endFadeOut большим (например, 3000) и полагается на эти ограничения, управляя реальным затуханием через значение far.

Типичные значения в официальных уровнях:

Поле Распространённое значение Значение
macroDistances [0, 10, 100, 3000] Macro полностью виден до ~100 м, присутствует до ограничения 1000 м.
detailDistances [0, 0, 30, 60] (или большой endFadeOut) Detail полностью виден до ~30 м, исчезает задолго до ограничения 250 м.
*MacroStrength [0.2, 0.4] (цвет), [0.5, 0.6] (нормаль) Слабый для цвета, сильнее для нормали.
*DetailStrength [0.3, 0] (цвет), [0.8, 0.15] (нормаль) Сильный вблизи, затухает до нуля вдали.
macroDistAtten / detailDistAtten [1, 1] Плавное включение из нуля и выключение до нуля.

Практические рекомендации:

  • Держите значение far для detail коротким (обычно 20–50 м). Он высокочастотный, полезен только вблизи, а его раннее отключение также экономит производительность.
  • Используйте macro для устранения очевидной тайловости базовой текстуры на средней и большой дистанции.
  • Значения силы macro/detail для height обычно оставляют равными 0 — высота управляет смешиванием слоёв , поэтому её модуляция через macro/detail редко нужна.

Проекция на склоны и глубина

  • useSideProjection — проецирует текстуру на вертикальные грани вместо проекции прямо вниз, чтобы скалы и крутые камни не выглядели растянутыми.
  • parallaxScale — добавляет эффект параллакса/самоперекрытия глубины из данных карты высот/нормалей.
Параметры useSideProjection и parallaxScale относятся к классическому пути v1 террейна. Рендерер v1.5 (и редактор материалов v1.5) их не использует, поэтому они не оказывают эффекта на террейн с назначенным materialTextureSet.

Граундмодель и аннотация

  • groundmodelName — связывает закрашенную область с граундмоделью, которая определяет физическую поверхность: трение, частицы шин, звуки заноса, сопротивление качению и т.д. Соотносите с внешним видом (GRASS, ROCK, ASPHALT, …). См. Groundmodels .
  • annotation — семантический/отладочный класс (например, GRASS, NATURE), используемый проходом рендеринга аннотаций для инструментов, таких как семантическая сегментация и сенсоры. По умолчанию NATURE.

Оба поля используются как классическими материалами v1, так и v1.5.

Поля TerrainMaterial v1.5

Объект TerrainMaterial версии v1.5 определяется полями текстур для каждой группы и несколькими полями уровня материала. О том, что делает каждое поле и как его использовать, см. в разделе Принцип работы материалов террейна v1.5 ; данный раздел — краткий справочник по именам и типам.

Значения хранятся как числа и массивы JSON: одиночные значения — как числа (512, 0.3), векторы — как массивы ([0.3, 0], [0, 10, 100, 3000]).

Поля групп текстур

Каждая из пяти групп — baseColor, normal, roughness, ao, height — предоставляет одинаковые поля. Имя поля формируется из названия группы и масштаба (Base, Macro, Detail):

Шаблон поля Тип Примечания
<group><Scale>Tex path Исходная текстура для данной группы/масштаба, например baseColorBaseTex, normalDetailTex.
<group><Scale>TexSize number Мировой размер маппинга в метрах (как правило, весь террейн для base, 30–80 для macro, 2–8 для detail).
<group>MacroStrength number[2] [near, far] Интенсивность наложения macro, 0..1.
<group>DetailStrength number[2] [near, far] Интенсивность наложения detail, 0..1.

Например, группа baseColor раскрывается в baseColorBaseTex, baseColorMacroTex, baseColorDetailTex, baseColorBaseTexSize, baseColorMacroTexSize, baseColorDetailTexSize, baseColorMacroStrength и baseColorDetailStrength.

Поля уровня материала

Поле Тип Примечания
internalName string Уникальное имя, на которое ссылается список материалов файла .ter.
macroDistances number[4] [startFadeIn, near, far, endFadeOut] Кривая затухания по дистанции (м), общая для всех текстур macro. Сэмплирование останавливается на ограничении 1000 м. Типичное значение: [0, 10, 100, 3000].
detailDistances number[4] [startFadeIn, near, far, endFadeOut] Кривая затухания по дистанции (м), общая для всех текстур detail. Сэмплирование останавливается на ограничении 250 м. Типичное значение: [0, 0, 30, 60].
macroDistAtten number[2] [near, far] Степень затухания macro на краях затухания, 0..1. Типичное значение: [1, 1].
detailDistAtten number[2] [near, far] Степень затухания detail на краях затухания, 0..1. Типичное значение: [1, 1].
groundmodelName string Физическая граундмодель (трение, частицы, звуки).
annotation string Семантический/отладочный класс для прохода аннотаций. По умолчанию NATURE.
useSideProjection bool Только для классического v1 — игнорируется рендерером v1.5.
parallaxScale number Только для классического v1 — игнорируется рендерером v1.5.

TerrainMaterialTextureSet

Объект TerrainMaterialTextureSet определяет ожидаемые размеры текстурных массивов, используемых рендерером материалов террейна v1.5.

Все материалы террейна, нанесённые на террейн, упаковываются в небольшой набор общих текстурных массивов GPU (по одному срезу на материал), а текстурный массив требует, чтобы каждый срез имел одинаковые размеры. Объект TerrainMaterialTextureSet объявляет эти размеры для слотов base, macro и detail. Именно поэтому каждая текстура, назначенная в данный слот, должна точно соответствовать объявленному размеру — несовпадение не позволяет упаковать материал, и он переключается на предупреждающую текстуру.

Как правило, на уровень приходится один объект TerrainMaterialTextureSet (с именем вида <levelName>TerrainMaterialTextureSet), общий для всех материалов террейна этого уровня.

Объект TerrainBlock ссылается на него через:

"materialTextureSet": "myTerrainTextureSet"

Важные поля (размер в пикселях как [width, height]):

Поле Описание Типичное значение
baseTexSize Размер в пикселях для массива базовых текстур. [512, 512] – [4096, 4096] (выше, когда base — уникальная карта на весь террейн)
macroTexSize Размер в пикселях для массива текстур macro. [1024, 1024]
detailTexSize Размер в пикселях для массива текстур detail. [1024, 1024]

Пример (из официальной структуры уровня):

{
  "templateTerrainMaterialTextureSet": {
    "class": "TerrainMaterialTextureSet",
    "name": "templateTerrainMaterialTextureSet",
    "baseTexSize": [512, 512],
    "macroTexSize": [1024, 1024],
    "detailTexSize": [1024, 1024]
  }
}

Рендерер использует эти размеры при упаковке текстур материалов террейна в текстурные массивы GPU.

Все текстуры, назначенные в один тип слота, должны соответствовать ожидаемому размеру, заданному в TerrainMaterialTextureSet. Например, все базовые текстуры должны соответствовать baseTexSize.

Упаковка текстур террейна

В пути материалов террейна v1.5 исходные текстуры упаковываются в генерируемые кэшированные текстуры и копируются в текстурные массивы террейна.

Сгенерированные файлы кэша могут храниться по путям вида:

/temp/art/terrainMaterialCache/<hash>.dds

Хэш основан на путях исходных текстур, поэтому одна и та же комбинация текстур может повторно использоваться.

Рендерер упаковывает данные террейна в две основные группы текстур:

Группа base color

Эта группа объединяет:

  • RGB базового цвета
  • Ambient occlusion

Типичное использование каналов:

R/G/B = base color
A     = ambient occlusion

Группа normal/data

Эта группа объединяет:

  • Roughness
  • Height
  • Данные нормали

Типичное использование каналов:

R = roughness
G = height
B/A = normal-related data
Упакованная текстура генерируется движком автоматически. Авторам следует предоставлять корректные исходные текстуры с совпадающими размерами и правильным использованием цвета/данных.

Если исходная текстура отсутствует, имеет неверный размер или не содержит ожидаемого канала, упаковка материала террейна может завершиться сбоем или вывести предупреждающий материал.

Требования к исходным текстурам

При создании текстур террейна v1.5:

  • Используйте исходные текстуры в формате PNG.
  • Размеры текстур в пикселях должны быть степенью двойки и точно совпадать с соответствующим размером слота в TerrainMaterialTextureSet (baseTexSize, macroTexSize или detailTexSize).
  • Текстуры base color должны быть в цветовом пространстве sRGB. Текстуры normal, roughness, ambient occlusion и height должны быть в линейном пространстве.

Рекомендуемые форматы источников для каждого слота:

Группа слота Цветовое пространство Рекомендуемый формат источника
Base color sRGB R8G8B8 / R8G8B8A8
Normal Linear R8G8B8 / R8G8B8A8
Roughness / AO / Height Linear R8 (оттенки серого)

Библиотека материалов террейна в World Editor проверяет эти правила и сообщает о несовпадениях размеров, отсутствующих текстурах и неожиданных форматах.

Рендеринг террейна

Рендерер террейна делит террейн на ячейки и строит квадродерево.

Важное значение:

minimum terrain cell size = 64

Система ячеек используется для:

  • Отсечения
  • Выбора LOD
  • Ограничивающих объёмов
  • Отслеживания материалов
  • Рендеринга теней
  • Рендеринга отражений
  • Проходов prepass / G-buffer / аннотации

Данные ячеек террейна загружаются в GPU-буферы для рендеринга.

LOD террейна

LOD террейна основан на ошибке экрана и расстоянии до ячейки.

Соответствующие поля/настройки:

Поле Описание
screenError Настройка ошибки экрана террейна.
$pref::Terrain::lodScale Глобальный масштаб LOD террейна.
$pref::Terrain::detailScale Глобальный масштаб дистанции деталей террейна.

Юбки террейна

Ячейки террейна генерируют дополнительную геометрию «юбки» вокруг краёв ячеек.

Юбки помогают скрыть трещины между ячейками террейна при использовании разных уровней LOD.

Эта геометрия генерируется автоматически и не хранится в файле .ter.

Пустой террейн / отверстия

Значение индекса слоя 255 (U8_MAX) отмечает пустой террейн.

Пустой террейн влияет на:

  • Рендеринг
  • Коллизии
  • Генерацию примитивов ячеек террейна
  • Операции SDF/булевого террейна

Рендерер может пропускать пустые квадраты террейна, генерируя пользовательский буфер примитивов для затронутых ячеек.

Коллизии

Коллизии террейна строятся на основе карты высот.

При изменении высоты террейна:

  • Данные сетки обновляются
  • Ограничивающие объёмы пересчитываются
  • Ячейки рендеринга обновляются
  • Пересборка физических коллизий может быть поставлена в очередь

Коллизии террейна используются:

  • Транспортными средствами
  • Пропами
  • Рейкастами
  • Определением земли
  • Системами ИИ/навигации

Импорт террейна

Террейн можно импортировать из карты высот и карт непрозрачности/слоёв.

Требования:

  • Карта высот должна быть квадратной
  • Разрешение карты высот должно быть степенью двойки
  • Размер карты высот должен быть от 128 до 8192
  • Карты непрозрачности/слоёв должны совпадать по размеру с картой высот
  • Количество имён материалов должно совпадать с количеством слоёв непрозрачности

Импорт карты высот

Поддерживаемое поведение при импорте карты высот:

  • Карты высот R16 читаются непосредственно как 16-битные данные высоты.
  • Другие форматы изображений конвертируются из 8-битных значений в полный диапазон u16.

Конвертация из 8-битного формата:

storedHeight = pixelValue / 255 x 65535

Импорт слоёв непрозрачности

Импорт использует слои непрозрачности для построения карты слоёв террейна.

Каждый слой непрозрачности может поступать из канала текстуры:

R
G
B
A

Для каждого образца террейна побеждает слой материала с наибольшим значением непрозрачности.

Псевдокод:

for each sample:
    bestLayer = 0
    bestValue = 0

    for each opacity layer:
        if opacityValue >= bestValue:
            bestLayer = layer
            bestValue = opacityValue

    layerMap[sample] = bestLayer

Импорт карты отверстий

Необязательная карта отверстий может пометить образцы террейна как пустые.

Если значение карты отверстий равно 0xFF, значение карты слоёв становится:

255

что означает пустой террейн.

Переворот по оси Y

Импорт поддерживает параметр flipYAxis.

Если он отключён, импортёр переворачивает входные данные по вертикали при копировании в память террейна.

Используйте этот параметр, если импортированный террейн выглядит перевёрнутым по вертикали.

Создание террейна

При создании нового террейна движок:

  1. Выбирает уникальное имя файла .ter.
  2. Создаёт объект TerrainFile.
  3. Добавляет начальный материал.
  4. Округляет разрешение террейна до следующей степени двойки.
  5. Инициализирует карту высот и карту слоёв.
  6. Сохраняет файл .ter.
  7. Записывает файл метаданных .terrain.json.
  8. Создаёт/регистрирует объект TerrainBlock.

Начальная высота нового террейна устанавливается выше нуля, чтобы у редактора было пространство для выемки грунта.

Минимальный пример TerrainBlock 

{
  "class": "TerrainBlock",
  "name": "theTerrain",
  "terrainFile": "/levels/example/art/terrains/terrain.ter",
  "squareSize": 1,
  "maxHeight": 2048,
  "baseTexSize": 1024,
  "lightMapSize": 256,
  "screenError": 16,
  "castShadows": true,
  "position": [-1024, -1024, 0],
  "rotationMatrix": [1, 0, 0, 0, 1, 0, 0, 0, 1],
  "scale": [1, 1, 1]
}

Минимальный пример .terrain.json 

{
  "version": 8,
  "datafile": "/levels/example/art/terrains/terrain.ter",
  "heightmapImage": "/levels/example/art/terrains/terrain.terrainheightmap.png",
  "size": 1024,
  "binaryFormat": "version(char), size(unsigned int), heightMap(heightMapSize * heightMapItemSize), layerMap(layerMapSize * layerMapItemSize), materialNames",
  "heightMapSize": 1048576,
  "heightMapItemSize": 2,
  "layerMapSize": 1048576,
  "layerMapItemSize": 1,
  "materials": [
    "grass",
    "rock"
  ]
}

Рекомендации

  • Используйте разрешение террейна, кратное степени двойки.
  • Используйте размеры террейна от 128 до 8192.
  • Используйте squareSize для управления масштабом в мировом пространстве.
  • Используйте maxHeight для управления вертикальным диапазоном.
  • Не масштабируйте объект TerrainBlock.
  • Держите количество материалов террейна управляемым.
  • Используйте допустимые внутренние имена TerrainMaterial.
  • Держите карты непрозрачности/слоёв того же размера, что и карта высот.
  • Используйте значения слоя 255 только для намеренных отверстий.
  • Пересохраняйте старые файлы террейна для их обновления.
  • Проверяйте коллизии террейна после значительного редактирования.
  • Используйте подготовленные текстуры DDS для классических материалов террейна.
  • Держите размеры текстурных наборов материалов террейна единообразными.

Распространённые проблемы

Террейн не загружается

Возможные причины:

  • Недопустимый путь terrainFile
  • Отсутствует файл .ter
  • Версия файла террейна новее, чем поддерживает движок
  • Повреждённые бинарные данные

Террейн отображается с предупреждающим материалом

Возможные причины:

  • Имя материала в .ter не удаётся разрешить
  • Отсутствует определение TerrainMaterial
  • Путь к текстуре материала террейна недопустим
  • Не удаётся найти materialTextureSet

Высота террейна неверна

Проверьте:

  • maxHeight
  • squareSize
  • Позицию объекта террейна
  • Масштаб импорта карты высот
  • Настройку переворота по оси Y

Закраска террейна выглядит неправильно

Возможные причины:

  • Индекс карты слоёв не совпадает с порядком материалов
  • Порядок материалов изменился
  • Карты непрозрачности неверны
  • Кэш текстур устарел
  • Исходные текстуры имеют неверный размер

В террейне есть отверстия

Значения карты слоёв могут быть равны 255, что отмечает террейн как пустой.

Коллизии террейна устарели

Пересоберите коллизии террейна или сохраните/перезагрузите уровень после значительного редактирования высот.

Импорт завершается с ошибкой

Проверьте, что:

  • Карта высот квадратная
  • Разрешение карты высот является степенью двойки
  • Размер карты высот находится в диапазоне от 128 до 8192
  • Карты непрозрачности совпадают по размеру с картой высот
  • Количество материалов совпадает с количеством слоёв непрозрачности

Итог

Террейн BeamNG использует:

  • Бинарный файл .ter для карты высот, карты слоёв и имён материалов
  • Файл метаданных .terrain.json для документации/инструментов
  • Объект сцены TerrainBlock для размещения террейна в уровне
  • Отдельные определения TerrainMaterial для визуальных и физических данных материала

Файл .ter компактен и быстро загружается, тогда как файл .terrain.json помогает инструментам понять структуру данных террейна.

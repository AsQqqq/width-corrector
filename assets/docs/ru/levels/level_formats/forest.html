В этом документе описывается современный формат расстановки леса BeamNG, используемый уровнями:

  • forest/*.forest4.json - файлы расстановки объектов леса
  • art/forest/managedItemData.json - определения типов объектов леса

Эти файлы работают совместно. managedItemData.json определяет что можно разместить, тогда как файлы .forest4.json определяют где на уровне размещается каждый объект.

Обзор

Система леса используется для большого количества повторяющихся статичных объектов, таких как:

  • Деревья
  • Кусты
  • Пучки травы
  • Камни
  • Брёвна
  • Мелкие декоративные растения

Объекты леса оптимизированы для рендеринга и обработки столкновений. Как правило, они не предназначены для индивидуального перемещения или анимации в процессе игры, однако могут поддерживать анимацию ветра через настройки ассета и параметры объекта леса.

Типичный уровень использует:

levels/example_level/
├── art/
│   └── forest/
│       └── managedItemData.json
└── forest/
    ├── trees.forest4.json
    ├── rocks.forest4.json
    └── bushes.forest4.json

Роли файлов

Файл Назначение
art/forest/managedItemData.json Определяет типы объектов леса, например описания деревьев или камней.
forest/*.forest4.json Хранит размещённые экземпляры этих типов объектов.
art/forest/managedItemData.cs Устаревший формат TorqueScript, по-прежнему поддерживается как запасной вариант.
*.forest / *.forest.json Старые форматы расстановки леса, устаревшие.
Движок отдаёт предпочтение managedItemData.json, если этот файл существует. Если он отсутствует, используется managedItemData.cs.

managedItemData.json

managedItemData.json определяет многократно используемые типы объектов леса. Каждый тип объекта ссылается на файл формы и необязательные физические параметры и параметры ветра.

Расположение 

levels/<levelName>/art/forest/managedItemData.json

Базовая структура

Файл представляет собой JSON-объект. Каждая запись верхнего уровня содержит блок данных объекта леса.

{
  "oak_large": {
    "class": "ForestItemData",
    "internalName": "oak_large",
    "shapeFile": "/levels/example/art/shapes/trees/oak_large.dae",
    "mass": 100,
    "rigidity": 10,
    "tightnessCoefficient": 0.5,
    "dampingCoefficient": 0.2,
    "windScale": 1,
    "trunkBendScale": 0.5,
    "branchAmp": 1,
    "detailAmp": 0.25,
    "detailFreq": 1
  }
}

Поддерживаемые имена классов:

ForestItemData
TSForestItemData

Оба варианта принимаются загрузчиком.

Поля managedItemData.json

Поле Тип Описание
class string Должно быть ForestItemData или TSForestItemData.
internalName string Уникальное имя типа объекта, используемое файлами расстановки .forest4.json.
shapeFile string Путь к форме, используемой для данного объекта леса.
mass number Физическая масса объекта.
rigidity number Управляет жёсткостью при взаимодействии с ветром.
tightnessCoefficient number Управляет тем, насколько быстро объект возвращается в состояние покоя.
dampingCoefficient number Управляет затуханием движения.
windScale number Масштабирует силу воздействия ветра на объект.
trunkBendScale number Управляет величиной изгиба ствола.
branchAmp number Управляет амплитудой анимации ветра для веток.
detailAmp number Управляет амплитудой анимации ветра для мелких деталей.
detailFreq number Управляет частотой анимации ветра для мелких деталей.

Не все поля обязательны, однако internalName и shapeFile являются необходимыми.

Имена объектов леса

internalName — это связующее звено между определениями объектов и размещёнными экземплярами.

Пример определения объекта:

"pine_small": {
  "class": "ForestItemData",
  "internalName": "pine_small",
  "shapeFile": "/levels/example/art/shapes/trees/pine_small.dae"
}

Пример расстановки, ссылающейся на него:

{"type":"pine_small","pos":[10,20,0],"rotationMatrix":[1,0,0,0,1,0,0,0,1],"scale":1}

Если расстановка ссылается на type, которого не существует в managedItemData.json, этот экземпляр не может быть разрешён.

Файлы расстановки .forest4.json

Файлы .forest4.json хранят размещённые экземпляры объектов леса.

Расположение 

levels/<levelName>/forest/*.forest4.json

Пример:

levels/example/forest/trees.forest4.json

Формат файла

Файлы .forest4.json представляют собой JSON с разделением по строкам.

Это означает, что каждая строка является отдельным JSON-объектом. Файл не является единым JSON-массивом.

Пример:

{"type":"pine_small","pos":[10,20,0],"rotationMatrix":[1,0,0,0,1,0,0,0,1],"scale":1}
{"type":"pine_small","pos":[15,25,0],"rotationMatrix":[0,-1,0,1,0,0,0,0,1],"scale":0.85}
{"type":"rock_large","pos":[30,12,0],"rotationMatrix":[1,0,0,0,1,0,0,0,1],"scale":1.4}
Не оборачивайте содержимое .forest4.json в [ ]. Каждая строка должна быть самостоятельным JSON-объектом.

Поля .forest4.json

Поле Тип Описание
type string Тип объекта леса. Должен совпадать с internalName из managedItemData.json.
pos array[3] Мировая позиция [x, y, z].
rotationMatrix array[9] Матрица поворота 3×3, хранящаяся в виде 9 чисел.
scale number Равномерный масштаб объекта.

Пример экземпляра 

{
  "type": "oak_large",
  "pos": [100.5, 42.0, 12.3],
  "rotationMatrix": [
    1, 0, 0,
    0, 1, 0,
    0, 0, 1
  ],
  "scale": 1.0
}

Процесс загрузки

При загрузке уровня движок примерно следует этому процессу:

  1. Поиск данных объектов леса:

    • art/forest/managedItemData.json
    • запасной вариант — art/forest/managedItemData.cs

  2. Разбор всех допустимых определений объектов леса.

  3. Поиск в папке леса уровня:

forest/*.forest4.json

  1. Построчное чтение каждого файла .forest4.json.

  2. Для каждой расстановки:

    • Чтение type
    • Поиск совпадающего internalName в данных объектов
    • Разрешение shapeFile
    • Чтение позиции, поворота и масштаба
    • Добавление экземпляра в систему леса

Устаревшие форматы леса

Движок по-прежнему может обнаруживать старые файлы леса:

*.forest
*.forest.json

Эти форматы устарели. Если присутствуют старые файлы, движок может выводить предупреждения вида:

Old forest file format detected. Please resave the forest and delete the old files.

Современные уровни должны использовать:

*.forest4.json

Устаревший managedItemData.cs

Если файл managedItemData.json отсутствует, движок может перейти на использование:

art/forest/managedItemData.cs

Это более старый формат в стиле TorqueScript.

Пример:

datablock ForestItemData(oak_large)
{
   internalName = "oak_large";
   shapeFile = "/levels/example/art/shapes/trees/oak_large.dae";
   mass = "100";
   rigidity = "10";
   windScale = "1";
};

Для нового контента предпочтительно использовать managedItemData.json.

Параметры ветра

Объекты леса могут реагировать на ветер, если ассет и материал поддерживают это. Значения данных объекта управляют силой реакции объекта на ветер.

Основные поля, связанные с ветром:

Поле Описание
windScale Общее влияние ветра. Используйте 0 для статичных объектов, таких как камни.
trunkBendScale Величина крупномасштабного изгиба ствола.
branchAmp Амплитуда движения веток.
detailAmp Амплитуда движения мелких деталей.
detailFreq Частота мелких колебаний от ветра.

Пример для дерева:

{
  "class": "ForestItemData",
  "internalName": "pine_large",
  "shapeFile": "/levels/example/art/shapes/trees/pine_large.dae",
  "windScale": 1,
  "trunkBendScale": 0.6,
  "branchAmp": 1,
  "detailAmp": 0.25,
  "detailFreq": 1.5
}

Пример для камня:

{
  "class": "ForestItemData",
  "internalName": "rock_large",
  "shapeFile": "/levels/example/art/shapes/rocks/rock_large.dae",
  "windScale": 0
}

Коллизия и физика

Объекты леса могут создавать коллизию в зависимости от своей формы и настроек объекта. Движок создаёт данные коллизии для каждой ячейки леса и может повторно использовать данные коллизии для одинаковых форм.

Это важно для производительности: множество экземпляров, использующих одинаковый shapeFile, могут совместно использовать кэшированные данные коллизии.

Для достижения наилучших результатов:

  • По возможности используйте простые сетки коллизии.
  • Избегайте чрезмерно сложной коллизии для густой растительности.
  • Отключайте коллизию для мелких декоративных элементов растительности, когда она не нужна.
  • Используйте правильные типы поверхностей материалов для поверхностей коллизии.

Замечания по производительности

Система леса разработана для большого количества повторяющихся статичных объектов, однако производительность по-прежнему зависит от качества ассетов.

Важные факторы:

  • Количество уникальных ассетов shapeFile
  • Количество треугольников
  • Количество материалов
  • Сложность коллизии
  • Плотность расстановки
  • Использование анимации ветра
  • Дальность прорисовки и LOD
  • Поддержка импостеров

Используйте редактор леса и инструменты отладки для проверки плотности расстановки и производительности.

Рекомендации

  • Используйте managedItemData.json для новых уровней.
  • Используйте файлы расстановки .forest4.json.
  • Храните каждый .forest4.json в формате JSON с разделением по строкам.
  • Убедитесь, что каждый type расстановки совпадает с каким-либо internalName.
  • Повторно используйте типы объектов вместо дублирования похожих определений.
  • Используйте LOD и импостеры для деревьев и крупной растительности.
  • Делайте коллизию простой.
  • Устанавливайте windScale в 0 для объектов, которые не должны двигаться.
  • Удаляйте старые файлы .forest и .forest.json после обновления.
  • Следите за тем, чтобы пути к формам были корректными и согласованными.

Распространённые проблемы

Объекты леса не отображаются

Возможные причины:

  • Отсутствует managedItemData.json
  • type расстановки не совпадает ни с одним internalName
  • Недопустимый путь shapeFile
  • Недопустимая JSON-строка в .forest4.json
  • Файл сохранён как JSON-массив вместо JSON с разделением по строкам

Некоторые объекты леса отсутствуют

Проверьте вывод консоли/журнала на наличие недопустимых строк или отсутствующих типов объектов. Одна неверная строка может привести к ошибке, тогда как остальные строки загрузятся успешно.

Объект леса имеет неправильный поворот

Убедитесь, что rotationMatrix содержит 9 допустимых числовых значений.

Объект леса имеет неправильный размер

Проверьте поле scale. В .forest4.json используется единственное значение равномерного масштаба.

Ветер не воздействует на объект

Возможные причины:

  • windScale равен 0
  • Ассет не поддерживает деформацию от ветра
  • Настройка цвета вершин не обеспечивает правильное управление ветром
  • Материал/шейдер не поддерживает функции ветра

Старые файлы леса по-прежнему загружаются

Если уровень содержит как старые, так и новые файлы леса, удалите устаревшие файлы после проверки обновлённых данных .forest4.json.

Минимальный пример

art/forest/managedItemData.json 

{
  "pine_small": {
    "class": "ForestItemData",
    "internalName": "pine_small",
    "shapeFile": "/levels/example/art/shapes/trees/pine_small.dae",
    "windScale": 1,
    "trunkBendScale": 0.4,
    "branchAmp": 1,
    "detailAmp": 0.25,
    "detailFreq": 1
  },
  "rock_large": {
    "class": "ForestItemData",
    "internalName": "rock_large",
    "shapeFile": "/levels/example/art/shapes/rocks/rock_large.dae",
    "windScale": 0
  }
}

forest/forest.forest4.json 

{"type":"pine_small","pos":[0,0,0],"rotationMatrix":[1,0,0,0,1,0,0,0,1],"scale":1}
{"type":"pine_small","pos":[5,2,0],"rotationMatrix":[0,-1,0,1,0,0,0,0,1],"scale":0.9}
{"type":"rock_large","pos":[10,4,0],"rotationMatrix":[1,0,0,0,1,0,0,0,1],"scale":1.5}

Итог

Современный формат леса использует две составляющие:

  • managedItemData.json определяет многократно используемые типы объектов леса.
  • Файлы .forest4.json хранят размещённые экземпляры в формате JSON с разделением по строкам.

Такая структура позволяет движку эффективно рендерить, группировать, обрабатывать столкновения и управлять большим количеством повторяющихся статичных объектов на уровне.

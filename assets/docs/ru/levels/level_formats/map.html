Этот документ описывает необязательный файл map.json, используемый уровнями BeamNG для добавления вручную заданных сегментов навигационного графа.

Навигационная карта используется системами ИИ, трафика, навигации, генерации маршрутов и смежными игровыми механиками. Большинство навигационных данных генерируется автоматически из объектов DecalRoad и BeamNGWaypoint, однако map.json позволяет создателям уровней вручную задавать дополнительные сегменты дорог/путей.

Расположение файла

map.json помещается в папку уровня:

levels/<levelName>/map.json

Пример:

levels/west_coast_usa/map.json

Если файл map.json отсутствует, навигационный граф всё равно может быть сгенерирован из объектов DecalRoad и BeamNGWaypoint.

Назначение

Используйте map.json, когда автоматической генерации дорожного графа недостаточно.

Типичные сценарии использования:

  • Тоннели
  • Мосты
  • Путепроводы
  • Сложные перекрёстки
  • Дороги, плохо представленные объектами DecalRoad
  • Пути с ручным управлением направлением движения
  • Частные/закрытые дороги
  • Специальные коннекторы для ИИ/навигации
  • Скрытые навигационные связи

map.json не определяет визуальную геометрию дороги. Он задаёт только навигационные связи между существующими нодами в графе.

Связь с DecalRoad и BeamNGWaypoint

Навигационный граф строится из нескольких источников:

  1. Объекты BeamNGWaypoint
  2. Объекты DecalRoad с drivability > 0
  3. Ручные сегменты из map.json

Сегменты map.json ссылаются на существующие имена нод. Ими могут быть:

  • Имена объектов BeamNGWaypoint
  • Ноды, сгенерированные из объектов DecalRoad
  • Другие существующие навигационные ноды

Если нода, указанная в map.json, не существует, сегмент не может быть создан в полном объёме, и в лог может быть записана ошибка.

Базовая структура

map.json содержит объект верхнего уровня segments.

{
  "segments": {
    "bridge1": {
      "nodes": ["Bridge1_A", "Bridge1_B"],
      "drivability": 1
    },

    "tunnel_A": {
      "nodes": ["tunnel_A_1", "tunnel_A_2", "tunnel_A_3"],
      "oneWay": true,
      "flipDirection": false,
      "lanesLeft": 1,
      "drivability": 1
    }
  }
}

Каждый ключ внутри segments является именем сегмента. Имя используется главным образом для организации и отладки.

Поля сегмента

Поле Тип По умолчанию Описание
nodes array/string обязательное Упорядоченный список имён нод, образующих сегмент.
drivability number 1 Предпочтение/множитель стоимости при прокладке пути. Меньшие значения — менее предпочтительный путь.
oneWay bool false Является ли сегмент односторонним.
flipDirection bool false Разворачивает логическое направление сегмента.
speedLimit number авто Необязательное значение скорости в метрах в секунду, используемое системами ИИ/навигации. Значения <= 0 трактуются как «не задано».
type string nil Метаданные типа дороги, например private.
gatedRoad bool false Обрабатывает сегмент как данные частной/закрытой дороги.
hiddenInNavi bool false Скрывает или убирает акцент на сегмент в навигационном отображении там, где это поддерживается.
autoLanes bool true Если задано false, количество полос читается из lanesLeft и lanesRight.
lanesLeft integer 0 Ручное количество полос для одной стороны, когда autoLanes равно false.
lanesRight integer 0 Ручное количество полос для другой стороны, когда autoLanes равно false.
autoJunction bool true Если задано false, предотвращает автоматическое слияние на нодах этого сегмента.

Ноды

Поле nodes задаёт упорядоченный путь сегмента.

Пример:

"nodes": ["Bridge1_A", "Bridge1_B", "Bridge1_C"]

Это создаёт следующие связи:

Bridge1_A → Bridge1_B
Bridge1_B → Bridge1_C

Для двусторонних сегментов прокладка пути возможна в обоих направлениях. Для односторонних — направление движения зависит от oneWay и flipDirection.

Краткая запись нод в виде строки

Поле nodes также может быть записано в виде строки.

Пример:

"nodes": "tunnel_A_1-tunnel_A_10"

Это разворачивается в:

tunnel_A_1
tunnel_A_2
tunnel_A_3
...
tunnel_A_10

Несколько диапазонов или отдельных нод можно разделить запятыми:

"nodes": "Bridge1_A, tunnel_A_1-tunnel_A_10, Bridge1_B"
Для раскрытия диапазона оба конца должны использовать одинаковый текстовый префикс и числовой суффикс.

Направление

oneWay

oneWay помечает сегмент как односторонний.

"oneWay": true

По умолчанию направление движения следует порядку списка nodes.

Пример:

"nodes": ["A", "B", "C"],
"oneWay": true

Это создаёт односторонний путь от:

A → B → C

flipDirection

flipDirection разворачивает логическое направление сегмента.

"flipDirection": true

Пример:

"nodes": ["A", "B", "C"],
"oneWay": true,
"flipDirection": true

Это создаёт односторонний путь от:

C → B → A

Используйте это, когда порядок нод удобен для редактирования, но разрешённое направление движения — противоположное.

Проходимость (Drivability)

drivability определяет, насколько желателен сегмент при прокладке пути.

"drivability": 1

Типичные значения:

Значение Смысл
1 Обычная дорога/путь.
0.5 Менее предпочтительный путь.
< 0.5 Настоятельно нежелательный путь.
0 Крайне нежелательный, однако внутренняя защита предотвращает нулевую стоимость.

Стоимость навигации возрастает по мере снижения проходимости, поэтому системы ИИ и маршрутизации при возможности предпочитают дороги с более высокой проходимостью.

Ограничение скорости

speedLimit может задаваться для каждого сегмента отдельно:

"speedLimit": 13.89

Если speedLimit отсутствует или <= 0, движок автоматически оценивает ограничение скорости на основе ширины/радиуса дороги, её проходимости и направления.

"speedLimit": 0

означает «рассчитать автоматически».

Если вы не уверены, опустите speedLimit и позвольте движку рассчитать его самостоятельно. Ограничение скорости задаётся в метрах в секунду.

Тип дороги и частные дороги

Сегмент может задавать тип дороги:

"type": "private"

Также можно использовать:

"gatedRoad": true

Если gatedRoad равно true, сегмент обрабатывается как данные частной/закрытой дороги.

Частные дороги влияют на поведение маршрутизации и трафика. Когда частные дороги соединяются с общественными, навигационный граф может помечать соединяющие рёбра как закрытые, чтобы прокладка пути обрабатывала их иначе.

Используйте частные/закрытые дороги для:

  • Ограниченных зон
  • Грунтовых дорог
  • Служебных дорог
  • Парковок
  • Гоночных трасс
  • Дорог, не предназначенных для общего пользования

Пример:

"dirttrack": {
  "nodes": ["dirttrack_1", "dirttrack_2", "dirttrack_3"],
  "oneWay": true,
  "drivability": 1,
  "gatedRoad": true
}

Скрытые навигационные сегменты

hiddenInNavi может скрывать сегмент из навигационного отображения там, где это поддерживается:

"hiddenInNavi": true

Это может быть полезно для:

  • Внутренних коннекторов
  • Служебных связей
  • Вспомогательных путей только для ИИ
  • Скрытых связей в тоннелях
  • Навигационных маршрутов, не предназначенных для пользователя

Сегмент может по-прежнему существовать в графе и использоваться системами в зависимости от контекста.

Данные о полосах

Информация о полосах может генерироваться автоматически или задаваться вручную.

Автоматические полосы

По умолчанию полосы генерируются исходя из ширины/радиуса дороги и правил дорожного движения.

"autoLanes": true

или не указываются вовсе.

Движок использует правила дорожного движения уровня из info.json:

"roadRules": {
  "rightHandDrive": false
}

для определения допущений о стороне полосы.

Ручные полосы

Задайте autoLanes равным false, чтобы использовать количество полос вручную:

"autoLanes": false,
"lanesLeft": 1,
"lanesRight": 1

Для односторонних дорог данные о полосах могут внутренне формировать строку односторонних полос.

Внутренняя строка полос использует:

+ = direction from inNode to outNode
- = direction from outNode to inNode

Порядок полос зависит от правила rightHandDrive уровня.

lanesLeft и lanesRight интерпретируются совместно с roadRules.rightHandDrive из info.json.

Взаимодействие с правилами дорожного движения

map.json использует правила дорожного движения из info.json.

Пример:

"roadRules": {
  "rightHandDrive": false,
  "turnOnRed": true
}

Соответствующее поле:

Поле Используется картой графа? Описание
rightHandDrive Да Влияет на интерпретацию автоматического/ручного направления полос.
turnOnRed Косвенно Используется системами трафика/навигации там, где применимо.

rightHandDrive не изменяет визуальный вид дороги. Это поле влияет на то, как интерпретируются или генерируются данные о навигационных полосах.

Автоматическое слияние на перекрёстках

По умолчанию навигационный граф выполняет автоматическое слияние и обработку перекрёстков.

Задайте:

"autoJunction": false

чтобы предотвратить автоматическое слияние на нодах сегмента.

Внутренне это помечает данные сегмента/ноды как noMerge.

Используйте это для:

  • Сложных перекрёстков
  • Параллельных дорог, расположенных близко друг к другу
  • Въездных пандусов тоннелей
  • Мостов/путепроводов
  • Мест, где автоматическое слияние создаёт некорректные соединения

Пример:

"junction_helper": {
  "nodes": ["j1", "j2", "j3"],
  "autoJunction": false,
  "drivability": 1
}

Обработка сгенерированного графа

После загрузки данных из DecalRoad, BeamNGWaypoint и map.json движок обрабатывает навигационный граф.

Процесс включает:

  1. Загрузку данных дорог и вейпойнтов
  2. Добавление ручных сегментов из map.json
  3. Создание/оценку ограничений скорости
  4. Слияние перекрывающихся нод
  5. Разрешение T-образных перекрёстков
  6. Разрешение X-образных перекрёстков
  7. Слияние нод с ближайшими отрезками
  8. Обработку частных/закрытых дорог
  9. Оптимизацию/упрощение нод
  10. Преобразование графа в одностороннее хранение рёбер
  11. Построение пространственных ускоряющих структур для прокладки пути

В связи с этим итоговый граф во время выполнения может не в точности совпадать с исходным файлом.

Пример: простой двусторонний мост 

{
  "segments": {
    "bridge1": {
      "nodes": ["Bridge1_A", "Bridge1_B"],
      "drivability": 1
    }
  }
}

Это создаёт обычный двусторонний сегмент между Bridge1_A и Bridge1_B.

Пример: односторонний мост 

{
  "segments": {
    "bridge2": {
      "nodes": ["Bridge2_A", "Bridge2_B"],
      "oneWay": true,
      "flipDirection": true,
      "drivability": 1
    }
  }
}

Это создаёт односторонний сегмент в направлении, обратном порядку указанных нод.

Пример: тоннель с ручными полосами 

{
  "segments": {
    "tunnel_A": {
      "nodes": [
        "tunnel_A_1",
        "tunnel_A_2",
        "tunnel_A_3",
        "tunnel_A_4"
      ],
      "oneWay": true,
      "autoLanes": false,
      "lanesLeft": 1,
      "lanesRight": 0,
      "drivability": 1
    }
  }
}

Пример: частная грунтовая дорога 

{
  "segments": {
    "dirttrack": {
      "nodes": ["dirttrack_1", "dirttrack_2", "dirttrack_3"],
      "oneWay": true,
      "flipDirection": false,
      "drivability": 1,
      "gatedRoad": true
    }
  }
}

Этот сегмент обрабатывается как данные частной/закрытой дороги.

Пример: краткая запись диапазона 

{
  "segments": {
    "tunnel_city_A": {
      "nodes": "tunnel_city_A_1-tunnel_city_A_63",
      "oneWay": true,
      "lanesLeft": 1,
      "drivability": 1
    }
  }
}

Это эквивалентно ручному перечислению всех нод от tunnel_city_A_1 до tunnel_city_A_63.

Связь с BeamNGWaypoint

Ручные сегменты нередко опираются на объекты BeamNGWaypoint, размещённые на уровне.

Пример объекта вейпойнта:

{
  "class": "BeamNGWaypoint",
  "name": "tunnel_A_1",
  "position": [100, 200, 5],
  "scale": [1, 1, 1]
}

Сегмент в map.json может затем ссылаться на него:

"nodes": ["tunnel_A_1", "tunnel_A_2"]

Если у вейпойнта задан параметр excludeFromMap, он не добавляется в навигационный граф автоматически.

Отладка

Полезные инструменты/функции отладки включают:

  • Визуализацию навигационного графа
  • Логирование хэша графа
  • Запросы ближайшей дороги
  • Запросы пути
  • Экспорт графа карты в SVG

Система карты может экспортировать упрощённый SVG-вид графа для его проверки.

Стандартная визуализация отображает:

  • Ноды перекрёстков
  • Ширину дорог
  • Направления полос
  • Цвета проходимости
  • Закрытые/частные дороги

Рекомендации

  • По возможности используйте DecalRoad для обычных дорог.
  • Используйте map.json только для ручных исправлений и специальных соединений.
  • Убедитесь, что каждая нода, на которую ссылается map.json, существует.
  • Избегайте дублирующихся последовательных нод в сегменте.
  • Внимательно используйте oneWay и flipDirection.
  • Используйте autoJunction: false вблизи сложных путепроводов, тоннелей или близко расположенных параллельных дорог.
  • Используйте gatedRoad или type: "private" для маршрутов, не предназначенных для общего пользования.
  • Держите drivability близким к 1 для обычных дорог.
  • Используйте более низкую drivability для труднопроходимых или менее предпочтительных маршрутов.
  • Опускайте speedLimit, если только конкретное значение не является обязательным.
  • Давайте сегментам понятные описательные имена.

Частые проблемы

Сегмент не отображается в навигации

Возможные причины:

  • Файл map.json отсутствует или содержит ошибки
  • Объект segments отсутствует
  • Первая нода в сегменте не существует
  • Одна из последующих нод не существует, из-за чего сегмент обрывается
  • Указанные вейпойнты исключены из карты

ИИ едет в неправильном направлении

Проверьте:

"oneWay": true,
"flipDirection": true

Если направление перевёрнуто, измените значение flipDirection.

Полосы работают некорректно

Проверьте:

  • autoLanes
  • lanesLeft
  • lanesRight
  • roadRules.rightHandDrive в info.json

Дороги сливаются некорректно

Задайте:

"autoJunction": false

для затронутых ручных сегментов.

Маршрут неожиданно использует частные дороги

Проверьте gatedRoad, type и то, как частные дороги соединяются с общественными.

Скорость ощущается неправильной

Удалите speedLimit, чтобы включить автоматический расчёт, или убедитесь, что заданное значение соответствует предполагаемому типу дороги.

Итог

map.json — необязательный файл ручного навигационного графа.

Он задаёт дополнительные сегменты путей между существующими навигационными нодами и используется преимущественно в особых случаях: тоннели, мосты, путепроводы, частные дороги и сложные перекрёстки.

Основная структура:

{
  "segments": {
    "segment_name": {
      "nodes": ["node_a", "node_b"],
      "drivability": 1
    }
  }
}

Большинство дорог должны генерироваться из объектов DecalRoad. Используйте map.json, когда требуется точный ручной контроль над связностью ИИ/навигации или когда использование декальных дорог на вашем уровне невозможно.

Этот документ описывает формат файла items.level.json, используемого уровнями BeamNG для хранения объектов сцены.

Файлы items.level.json размещаются внутри папок объектов уровня, например:

levels/<levelName>/main/items.level.json

В них хранятся сериализованные объекты сцены в формате JSON с разделением по строкам: каждая строка представляет собой один полный JSON-объект.

На этой странице описан формат файла. Поведение объектов и поля, специфичные для конкретных классов, описаны в разделе Классы объектов уровня .

Обзор

Уровни BeamNG хранят множество объектов сцены в виде JSON-объектов. Вместо одного большого JSON-массива движок записывает по одному объекту в каждую строку.

Пример:

{"class":"SimGroup","name":"Environment"}
{"class":"TSStatic","name":"garage","shapeName":"/levels/example/art/shapes/garage.dae","position":[0,0,0],"rotationMatrix":[1,0,0,0,1,0,0,0,1],"scale":[1,1,1]}
{"class":"PointLight","name":"garage_light","position":[2,4,3],"rotationMatrix":[1,0,0,0,1,0,0,0,1],"scale":[1,1,1],"intensity":800}

Такой формат используется потому, что его легко читать потоком, редактировать, сравнивать (diff), объединять (merge) и сериализовывать постепенно.

Расположение файла

Основной файл объектов уровня обычно находится по пути:

levels/<levelName>/main/items.level.json

Дополнительные группы объектов могут храниться во вложенных папках:

levels/<levelName>/main/
├── items.level.json
├── Environment/
│   └── items.level.json
├── Buildings/
│   └── items.level.json
└── Gameplay/
    └── items.level.json

Каждая папка может содержать собственный файл items.level.json.

JSON с разделением по строкам

items.level.json — это не обычный JSON-массив.

Правильно:

{"class":"SimGroup","name":"Buildings"}
{"class":"TSStatic","name":"building_01","shapeName":"/levels/example/art/shapes/building_01.dae"}
{"class":"TSStatic","name":"building_02","shapeName":"/levels/example/art/shapes/building_02.dae"}

Неправильно:

[
  {"class":"SimGroup","name":"Buildings"},
  {"class":"TSStatic","name":"building_01"},
  {"class":"TSStatic","name":"building_02"}
]
Не оборачивайте содержимое items.level.json в [ ]. Каждая строка должна быть самостоятельным JSON-объектом.

Рекурсивная загрузка

Движок загружает items.level.json рекурсивно.

Когда он находит объект SimGroup, то проверяет, существует ли папка с таким же именем. Если да — движок загружает и файл items.level.json из этой папки.

Пример корневого файла:

{"class":"SimGroup","name":"Buildings"}
{"class":"SimGroup","name":"Gameplay"}

В результате загрузчик будет искать:

main/Buildings/items.level.json
main/Gameplay/items.level.json

Это позволяет организовывать объекты крупных уровней по папкам и группам.

Структура объекта

Каждая строка — это JSON-объект, представляющий один сериализованный объект сцены.

Большинство объектов содержат как минимум следующие поля:

Поле Тип Описание
class string Имя класса объекта, например TSStatic, SimGroup, PointLight, TerrainBlock и т.д.
name string Имя объекта в дереве сцены.
position array[3] Позиция объекта [x, y, z], если применимо.
rotationMatrix array[9] Поворот объекта в виде матрицы 3×3, если применимо.
scale array[3] Масштаб объекта [x, y, z], если применимо.

Различные классы объектов могут содержать множество дополнительных полей.

Примеры распространённых объектов

В этих примерах показано, как различные классы объектов выглядят в файле items.level.json. За подробным описанием поведения обращайтесь к соответствующей странице справки по классу.

SimGroup

Объект-группа для организации структуры.

{"class":"SimGroup","name":"Buildings"}

Если рядом с текущим файлом items.level.json существует папка с именем Buildings, она может содержать собственный вложенный файл объектов:

Buildings/items.level.json

TSStatic

Статический меш-объект.

{
  "class": "TSStatic",
  "name": "garage",
  "shapeName": "/levels/example/art/shapes/garage.dae",
  "position": [0, 0, 0],
  "rotationMatrix": [1,0,0,0,1,0,0,0,1],
  "scale": [1,1,1]
}

Важные поля:

Поле Описание
shapeName Путь к файлу формы (shape).
position Позиция объекта.
rotationMatrix Поворот объекта.
scale Масштаб объекта.

TerrainBlock

Объект рельефа.

{
  "class": "TerrainBlock",
  "name": "theTerrain",
  "terrainFile": "/levels/example/terrain/main.ter",
  "position": [0,0,0],
  "rotationMatrix": [1,0,0,0,1,0,0,0,1],
  "scale": [1,1,1],
  "maxHeight": 2048,
  "squareSize": 1
}

Важные поля:

Поле Описание
terrainFile Путь к файлу данных рельефа.
maxHeight Максимальная высота рельефа. По умолчанию 2048, если не задано.
squareSize Размер одной ячейки рельефа. По умолчанию 1, если не задано.

SpawnSphere

Точка появления.

{
  "class": "SpawnSphere",
  "name": "default_spawn",
  "position": [10, 20, 1],
  "rotationMatrix": [1,0,0,0,1,0,0,0,1]
}

Точки появления собираются по имени и позиции. Движок также добавляет запасную точку появления с именем Zero в начале координат.

WaterPlane

Объект водной поверхности.

{
  "class": "WaterPlane",
  "name": "lake",
  "position": [0,0,0],
  "rotationMatrix": [1,0,0,0,1,0,0,0,1],
  "scale": [100,100,1]
}

Поля трансформации

Многие объекты сцены используют следующие поля трансформации:

position 

"position": [0, 0, 0]

Массив из 3 значений:

[x, y, z]

rotationMatrix 

"rotationMatrix": [
  1,0,0,
  0,1,0,
  0,0,1
]

Матрица поворота 3×3 из 9 значений.

scale 

"scale": [1, 1, 1]

Вектор масштаба из 3 значений:

[x, y, z]

Если обнаружены некорректные значения трансформации, загрузчик может заменить их безопасными значениями по умолчанию, например:

position = [0, 0, 0]
rotation = identity matrix
scale = [1, 1, 1]

Порядок сериализации

При записи файлов с объектами по строкам движок может сортировать объекты перед сохранением.

Существует два режима сортировки:

Сортировка по serializeOrder

Если режим включён, объекты с полем serializeOrder типа string сортируются в числовом порядке.

{"class":"TSStatic","name":"a","serializeOrder":"10"}
{"class":"TSStatic","name":"b","serializeOrder":"20"}

Поле serializeOrder удаляется перед записью итоговой строки объекта.

Сортировка по классу и имени

Если сортировка по serializeOrder не используется, объекты сортируются по:

  1. class
  2. name

Это обеспечивает стабильный вывод и упрощает сравнение файлов (diff).

Сохранение с пространством имён

Движок может сохранять объекты в папки пространства имён.

Например, список объектов с пространством имён может быть сохранён как:

main/
├── Environment/
│   └── items.level.json
├── Gameplay/
│   └── items.level.json
└── Buildings/
    └── items.level.json

Каждое пространство имён становится папкой, а содержащиеся в нём записи записываются в файл items.level.json этой папки.

Связь с загрузкой уровня

При загрузке уровня движок начинает с:

levels/<levelName>/main/

Затем он рекурсивно загружает:

items.level.json

из основной папки и всех обнаруженных вложенных папок SimGroup.

Полученный список объектов используется для создания объектов сцены.

Требования к корректным JSON-объектам

Каждая строка должна:

  • Содержать ровно один JSON-объект
  • Включать корректное поле class
  • Использовать синтаксически верный JSON
  • Использовать массивы для данных трансформации
  • Не содержать замыкающих запятых
  • Не оборачивать весь файл в массив

Некорректные строки пропускаются и записываются в журнал.

Рекомендации по редактированию

Ручное редактирование возможно, но требует осторожности.

Рекомендуемый рабочий процесс:

  1. Редактируйте уровень в редакторе мира (World Editor).
  2. Сохраните уровень.
  3. Используйте ручное редактирование только для массовой очистки или продвинутых инструментов.
  4. Проверяйте синтаксис JSON построчно.
  5. По возможности сохраняйте уникальность имён объектов.
  6. Не изменяйте имена классов, если не знаете целевой тип объекта.
Одна некорректная строка может помешать загрузке этого объекта. Проверяйте консоль или журнал, если объекты пропадают после ручного редактирования.

Распространённые проблемы

Объекты не загружаются

Возможные причины:

  • Файл отсутствует
  • Некорректная строка JSON
  • Объект обёрнут в массив
  • Поле class отсутствует или некорректно
  • Неправильная структура папок
  • Имя вложенной папки не совпадает с именем SimGroup

Вложенная группа не загружается

Убедитесь, что родительский файл items.level.json содержит:

{"class":"SimGroup","name":"GroupName"}

и что папка существует:

GroupName/items.level.json

Объект появляется в начале координат

Возможные причины:

  • Некорректный массив position
  • Нечисловые значения позиции
  • Отсутствуют данные трансформации

Объект имеет неверный поворот

Убедитесь, что rotationMatrix содержит ровно 9 числовых значений.

Объект имеет неверный масштаб

Убедитесь, что scale содержит ровно 3 числовых значения.

Итог

Файлы items.level.json хранят объекты уровня в формате JSON с разделением по строкам.

Ключевые моменты:

  • Каждая строка — один JSON-объект.
  • Файлы загружаются рекурсивно через папки SimGroup.
  • Не оборачивайте содержимое в массив.
  • Данные трансформации обычно используют поля position, rotationMatrix и scale.
  • Объекты могут сортироваться по serializeOrder или по class и name.
  • Некорректные строки записываются в журнал и пропускаются.

Этот формат компактен, удобен для инструментов и подходит для крупных уровней с большим количеством объектов сцены.

В этом документе описаны файлы префабов BeamNG и способ их загрузки движком.

Префабы — это повторно используемые группы объектов сцены. Они позволяют хранить несколько объектов в отдельном файле и размещать их в уровне как единый объект.

На этой странице описаны форматы файлов префабов. Описание объекта сцены, размещающего и загружающего префаб в уровне, см. в Prefab .

BeamNG поддерживает два формата префабов:

  • Устаревший формат TorqueScript .prefab
  • Современный JSON-формат .prefab.json

Обзор

Префаб представлен в уровне объектом Prefab. Объект Prefab указывает на внешний файл через поле filename. При загрузке префаб читает этот файл и создаёт дочернюю группу SimGroup, содержащую хранящиеся внутри объекты.

Типичные варианты использования:

  • Повторно используемые здания
  • Группы придорожных объектов
  • Повторяющиеся элементы декораций
  • Объекты миссий
  • Группы статичных мешей
  • Точки спауна
  • Сцены для демонстрации транспортных средств

Пример объекта уровня:

{
  "class": "Prefab",
  "name": "gas_station_prefab",
  "filename": "/levels/example/prefabs/gas_station.json",
  "position": [100, 200, 0],
  "rotationMatrix": [1,0,0,0,1,0,0,0,1],
  "scale": [1, 1, 1],
  "loadMode": "Automatic"
}

Роли файлов

Файл Назначение
*.prefab.json Современный JSON-файл префаба.
*.prefab Устаревший файл префаба в формате TorqueScript.
Объект Prefab Объект сцены, ссылающийся на файл префаба и загружающий его.
Корневая группа SimGroup Обязательная группа, создаваемая файлом префаба. Содержит все дочерние объекты префаба.

Поля объекта Prefab

Поле Тип Описание
filename string Путь к файлу префаба. Может указывать на .json или устаревший формат .prefab.
loadMode enum Управляет тем, когда загружается префаб.
useGlobalTranslation bool Изменяет способ применения трансформаций дочерних объектов. Используется преимущественно для совместимости и особых случаев.

loadMode

Поддерживаемые значения:

Значение Описание
Automatic Префаб загружается автоматически при добавлении объекта Prefab.
Manual Префаб не загружается до явного запроса загрузки.

Современные JSON-префабы

Современные JSON-префабы загружаются, если filename содержит .json.

JSON-префаб должен создавать корневую группу SimGroup. Движок ожидает, что один из объектов в файле является SimGroup, и эта группа становится дочерней группой префаба.

Минимальный JSON-префаб 

{"class":"SimGroup","name":"example_prefab_group"}
{"class":"TSStatic","name":"example_prop","shapeName":"/levels/example/art/shapes/prop.dae","position":[0,0,0],"rotationMatrix":[1,0,0,0,1,0,0,0,1],"scale":[1,1,1]}
JSON-префаб должен создавать корневую группу SimGroup. Если SimGroup не создана, загрузка префаба завершится ошибкой.

Структура JSON-префаба

JSON-префабы, как правило, хранятся в виде построчных JSON-данных объектов, аналогично файлам объектов основного уровня (main). Каждый объект сериализуется как JSON-объект.

Типичный префаб содержит:

  1. Корневую группу SimGroup
  2. Один или несколько объектов сцены
  3. Опциональные вложенные группы
  4. Опциональные транспортные средства или другие поддерживаемые объекты

Пример:

{"class":"SimGroup","name":"garage_prefab_group"}
{"class":"TSStatic","name":"garage_building","shapeName":"/levels/example/art/shapes/garage.dae","position":[0,0,0],"rotationMatrix":[1,0,0,0,1,0,0,0,1],"scale":[1,1,1]}
{"class":"PointLight","name":"garage_light","position":[0,2,3],"rotationMatrix":[1,0,0,0,1,0,0,0,1],"scale":[1,1,1],"intensity":800}

Корневая SimGroup

Корневая группа SimGroup служит контейнером для всех объектов префаба.

При загрузке префаба:

  • Движок находит SimGroup
  • При необходимости удаляет её из некорректного родителя
  • Назначает её дочерней группой префаба
  • Применяет трансформацию префаба ко всем дочерним объектам SceneObject
  • Отслеживает дочерние объекты как принадлежащие данному префабу

Для JSON-префабов корневая группа может быть автоматически переименована в соответствии с именем префаба.

Пример:

{"class":"SimGroup","name":"garage_group"}

noAutoReload

Корневая группа SimGroup может содержать поле noAutoReload.

{"class":"SimGroup","name":"garage_group","noAutoReload":true}

Если значение noAutoReload равно true, файл префаба не отслеживается для автоматической перезагрузки.

Это может быть полезно при работе с генерируемыми или специальными префабами, которые не должны автоматически перезагружаться при изменении файла.

Трансформации дочерних объектов

Дочерние объекты префаба хранятся с трансформациями относительно префаба.

При перемещении, повороте или масштабировании префаба дочерние объекты обновляются.

Поведение по умолчанию:

child final transform = prefab transform x stored child transform
child final scale = stored child scale x prefab scale

Это позволяет перемещать весь префаб как единый объект.

useGlobalTranslation

Опция useGlobalTranslation изменяет способ применения трансформаций дочерних объектов.

При включении префаб может использовать поле данных groupPosition из дочерней группы и применять трансформации дочерних объектов более глобально, а не строго относительно трансформации префаба.

Это преимущественно опция совместимости для особых случаев. В большинстве префабов её следует оставлять отключённой.

Автоматическая перезагрузка файла

После загрузки префаба движок может отслеживать изменения файла префаба. При изменении файла префаб перезагружается:

  1. Существующие дочерние объекты удаляются
  2. Файл префаба загружается повторно
  3. Создаются новые дочерние объекты
  4. Трансформации применяются заново

Это удобно при редактировании файлов префабов во внешних редакторах.

Поле noAutoReload в корневой группе SimGroup отключает это поведение.

Защита от рекурсивной загрузки префабов

Движок предотвращает бесконечную рекурсию при загрузке префабов.

Например, следующее является недопустимым:

Prefab A loads Prefab B
Prefab B loads Prefab A

Если файл префаба уже находится в стеке активной загрузки префабов, загрузка прекращается и в лог записывается ошибка.

Избегайте циклических ссылок между префабами. Префаб не должен прямо или косвенно загружать сам себя.

Устаревшие файлы .prefab

Устаревшие файлы .prefab основаны на TorqueScript.

Они загружаются путём выполнения файла:

exec("path/to/file.prefab");

Файл должен создать группу SimGroup и присвоить её переменной $ThisPrefab.

Пример:

$ThisPrefab = new SimGroup(GaragePrefabGroup) {
   new TSStatic(garage_building) {
      shapeName = "/levels/example/art/shapes/garage.dae";
      position = "0 0 0";
      rotationMatrix = "1 0 0 0 1 0 0 0 1";
      scale = "1 1 1";
   };
};

Если переменная $ThisPrefab не задана или не указывает на допустимую группу SimGroup, загрузка префаба завершается ошибкой.

Устаревшие файлы .prefab по-прежнему поддерживаются, однако для нового контента рекомендуется использовать JSON-префабы там, где это возможно.

JSON-префабы и транспортные средства

При загрузке JSON-префабов движок выполняет специальную обработку объектов BeamNGVehicle.

Если объект транспортного средства задаёт поле jBeam/jbeam, но не задаёт partConfig, движок пытается загрузить файл info.json транспортного средства и использовать конфигурацию деталей по умолчанию.

Также могут быть заполнены отсутствующие данные о цвете по умолчанию:

  • color
  • metallicPaintData

Пример:

{
  "class": "BeamNGVehicle",
  "name": "display_vehicle",
  "jBeam": "vivace",
  "position": [0,0,0],
  "rotationMatrix": [1,0,0,0,1,0,0,0,1],
  "scale": [1,1,1]
}

Если у транспортного средства есть допустимая конфигурация по умолчанию, отсутствующие поля могут быть заполнены автоматически в процессе загрузки.

Допустимые и недопустимые дочерние объекты префаба

Не все объекты допустимы внутри префабов.

Допустимыми объектами, как правило, являются обычные объекты сцены, статичные формы, источники света, группы и аналогичный размещаемый контент.

Недопустимые или нежелательные объекты

Движок отклоняет или выдаёт предупреждение для нескольких типов объектов:

Тип объекта Причина
MissionGroup Префабы не должны содержать основную группу миссии.
LevelInfo Глобальный объект уровня; недопустим внутри префаба.
TimeOfDay Глобальный объект уровня; недопустим внутри префаба.
TerrainBlock Рельеф является глобальным для уровня и не должен находиться внутри префаба.
Player Объекты игрока недопустимы в качестве дочерних элементов префаба.
Объекты SceneObject с глобальными границами Глобальные объекты недопустимы в качестве дочерних элементов префаба.

Объект SimPath явно разрешён.

Префабы должны содержать локальные, повторно используемые объекты. Не храните в префабе глобальные объекты уровня, такие как рельеф, время суток или информацию об уровне.

Разворачивание префаба

Префаб можно «развернуть» или распаковать в обычные объекты сцены.

При разворачивании:

  • Дочерняя группа перемещается в иерархию миссии/групп
  • Трансформации дочерних объектов применяются
  • Объекты более не управляются префабом
  • В результирующую группу добавляются метаданные

Результирующая группа может получить следующие поля:

Поле Описание
unpacked_prefab Помечает группу как распакованную из префаба.
prefab_filename Путь к исходному файлу префаба.
prefab_name Исходное имя объекта префаба.
prefab_loadmode Исходный режим загрузки.
prefab_position Исходная позиция префаба.

Это позволяет инструментам определить, что группа была получена из префаба.

Конвертация в PrefabV2

Движок поддерживает конвертацию старых префабов в новое представление префабов (PrefabV2).

Процесс конвертации в общих чертах:

  1. Загружает/разворачивает старый префаб
  2. Собирает идентификаторы объектов
  3. Упаковывает их в PrefabV2
  4. Сохраняет результат обратно в файл префаба

Обычно это выполняется инструментами и рабочими процессами редактора.

Сохранение префабов

При сохранении устаревшего префаба его дочерняя группа SimGroup записывается в целевой файл.

При сохранении в устаревшем формате обычно используется префикс вида:

$ThisPrefab =

Это гарантирует, что при загрузке файла .prefab созданная корневая группа будет восстановлена.

Для работы с JSON используйте инструменты редактора, которые сериализуют объекты в ожидаемом формате JSON.

Процесс загрузки

При добавлении объекта Prefab в сцену:

  1. Создаётся объект префаба.
  2. Если значение loadMode равно Automatic, вызывается метод load().
  3. Движок проверяет поле filename.
  4. Если имя файла содержит .json, используется загрузка JSON-префаба.
  5. В противном случае используется загрузка устаревшего TorqueScript-префаба.
  6. Находится корневая группа SimGroup.
  7. Дочерние объекты регистрируются и связываются с префабом.
  8. Трансформации дочерних объектов обновляются относительно префаба.
  9. Границы префаба обновляются на основе границ дочерних объектов.
  10. Включается отслеживание изменений файла (если не отключено).

Рекомендации

  • Для нового контента используйте JSON-префабы.
  • Всегда включайте корневую группу SimGroup.
  • Избегайте циклических ссылок между префабами.
  • Делайте префабы самодостаточными и повторно используемыми.
  • Не включайте глобальные объекты уровня, такие как TerrainBlock, LevelInfo или TimeOfDay.
  • Используйте относительные трансформации дочерних объектов, чтобы префаб можно было легко перемещать.
  • Используйте режим загрузки Automatic для обычного контента уровня.
  • Используйте режим Manual только когда загрузкой управляет инструмент или скрипт.
  • Избегайте размещения в префабах очень крупных или глобальных систем.
  • Используйте понятные имена для файлов префабов и корневых групп.

Частые проблемы

Префаб не загружается

Возможные причины:

  • Недопустимое значение поля filename
  • Файл не существует
  • Ошибка разбора JSON
  • Отсутствует корневая группа SimGroup
  • В устаревшем файле .prefab не задана переменная $ThisPrefab

Префаб загружается рекурсивно

Префаб прямо или косвенно ссылается на себя. Удалите циклическую ссылку.

Дочерние объекты находятся в неверной позиции

Возможные причины:

  • Неверная трансформация префаба
  • Неверная трансформация дочернего объекта
  • Непредвиденное значение useGlobalTranslation
  • Неверное значение поля данных groupPosition

Префаб перезагружается неожиданно

Файл отслеживается на предмет изменений. Используйте поле noAutoReload в корневой группе SimGroup, если автоматическую перезагрузку необходимо отключить.

Объект не может быть добавлен в префаб

Объект может быть недопустимым для префабов, например TerrainBlock, LevelInfo, TimeOfDay, Player или объект с глобальными границами.

Итог

Префабы — это повторно используемые группы объектов, хранящиеся за пределами основного файла уровня.

Современные JSON-префабы содержат корневую группу SimGroup и один или несколько дочерних объектов. Объект сцены Prefab загружает файл, управляет трансформациями дочерних объектов, обновляет их при перемещении и может выполнять перезагрузку при изменении файла.

Устаревшие файлы .prefab по-прежнему поддерживаются, однако для нового контента рекомендуется использовать JSON-префабы там, где это возможно.

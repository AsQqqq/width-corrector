В этом документе описывается файл info.json, используемый уровнями BeamNG.

info.json — основной файл метаданных уровня. Он используется селектором уровней, логикой загрузки, интерфейсом точек спауна, системами большой карты и POI, пресетами окружения, системами трафика и геймплея, отображением миникарты, а также частичной логикой геймплея, зависящей от региона.

Он не хранит геометрию уровня. Объекты сцены хранятся отдельно в таких файлах, как main/items.level.json.

Расположение файла

Каждый уровень должен содержать файл info.json в корневой папке уровня:

levels/<levelName>/info.json

Пример:

levels/west_coast_usa/info.json

Если файл info.json отсутствует, уровень всё равно может быть обнаружен при наличии допустимой точки входа, однако метаданные интерфейса будут неполными и в журнал могут быть записаны предупреждения.

Обнаружение уровней

Менеджер уровней сканирует:

/levels/

Каждая директория рассматривается как потенциальный уровень.

Уровень считается работоспособным, если загрузчик может найти допустимую точку входа. Предпочтительный формат — папка дерева сцены main/:

levels/<levelName>/main/

Старые точки входа, такие как файлы .mis или .level.json, могут по-прежнему поддерживаться для обратной совместимости, однако современные уровни должны использовать структуру папки main/.

В процессе обнаружения загрузчик читает info.json, добавляет метаданные времени выполнения, разрешает пути к превью, обрабатывает данные точек спауна и применяет значения по умолчанию для отсутствующих необязательных полей.

Базовая структура

Упрощённый файл info.json:

{
  "title": "levels.example.info.title",
  "description": "levels.example.info.description",
  "authors": "BeamNG",
  "defaultSpawnPointName": "spawns_default",
  "country": "levels.common.country.usa",
  "region": "northAmerica",
  "size": [2048, 2048],
  "previews": [
    "preview.jpg"
  ],
  "spawnPoints": [
    {
      "translationId": "levels.example.spawnpoints.default",
      "description": "levels.example.spawnpoints.default.description",
      "objectname": "spawns_default",
      "preview": "spawns_default.jpg"
    }
  ]
}

Основные поля

Поле Тип Описание
title string Название уровня. Обычно идентификатор перевода.
description string Описание уровня. Обычно идентификатор перевода.
authors string Имя автора или авторов.
features string Идентификатор перевода или строка, описывающая особенности уровня.
suitablefor string Идентификатор перевода или строка, описывающая, для чего подходит уровень.
roads string Идентификатор перевода или строка, описывающая дорожную сеть или типы дорог.
biome string Идентификатор перевода или строка, описывающая биом/тип окружения.
country string Идентификатор перевода страны.
region string Идентификатор региона, например northAmerica.
size array[2] Приблизительный размер уровня в метрах, обычно [width, height].
defaultSpawnPointName string Имя объекта точки спауна по умолчанию.
previews array[string] Пути к превью-изображениям уровня относительно папки уровня.
spawnPoints array[object] Метаданные точек спауна, отображаемые в интерфейсе и на большой карте.
minimap array[object] Определения тайлов изображений миникарты.
roadRules object Метаданные правил дорожного движения для уровня.
localUnits object Предпочтения локальных единиц измерения, используемые системами геймплея и интерфейса.
supportsTraffic bool Поддерживает ли уровень трафик. По умолчанию true, если не указано.
supportsTimeOfDay bool Поддерживает ли уровень параметры времени суток. По умолчанию true, если не указано.
timeOfDayPresets array[object] Необязательные пресеты времени суток для данного уровня.
excludeDefaultTimeOfDayOptions bool Если true, стандартные пресеты времени суток не добавляются.
isAuxiliary bool Помечает уровень как вспомогательный/внутренний в зависимости от контекста использования в интерфейсе.

Идентификаторы перевода

Большинство текстовых полей, отображаемых пользователю, должны использовать идентификаторы перевода вместо жёстко заданного текста.

Пример:

"title": "levels.west_coast_usa.info.title",
"description": "levels.west_coast_usa.info.description"

Это позволяет интерфейсу локализовать названия уровней, описания, имена точек спауна и другие тексты.

К часто переводимым полям относятся:

  • title
  • description
  • features
  • suitablefor
  • roads
  • biome
  • country
  • spawnPoints[].translationId
  • spawnPoints[].description
  • spawnPoints[].logbookEntry

Превью-изображения

Массив previews определяет превью-изображения уровня.

"previews": [
  "spawns_industrial.jpg",
  "west_coast_usa_preview1.jpg",
  "west_coast_usa_preview2.jpg"
]

Пути указываются относительно папки уровня. При загрузке они преобразуются в полные виртуальные пути.

Пример:

"west_coast_usa_preview1.jpg"

становится:

/levels/west_coast_usa/west_coast_usa_preview1.jpg

Первое превью-изображение, как правило, используется в качестве основной миниатюры уровня.

Если массив previews отсутствует или пуст, система пытается найти резервные пути к превью из точки входа уровня.

Точки спауна

Массив spawnPoints определяет варианты спауна, отображаемые в интерфейсе и на большой карте.

Пример:

"spawnPoints": [
  {
    "translationId": "levels.west_coast_usa.spawnpoints.industrial",
    "description": "levels.west_coast_usa.spawnpoints.industrial.description",
    "logbookEntry": "levels.west_coast_usa.spawnpoints.industrial.logbookEntry",
    "objectname": "spawns_industrial",
    "preview": "spawns_industrial.jpg"
  }
]

Поля точки спауна

Поле Тип Описание
translationId string Идентификатор перевода отображаемого имени точки спауна.
description string Идентификатор перевода описания точки спауна.
logbookEntry string Необязательный идентификатор перевода текста журнала/открытия.
objectname string Имя объекта сцены, используемого в качестве трансформа спауна.
preview string Путь к превью-изображению относительно папки уровня.
previews array[string] Необязательный список превью-изображений. Генерируется автоматически, если используется только preview.

Значение objectname должно соответствовать объекту уровня, как правило SpawnSphere.

Пример объекта сцены:

{"class":"SpawnSphere","name":"spawns_industrial","position":[0,0,1],"rotationMatrix":[1,0,0,0,1,0,0,0,1]}

Точка спауна по умолчанию

Точка спауна по умолчанию выбирается с помощью:

"defaultSpawnPointName": "spawns_industrial"

Значение должно соответствовать одному из значений spawnPoints[].objectname.

Когда точка спауна совпадает с defaultSpawnPointName, она внутренне помечается как спаун по умолчанию и получает основной список превью уровня.

Если совпадающая точка спауна не найдена, загрузчик вставляет резервную запись спауна по умолчанию, используя превью-изображения уровня.

POI точек спауна

Точки спауна могут отображаться как POI на большой карте.

Для каждой точки спауна с полем objectname система находит соответствующий объект сцены и использует его позицию в качестве положения маркера.

Если объект не найден, в журнал может быть записана ошибка.

Миникарта

Массив minimap определяет один или несколько тайлов изображений миникарты.

Пример:

"minimap": [
  {
    "file": "minimap/terrain.png",
    "size": [2048, 2048],
    "offset": [-1024, 1024]
  },
  {
    "file": "minimap/island.png",
    "size": [4096, 2048],
    "offset": [-1536, 3072]
  }
]

Поля миникарты

Поле Тип Описание
file string Путь к изображению миникарты относительно папки уровня.
size array[2] Размер тайла в единицах мира/карты.
offset array[2] Смещение тайла в единицах мира/карты.

При загрузке к пути file добавляется префикс директории уровня.

Пример:

minimap/terrain.png

становится:

/levels/west_coast_usa/minimap/terrain.png
Файл info.json должен быть валидным JSON. Не используйте завершающие запятые в записях миникарты и в любых других местах файла.

Правила дорожного движения

Объект roadRules хранит специфичные для уровня правила дорожного движения, используемые навигацией, ИИ и системами трафика.

Пример:

"roadRules": {
  "rightHandDrive": false,
  "turnOnRed": true
}

Если roadRules отсутствует, применяются значения по умолчанию:

{
  "rightHandDrive": false,
  "turnOnRed": false
}

Поддерживаемые поля правил дорожного движения

Поле Тип По умолчанию Описание
rightHandDrive bool false Управляет предположениями о стороне полосы при генерации или интерпретации данных полос. false означает правостороннее движение, true — левостороннее движение.
turnOnRed bool false Указывает, разрешён ли поворот на красный свет для данного уровня. Используется логикой трафика и навигации там, где применимо.

rightHandDrive

rightHandDrive влияет на то, как граф навигации интерпретирует/генерирует направления полос.

Внутренне направления полос представлены строками:

+ = lane direction from inNode to outNode
- = lane direction from outNode to inNode

Для двусторонних дорог порядок полос зависит от того, используется ли на уровне правостороннее или левостороннее движение.

Используйте:

"rightHandDrive": false

для регионов с правосторонним движением.

Используйте:

"rightHandDrive": true

для регионов с левосторонним движением.

rightHandDrive не изменяет геометрию дорог визуально. Это поле влияет на навигацию/интерпретацию полос и автоматическую генерацию полос.

Локальные единицы измерения

Объект localUnits определяет специфичные для уровня предпочтения единиц измерения, используемые системами геймплея и интерфейса.

Пример:

"localUnits": {
  "gasoline": "gallonUS",
  "diesel": "gallonUS"
}

В настоящее время это используется такими системами, как логика отображения цен на топливо. Внутренне цены могут храниться в метрических единицах, тогда как объекты отображения могут преобразовывать значения в зависимости от настроек локальных единиц уровня.

Например:

"gasoline": "gallonUS"

позволяет вывескам с ценами на бензин использовать американские галлоны.

Пример поддерживаемых единиц топлива

Поле Значение Описание
gasoline gallonUS Отображать значения, связанные с бензином, в американских галлонах.
diesel gallonUS Отображать значения, связанные с дизелем, в американских галлонах.

Если переопределение локальных единиц не задано, системы используют поведение по умолчанию.

localUnits влияет на представление данных и логику геймплея/интерфейса. По возможности это значение должно соответствовать реальному региону, представленному уровнем.

Поддержка трафика

Если supportsTraffic не указан, по умолчанию применяется:

"supportsTraffic": true

Установите значение false, если уровень не должен использовать системы, связанные с трафиком.

Пример:

"supportsTraffic": false

Поддержка времени суток

Если supportsTimeOfDay не указан, по умолчанию применяется:

"supportsTimeOfDay": true

Установите значение false, если уровень не должен предоставлять параметры времени суток.

Пример:

"supportsTimeOfDay": false

Пресеты времени суток

Уровни могут определять пользовательские пресеты времени суток.

Пример:

"timeOfDayPresets": [
  {
    "key": "sunrise",
    "value": 0.23,
    "label": "ui.environment.time.sunrise"
  },
  {
    "key": "noon",
    "value": 0.5,
    "label": "ui.environment.time.noon"
  }
]

Поля пресета

Поле Тип Описание
key string Уникальный ключ пресета.
value number Значение времени, используемое системой окружения.
label string Отображаемая метка или идентификатор перевода.

По умолчанию игра запускается с набором стандартных пресетов времени суток, после чего применяет или переопределяет специфичные для уровня пресеты.

Чтобы исключить стандартные пресеты:

"excludeDefaultTimeOfDayOptions": true

Если это поле задано, предоставьте собственные timeOfDayPresets, иначе у уровня может не быть доступных для выбора вариантов времени суток.

Вспомогательные уровни

В более старых метаданных может использоваться:

"hidden": true

Это поле устарело.

Используйте:

"isAuxiliary": true

Если обнаружено поле hidden, загрузчик сопоставляет его с isAuxiliary для обратной совместимости и записывает предупреждение.

Полный пример

  "region": "northAmerica",
  "biome": "levels.example.info.biome",
  "size": [2048, 2048],
  "localUnits": {
    "gasoline": "gallonUS",
    "diesel": "gallonUS"
  },
  "previews": [
    "preview.jpg",
    "preview_1.jpg"
  ],
  "spawnPoints": [
    {
      "translationId": "levels.example.spawnpoints.default",
      "description": "levels.example.spawnpoints.default.description",
      "logbookEntry": "levels.example.spawnpoints.default.logbookEntry",
      "objectname": "spawns_default",
      "preview": "spawns_default.jpg"
    }
  ],

    "rightHandDrive": false,

  },

Поведение при загрузке

При формировании списка уровней менеджер уровней:

  1. Сканирует /levels/.
  2. Находит допустимые директории уровней.
  3. Определяет точку входа уровня.
  4. Читает info.json.
  5. Добавляет метаданные модов/официального контента.
  6. Добавляет поля путей времени выполнения.
  7. Добавляет префикс директории уровня к путям превью.
  8. Обрабатывает превью точек спауна.
  9. Помечает точку спауна по умолчанию, если она найдена.
  10. Добавляет резервную точку спауна по умолчанию при необходимости.
  11. Добавляет префикс к путям файлов миникарты.
  12. Применяет значения по умолчанию для отсутствующих флагов поддержки.

Рекомендации

  • Всегда включайте info.json в корень уровня.
  • Используйте идентификаторы перевода для текстов, отображаемых в интерфейсе.
  • Предоставьте хотя бы одно превью-изображение.
  • Убедитесь, что пути к превью указаны относительно папки уровня.
  • Убедитесь, что defaultSpawnPointName соответствует допустимому объекту точки спауна.
  • Сохраняйте стабильными значения objectname точек спауна после их использования в данных геймплея/карьеры.
  • Используйте roadRules в соответствии с реальной стороной движения в регионе.
  • Используйте localUnits для регионального отображения единиц измерения.
  • Избегайте устаревшего поля hidden; используйте isAuxiliary.
  • Проверяйте синтаксис JSON перед выпуском.
  • Не используйте завершающие запятые.
  • Предоставьте данные миникарты, если уровень поддерживает отображение карты/большой карты.
  • Устанавливайте supportsTraffic или supportsTimeOfDay в false только тогда, когда соответствующие системы не поддерживаются.

Распространённые проблемы

Уровень отображается с именем папки вместо названия

Возможные причины:

  • Отсутствует поле title
  • Недопустимый идентификатор перевода
  • Отсутствующий или недопустимый файл info.json

Превью-изображение отсутствует

Возможные причины:

  • Неверный путь
  • Изображение не включено
  • Массив previews пуст
  • Недопустимое имя файла или расширение

Точка спауна не работает

Возможные причины:

  • objectname не соответствует объекту сцены
  • Объект спауна отсутствует в items.level.json
  • defaultSpawnPointName не совпадает ни с одной точкой спауна
  • Объект точки спауна существует, но не загружен

POI спауна на большой карте отсутствует

Возможные причины:

  • У точки спауна нет поля objectname
  • Указанный объект не найден
  • Точка спауна не открыта в режиме карьеры там, где это применимо

Тайл миникарты отсутствует

Возможные причины:

  • Неверный путь в поле file
  • Отсутствующий файл изображения
  • Недопустимая запись миникарты
  • Недопустимый JSON из-за завершающей запятой

Параметры времени суток отсутствуют

Возможные причины:

  • supportsTimeOfDay имеет значение false
  • excludeDefaultTimeOfDayOptions имеет значение true и пользовательские пресеты не заданы

Поведение полос ощущается неправильным

Проверьте:

"roadRules": {
  "rightHandDrive": false
}

Используйте правильное значение для стороны движения, представленной уровнем.

Отображение цены на топливо использует неверные единицы

Проверьте:

"localUnits": {
  "gasoline": "gallonUS",
  "diesel": "gallonUS"
}

Задавайте переопределения локальных единиц только тогда, когда этого требует регион уровня.

Итог

info.json — основной файл метаданных уровня.

Он определяет:

  • Название и описание в интерфейсе
  • Метаданные автора и региона
  • Превью-изображения
  • Метаданные точек спауна
  • Точку спауна по умолчанию
  • Тайлы миникарты
  • Правила дорожного движения
  • Предпочтения локальных единиц измерения
  • Поддержку трафика
  • Поддержку и пресеты времени суток

Полный и валидный файл info.json необходим для корректного обнаружения уровня, отображения в интерфейсе, спауна, поддержки миникарты и интеграции с геймплеем.

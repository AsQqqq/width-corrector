На этой странице описываются локальные для уровня папки с дрифт-спотами.

Дрифт-споты определяют испытания по дрифту в режиме свободного катания. Они объединяют UI-метаданные, цели для подсчёта очков, пространственные настройки, данные маршрута, граничные данные и объекты-маркеры сцены.

Расположение файлов

Дрифт-споты хранятся в корне уровня:

levels/<levelName>/driftSpots/<spotName>/

Общие файлы:

info.json
spot.driftSpot.json
race.race.json
bounds.sites.json
preview.jpg

Система дрифта обнаруживает споты, сканируя каталоги в поисках spot.driftSpot.json.

Поведение при обнаружении

При загрузке уровня система дрифт-спотов сканирует:

levels/<levelName>/driftSpots/

Каждая папка, содержащая spot.driftSpot.json, считается одним дрифт-спотом. Имя папки становится частью идентификатора дрифт-спота, поэтому оставляйте его неизменным после релиза или после того, как на него начнут ссылаться данные карьеры/прогресса.

Роли файлов

Файл Назначение
info.json Отображаемое название, данные разблокировки, цели, награды и UI-метаданные.
spot.driftSpot.json Пространственная настройка дрифт-спота, трансформация телепорта на глобальной карте, определения линий дрифта и ссылки на объекты-маркеры сцены.
race.race.json Данные пути/маршрута, используемые геймплеем дрифта.
bounds.sites.json Граничные зоны игровой или зачётной области.
preview.* Необязательное изображение предпросмотра для резервного отображения в UI.

info.json

Файл info.json дрифт-спота — это не то же самое, что info.json уровня. Он является локальным для конкретного дрифт-спота и обычно содержит данные UI и прогресса:

Поле Назначение
name Отображаемое название или ключ перевода.
objectives Пороги очков и награды: бронза, серебро и золото.
unlock Необязательные данные разблокировки/прогресса.

Пороги очков следует подбирать в ходе реального вождения. Если они слишком низкие, прогресс ощущается случайным; если слишком высокие, спот может казаться игрокам неработающим.

Минимальный info.json:

{
  "name": "levels.example.driftSpots.harbor.name",
  "objectives": [
    {
      "id": "bronze",
      "score": 5000,
      "rewards": {
        "money": 500,
        "beamXP": 25
      }
    },
    {
      "id": "silver",
      "score": 10000,
      "rewards": {
        "money": 1000,
        "beamXP": 50
      }
    },
    {
      "id": "gold",
      "score": 15000,
      "rewards": {
        "money": 1500,
        "beamXP": 75
      }
    }
  ],
  "unlock": {}
}

Используйте стабильные идентификаторы целей. Системы прогресса могут хранить информацию о выполнении по идентификатору цели.

spot.driftSpot.json

Этот файл хранит пространственную настройку спота. Основные концепции:

Концепция Назначение
bigMapTp Позиция/поворот/масштаб телепорта дрифт-спота на глобальной карте.
lines Одно или несколько определений стартовой/зачётной линии.
markerObjects Объекты сцены, используемые как видимые маркеры или управляемые маркеры.
startDir Направление, используемое линией дрифта или логикой старта.

Конкретная схема расположения линий может варьироваться в зависимости от спота. По возможности сохраняйте файл, созданный редактором Drift Data Editor.

Минимальная структура spot.driftSpot.json:

{
  "spatialInfo": {
    "bigMapTp": {
      "pos": [0, 0, 0],
      "rot": [0, 0, 0, 1],
      "scl": [1, 1, 1]
    },
    "lines": {
      "lineOne": {
        "markerObjects": [
          "harbor_line_1_a",
          "harbor_line_1_b",
          "harbor_drift_sign"
        ],
        "pos": [0, 0, 0],
        "rot": [0, 0, 0, 1],
        "scl": [6, 2, 2],
        "startDir": [1, 0, 0]
      }
    }
  }
}

Поля spot.driftSpot.json

Поле Обязательное Примечания
spatialInfo Да Контейнер для данных телепорта и линий.
spatialInfo.bigMapTp Рекомендуется Трансформация телепорта на глобальной карте.
spatialInfo.lines Да Одно или несколько определений линий дрифта.
markerObjects Рекомендуется Имена объектов сцены, управляемых дрифт-спотом.
pos / rot / scl Да, для каждой линии Трансформация триггера/объёма линии.
startDir Рекомендуется Ожидаемое направление въезда для линии.

rot хранится в виде массива кватернионов. scl управляет размером объёма линии.

race.race.json и bounds.sites.json

Загрузчик дрифт-спота ожидает эти файлы рядом с spot.driftSpot.json:

race.race.json
bounds.sites.json

race.race.json описывает путь, используемый геймплеем дрифта. bounds.sites.json содержит игровую/зачётную область. Эти форматы используются совместно с другими игровыми системами, поэтому предпочтительнее создавать их с помощью существующих инструментов для путей и сайтов, а не вручную составлять большие файлы.

Идентификация папки

Идентификатор спота формируется на основе текущего уровня и имени папки:

<levelName>/<spotName>

Сохраняйте имена папок стабильными после того, как появятся ссылки на них в данных прогресса, карьеры, лигах или UI.

Ссылки на объекты сцены

spot.driftSpot.json может ссылаться на объекты-маркеры сцены для линий дрифта и знаков. Во время выполнения эти объекты обычно скрыты или управляются системой дрифта.

При перемещении или переименовании объектов-маркеров дрифта одновременно обновляйте файл дрифт-спота.

Объекты-маркеры следует чётко группировать в сцене, чтобы можно было обслуживать дрифт-спот без поиска по всему дереву уровня.

Связь с миссиями и POI

Дрифт-споты могут отображаться как POI на глобальной карте и могут использовать те же паттерны, что и данные путей миссий. Файлы race.race.json и bounds.sites.json следует считать частью спота, даже несмотря на то что аналогичные имена файлов используются и в миссиях.

Система дрифта также может сохранять прогресс игрока отдельно от файлов уровня. Изменение имён папок, идентификаторов целей или идентификаторов спотов после релиза может нарушить привязку существующих результатов или данных о выполнении.

Рабочий процесс создания

  1. Создайте или выберите дорогу/область для использования.
  2. Создайте объекты-маркеры и расставьте линии в сцене уровня.
  3. По возможности создайте spot.driftSpot.json с помощью редактора Drift Data Editor.
  4. Добавьте или обновите race.race.json для нужного маршрута.
  5. Добавьте bounds.sites.json для зачётной/игровой области.
  6. Настройте пороги целей в info.json на основе реальных тестовых заездов.
  7. Проверьте маркер на глобальной карте и цель телепорта.

Создание дрифт-спота с помощью инструментов редактора

При наличии используйте редактор Drift Data Editor. Он помогает поддерживать согласованность пространственных данных и ссылок на маркеры.

Рекомендуемый рабочий процесс с приоритетом редактора:

  1. Откройте уровень в World Editor.
  2. Создайте отдельную группу сцены для маркеров дрифт-спота.
  3. Разместите видимые меши-маркеры или знаки для стартовых/зачётных линий дрифта.
  4. Используйте Drift Data Editor для определения линий дрифта и пространственных данных.
  5. Сохраните спот в папку levels/<levelName>/driftSpots/<spotName>/.
  6. Создайте или обновите race.race.json для нужного пути дрифта.
  7. Создайте или обновите bounds.sites.json для зачётной области.
  8. Добавьте info.json с названием, данными разблокировки и целями.
  9. Перезагрузите уровень или данные дрифта и протестируйте с глобальной карты.

Делайте имена объектов-маркеров короткими, но понятными. Например:

garage_line_1_a
garage_line_1_b
garage_drift_sign

Создание файлов вручную

Ручное создание возможно, но при этом легче нарушить ссылки на маркеры или трансформации. При редактировании вручную действуйте в следующем порядке:

  1. Сначала создайте папку.
  2. Напишите info.json с названием и небольшим набором целей.
  3. Создайте bounds.sites.json в редакторе Sites Editor.
  4. Создайте или скопируйте путь race.race.json, затем настройте его для данного спота.
  5. В последнюю очередь напишите spot.driftSpot.json — после того как имена объектов-маркеров и трансформации окончательно установлены.

Минимальная полезная папка дрифт-спота обычно требует:

info.json
spot.driftSpot.json
race.race.json
bounds.sites.json

Не настраивайте пороги целей до тех пор, пока расстановка линий, маршрут и граница не будут окончательными.

Обновление существующего дрифт-спота

При редактировании существующего спота:

  • Сохраняйте имя папки, если может существовать прогресс игрока или данные карьеры.
  • Сохраняйте идентификаторы целей, такие как bronze, silver и gold, если только не планируется намеренный сброс прогресса.
  • Если объекты-маркеры переименованы, обновите все записи markerObjects.
  • Если маршрут существенно изменился, пересмотрите пороги очков.
  • Если телепорт на глобальной карте изменился, протестируйте спавн на нескольких транспортных средствах.

Устранение неполадок

Спот не отображается

Проверьте:

  • Папка находится в levels/<levelName>/driftSpots/.
  • Файл spot.driftSpot.json существует.
  • Файл info.json существует, или резервные данные отображения приемлемы.
  • Функция включена в текущем игровом режиме.

Маркеры отсутствуют или постоянно отображаются

Проверьте:

  • Каждое имя маркера в spot.driftSpot.json существует в сцене уровня.
  • Объекты-маркеры не были переименованы при распаковке префаба или очистке сцены.
  • Спот был перезагружен после редактирования имён маркеров.

Очки или цели кажутся неверными

Проверьте:

  • race.race.json соответствует нужному маршруту.
  • bounds.sites.json охватывает нужную зачётную область.
  • Пороги очков целей в info.json были настроены с использованием нужного класса транспортных средств.
  • Игрок въезжает на линию в ожидаемом направлении.

Чеклист проверки

  • Папка содержит spot.driftSpot.json и info.json.
  • Имя папки спота стабильно и уникально в пределах уровня.
  • race.race.json существует и соответствует нужному маршруту.
  • bounds.sites.json существует и охватывает нужную область.
  • Имена объектов-маркеров, на которые ссылается spot.driftSpot.json, существуют в сцене уровня.
  • Трансформация телепорта на глобальной карте безопасна и указывает на нужную стартовую область.
  • Пороги целей в info.json достижимы в обычном геймплее.
  • Имена папок и идентификаторы целей стабильны после того, как данные прогресса начинают зависеть от них.

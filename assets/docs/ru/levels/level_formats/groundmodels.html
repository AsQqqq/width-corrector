Этот документ описывает groundmodels.json — формат файла, используемый для определения физического поведения поверхностей в BeamNG.

Модели поверхностей описывают, как шины, колёса, реквизит и другие физические объекты взаимодействуют с поверхностями: асфальтом, гравием, песком, грязью, снегом, травой или материалами, похожими на воду.

Они влияют на:

  • Сцепление шин
  • Скользящее трение
  • Сопротивление качению / шероховатость
  • Деформацию и погружение
  • Гидродинамическое/жидкостное сопротивление
  • Тип частиц/материала столкновения
  • Поведение следов торможения

Расположение файлов

Общий/глобальный файл моделей поверхностей находится по пути:

/art/groundmodels.json

Уровни могут добавлять или переопределять модели поверхностей с помощью JSON-файлов, расположенных в:

levels/<levelName>/groundModels/*.json

При загрузке уровня движок:

  1. Загружает общий файл:
/art/groundmodels.json
  1. Выполняет поиск в папке текущего уровня для:
groundModels/*.json
  1. Объединяет модели поверхностей уровня с общим набором.
  2. Передаёт все модели поверхностей в физический движок.

Файлы уровней могут переопределять общие модели поверхностей, используя то же имя.

Базовая структура

Файл модели поверхности — это JSON-объект, в котором каждый ключ является именем модели поверхности.

Пример:

{
  "asphalt_special": {
    "roughnessCoefficient": 0,
    "staticFrictionCoefficient": 1.0,
    "slidingFrictionCoefficient": 0.9,
    "hydrodynamicFriction": 0.01,
    "stribeckVelocity": 6,
    "strength": 1,
    "collisiontype": "ASPHALT",
    "fluidDensity": 200,
    "flowConsistencyIndex": 10000,
    "flowBehaviorIndex": 0.5,
    "dragAnisotropy": 0,
    "skidMarks": true,
    "defaultDepth": 0,
    "shearStrength": 0
  }
}

Имена моделей поверхностей

Имена моделей поверхностей преобразуются в верхний регистр при загрузке.

Пример:

"asphalt_special": {}

становится:

ASPHALT_SPECIAL

Имена длиннее 31 символа усекаются.

Используйте короткие и уникальные имена моделей поверхностей. Имена длиннее 31 символа обрезаются при загрузке.

Псевдонимы

Модель поверхности может определять псевдонимы:

{
  "asphalt_special": {
    "aliases": ["road_special", "tarmac_extra"],
    "staticFrictionCoefficient": 1.0,
    "slidingFrictionCoefficient": 0.9
  }
}

Это регистрирует несколько имён для одних и тех же данных модели поверхности:

ASPHALT_SPECIAL
ROAD_SPECIAL
TARMAC_EXTRA

Псевдонимы удобны, когда различные системы, материалы рельефа или устаревший контент используют разные имена для одного и того же поведения поверхности.

Поля

Поле Тип По умолчанию Описание
aliases array[string] [] Дополнительные имена для данной модели поверхности.
roughnessCoefficient number 0 Шероховатость поверхности, используемая при физическом взаимодействии.
defaultDepth number 0 Глубина поверхности по умолчанию, используется для поведения деформируемого/мягкого рельефа.
staticFrictionCoefficient number 1 Коэффициент статического трения. Влияет на сцепление до начала скольжения.
slidingFrictionCoefficient number 0.7 Коэффициент скользящего/кинетического трения. Влияет на сцепление во время скольжения.
hydrodynamicFriction number 0.01 Сопротивление, обусловленное жидкостным/деформируемым поведением поверхности.
stribeckVelocity number 6 Параметр скорости для поведения перехода трения.
strength number 1 Прочность/сопротивление поверхности.
collisiontype string 0 Имя материала частиц/столкновения. Разрешается через таблицу материалов частиц.
fluidDensity number 200 Плотность, используемая для жидкостного взаимодействия с поверхностью.
flowConsistencyIndex number 10000 Параметр консистентности потока для неньютоновских/деформируемых поверхностей.
flowBehaviorIndex number 0.5 Параметр поведения потока для неньютоновских/деформируемых поверхностей.
dragAnisotropy number 0 Анизотропия сопротивления по направлениям.
skidMarks bool false Определяет, могут ли на данной поверхности оставаться следы торможения.
shearStrength number 0 Прочность поверхности на сдвиг.

Устаревшие / неправильно написанные поля

Загрузчик обнаруживает некоторые старые поля с опечатками и записывает ошибки в журнал.

Используйте:

"hydrodynamicFriction"

а не:

"hydrodnamicFriction"

Используйте:

"flowBehaviorIndex"

а не:

"flowBehaviourIndex"

Старые имена могут по-прежнему считываться для обратной совместимости, однако их следует исправить.

Неизвестные поля

Неизвестные атрибуты игнорируются и записываются в журнал как ошибки.

Пример:

Unknown ground model attribute: myCustomValue - IGNORED

Следует использовать только поддерживаемые поля.

Поля трения

staticFrictionCoefficient

Управляет сцеплением, когда шина/пятно контакта не скользит.

Более высокие значения, как правило, означают лучшее сцепление.

Пример:

"staticFrictionCoefficient": 1.0

slidingFrictionCoefficient

Управляет сцеплением во время скольжения.

Пример:

"slidingFrictionCoefficient": 0.8

Как правило, это значение ниже, чем статическое трение.

Шероховатость

roughnessCoefficient управляет физической шероховатостью поверхности.

Пример:

"roughnessCoefficient": 0.05

Более шероховатые поверхности могут делать езду более тряской или менее плавной в зависимости от автомобиля и взаимодействия шин.

Глубина и мягкий грунт

defaultDepth используется для моделей поверхностей, представляющих мягкий или деформируемый грунт.

Пример:

"defaultDepth": 0.15

Это полезно для:

  • Грязи
  • Песка
  • Снега
  • Гравия
  • Глубокой земли

Параметры жидкости/деформируемого грунта

Эти поля управляют более продвинутым жидкостным или деформируемым поведением поверхности:

"fluidDensity": 800,
"flowConsistencyIndex": 12000,
"flowBehaviorIndex": 0.6,
"hydrodynamicFriction": 0.05

Они полезны для таких поверхностей, как:

  • Грязь
  • Влажный песок
  • Глубокий снег
  • Зоны сопротивления, похожие на воду

Тип столкновения

collisiontype связывает модель поверхности с именем материала частиц/столкновения.

Пример:

"collisiontype": "DIRT"

Значение разрешается через таблицу материалов частиц.

Это может влиять на такие эффекты, как:

  • Частицы
  • Пыль
  • Обломки
  • Визуальное взаимодействие с поверхностью

Следы торможения

skidMarks управляет тем, допускаются ли следы торможения на данной модели поверхности.

Пример:

"skidMarks": true

Типичное использование:

Поверхность skidMarks
Асфальт true
Бетон true
Грунт зависит от настроек
Трава обычно false
Вода false

Модели поверхностей, специфичные для уровня

Уровни могут переопределять или добавлять модели поверхностей, размещая JSON-файлы в:

levels/<levelName>/groundModels/

Пример:

levels/example/groundModels/wet_surfaces.json

{
  "wet_asphalt": {
    "aliases": ["rain_asphalt"],
    "staticFrictionCoefficient": 0.75,
    "slidingFrictionCoefficient": 0.55,
    "roughnessCoefficient": 0.01,
    "hydrodynamicFriction": 0.03,
    "collisiontype": "ASPHALT",
    "skidMarks": true
  }
}

Файлы уровней объединяются после общего файла, поэтому они могут заменять общие определения при использовании того же имени.

Связь с материалами рельефа

Материалы рельефа могут ссылаться на модель поверхности с помощью:

groundmodelName

Пример поля материала рельефа:

"groundmodelName": "grass"

Когда транспортное средство едет по рельефу, использующему этот материал, соответствующая модель поверхности управляет физическим поведением поверхности.

Связь с материалами объектов

Материалы также могут определять поведение поверхности/столкновения через настройки физического материала в зависимости от типа объекта и настройки столкновения.

Для рельефа в частности, TerrainMaterial.groundmodelName является ключевой связью.

Перезагрузка

Модели поверхностей загружаются при запуске миссии.

Они также могут перезагружаться при изменении одного из загруженных JSON-файлов моделей поверхностей. Когда это происходит, система окружения перезагружает модели поверхностей, и коллизии могут быть перезагружены, чтобы новые значения вступили в силу.

Рекомендации

  • Используйте короткие и описательные имена.
  • Используйте псевдонимы для обратной совместимости или альтернативных имён.
  • Используйте файлы, специфичные для уровня, только для поведения, специфичного для данного уровня.
  • В большинстве случаев статическое трение должно быть выше скользящего.
  • Используйте skidMarks только там, где следы имеют смысл.
  • Используйте defaultDepth и параметры жидкости/деформируемого грунта для мягкого рельефа.
  • Избегайте неизвестных полей — они игнорируются.
  • Исправляйте поля с опечатками/устаревшие поля при обнаружении в журналах.
  • Проверяйте поведение транспортного средства после изменения значений трения или глубины.

Распространённые проблемы

Модель поверхности не применяется

Возможные причины:

  • В материале рельефа указано неверное значение groundmodelName
  • Имя модели поверхности было усечено
  • Файл модели поверхности не был загружен
  • Несоответствие имён после преобразования в верхний регистр
  • Материал/настройка столкновения не использует данную модель поверхности

Предупреждение о неизвестном атрибуте

JSON содержит неподдерживаемое поле. Удалите или переименуйте его.

Предупреждение о поле с опечаткой

Замените старые поля с опечатками:

hydrodnamicFriction → hydrodynamicFriction
flowBehaviourIndex → flowBehaviorIndex

Поверхность имеет слишком высокое сцепление

Уменьшите:

"staticFrictionCoefficient"
"slidingFrictionCoefficient"

Поверхность кажется слишком мягкой или глубокой

Уменьшите:

"defaultDepth"
"hydrodynamicFriction"
"fluidDensity"
"flowConsistencyIndex"

Следы торможения отсутствуют

Проверьте:

"skidMarks": true

и убедитесь, что взаимодействие материала/поверхности поддерживает следы торможения.

Итог

groundmodels.json определяет физическое поведение поверхностей.

Файл управляет тем, как транспортные средства и физические объекты взаимодействуют с поверхностями через трение, шероховатость, глубину, сопротивление, тип материала столкновения частиц и настройки следов торможения.

Общий файл загружается первым:

/art/groundmodels.json

Специфичные для уровня переопределения загружаются из:

levels/<levelName>/groundModels/*.json

Модели поверхностей связываются с рельефом через TerrainMaterial.groundmodelName.

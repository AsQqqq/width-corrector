Blender 2.80 — экспорт статичных объектов

  1. выделите все объекты для экспорта

  2. Меш коллизии должен называться "Colmesh-1". «Вырежьте» имя модели из названия объекта коллизии и удалите нижнее подчёркивание, оставив "Colmesh-1" (это имя меша, которое игра будет использовать для коллизий)

  3. Shift + S, «Cursor to World Origin»

  4. Shift + S, «Selection to Cursor»

  5. Ctrl + левый клик по "start01"

  6. Наведите курсор на окно 3D-вида и нажмите Ctrl + P, «Object»

  7. Рамкой выделите все пустые объекты (empties), необходимые для экспорта

  8. File > Export > Collada

  9. Откройте правую панель с иконкой шестерёнки (верхний правый угол) или нажмите «N»

  10. Настройте параметры экспорта в соответствии с exporting_statics_3.jpg, в первую очередь:

  • Selection Only
  • Снимите галочку «Copy» в разделе «Texture Options»

Несколько важных моментов
  • Все объекты должны быть дочерними к "start01"
  • "start01" должен быть дочерним к "base00"
  • Число в конце имени объекта ("800", "300" и т. д.) — это размер в пикселях, при котором объект переключится на следующий уровень детализации; чем больше число, тем раньше произойдёт переход к следующему LOD
  • Перед расстоянием LOD ОБЯЗАТЕЛЬНО должна стоять буква, иначе импорт завершится ошибкой:
    • "example_model_a800" — правильно
    • "example_model_800" — неправильно
  • Правильно называйте объекты перед экспортом
  • Blender не допускает дублирующихся имён объектов, поэтому если в одном файле Blender находится несколько объектов коллизии (при экспорте все они должны называться "Colmesh-1"), добавьте в начало имя объекта — его можно «вырезать» перед экспортом и вставить в имя файла при сохранении, чтобы ускорить процесс
  • Упорядочивайте файл:
    • Группируйте вспомогательные компоненты и основные объекты раздельно
    • Придерживайтесь единообразного расположения объектов
  • nulldetail# — это размер в пикселях, при котором игра перестаёт рендерить объект; он работает как дополнительный LOD и должен идти после всех остальных
  • Расположение вторичных пустых объектов не имеет значения (bb__autobillboard25, nulldetail10), однако, разместив их вблизи точки начала координат, вы сможете выбирать нужные из них в зависимости от того, как рисуете рамку выделения перед экспортом

На этой странице описано, как добавить радары контроля скорости и камеры фиксации проезда на красный свет на уровень.

Радары контроля скорости создаются из стандартных объектов уровня:

  • Объект BeamNGTrigger, обнаруживающий транспортные средства.
  • Необязательный световой объект, как правило SpotLight, используемый в качестве вспышки камеры.

Триггер содержит динамические поля, которые сообщают игре, какого типа радар и как он должен работать.

Обзор

Типичная конфигурация радара контроля скорости включает:

BeamNGTrigger
SpotLight

Триггер является функциональной частью. Световой объект необязателен и используется только для визуального эффекта вспышки.

Когда транспортное средство въезжает в триггер, игра проверяет:

  1. Настроен ли триггер как радар контроля скорости или камера фиксации проезда на красный свет.
  2. Движется ли транспортное средство в направлении положительной локальной оси Y триггера.
  3. Превышает ли скорость транспортного средства заданный порог.
  4. Для камер фиксации проезда на красный свет — горит ли в данный момент красный сигнал у связанного светофора.
  5. Нужно ли активировать необязательную вспышку света.
Значения скорости для радаров контроля скорости задаются в метрах в секунду.

Создание триггера

В World Editor создайте объект BeamNGTrigger и разместите его там, где радар должен фиксировать транспортные средства.

Триггер должен:

  • Перекрывать полную ширину дороги или полосы движения.
  • Иметь глубину в несколько метров по направлению движения.
  • Быть повёрнут таким образом, чтобы положительная локальная ось Y указывала в направлении движения транспортных средств.
  • Иметь уникальное имя объекта.
Если триггер слишком тонкий, быстрые транспортные средства могут проскочить сквозь него, не будучи надёжно зафиксированными.

Направление триггера

Триггер срабатывает только для транспортных средств, движущихся в направлении положительной локальной оси Y.

Иными словами, триггер имеет направление «вперёд». Транспортные средства, движущиеся в противоположном направлении, игнорируются.

Убедитесь, что триггер повёрнут правильно:

Positive local Y axis = valid travel direction

Если радар не срабатывает, возможно, триггер направлен в неверную сторону.

Поля триггера радара контроля скорости

Выберите BeamNGTrigger и добавьте следующие динамические поля.

Обязательные поля

Поле Значение Описание
speedTrapType speed Отмечает данный триггер как радар контроля скорости.
speedLimit Число, в м/с Ограничение скорости, используемое радаром.

Необязательные поля

Поле Значение Описание
speedLimitTolerance Число, в м/с Дополнительная скорость сверх ограничения, после которой срабатывает радар.
speedTrapLightName Имя объекта Имя светового объекта, используемого для эффекта вспышки.

Пример:

speedTrapType = speed
speedLimit = 13.89
speedLimitTolerance = 2.78
speedTrapLightName = speedTrapLight_01

Этот пример означает:

Speed limit: 13.89 m/s ≈ 50 km/h
Tolerance:   2.78 m/s ≈ 10 km/h
Trigger at:  16.67 m/s ≈ 60 km/h

Если speedLimitTolerance не задано, игра использует допуск по умолчанию — 10% от ограничения скорости.

Поля триггера камеры фиксации проезда на красный свет

Камера фиксации проезда на красный свет также использует BeamNGTrigger, но с другими динамическими полями.

Обязательные поля

Поле Значение Описание
speedTrapType redLight Отмечает данный триггер как камеру фиксации проезда на красный свет.
trafficSignalId Имя/идентификатор сигнала Светофор, к которому относится данная камера.

Необязательные поля

Поле Значение Описание
triggerSpeed Число, в м/с Минимальная скорость, необходимая для срабатывания камеры.
speedTrapLightName Имя объекта Имя светового объекта, используемого для эффекта вспышки.

Пример:

speedTrapType = redLight
trafficSignalId = trafficSignal_01
triggerSpeed = 5
speedTrapLightName = redLightCameraFlash_01

Если triggerSpeed не задано, используется значение по умолчанию:

5 m/s

Камера срабатывает только в том случае, если связанный светофор в данный момент показывает красный сигнал.

Размещение камеры фиксации проезда на красный свет

Разместите триггер камеры фиксации проезда на красный свет вблизи стоп-линии или сразу за ней — в зависимости от планировки перекрёстка.

Триггер должен фиксировать транспортные средства, въезжающие на перекрёсток при движении на красный свет.

Рекомендуемая конфигурация:

  • Триггер перекрывает контролируемую полосу или полосы движения.
  • Триггер выровнен по допустимому направлению движения.
  • Триггер имеет достаточную глубину для надёжного обнаружения быстрых транспортных средств.
  • Триггер расположен так, чтобы транспортные средства, правильно остановившиеся перед стоп-линией, его не активировали.

Определение идентификатора светофора

Для камер фиксации проезда на красный свет триггер должен знать, к какому светофору он относится.

Идентификаторы светофоров можно просмотреть, включив отладочный вывод светофоров из консоли:

core_trafficSignals.debugLevel = 2

Используйте отображаемое имя/идентификатор сигнала в качестве значения для:

trafficSignalId

Пример:

trafficSignalId = trafficSignal_01

Создание вспышки света

Создайте световой объект вблизи радара контроля скорости или камеры.

Чаще всего используется SpotLight, однако подойдёт любой световой объект, поддерживающий включение и отключение.

Световой объект должен:

  • Иметь уникальное имя объекта.
  • Быть указан в поле speedTrapLightName триггера.
  • По умолчанию быть отключён.

Установите для поля enabled светового объекта значение:

enabled = false

При необходимости световой объект можно настроить визуально:

  • Цвет
  • Яркость
  • Дальность
  • Поворот
  • Угол конуса
  • Тип блика (flare)

Для вспышки камеры рекомендуется использовать яркий белый свет с бликом.

Пример имени объекта:

speedTrapLight_01

Затем укажите его в триггере:

speedTrapLightName = speedTrapLight_01
Световой объект необязателен. Радар контроля скорости может работать без объекта вспышки.

Единицы измерения скорости

Значения скорости для радаров задаются в метрах в секунду.

Полезные значения для перевода:

Скорость Метры в секунду
30 km/h 8.33 m/s
50 km/h 13.89 m/s
60 km/h 16.67 m/s
80 km/h 22.22 m/s
100 km/h 27.78 m/s
120 km/h 33.33 m/s

Формула:

km/h ÷ 3.6 = m/s

Для миль в час:

mph × 0.44704 = m/s

Пример: базовый радар контроля скорости

Объекты:

BeamNGTrigger: speedTrapTrigger_01
SpotLight:     speedTrapLight_01

Динамические поля триггера:

speedTrapType = speed
speedLimit = 22.22
speedLimitTolerance = 2.78
speedTrapLightName = speedTrapLight_01

В результате создаётся радар контроля скорости со следующими параметрами:

Speed limit: 80 km/h
Tolerance:   10 km/h
Trigger at:  90 km/h

Пример: камера фиксации проезда на красный свет

Объекты:

BeamNGTrigger: redLightCameraTrigger_01
SpotLight:     redLightCameraFlash_01
TrafficSignal: trafficSignal_01

Динамические поля триггера:

speedTrapType = redLight
trafficSignalId = trafficSignal_01
triggerSpeed = 5
speedTrapLightName = redLightCameraFlash_01

Камера срабатывает при выполнении следующих условий:

  • Транспортное средство въезжает в триггер.
  • Транспортное средство движется в направлении положительной локальной оси Y триггера.
  • Скорость транспортного средства превышает 5 m/s.
  • Связанный светофор показывает красный сигнал.

Поведение во время работы

Когда транспортное средство въезжает в настроенный триггер, игра проверяет данные радара контроля скорости.

Для обычных радаров контроля скорости:

  • Скорость транспортного средства сравнивается со значением speedLimit + speedLimitTolerance.
  • Если транспортное средство едет слишком быстро, срабатывает событие радара контроля скорости.
  • Если настроена вспышка, свет кратковременно включается, а затем выключается.

Для камер фиксации проезда на красный свет:

  • Скорость транспортного средства сравнивается со значением triggerSpeed.
  • Проверяется состояние связанного светофора.
  • Камера срабатывает только при красном сигнале светофора.
  • Если настроена вспышка, свет вспыхивает.

Радары контроля скорости игнорируются, если транспортное средство находится слишком далеко от активной камеры.

Таблицы рекордов

В режиме свободной езды (freeroam) рекорды радаров контроля скорости могут сохраняться отдельно для каждого уровня.

Рекорды хранятся по уровню и имени триггера. В связи с этим имена триггеров радаров контроля скорости должны быть уникальными и неизменными.

Не переименовывайте триггер радара контроля скорости после выпуска, если только вы намеренно не хотите начать новую запись в таблице рекордов.

Рекомендации

  • Используйте один объект BeamNGTrigger на каждый радар контроля скорости или камеру фиксации проезда на красный свет.
  • Давайте каждому триггеру и световому объекту понятное уникальное имя.
  • Делайте триггер достаточно широким, чтобы он перекрывал всю зону обнаружения.
  • Делайте триггер глубиной в несколько метров для надёжного обнаружения.
  • Поворачивайте триггер так, чтобы положительная локальная ось Y указывала в допустимое направление движения.
  • Используйте метры в секунду для всех значений скорости.
  • Следите за тем, чтобы значение speedTrapLightName точно совпадало с именем светового объекта.
  • По умолчанию отключайте световой объект вспышки.
  • Используйте видимый тип блика для вспышек камеры.
  • Проверяйте работу с медленными, обычными и быстрыми транспортными средствами.
  • Проверяйте оба направления движения, чтобы убедиться, что срабатывает только нужное направление.

Распространённые проблемы

Радар контроля скорости не срабатывает

Возможные причины:

  • Поле speedTrapType отсутствует или написано с ошибкой.
  • Транспортное средство движется противоположно положительной оси Y триггера.
  • Скорость транспортного средства ниже порога срабатывания триггера.
  • Триггер слишком тонкий.
  • Триггер не перекрывает полосу движения.
  • На уровне выполняется активная миссия на переднем плане, которая подавляет поведение радаров контроля скорости в режиме свободной езды.

Радар контроля скорости срабатывает с неправильного направления

Триггер повёрнут неправильно.

Поверните BeamNGTrigger так, чтобы его положительная локальная ось Y указывала в нужное направление движения.

Вспышка не появляется

Возможные причины:

  • Значение speedTrapLightName не совпадает с именем светового объекта.
  • Световой объект не существует.
  • Свет слишком тусклый или расположен слишком далеко.
  • Световой объект не имеет подходящего блика или дальности.
  • Свет уже включён, что делает вспышку незаметной.

Камера фиксации проезда на красный свет не срабатывает

Возможные причины:

  • Поле speedTrapType не установлено в значение redLight.
  • Значение trafficSignalId неверно.
  • Светофор не показывает красный сигнал.
  • Значение triggerSpeed слишком высокое.
  • Триггер направлен в неверную сторону.
  • Триггер размещён не на нужной стоп-линии или полосе движения.

Итог

Для добавления радара контроля скорости:

  1. Разместите BeamNGTrigger.
  2. Поверните его так, чтобы положительная локальная ось Y указывала в направлении движения.
  3. Добавьте speedTrapType = speed.
  4. Установите speedLimit в метрах в секунду.
  5. При необходимости добавьте speedLimitTolerance.
  6. При необходимости добавьте отключённый SpotLight и укажите его имя в поле speedTrapLightName.

Для добавления камеры фиксации проезда на красный свет:

  1. Разместите BeamNGTrigger вблизи стоп-линии.
  2. Поверните его так, чтобы положительная локальная ось Y указывала в контролируемом направлении.
  3. Добавьте speedTrapType = redLight.
  4. Установите trafficSignalId.
  5. При необходимости установите triggerSpeed.
  6. При необходимости добавьте отключённый световой объект вспышки и укажите его имя в поле speedTrapLightName.

6.1 Назначение

Описывает, как определяются и организуются материалы, включая текстурные входы, обработку отражений и оптимизацию.

Редактор материалов можно открыть из главного меню: Window > Material Editor

Окно редактора материалов

6.2 Основные концепции

Материалы определяются в JSON (main.materials.json) и ссылаются на PNG-текстуры по относительным путям.

Имена материалов должны совпадать с записями mapTo в main.materials.json.

main.materials.json, пример определения материала

Отражения управляются и назначаются для каждого материала через выбор Reflection Mode в разделе Advanced - All Layers окна редактора материалов. Доступны 3 основных варианта.

  • None: кубическая карта не используется. Установлено по умолчанию и применяется для неметаллических и непрозрачных материалов
  • Level: материал использует информацию об отражениях из уровня
  • Cubemap: материал использует информацию об отражениях из пользовательской кубической карты (некоторые кубические карты уже доступны в игре)

Кубическая карта — это GPU-текстура, хранящая шесть квадратных изображений одинакового размера, соответствующих граням куба (±X, ±Y, ±Z). Она сэмплируется трёхмерным вектором направления.

Level — лучший вариант для создания корректных отражений от вашего уровня.

Кубические карты можно найти готовыми к использованию в интернете или создать самостоятельно.

Используйте кубические карты, если хотите задать пользовательские отражения для материала.

Вот пример пользовательской кубической карты, созданной для карты West Coast.

Пользовательская кубическая карта, созданная для карты West Coast

Кубические карты используются для объектов водной плоскости (в основном применяются в игре как Ocean) для симуляции отражений уровня на поверхности воды

6.3 Импорт материалов

Если вы хотите применить собственные материалы к мешу, следуйте описанному ниже процессу.

Нельзя напрямую применить материалы к мешу внутри движка. Вместо этого необходимо использовать внешнее программное обеспечение (Blender, 3Ds Max, Maya и т.д.), назначить материалы там и экспортировать меш.

Есть 2 возможных пути:

  1. Применить уже имеющийся в игре материал
  2. Создать собственный материал

6.5 Текстурные входы и калибровка

Основные текстурные входы в окне редактора материалов включают карты Base Color, Metallic, Normal, Roughness, Opacity и Ambient Occlusion

Для более детального контроля над материалом предусмотрены карты Base Color Detail и Normal Detail

Другие расширенные каналы карт: Base Color Palette и Emissive

Поддерживайте разумные диапазоны значений цвета и затенения, избегая крайностей.

Движок BeamNG использует формат карт нормалей OpenGL. При импорте текстур убедитесь, что ориентация карты нормалей соответствует рендереру.

6.6 Оптимизация

Карты BeamNG используют широкий спектр разрешений текстур в зависимости от применения материала.

Большинство текстур в игре создано с базовым разрешением 512 пикс./м.

Используйте размеры текстур, кратные степени двойки (POT), для сохранения корректных пропорций. Допускаются также пропорции 1:2 или 2:1, но всегда кратные степени двойки (напр. 512x512 пикс., 1024x512 пикс., 2048x4096 пикс.)

По возможности используйте общие материалы для снижения количества draw call-ов.

Следует контролировать использование текстурной памяти, выбирая подходящие разрешения. Resource checker > Assets Stat помогает оценить затраты памяти на ассеты.

6.7 Расположение данных

  • main.materials.json в домене уровня или арта.
  • PNG-текстуры в /art/textures и тематических папках (/art/roads, /art/terrains и т.д.).

См. также: Формат файла материала .

6.8 Типичные проблемы

  • Отсутствующие текстуры из-за ошибок в путях.
  • Слишком сильные отражения, выравнивающие затенение.
  • Непоследовательное именование материалов, приводящее к несоответствиям mapTo.

6.9 Критерии проверки

  • Отсутствие пропущенных или временных материалов в репрезентативных сценах.
  • Декали отображаются чисто без мерцания; атласы сокращают переключения материалов.
  • Отражения выглядят согласованно с типом поверхности и задуманным окружением.

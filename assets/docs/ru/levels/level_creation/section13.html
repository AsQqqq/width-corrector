13.1 Ключевые файлы и папки

  • level.json: конфигурация уровня / данные точки входа.

  • info.json: метаданные для выбора уровня, точек появления, представления в репозитории и связанных данных интерфейса.

  • main.materials.json: определения материалов со ссылками на текстуры.

  • main/items.level.json: данные объектов сцены, сохранённые редактором World Editor.

  • /levels/YourMap: домен уровня (данные сцены, специфичные для уровня ресурсы, метаданные).

  • /levels/YourMap/art/...: специфичные для уровня материалы, текстуры, меши, декали, данные леса и префабы.

  • /levels/YourMap/source: редактируемые оригиналы; не включайте их в пакетируемые релизы.

13.2 Основные объекты и системы

  • Terrain: земля на основе карты высот.

  • DecalRoad: дорожное покрытие на основе сплайна, проецируемое на поверхность.

  • MeshRoad: дорога на основе сплайна, генерирующая геометрию.

  • Forest: система инстансированной растительности.

  • TSStatic: стандартный класс статичного объекта.

  • WaterPlane: объект водной поверхности.

  • Groundmodels: поведение поверхности, привязанное к материалам.

  • World Editor: внутриигровой инструмент редактирования уровней.

  • SpawnSphere: объект сцены, используемый записями точек появления в info.json.

  • LevelInfo: общие для уровня настройки тумана, гравитации, звукового окружения, карты окружения и карты.

  • TimeOfDay / объекты неба: настройка времени суток, солнца, неба, облаков и освещения.

13.3 Термины разработки и пайплайна

  • Collada (.DAE): формат обмена мешами, используемый в пайплайне.

  • LOD (Level of Detail): упрощённые меши, используемые на расстоянии.

  • Atlas: текстура, содержащая несколько элементов для сокращения количества draw call-ов.

  • Cubemap: текстура, применяемая к материалу для обеспечения отражений.

  • PNG: обязательный формат текстур для данного пайплайна.

  • LOD: упрощённый вариант меша, используемый на расстоянии.

  • Impostor: представление на большом расстоянии для растительности или повторяющихся ресурсов.

  • Draw call: одна операция передачи данных от CPU для рендеринга пакета геометрии с материалом/состоянием рендеринга.

  • Overdraw: повторное затенение одних и тех же пикселей экрана, часто вызываемое прозрачностью, декалями или густой листвой.

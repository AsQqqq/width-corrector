Forest — это объект сцены, отвечающий за рендеринг, отсечение, коллизии и взаимодействие с ветром для элементов леса на уровне.

Система Forest предназначена для очень большого количества повторяющихся статических объектов, например:

  • Деревья
  • Кусты
  • Камни
  • Пучки травы
  • Брёвна
  • Небольшие природные пропы
  • Повторяющиеся рассеянные ассеты

Элементы Forest, как правило, эффективнее, чем размещение тысяч отдельных объектов TSStatic. Для повторяющейся растительности и рассеянных ассетов используйте систему Forest по возможности.

Связь с файлами леса

Объект Forest обычно не хранит расположение отдельных элементов непосредственно внутри items.level.json.

Вместо этого он загружает файлы расположения леса из папки уровня:

levels/<levelName>/forest/*.forest4.json

Определения типов элементов леса хранятся отдельно в:

levels/<levelName>/art/forest/managedItemData.json

Обе системы работают совместно:

Файл / Объект Назначение
Forest object Объект сцены, загружающий и управляющий данными леса.
managedItemData.json Определяет типы элементов леса: деревья, камни, кусты и пр.
*.forest4.json Хранит размещённые экземпляры элементов леса.
ForestItemData Объект рантайма/редактора, представляющий один тип элемента леса.
ForestWindEmitter Применяет ветер к ближайшим или ко всем элементам леса.
ForestBrush / ForestBrushElement Только для редактора: настройка кисти для рисования элементов леса.

Подробнее о самом формате файла смотрите в документации Forest Data (.forest4.json) and managedItemData.json.

Базовый объект Forest

Обычно уровень содержит один объект Forest со стандартным именем:

theForest

Пример:

{
  "class": "Forest",
  "name": "theForest",
  "position": [0, 0, 0],
  "rotationMatrix": [1, 0, 0, 0, 1, 0, 0, 0, 1],
  "scale": [1, 1, 1],
  "lodReflectScalar": 2
}

Объект Forest использует глобальные границы, и позиция его трансформа обычно не влияет на размещение элементов.

Элементы леса хранятся в файлах данных леса. Перемещение объекта Forest не перемещает элементы леса.

Важные поля объекта Forest

Поле Тип Описание
dataFile string Устаревшее поле источника данных леса. Скрыто / не сериализуется в современном использовании.
lodReflectScalar number Скаляр LOD, применяемый при рендеринге леса в отражениях.

lodReflectScalar 

"lodReflectScalar": 2

lodReflectScalar изменяет поведение дальнего обрезания / LOD при рендеринге леса в проходах отражений.

Более высокие значения могут заставить отражения леса использовать меньший уровень детализации или иначе отсекать объекты, улучшая производительность отражений.

Это главным образом инструмент управления производительностью и качеством для отражённых лесов.

Поведение при загрузке

При добавлении объекта Forest в сцену происходит следующее:

  1. Устанавливаются глобальные границы.
  2. Объект добавляет себя в сцену.
  3. Выполняется поиск файлов расположения леса.
  4. При наличии загружаются современные файлы .forest4.json.
  5. При необходимости выполняется откат к более старым форматам.
  6. Предварительно загружаются данные элементов леса.
  7. При необходимости строятся внутренние пространственные ячейки и пакеты рендеринга.

Приоритет отдаётся современным файлам:

*.forest4.json

Более старые форматы устарели:

*.forest
*.forest.json

При обнаружении старых файлов движок выводит предупреждения с просьбой к автору уровня пересохранить и удалить их.

Несколько файлов леса

Современные данные леса обычно разбиты по типам элементов.

Пример:

levels/example/forest/oak_large.forest4.json
levels/example/forest/pine_small.forest4.json
levels/example/forest/rock_large.forest4.json

При сохранении система Forest группирует элементы по внутреннему имени ForestItemData и записывает по одному файлу .forest4.json на каждый тип.

Неиспользуемые файлы леса могут быть удалены или заархивированы при сохранении — в зависимости от того, остались ли элементы леса данного типа.

Ячейки леса

Элементы леса организованы в пространственные ячейки.

Система ячеек используется для:

  • Быстрого поиска
  • Отсечения
  • Пакетного рендеринга
  • Коллизий
  • Рейкастов
  • Выделения в редакторе
  • Запросов ветра

Ячейки могут делиться, если содержат слишком много элементов. Это формирует пространственную иерархию, позволяя движку быстро пропускать области за пределами обзора камеры или области запроса.

Бакеты и пространственный поиск

Данные леса используют бакеты верхнего уровня, основанные на позиции элемента. Каждый бакет содержит ForestCell.

При добавлении элемента:

  1. Позиция элемента преобразуется в ключ бакета.
  2. Бакет/ячейка находится или создаётся.
  3. Элемент вставляется в эту ячейку.
  4. Если ячейка содержит слишком много элементов, она делится на подячейки.

Это делает запросы быстрее, чем перебор каждого элемента леса на уровне.

Поиск элементов

Элементы леса можно найти по:

  • Внутреннему ключу леса
  • UID
  • Позиции/бакету
  • Запросу по прямоугольнику
  • Запросу по радиусу
  • Запросу по фрустуму
  • Запросу по полигону

Используется в:

  • Рендеринге
  • Коллизиях
  • Выделении
  • Редактировании кистью
  • Инструментах биомов
  • Обновлениях ветра
  • Скриптах/инструментах

ForestItemData

ForestItemData определяет один тип элемента леса.

Включает:

  • Файл шейпа
  • Настройки коллизий
  • Радиус размещения
  • Настройки ветра
  • Настройки выравнивания по рельефу
  • Аннотацию

Важные поля:

Поле Описание
shapeFile Шейп, используемый данным типом элемента леса.
collidable Участвуют ли элементы данного типа в коллизиях.
radius Радиус размещения, используемый для предотвращения скученности.
snapRotationToTerrain Выравнивает размещённые элементы по нормали поверхности.
windScale Общее влияние ветра.
trunkBendScale Величина изгиба ствола.
branchAmp Амплитуда колебания веток от ветра.
detailAmp Амплитуда колебания листьев/кистей/деталей от ветра.
detailFreq Частота колебания листьев/кистей/деталей от ветра.
mass Используется в пружинной симуляции ветра.
rigidity Сопротивление силе ветра.
tightnessCoefficient Сопротивление изгибу.
dampingCoefficient Затухание колебаний со временем.
annotation Аннотация/классификация для отладки.

Загрузка шейпа

Каждый ForestItemData ссылается на шейп:

"shapeFile": "/levels/example/art/shapes/trees/oak_large.dae"

Шейп загружается один раз и используется совместно всеми элементами данного типа.

Это одна из главных причин эффективности Forest для повторяющихся ассетов.

Рендеринг

Рендеринг леса оптимизирован для большого числа повторяющихся экземпляров.

Рендерер:

  1. Находит видимые ячейки леса.
  2. Отсекает ячейки за пределами камеры/фрустума.
  3. Группирует видимые элементы по типу.
  4. Выбирает LOD для каждого элемента.
  5. Использует инстансинг/пакетирование для повторяющихся элементов.
  6. Использует импостеры/билборды там, где это возможно.

При включённом Scene Static Manager рендеринг леса может направляться через этот оптимизированный путь.

Forest vs TSStatic

Используйте Forest для большого числа повторяющихся объектов.

Используйте TSStatic для индивидуально размещённых объектов.

Сценарий использования Рекомендация
Тысячи деревьев Forest
Повторяющиеся камни/кусты Forest
Рассеивание травы/растительности Forest / GroundCover depending use
Одиночное здание TSStatic
Уникальный мост TSStatic
Вручную размещённый проп TSStatic
Объект, требующий особого трансформа или поведения материала TSStatic
Forest — как правило, наиболее лёгкий вариант для большого числа повторяющихся статических объектов. TSStatic лучше подходит для индивидуально размещённых мешей.

LOD и билборды

Элементы леса используют LOD, хранящиеся в их файлах шейпов.

Во время рендеринга движок оценивает размер элемента в пространстве экрана и выбирает подходящий уровень детализации. Если наименее детализированный уровень — это билборд/импостер, элемент может быть отрисован в виде пакетного импостера.

Вся ячейка леса может быть отрисована как пакет импостеров, если наибольший элемент в ячейке может быть переведён в билборд на текущей дистанции.

Это существенно повышает производительность для удалённой растительности.

Пакетирование импостеров

Ячейки леса могут формировать пакеты импостеров.

Пакет объединяет элементы, использующие одинаковый последний уровень детализации/тип импостера.

Это снижает накладные расходы на отрисовку для удалённых лесов.

Пакетирование используется, когда:

  • Тип элемента поддерживает уровень детализации билборд/импостер.
  • Ячейка находится достаточно далеко.
  • Импостеры не отключены.

Отладочные настройки могут принудительно включать или отключать импостеры:

Forest::smForceImposters
Forest::smDisableImposters

Рендеринг в отражениях

Forest использует lodReflectScalar для регулировки рендеринга в проходах отражений.

В отражениях детализация леса может быть снижена или отсечение выполняться раньше — для повышения производительности.

Это важно, поскольку рендеринг лесов в проходах отражений может быть ресурсозатратным.

Коллизия

Forest может обеспечивать коллизии для элементов леса, если их ForestItemData является collidable.

Система коллизий:

  1. Запрашивает ячейки леса вблизи области коллизии.
  2. Находит соответствующие элементы леса.
  3. Использует данные коллизии каждого элемента.
  4. Строит или повторно использует кэшированные данные коллизии для каждого шейпа.
  5. Добавляет трансформированные меши коллизий в физику.

Данные коллизии кэшируются по файлу шейпа, чтобы избежать повторного построения одной и той же коллизии для каждого элемента.

Элементы леса с коллизией могут стать ресурсозатратными, если меш коллизии слишком сложен или плотность элементов очень высока.

Рейкасты

Forest поддерживает рейкасты против элементов леса.

Рейкасты могут использовать:

  • Детали коллизии/видимости
  • Детали рендерного меша, при запросе

Используется в:

  • Выделении
  • Запросах коллизий
  • Рейкастах геймплея
  • Инструментах редактора

Если луч попадает в элемент леса, возвращаемым объектом является объект Forest, а дополнительные внутренние данные элемента могут использоваться инструментами.

Декали

Forest не поддерживает проецирование декалей так же, как это делают рельеф и статические меши.

На запросы списка полигонов для декалей Forest возвращает false.

PLC_Decal → false

Это означает, что декали дорог и другие проецируемые декали не должны полагаться на геометрию элементов леса в качестве приёмников декалей.

Ветер

Элементы леса могут реагировать на ветер, если их шейп/настройка материала это поддерживают.

Для работы ветра требуется:

  • windScale > 0
  • Данные цвета вершин в шейпе для весов ветра
  • Функции материала, поддерживающие деформацию от ветра
  • Активный ForestWindEmitter или глобальный источник ветра

Ветер воздействует на:

  • Изгиб ствола
  • Движение веток
  • Движение листьев/кистей/деталей
  • Отклик на локальный импульс

ForestWindEmitter

ForestWindEmitter задаёт ветер на уровне.

Может действовать как:

  • Глобальный направленный ветер
  • Локальный радиальный источник ветра

Важные поля:

Поле Описание
windEnabled Включает данный эмиттер.
radialEmitter Если true, ветер является локальным/радиальным вместо глобального направленного.
strength Сила ветра.
radius Радиус для радиальных эмиттеров.
gustStrength Максимальная сила порыва.
gustFrequency Частота порывов в секундах.
gustYawAngle Угол отклонения направления.
gustYawFrequency Частота отклонения направления.
gustWobbleStrength Случайное покачивание, добавляемое к порывам/турбулентности.
turbulenceStrength Сила турбулентности.
turbulenceFrequency Частота турбулентности.

Пример:

{
  "class": "ForestWindEmitter",
  "name": "forest_wind",
  "position": [0, 0, 0],
  "windEnabled": true,
  "radialEmitter": false,
  "strength": 1,
  "gustStrength": 0.5,
  "gustFrequency": 3,
  "gustYawAngle": 10,
  "gustYawFrequency": 4,
  "turbulenceStrength": 0.25,
  "turbulenceFrequency": 2
}

Радиус обновления ветра

Менеджер ветра собирает только ближайшие к камере деревья, реагирующие на ветер.

Это управляется настройками движка, такими как радиус эффекта ветра.

Элементы с:

windScale < 0.001

пропускаются при заполнении информации о размещении для обновления ветра.

Это позволяет не тратить процессорное время на статичные камни или растительность без реакции на ветер.

Радиальные импульсы

Forest может применять радиальные импульсы к ближайшим элементам, реагирующим на ветер.

Используется для эффектов, таких как:

  • Взрывы
  • Мощные локальные силы
  • Взаимодействие с порывами ветра

Импульс воздействует на локальные аккумуляторы ветра и вызывает временное отклонение/реакцию ближайшей растительности.

Редактор леса

Редактор леса использует объект Forest с именем:

theForest

Если активный лес отсутствует, редактор может создать его.

Редактор изменяет данные леса с помощью таких инструментов, как:

  • Рисование
  • Стирание
  • Стирание выбранных
  • Привязка к рельефу
  • Инструменты размещения биомов

Изменения сохраняются в данных леса и записываются в файлы .forest4.json.

ForestBrush

ForestBrush — это контейнер только для редактора, содержащий объекты ForestBrushElement.

Хранится в:

ForestBrushGroup

Кисть определяет, какие типы элементов леса можно рисовать и с какими правилами размещения.

ForestBrushElement

ForestBrushElement определяет параметры размещения для одного типа элемента леса.

Важные поля:

Поле Описание
forestItemData Тип элемента леса для размещения.
probability Относительная вероятность при случайном выборе.
rotationRange Диапазон случайного вращения по рысканью.
scaleMin Минимальный случайный масштаб.
scaleMax Максимальный случайный масштаб.
scaleExponent Смещение между минимальным и максимальным масштабом.
sinkMin Минимальная величина погружения.
sinkMax Максимальная величина погружения.
sinkRadius Радиус, используемый для погружения на склонах.
slopeMin Минимально допустимый уклон.
slopeMax Максимально допустимый уклон.
elevationMin Минимально допустимая высота.
elevationMax Максимально допустимая высота.

ForestBrushTool

ForestBrushTool — инструмент редактора для рисования, стирания и корректировки элементов леса.

Важные поля инструмента кисти:

Поле Описание
mode Режим: рисование, стирание, стирание выбранных или привязка к рельефу.
size Радиус кисти.
pressure Плотность/интенсивность мазка кисти.
hardness Жёсткость/спад кисти.
depthOffset Смещение выше/ниже рельефа относительно опорной точки элемента.
forceAlignToTerrain Принудительное выравнивание по нормали рельефа.
snapSinkEnabled Включает погружение при привязке к рельефу.
snapSink Величина погружения при операции привязки.
snapAnyForestItemType Привязка всех типов элементов вместо только выбранных.

Режим рисования

Режим рисования случайным образом размещает элементы леса внутри окружности кисти.

Процесс в общих чертах:

  1. Вычислить область кисти.
  2. Умножить на pressure.
  3. Выбрать типы элементов на основе ForestBrushElement.probability.
  4. Случайным образом задать масштаб и поворот.
  5. Выполнить рейкаст по рельефу/статической поверхности.
  6. Проверить ограничения по уклону/высоте.
  7. Проверить ближайшие элементы по радиусу размещения.
  8. Применить погружение/смещение по глубине.
  9. Добавить элемент с поддержкой отмены.

Это создаёт естественное случайное рассеивание.

Режимы стирания

Режим стирания удаляет элементы леса внутри кисти.

Режимы:

Режим Описание
Erase Стирает любой элемент леса под кистью.
EraseSelected Стирает только выбранные/активные типы элементов леса.

Объём стирания определяется давлением кисти.

Привязка к рельефу

Режим привязки к рельефу перемещает существующие элементы леса на поверхность рельефа/статики.

Также может:

  • Применять погружение
  • Выравнивать по нормали рельефа
  • Учитывать выбранные типы элементов

Это полезно после редактирования рельефа или когда элементы неправильно висят в воздухе или врезаются в поверхность.

Размещение биомов

Инструменты Forest включают расширенные рабочие процессы размещения биомов.

Инструменты биомов могут размещать или заменять элементы леса на основе:

  • Индекса материала рельефа
  • Маски в оттенках серого
  • Областей лассо/полигона
  • Диапазона уклонов
  • Диапазона высот
  • Областей спада
  • Зон исключения
  • Правил размещения по краям

Эти инструменты полезны для крупномасштабной генерации растительности.

Примеры использования:

  • Заполнить материал травы кустами
  • Разместить камни вдоль краёв биома
  • Рассеять деревья только на определённых склонах
  • Заменить старые элементы биома в выбранной области
  • Использовать маски для управления плотностью растительности

Контекстные ID

Элементы леса могут хранить контекстный ID инструмента.

Примеры:

  • Обычное размещение в редакторе
  • Размещение инструментами биомов

Это позволяет инструментам определять, какие элементы созданы инструментами биомов, и выборочно удалять/заменять только эти элементы.

Сохранение

При сохранении элементы леса группируются по типу и записываются в:

levels/<levelName>/forest/<type>.forest4.json

Имя файла формируется из типа/внутреннего имени элемента леса с заменой недопустимых символов.

Если тип элемента больше не содержит элементов:

  • Если лес пуст, файл может быть удалён.
  • В противном случае неиспользуемые файлы могут быть переименованы/заархивированы.

Состояние изменений

Данные леса отслеживают, были ли они изменены.

Редактор использует это, чтобы определить, нужно ли сохранять данные леса.

Полезные команды/методы:

save()
reload()
clear()
regenCells()
isDirty()

Зонирование

Ячейки леса обновляют состояние видимости зоны при изменении зонирования сцены.

Это позволяет корректно отсекать ячейки леса для систем видимости внутри/снаружи помещений.

Отладочная статистика

Система Forest предоставляет некоторые отладочные данные:

$Forest::totalCells
$Forest::cellsRendered
$Forest::cellItemsRendered
$Forest::cellsBatched
$Forest::cellItemsBatched

Они могут помочь диагностировать производительность рендеринга леса.

Рекомендации

  • Используйте Forest для повторяющихся природных/рассеянных ассетов.
  • Используйте TSStatic для индивидуальных уникальных объектов.
  • Оптимизируйте шейпы элементов леса.
  • Используйте LOD и детали билборд/импостер для деревьев.
  • Держите меши коллизий простыми.
  • Отключайте коллизии для декоративной листвы, когда это возможно.
  • Используйте radius для предотвращения скученности.
  • Включайте ветер только для ассетов, которым это необходимо.
  • Устанавливайте windScale в 0 для камней/статичных пропов.
  • Используйте инструменты биомов для крупномасштабного размещения.
  • Сохраняйте данные леса после серьёзных правок.
  • Удаляйте старые файлы .forest / .forest.json после обновления.

Распространённые проблемы

Элементы леса не отображаются

Возможные причины:

  • Отсутствует объект Forest
  • Отсутствуют файлы .forest4.json
  • Отсутствует managedItemData.json
  • Несовпадение внутреннего имени ForestItemData
  • Некорректный shapeFile
  • Ошибка загрузки шейпа

Некоторые элементы отсутствуют

Возможные причины:

  • Отсутствует определение типа элемента
  • Переименованный/удалённый ForestItemData
  • Некорректные строки в .forest4.json
  • Масштаб ниже минимального

Коллизия леса слишком ресурсозатратна

Возможные причины:

  • Слишком много элементов с коллизией
  • Сложные меши коллизий
  • Высокая плотность размещения
  • Коллизия включена для мелкой декоративной листвы

Ветер не воздействует на деревья

Проверьте:

  • windScale больше 0
  • В шейпе есть данные цвета вершин для весов ветра
  • Материал поддерживает деформацию от ветра
  • ForestWindEmitter существует и включён
  • Элемент находится в радиусе эффекта ветра

Элементы висят над рельефом

Используйте привязку к рельефу, проверьте опорную точку элемента или скорректируйте погружение/смещение по глубине.

Элементы слишком глубоко уходят в землю

Уменьшите значения погружения или проверьте опорную точку ассета.

Редактор леса сообщает об отсутствии леса

Создайте или добавьте объект Forest с именем theForest.

Итог

Forest — главная система для рендеринга и управления большим числом повторяющихся статических элементов на уровне.

Она загружает данные расположения из файлов .forest4.json, использует определения ForestItemData из managedItemData.json, организует элементы в пространственные ячейки, поддерживает отсечение и пакетирование, может обеспечивать коллизии и поддерживает деформацию от ветра для растительности.

Используйте Forest для крупномасштабных повторяющихся природных ассетов, а TSStatic — для индивидуально размещённых статических мешей.

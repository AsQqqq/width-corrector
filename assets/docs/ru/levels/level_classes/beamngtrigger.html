BeamNGTrigger — это триггерный объём для игровой механики. Он вызывает Lua-код, когда транспортные средства или другие объекты входят, покидают, перекрываются или полностью находятся внутри триггерного объёма.

Чаще всего используется для автобусных остановок, ловушек скорости, камер контроля проезда на красный сигнал светофора, логики гаражей/автосалонов, вспомогательных элементов миссий и других скриптовых взаимодействий на уровне.

Базовый пример 

{
  "class": "BeamNGTrigger",
  "name": "speedTrapTrigger_01",
  "__parent": "speedCamera_001",
  "position": [100, 200, 5],
  "rotationMatrix": [1, 0, 0, 0, 1, 0, 0, 0, 1],
  "scale": [10, 8, 5],
  "luaFunction": "onBeamNGTrigger",
  "TriggerType": "Box",
  "TriggerMode": "Center",
  "TriggerTestType": "Race corners"
}

Важные поля

Поле Тип Описание
class string Должно быть "BeamNGTrigger".
name string Имя объекта сцены.
__parent string Родительская группа.
position array[3] Центр триггерного объёма.
rotationMatrix array[9] Ориентация триггера.
scale array[3] Размер триггерного объёма.
TriggerType enum/string Форма триггера, обычно "Box" или "Sphere".
TriggerMode enum/string Условие срабатывания: "Center", "Contains" или "Overlaps".
TriggerTestType enum/string Тип проверки, обычно "Race corners" или "Bounding box".
luaFunction string Вызываемая Lua-функция. Если пусто, используется onBeamNGTrigger.
tickPeriod integer Период тика в миллисекундах при включённом тикировании.
ticking bool Включает повторные вызовы тика триггера.
debug bool Отображает отладочную визуализацию триггера.
debugInEditor bool Отображает отладочную визуализацию триггера в редакторе.
triggerColor color Цвет отладочной отрисовки.
cameraOnEnter string Камера, на которую переключиться при входе.
defaultOnLeave bool Возврат к обычной камере при выходе.

Режимы триггера

Значение Смысл
Center Проверяет, находится ли центр объекта внутри триггера.
Contains Проверяет, полностью ли объект находится внутри триггера.
Overlaps Проверяет, перекрывается ли объект с триггерным объёмом.

Используйте Center для большинства игровых объёмов. Используйте Overlaps или Contains, когда размер транспортного средства или объекта должен иметь значение.

Lua-функция

luaFunction задаёт Lua-коллбэк:

"luaFunction": "onBeamNGTrigger"

Если поле пустое, триггер использует имя коллбэка по умолчанию:

onBeamNGTrigger

Скрипты уровней и игровые системы также могут считывать дополнительные динамические поля, сохранённые в объекте, например метаданные ловушки скорости или автобусной остановки.

Поля такие как speedLimit, speedTrapType, trafficSignalId или пользовательские данные автобусной остановки — это динамические поля, используемые игровыми скриптами. Сохраняйте их, даже если они не являются основными полями BeamNGTrigger.

Замечания для инструментов

Для инструментов, генерирующих триггеры:

  • Сохраняйте динамические поля, используемые игровыми скриптами.
  • Оставляйте имена триггеров стабильными, если на них ссылаются скрипты или файлы данных.
  • Используйте scale для задания триггерного объёма.
  • Поддерживайте luaFunction в соответствии с игровой системой, которая потребляет триггер.
  • Используйте debugInEditor в процессе разработки, затем отключите его, если триггер не должен отображаться в обычном режиме.

Распространённые проблемы

  • Триггер ничего не делает, потому что luaFunction указана неверно или отсутствует.
  • Триггер срабатывает слишком рано или слишком поздно из-за неправильного масштаба или режима.
  • Транспортное средство не активирует триггер, потому что тип проверки не соответствует предполагаемым границам объекта.
  • Сгенерированные триггеры теряют пользовательские игровые поля.

Критерии проверки

  • Триггерный объём охватывает предполагаемую игровую зону.
  • Lua-коллбэк существует и получает ожидаемые события триггера.
  • Динамические поля, требуемые владеющей игровой системой, присутствуют.
  • Отладочная отрисовка подтверждает размещение и размер в редакторе.

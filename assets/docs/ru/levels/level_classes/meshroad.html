MeshRoad — это дорожный объект на основе сплайна, генерирующий реальную меш-геометрию из узлов.

Используйте его, когда дороге нужна реальная толщина, боковые грани, мосты, пандусы или геометрия, которая не является просто декалью, проецируемой на рельеф.

Для окрашенных дорожных поверхностей, разметки, AI-маршрутов и дорог, проецируемых на рельеф, DecalRoad обычно является более предпочтительным выбором.

Basic example

Простой пример записи MeshRoad в items.level.json:

{
  "class": "MeshRoad",
  "name": "bridge_meshroad_01",
  "__parent": "MissionGroup",
  "topMaterial": "road_asphalt",
  "bottomMaterial": "road_concrete",
  "sideMaterial": "road_concrete",
  "textureLength": 8,
  "breakAngle": 3,
  "widthSubdivisions": 0,
  "nodes": [
    [0, 0, 5, 8, 0.5, 0, 0, 1],
    [40, 0, 5, 8, 0.5, 0, 0, 1],
    [80, 10, 7, 8, 0.5, 0, 0, 1]
  ]
}

Каждый узел хранится как:

[x, y, z, width, depth, normalX, normalY, normalZ]

Important fields

Поле Тип Описание
class string Должно быть "MeshRoad".
name string Имя объекта на сцене.
__parent string Родительская группа.
topMaterial string Материал верхней поверхности дороги.
bottomMaterial string Материал нижней поверхности.
sideMaterial string Материал боковых, передней и задней граней.
textureLength number Длина в метрах до повторения материала вдоль дороги.
breakAngle number Пороговый угол в градусах, используемый при генерации срезов дороги из сплайна.
widthSubdivisions integer Количество поперечных подразделений для генерируемых вершин.
nodes array Упорядоченный список узлов. Каждый узел имеет формат [x, y, z, width, depth, normalX, normalY, normalZ].

Node data

MeshRoad хранит свою форму в виде упорядоченного списка узлов.

Пример:

"nodes": [
  [0, 0, 5, 8, 0.5, 0, 0, 1],
  [40, 0, 5, 8, 0.5, 0, 0, 1],
  [80, 10, 7, 8, 0.5, 0, 0, 1]
]

Каждый узел содержит:

Индекс Значение
0 Позиция X
1 Позиция Y
2 Позиция Z
3 Ширина дороги
4 Глубина/толщина дороги
5 Normal X
6 Normal Y
7 Normal Z

Меш дороги генерируется между последовательными узлами.

Не обращайтесь с узлами MeshRoad как с узлами DecalRoad. Узлы MeshRoad включают данные о глубине и нормали.

Creating MeshRoads

В World Editor создавайте MeshRoad, когда вам нужна генерируемая геометрия дороги вместо проецируемой декали.

Типичный рабочий процесс:

  1. Создайте объект MeshRoad.
  2. Расставьте узлы вдоль предполагаемой центральной линии дороги.
  3. Задайте ширину и глубину узлов.
  4. Отрегулируйте высоту и нормаль узлов так, чтобы полотно дороги повторяло необходимый уклон и угол поперечного наклона.
  5. Назначьте topMaterial, bottomMaterial и sideMaterial.
  6. Настройте textureLength, breakAngle и widthSubdivisions.
  7. Проверьте визуальное сглаживание, коллизии и поведение транспортного средства в игре.

Используйте меньше, но хорошо расположенных узлов для плавных дорог. Добавляйте узлы там, где дорога должна намеренно менять направление, ширину, глубину, высоту или угол поперечного наклона.

MeshRoad игнорирует стандартное масштабирование объектов. Для формирования дороги используйте узлы, ширину, глубину и настройки генерируемой геометрии.

Generation and smoothing

MeshRoad не соединяет узлы прямыми меш-пролётами напрямую.

При регенерации движок:

  1. Копирует каждый узел в узел сплайна, содержащий позицию, ширину, глубину и нормаль.
  2. Строит сплайн CatmullRom<MeshRoadSplineNode> из этих узлов.
  3. Дискретизирует сплайн между каждой парой авторских узлов.
  4. Создаёт срезы дороги вдоль дискретизированного сплайна.
  5. Строит сегменты дороги между последовательными срезами.
  6. Генерирует буферы вершин и индексов верхней, нижней и боковых частей на основе этих срезов.

Это означает, что центральная линия, ширина, глубина и нормаль — всё интерполируется вдоль сплайна.

Catmull-Rom spline

Алгоритм сглаживания — интерполяция Catmull-Rom.

Движок строит сплайн на основе авторских узлов:

position + width + depth + normal

Для каждого отрезка между узлами он вычисляет сплайн и измеряет приблизительную длину отрезка. Отрезок разбивается на мелкие шаги перед принятием решения о расположении финальных срезов дороги.

Интерполяция Catmull-Rom даёт плавные кривые через контрольные узлы. Это удобно при создании дорог, однако плотное или неравномерное расположение узлов может приводить к нежелательным изгибам или выходу за пределы.

Хорошие практики создания:

  • Сохраняйте равномерное расстояние между узлами.
  • Добавляйте вспомогательные узлы до и после резких переходов.
  • Избегайте резких изменений ширины/глубины на очень коротких участках.
  • Проверяйте нормали угла поперечного наклона на поворотах и пандусах.

breakAngle

breakAngle управляет количеством генерируемых срезов, сохраняемых из дискретизированного сплайна.

В процессе генерации движок сравнивает текущее дискретизированное направление с направлением в последнем сохранённом срезе. Новый срез сохраняется, когда:

  • Это начало отрезка.
  • Это последний образец последнего отрезка.
  • Изменение угла больше, чем breakAngle.

Пример:

"breakAngle": 3

Меньшие значения создают больше срезов и более гладкую генерируемую геометрию, особенно на поворотах.

Большие значения создают меньше срезов, что может уменьшить объём геометрии, но сделает повороты более угловатыми.

Очень маленькие значения breakAngle могут увеличить количество генерируемых вершин, треугольников, данных коллизий и затраты на редактирование. Используйте наименьшее значение, обеспечивающее необходимую визуальную плавность и плавность езды.

Slice orientation

Каждый генерируемый срез получает локальную систему координат:

Вектор Значение
fvec Направление вперёд вдоль дороги.
rvec Направление вправо поперёк ширины дороги.
uvec Направление вверх от нормали узла/сплайна.

Движок использует нормаль среза и позиции соседних срезов для вычисления этих векторов.

Затем вычисляются:

Точка Значение
p1 Центральная точка среза дороги.
p0 Точка верхнего левого края.
p2 Точка верхнего правого края.
pb0 Точка нижнего левого края.
pb2 Точка нижнего правого края.

Ширина расширяется от центра вдоль rvec. Глубина уходит вниз вдоль uvec.

Если нормали непоследовательны, генерируемая система координат может скручиваться. Это наиболее распространённая причина артефактов угла поперечного наклона или боковых граней на генерируемых меш-дорогах.

widthSubdivisions

widthSubdivisions добавляет дополнительные вершины поперёк верхней поверхности.

Пример:

"widthSubdivisions": 2

Верхняя поверхность использует:

2 + widthSubdivisions

вершин на срез.

Это может помочь широким дорогам или секциям с поперечным наклоном более плавно затеняться и деформироваться по ширине. Это также увеличивает количество вершин и треугольников.

Нижняя и боковые поверхности не разбиваются поперечно так же, как верхняя поверхность.

Texture mapping

textureLength управляет координатой V вдоль дороги.

При генерации вершин движок измеряет расстояние между последовательными срезами и увеличивает координату текстуры на:

sliceDistance / textureLength

Пример:

"textureLength": 8

Меньшие значения заставляют текстуру повторяться чаще. Большие значения растягивают текстуру на большее расстояние вдоль дороги.

MeshRoad vs DecalRoad

Сценарий использования Рекомендуемый класс
Дорожная разметка или поверхность, проецируемая на рельеф DecalRoad
Данные AI/навигационного маршрута DecalRoad
Мост с видимыми боковыми гранями MeshRoad
Приподнятый пандус или пользовательская геометрия дороги MeshRoad
Дорога на существующем меше с материалом декали DecalRoad с overObjects

MeshRoad создаёт геометрию. DecalRoad отрисовывает декаль на существующей геометрии.

Materials

MeshRoad может использовать три отдельных материала:

"topMaterial": "road_asphalt",
"bottomMaterial": "road_concrete",
"sideMaterial": "road_concrete"

Материалы должны бесшовно тайлиться вдоль дороги.

textureLength управляет частотой повторения материала:

"textureLength": 8

Tool notes

Для инструментов, генерирующих дороги:

  • Сохраняйте все восемь значений узла.
  • Сохраняйте порядок узлов стабильным.
  • По возможности нормализуйте нормали узлов.
  • Генерируйте достаточно равномерное расстояние между узлами для более плавных результатов Catmull-Rom.
  • Добавляйте вспомогательные узлы вокруг острых поворотов, крутых пандусов или резких изменений ширины/глубины.
  • Настраивайте breakAngle вместо того, чтобы бездумно добавлять чрезмерное количество авторских узлов.
  • Используйте DecalRoad, если результат — только проецируемая визуальная дорога.
  • Используйте MeshRoad, когда требуется генерируемая геометрия, толщина или поведение наподобие моста.
  • Проверяйте коллизии и поведение транспортного средства в игре после изменения генерируемых узлов.

Common issues

  • Дорога скручивается, потому что нормали узлов неверны или непоследовательны.
  • Материалы растягиваются, потому что textureLength слишком велик.
  • У дороги отсутствуют боковые/нижние визуальные элементы, потому что sideMaterial или bottomMaterial не указан.
  • Генерируемые инструментом узлы используют четырёхзначный формат DecalRoad вместо восьмизначного формата MeshRoad.
  • Повороты выглядят угловатыми, потому что breakAngle слишком велик.
  • Меш слишком плотный, потому что breakAngle слишком мал или расстояние между узлами избыточно.

Validation criteria

  • Все узлы имеют восемь числовых значений.
  • Материалы верхней, боковых и нижней поверхностей существуют.
  • Геометрия дороги следует заданному маршруту без скручивания.
  • Повороты достаточно плавны на нормальной скорости езды.
  • Генерируемая геометрия и коллизии остаются разумными для длины дороги.
  • Транспортные средства могут ехать по генерируемой поверхности так, как задумано.
  • Дороги-декали и AI-маршруты добавляются отдельно, когда требуются навигационные данные.

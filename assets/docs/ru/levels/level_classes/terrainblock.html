TerrainBlock — это объект сцены, представляющий рельеф на основе карты высот в уровне.

Он загружает данные высот рельефа из файла .ter и связывает их с данными материалов рельефа, изображением миникарты, коллизией рельефа и его рендерингом.

В большинстве обычных уровней с открытым пространством есть один основной объект рельефа, которому обычно присваивают имя:

theTerrain

Basic example

Типичная запись TerrainBlock в файле items.level.json:

{
  "class": "TerrainBlock",
  "name": "theTerrain",
  "__parent": "LevelObjects",
  "position": [-1024, -1024, 0],
  "terrainFile": "/levels/example/theTerrain.ter",
  "materialTextureSet": "exampleTerrainMaterialTextureSet",
  "minimapImage": "levels/example/example_minimap.png",
  "squareSize": 1,
  "maxHeight": 256,
  "baseTexSize": 2048,
  "castShadows": true
}

Important fields

Поле Тип Описание
class string Должно быть "TerrainBlock".
name string Имя объекта сцены. Обычно "theTerrain".
__parent string Родительская группа, как правило MissionGroup или группа объектов уровня.
position array[3] Позиция блока рельефа в мировых координатах.
terrainFile string/path Исходный файл данных рельефа, обычно .ter.
minimapImage string/path Изображение миникарты для данного блока рельефа.
materialTextureSet string Набор текстур материала рельефа, используемый для базового, макро- и детализированного размеров текстур.
castShadows bool Позволяет рельефу отбрасывать тени.
squareSize number Шаг сетки рельефа в метрах на один замер высоты.
maxHeight number Максимальный диапазон высот в метрах.
baseTexSize integer Настройка размера базовой текстуры рельефа.
lightMapSize integer Устаревший размер карты освещения рельефа.
screenError integer Устаревший параметр; в целом не полезен при современном создании контента.
TerrainBlock не поддерживает обычное масштабирование объекта. Вместо масштабирования объекта используйте squareSize, maxHeight, карту высот и рабочий процесс импорта рельефа.

Terrain file

Наиболее важное поле:

"terrainFile": "/levels/example/theTerrain.ter"

Этот файл хранит данные карты высот рельефа, используемые движком.

Уровень также может использовать связанные изображения и данные рельефа:

  • Исходные изображения карты высот
  • Карты отверстий
  • Карты слоёв материалов рельефа
  • Карты глубины
  • Изображение миникарты
  • Набор текстур материала рельефа

Подробнее о форматах .ter, .terrain.json, материалах рельефа и упаковке текстур см. в документации Terrain format .

Position and size

position задаёт расположение блока рельефа в мировом пространстве.

"position": [-1024, -1024, 0]

squareSize управляет расстоянием между замерами высоты:

"squareSize": 1

maxHeight управляет диапазоном высот рельефа:

"maxHeight": 256

Изменение этих значений после создания контента может повлиять на дороги, расстановку объектов, выравнивание материалов рельефа и читаемость игрового пространства.

Terrain materials

materialTextureSet ссылается на настройку текстур рельефа, используемую рендерером.

"materialTextureSet": "exampleTerrainMaterialTextureSet"

Материалы рельефа отличаются от обычных материалов мешей. Они наносятся поверх рельефа и используют базовые, макро- и детализированные текстуры.

Поведение модели грунта обычно задаётся через настройку материалов рельефа, а не через сам объект TerrainBlock.

Tool notes

Для инструментов, создающих или изменяющих данные рельефа:

  • Следите за тем, чтобы пути в terrainFile были корректными и переносимыми.
  • Сохраняйте значения position, squareSize и maxHeight, если инструмент не перестраивает масштаб рельефа намеренно.
  • Там, где это требуется, выравнивайте изображения, связанные с рельефом, по разрешению рельефа.
  • При перемещении или изменении масштаба рельефа обновляйте дороги, данные леса, почвенное покрытие, путевые точки и точки появления.
  • Не добавляйте обычное поле scale к TerrainBlock — это не рабочий процесс масштабирования рельефа.

Common issues

  • Рельеф отображается не там, где нужно, потому что position не совпадает с расстановкой объектов и дорог.
  • Рельеф рендерится чёрным или некорректно, так как отсутствует materialTextureSet или материалы рельефа.
  • Дороги, леса или почвенное покрытие смещаются после изменения squareSize, maxHeight или позиции рельефа.
  • Миникарта не совпадает с рельефом, потому что minimapImage указывает на устаревшее изображение.

Validation criteria

  • terrainFile существует и загружается.
  • Набор текстур материала рельефа существует.
  • Позиция, размер и высота рельефа соответствуют задуманному масштабу мира.
  • Почвенное покрытие и расположение леса по-прежнему совпадают с рельефом.
  • Дороги, точки появления, путевые точки и основные реквизиты по-прежнему расположены на ожидаемых поверхностях.

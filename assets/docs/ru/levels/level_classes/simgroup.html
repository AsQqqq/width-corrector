SimGroup — это контейнерный объект, используемый для организации других объектов уровня в дереве сцены и в файлах items.level.json.

В большинстве уровней используется корневая группа с именем:

MissionGroup

Вложенные группы широко применяются для хранения пропсов, источников освещения, дорог, объектов ландшафта, маркеров и сгенерированного контента в удобных папках.

Базовый пример

Простая запись группы в items.level.json:

{
  "class": "SimGroup",
  "name": "road_props",
  "__parent": "MissionGroup"
}

Затем объекты могут использовать эту группу в качестве родителя:

{
  "class": "TSStatic",
  "name": "barrier_01",
  "__parent": "road_props",
  "shapeName": "/levels/example/art/shapes/barrier.dae",
  "position": [10, 20, 0],
  "rotationMatrix": [1, 0, 0, 0, 1, 0, 0, 0, 1],
  "scale": [1, 1, 1]
}

Важные поля

Поле Тип Описание
class string Должно быть "SimGroup".
name string Имя группы. Другие объекты используют это значение в поле __parent.
__parent string Имя родительской группы. Обычно "MissionGroup" или другая группа.
persistentId string Стабильный идентификатор для редактора/среды выполнения, если присутствует. Как правило, генерируется автоматически.
hidden bool Скрывает группу в поддерживаемых контекстах редактора/среды выполнения. Дочерние объекты при этом могут существовать.
enabled bool/string Состояние активности группы, если применимо.

Связь с __parent

Файлы объектов уровня обычно хранят иерархию через поле __parent.

Пример:

{
  "name": "lights_night",
  "class": "SimGroup",
  "__parent": "MissionGroup"
}

Дочерний объект:

{
  "name": "street_lamp_01",
  "class": "SpotLight",
  "__parent": "lights_night"
}

Группа должна быть загружена раньше дочерних объектов, которые на неё ссылаются, либо инструмент загрузки должен восстанавливать родительские связи самостоятельно.

Когда внешние инструменты перемещают объекты между файлами, сохраняйте значение __parent или намеренно восстанавливайте ту же иерархию групп.

MissionGroup

MissionGroup — главная группа уровня, используемая деревом сцены.

Типичный основной файл объектов содержит:

{
  "name": "MissionGroup",
  "class": "SimGroup",
  "enabled": "1"
}

Большинство объектов уровня являются прямыми или косвенными потомками MissionGroup.

Примечания для инструментов

Для инструментов, генерирующих или преобразующих уровни:

  • Сохраняйте имена групп, на которые ссылается __parent.
  • Сохраняйте значения persistentId, если только не создаётся новая группа.
  • Размещайте сгенерированные группы рядом с контентом, которым они управляют.
  • Не сводите всю сцену в MissionGroup: крупные уровни используют вложенные группы для удобства редактирования.
  • Не удаляйте группу, не переместив или не удалив намеренно все её дочерние объекты.

Типичные проблемы

  • Объекты оказываются не в том месте дерева сцены, потому что __parent ссылается на неверную группу.
  • Сгенерированные объекты не загружаются в нужную группу, поскольку запись этой группы отсутствует.
  • Переименование группы нарушает размещение дочерних объектов, если значения __parent не обновлены.

Критерии валидации

  • MissionGroup существует.
  • Каждое сгенерированное значение __parent разрешается в существующую SimGroup.
  • Большие наборы объектов организованы в осмысленные группы.
  • Имена групп достаточно стабильны для использования инструментами, скриптами и рабочими процессами редактора.

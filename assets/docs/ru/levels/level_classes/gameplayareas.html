Уровни BeamNG используют вспомогательные объекты для игровых зон взаимодействия и точек интереса:

Класс Назначение
BeamNGGameplayArea Видимая в редакторе прямоугольная зона, используемая игровыми системами.
BeamNGPointOfInterest Именованная точка/метаданные иконки, используемые картой, карьерой, объектами, доставками и другими игровыми системами.

Эти объекты широко применяются на заправочных станциях, в автосалонах, при доставках, в гаражах, на парковках и в других местах взаимодействия.

Пример BeamNGGameplayArea 

{
  "class": "BeamNGGameplayArea",
  "name": "garage_area_01",
  "__parent": "garage_interaction",
  "position": [100, 200, 5],
  "rotationMatrix": [1, 0, 0, 0, 1, 0, 0, 0, 1],
  "scale": [2, 4, 2],
  "area2D": false
}

Пример BeamNGPointOfInterest 

{
  "class": "BeamNGPointOfInterest",
  "name": "garage_icon_01",
  "__parent": "garage_interaction",
  "position": [100, 200, 7],
  "title": "Garage",
  "desc": "Vehicle service location",
  "type": "garage"
}

Важные поля

Класс Поле Тип Описание
BeamNGGameplayArea position array[3] Центр зоны.
BeamNGGameplayArea rotationMatrix array[9] Ориентация зоны.
BeamNGGameplayArea scale array[3] Размер объёма зоны.
BeamNGGameplayArea area2D bool Обрабатывает зону как двумерную там, где это поддерживается.
BeamNGGameplayArea color color Цвет отображения в редакторе/режиме отладки.
BeamNGPointOfInterest position array[3] Расположение точки интереса (POI).
BeamNGPointOfInterest title string Название POI или ключ локализации.
BeamNGPointOfInterest desc string Описание POI или ключ локализации.
BeamNGPointOfInterest type string Тип POI, используемый игровыми системами и интерфейсом.

Динамические поля

Многие встроенные игровые зоны и точки интереса управляются системами Lua и связанными файлами.

Дополнительные поля могут сохраняться на этих объектах в зависимости от функции, которой они принадлежат.

Внешние инструменты должны сохранять неизвестные поля вспомогательных объектов геймплея. Они могут потребоваться скриптам карьеры, доставки, заправочной станции, автосалона или миссий.

Замечания для инструментов

Для инструментов, генерирующих игровые зоны:

  • Размещайте связанные объекты зоны и POI в одной группе.
  • Сохраняйте значения persistentId, если не создаёте новые объекты.
  • Сохраняйте неизвестные динамические поля.
  • Выравнивайте позицию POI по видимой иконке/точке взаимодействия.
  • Делайте игровую зону достаточно большой для предполагаемого взаимодействия с транспортным средством.
  • Проверяйте связанные данные sites, объектов или миссий при перемещении этих объектов.

Распространённые проблемы

  • Взаимодействие не отображается, потому что отсутствует тип POI или связанные метаданные геймплея.
  • Транспортное средство не может взаимодействовать, потому что зона слишком мала или расположена неверно.
  • Иконки отображаются в неправильном месте, потому что позиция POI не совпадает с центром зоны.
  • Сгенерированные объекты теряют динамические поля, используемые игровыми скриптами.

Критерии проверки

  • Объём зоны охватывает предполагаемое пространство взаимодействия.
  • Позиция POI читаема на карте/в интерфейсе.
  • Связанные игровые данные ссылаются на правильные имена зоны/POI там, где это применимо.
  • Неизвестные динамические поля сохраняются инструментами.

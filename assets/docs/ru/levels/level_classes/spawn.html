SpawnSphere — это объект-маркер, используемый для задания точек появления (спавна) на уровне. В уровнях BeamNG он чаще всего применяется как именованная точка спавна, на которую ссылается info.json.

SpawnSphere является разновидностью MissionMarker, то есть представляет собой преимущественно маркер 3D-редактора, а не игровую геометрию. Он отображается в World Editor, но не предназначен для отображения как обычный объект уровня во время игры.

Базовый пример

Пример SpawnSphere в файле items.level.json:

{
  "class": "SpawnSphere",
  "name": "spawns_default",
  "position": [0, 0, 1],
  "rotationMatrix": [1, 0, 0, 0, 1, 0, 0, 0, 1],
  "scale": [1, 1, 1],
  "dataBlock": "SpawnSphereMarker"
}

Затем на этот объект можно сослаться из info.json:

{
  "defaultSpawnPointName": "spawns_default",
  "spawnPoints": [
    {
      "translationId": "levels.example.spawnpoints.default",
      "description": "levels.example.spawnpoints.default.description",
      "objectname": "spawns_default",
      "preview": "spawns_default.jpg"
    }
  ]
}

Основное назначение

При современном создании уровней SpawnSphere используется главным образом для:

  • Позиций появления игрока
  • Выбора точки спавна в интерфейсе уровня
  • Точек интереса (POI) на большой карте
  • Мест спавна в режиме свободного движения
  • Референсных позиций для сценариев/миссий
  • Маркеров спавна, видимых в редакторе

Наиболее важной частью является имя объекта. Записи точек спавна в info.json ссылаются на SpawnSphere по его objectname.

Важные поля

Поле Тип Описание
class string Должно быть равно "SpawnSphere".
name string Имя объекта на сцене. Используется записями точек спавна в info.json.
position array[3] Позиция маркера спавна в мировых координатах.
rotationMatrix array[9] Ориентация точки спавна. Используется для ориентации спавнящихся транспортных средств и камеры.
scale array[3] Масштаб маркера в редакторе. Обычно [1, 1, 1].
dataBlock string Датаблок маркера, используемый для визуализации в редакторе; обычно SpawnSphereMarker.
spawnClass string Устаревшее/общее поле для спавна объектов. Как правило, не нужно для точек спавна на уровне.
spawnDatablock string Устаревшее/общее поле для спавна объектов.
spawnProperties string Устаревшие/общие свойства спавна объектов.
spawnScript string Устаревший скрипт, выполняемый после спавна объекта.
autoSpawn bool Устаревшее поведение: спавнит объект при создании SpawnSphere.
spawnTransform bool Применяет трансформацию SpawnSphere к спавнящемуся объекту при использовании устаревшего поведения спавна.
radius number Устарело.

Связь с info.json

Точки спавна, отображаемые в селекторе уровней и на большой карте, задаются в info.json, однако их фактическая трансформация в мировых координатах берётся из объекта на сцене.

Поле objectname в info.json должно совпадать с именем объекта SpawnSphere.

Пример:

{
  "translationId": "levels.example.spawnpoints.docks",
  "description": "levels.example.spawnpoints.docks.description",
  "objectname": "spawns_docks",
  "preview": "spawns_docks.jpg"
}

Соответствующий объект на сцене:

{
  "class": "SpawnSphere",
  "name": "spawns_docks",
  "position": [125, 420, 3],
  "rotationMatrix": [0, -1, 0, 1, 0, 0, 0, 0, 1],
  "scale": [1, 1, 1],
  "dataBlock": "SpawnSphereMarker"
}
Если info.json ссылается на объект точки спавна, которого не существует, точка спавна может не отображаться или вызывать ошибку при выборе.

Точка спавна по умолчанию

Точка спавна по умолчанию задаётся в info.json с помощью:

"defaultSpawnPointName": "spawns_default"

Значение должно совпадать с именем объекта SpawnSphere.

{
  "class": "SpawnSphere",
  "name": "spawns_default",
  "position": [0, 0, 1],
  "rotationMatrix": [1, 0, 0, 0, 1, 0, 0, 0, 1]
}

Если подходящая точка спавна по умолчанию не найдена, система уровня может создать резервную запись точки спавна по умолчанию.

Позиция

position задаёт местоположение точки спавна.

"position": [100, 200, 2]

Размещайте точки спавна:

  • На проходимой поверхности или дорожном покрытии
  • С достаточным запасом пространства для транспортного средства по умолчанию
  • Вдали от стен, пропов и трафика
  • Вдали от крутых склонов, если это не задумано намеренно
  • В местах, безопасных сразу после загрузки уровня

Поворот

rotationMatrix задаёт ориентацию точки спавна.

Это влияет на то, в каком направлении будут смотреть транспортные средства при появлении.

Пример единичного поворота (identity rotation):

"rotationMatrix": [
  1, 0, 0,
  0, 1, 0,
  0, 0, 1
]

При задании точек спавна убедитесь, что вектор «вперёд» направлен в сторону безопасного направления движения.

Если транспортное средство появляется лицом в неправильную сторону, поверните объект SpawnSphere в World Editor и сохраните уровень.

Масштаб

scale управляет размером маркера в редакторе.

"scale": [1, 1, 1]

Для обычных точек спавна на уровне сохраняйте значение:

"scale": [1, 1, 1]

Устаревшее поле radius является устаревшим и не должно использоваться для современного поведения точек спавна.

dataBlock

Сферы спавна обычно визуализируются с помощью датаблока маркера:

"dataBlock": "SpawnSphereMarker"

Этот датаблок задаёт форму в редакторе, используемую для отображения маркера, — как правило, зелёный октаэдр.

Если датаблок отсутствует, точка спавна может логически существовать, но визуализация в редакторе может быть нарушена.

Поведение MissionMarker

SpawnSphere наследует от MissionMarker.

Маркеры миссий добавляются на сцену главным образом во время редактирования, чтобы их можно было видеть и ими можно было управлять в World Editor.

Они не предназначены для роли видимых игровых мешей.

Устаревшие поля общего спавна

SpawnSphere также поддерживает устаревшее общее поведение спавна через такие поля, как:

"spawnClass": "Player",
"spawnDatablock": "DefaultPlayerData",
"spawnProperties": "name = \"Bob\";",
"spawnScript": "echo(\"Spawned object\");",
"autoSpawn": true,
"spawnTransform": true

Эта система может создавать объект, используя spawnClass и spawnDatablock, с опциональным применением свойств и скрипта.

Однако для современных точек спавна на уровнях BeamNG эти поля обычно не требуются.

Для обычных точек спавна на уровне используйте SpawnSphere как именованный маркер, на который ссылается info.json. Не используйте autoSpawn, если вам не требуется именно устаревшее поведение спавна объектов.

spawnObject()

У SpawnSphere есть функция spawnObject(), которая может создавать объект на основе его устаревших полей спавна.

Создаваемый объект может опционально получить трансформацию SpawnSphere, если:

"spawnTransform": true

Объект добавляется в MissionCleanup, если этот контейнер доступен.

Это преимущественно устаревшая функциональность, которую не следует путать с обычным выбором точки спавна игрока.

Интеграция с большой картой и POI

Точки спавна из info.json могут отображаться как POI на большой карте.

Система уровня:

  1. Читает info.json.
  2. Перебирает записи в spawnPoints.
  3. Находит объект с именем, указанным в objectname.
  4. Использует позицию этого объекта для маркера POI.
  5. Использует данные предпросмотра/описания из info.json.

Если объект SpawnSphere отсутствует, POI может не быть создан.

Пример: несколько точек спавна

items.level.json:

{"class":"SpawnSphere","name":"spawns_city","position":[100,200,1],"rotationMatrix":[1,0,0,0,1,0,0,0,1],"scale":[1,1,1],"dataBlock":"SpawnSphereMarker"}
{"class":"SpawnSphere","name":"spawns_highway","position":[500,100,1],"rotationMatrix":[0,-1,0,1,0,0,0,0,1],"scale":[1,1,1],"dataBlock":"SpawnSphereMarker"}
{"class":"SpawnSphere","name":"spawns_docks","position":[-250,400,1],"rotationMatrix":[1,0,0,0,1,0,0,0,1],"scale":[1,1,1],"dataBlock":"SpawnSphereMarker"}

info.json:

{
  "defaultSpawnPointName": "spawns_city",
  "spawnPoints": [
    {
      "translationId": "levels.example.spawnpoints.city",
      "description": "levels.example.spawnpoints.city.description",
      "objectname": "spawns_city",
      "preview": "spawns_city.jpg"
    },
    {
      "translationId": "levels.example.spawnpoints.highway",
      "description": "levels.example.spawnpoints.highway.description",
      "objectname": "spawns_highway",
      "preview": "spawns_highway.jpg"
    },
    {
      "translationId": "levels.example.spawnpoints.docks",
      "description": "levels.example.spawnpoints.docks.description",
      "objectname": "spawns_docks",
      "preview": "spawns_docks.jpg"
    }
  ]
}

Рекомендации

  • Используйте SpawnSphere для точек спавна на уровне.
  • Давайте каждой точке спавна понятное, неизменное имя.
  • Ссылайтесь на точки спавна из info.json через objectname.
  • Убедитесь, что defaultSpawnPointName соответствует существующей точке спавна.
  • Размещайте точки спавна на безопасных, достаточно ровных поверхностях.
  • Ориентируйте точки спавна в направлении безопасного движения.
  • Сохраняйте scale равным [1, 1, 1], если не нужен увеличенный маркер в редакторе.
  • Не используйте устаревшее поле radius.
  • Избегайте устаревшего поведения autoSpawn для обычных точек спавна на уровне.
  • Используйте изображения предпросмотра для каждой точки спавна в info.json.

Распространённые проблемы

Точка спавна не отображается в селекторе уровней

Возможные причины:

  • Отсутствует запись в info.json
  • В записи spawnPoints не указан objectname
  • objectname не совпадает с именем SpawnSphere
  • Объект точки спавна отсутствует в items.level.json

Неверная точка спавна по умолчанию

Проверьте:

"defaultSpawnPointName": "spawns_default"

и убедитесь, что соответствующий объект SpawnSphere существует.

Транспортное средство появляется лицом в неверном направлении

Поверните SpawnSphere в World Editor и сохраните уровень.

Транспортное средство появляется внутри геометрии

Переместите SpawnSphere в более безопасное место с достаточным запасом пространства.

Маркер спавна не виден в редакторе

Возможные причины:

  • Отсутствует dataBlock
  • Визуализация маркеров в редакторе отключена
  • Редактор не активен
  • Объект-маркер скрыт

Неожиданный автоматический спавн объекта

Проверьте, не активирован ли устаревший спавн:

"autoSpawn": true

Для обычных маркеров точек спавна на уровне это значение должно быть false или поле должно быть опущено.

Итог

SpawnSphere — стандартный объект-маркер для точек спавна на уровне.

В современных уровнях BeamNG он используется главным образом как именованная трансформация, на которую ссылаются записи точек спавна в info.json. Объект предоставляет позицию и поворот, тогда как info.json предоставляет текст интерфейса, изображения предпросмотра и выбор точки спавна по умолчанию.

Устаревшие поля общего спавна объектов по-прежнему существуют, но для настройки точек спавна на уровне они обычно не требуются.

Prefab — это объект сцены, который размещает и загружает повторно используемую группу объектов из файла префаба.

Объект Prefab хранит трансформацию размещения и ссылку на файл. Дочерние объекты хранятся в самом файле префаба.

Описание формата файла префаба см. в документации Prefab format.

Простой пример

Простая запись Prefab в файле items.level.json:

{
  "class": "Prefab",
  "name": "gas_station_prefab",
  "__parent": "MissionGroup",
  "filename": "/levels/example/art/prefabs/gas_station.prefab.json",
  "position": [100, 200, 0],
  "rotationMatrix": [1, 0, 0, 0, 1, 0, 0, 0, 1],
  "scale": [1, 1, 1],
  "loadMode": "Automatic"
}

Важные поля

Поле Тип Описание
class string Должно быть "Prefab".
name string Имя объекта сцены.
__parent string Родительская группа.
filename string/path Загружаемый файл префаба. Может указывать на .prefab или .prefab.json.
position array[3] Позиция размещения префаба.
rotationMatrix array[9] Поворот при размещении префаба.
scale array[3] Масштаб размещения префаба.
loadMode enum/string "Automatic" или "Manual".
useGlobalTranslation bool Поведение трансформации для совместимости и особых случаев, применяемое к дочерним объектам.

filename

filename указывает на файл префаба:

"filename": "/levels/example/art/prefabs/gas_station.prefab.json"

Современные файлы префабов обычно используют:

*.prefab.json

Устаревший контент может использовать:

*.prefab
Если файл отсутствует или файл префаба не содержит корректной корневой группы, префаб не сможет создать свои дочерние объекты.

loadMode

Поддерживаемые значения:

Значение Описание
Automatic Загружает префаб при добавлении объекта Prefab.
Manual Не загружается автоматически. Префаб должен быть загружен скриптом или инструментом.

Для большинства контента уровня следует использовать:

"loadMode": "Automatic"

Поведение трансформации

Трансформация объекта Prefab применяется к дочерним объектам, загруженным из файла префаба.

Это позволяет размещать повторно используемую группу объектов несколько раз с различными позицией, поворотом или масштабом.

Для большинства контента оставляйте:

"useGlobalTranslation": false

Используйте useGlobalTranslation только при работе с существующим контентом или специфическим рабочим процессом инструмента, который этого требует.

Связь с файлами префабов

Класс и формат файла выполняют различные функции:

Компонент Ответственность
Объект Prefab Размещение, трансформация, ссылка на файл, режим загрузки.
Файл .prefab.json / .prefab Данные дочерних объектов и корневая группа.

Не дублируйте все дочерние объекты в файле элементов уровня, если они должны находиться внутри файла префаба.

Примечания для инструментов

Для инструментов, создающих или изменяющих префабы:

  • Сохраняйте пути filename и обеспечивайте их переносимость.
  • Храните трансформацию Prefab отдельно от трансформаций дочерних объектов.
  • Используйте Prefab для повторно используемых групп объектов, а не для отдельных статических мешей.
  • Используйте TSStatic или Forest для размещения отдельных или повторяющихся мешей там, где группировка с помощью префаба не нужна.
  • Следуйте структуре файла префаба, описанной на странице формата префаба.

Типичные проблемы

  • Префаб не отображается, поскольку значение filename задано неверно или отсутствует.
  • Префаб загружается не в том месте, потому что трансформации дочерних объектов и трансформация самого префаба были «запечены» одновременно.
  • Несколько экземпляров префаба конфликтуют между собой, поскольку имена дочерних объектов не обрабатываются корректно в рамках рабочего процесса префаба.
  • Используется loadMode равный Manual, однако ни один скрипт или инструмент не загружает префаб.

Критерии валидации

  • filename указывает на существующий файл префаба.
  • Значение loadMode задано намеренно.
  • Дочерние объекты отображаются в ожидаемой позиции в мировых координатах.
  • Перемещение или поворот объекта Prefab перемещает дочерние объекты, как ожидается.
  • Сам файл префаба соответствует правилам формата префаба.

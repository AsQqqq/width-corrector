GroundCover процедурно покрывает ландшафт множеством мелких элементов растительности и деталей: трава, цветы, сорняки, тростник, камни, кустарники и другой низкозатратный декор среды.

Объект GroundCover не размещает каждый элемент как отдельный объект сцены. Вместо этого он генерирует и отрисовывает ячейки вокруг камеры во время выполнения.

Элементы наземного покрытия могут отрисовываться как:

  • Биллборды с использованием общего материала/атласа текстур растительности.
  • Инстансные TS-формы с использованием записей shapeFilename.

GroundCover привязан к ландшафту. Он размещает элементы на поверхностях TerrainBlock и может фильтровать размещение по слою материала ландшафта, высоте и уклону.

Базовый пример

Простой объект наземного покрытия из травы и цветов:

{
  "class": "GroundCover",
  "name": "grass_groundcover",
  "material": "grass_atlas",
  "radius": 80,
  "dissolveRadius": 65,
  "gridSize": 7,
  "maxElements": 12000,
  "zOffset": 0.02,
  "seed": 12345,

  "billboardUVs": [
    [0.0, 0.0, 0.25, 0.5],
    [0.25, 0.0, 0.25, 0.5]
  ],

  "probability": [0.85, 0.15],
  "sizeMin": [0.4, 0.25],
  "sizeMax": [1.1, 0.7],
  "sizeExponent": [1.5, 1.2],
  "windScale": [1.0, 0.6],
  "maxSlope": [35, 30],
  "minElevation": [-99999, -99999],
  "maxElevation": [99999, 99999],

  "minClumpCount": [3, 1],
  "maxClumpCount": [10, 4],
  "clumpExponent": [1.5, 1.0],
  "clumpRadius": [2.0, 1.0],

  "windDirection": [1, 0],
  "windGustLength": 20,
  "windGustFrequency": 0.15,
  "windGustStrength": 0.25,
  "windTurbulenceFrequency": 1.5,
  "windTurbulenceStrength": 0.05
}

Как работает GroundCover

GroundCover поддерживает квадратную сетку ячеек вокруг камеры.

Для каждой активной ячейки он:

  1. Выбирает случайные точки размещения.
  2. Считывает высоту и нормаль ландшафта.
  3. Проверяет фильтры слоёв материала ландшафта.
  4. Проверяет ограничения по высоте и уклону.
  5. Выбирает тип элемента по вероятностям для каждого типа.
  6. Применяет случайный масштаб, поворот, кластеризацию, коэффициент ветра и цвет лайтмапа.
  7. Формирует буферы биллбордов и/или данные инстансов форм.
  8. Отрисовывает только видимые ячейки.

При движении камеры ячейки сдвигаются и переиспользуются. Новые ячейки генерируются на краях сетки.

Поля для каждого типа

Наземное покрытие поддерживает несколько типов покрытия. Каждый тип имеет собственные UV-координаты биллборда, форму, вероятность, диапазон размеров, фильтры слоёв и настройки размещения.

В JSON эти поля обычно представлены в виде массивов. Индекс 0 описывает тип 0, индекс 1 — тип 1 и т. д.

Поле Тип Описание

useWorldRandomYaw array[bool] Использует случайный рыскание в мировом пространстве для ориентации биллборда там, где это поддерживается.
shapeFilename array[string] Необязательный файл формы для данного типа. Если задан, тип отрисовывается как инстансные формы вместо биллбордов.
layer array[string] Внутреннее имя материала ландшафта, которым ограничивается размещение. Пустое значение означает любой слой ландшафта.
invertLayer array[bool] Обрабатывает layer как маску исключения вместо маски включения.
probability array[number] Относительная вероятность каждого типа. Значения нормализуются внутри движка.
sizeMin array[number] Минимальный случайный размер/масштаб.
sizeMax array[number] Максимальный случайный размер/масштаб.
sizeExponent array[number] Показатель смещения для случайного выбора размера.
windScale array[number] Множитель смещения от ветра для каждого типа.
maxSlope array[number] Максимальный уклон ландшафта в градусах. 0 отключает ограничение по уклону.
minElevation array[number] Минимальная высота в мировом пространстве для размещения.
maxElevation array[number] Максимальная высота в мировом пространстве для размещения.
minClumpCount array[integer] Минимальное количество элементов в кластере.
maxClumpCount array[integer] Максимальное количество элементов в кластере.
clumpExponent array[number] Показатель смещения для размера/количества кластера.
clumpRadius array[number] Максимальный радиус кластера.

Типы биллбордов

Биллбордное наземное покрытие использует поле material и billboardUVs.

Пример с двумя типами биллбордов в одном атласе текстур:

{
  "class": "GroundCover",
  "name": "meadow_grass",
  "material": "meadow_grass_atlas",
  "radius": 90,
  "dissolveRadius": 75,
  "gridSize": 7,
  "maxElements": 16000,

  "billboardUVs": [
    [0.0, 0.0, 0.25, 0.5],
    [0.25, 0.0, 0.25, 0.5]
  ],

  "probability": [0.8, 0.2],
  "sizeMin": [0.4, 0.3],
  "sizeMax": [1.2, 0.8],
  "windScale": [1.0, 0.8]
}

Каждая запись billboardUVs — это прямоугольник в атласе текстур материала:

[u, v, width, height]

Движок использует размеры диффузной текстуры материала для вычисления соотношения сторон биллборда по UV-прямоугольнику.

Типы форм

Если для типа задан shapeFilename, этот тип отрисовывается как инстансная геометрия TS-формы вместо биллборда.

Пример:

{
  "class": "GroundCover",
  "name": "rocks_and_grass",
  "material": "grass_atlas",
  "radius": 70,
  "dissolveRadius": 55,
  "gridSize": 6,
  "maxElements": 8000,
  "shapeCullRadius": 60,
  "shapesCastShadows": true,

  "billboardUVs": [
    [0.0, 0.0, 0.25, 0.5],
    [0.0, 0.0, 1.0, 1.0]
  ],

  "shapeFilename": [
    "",
    "/levels/example/art/shapes/rocks/small_rock.dae"
  ],

  "probability": [0.9, 0.1],
  "sizeMin": [0.4, 0.4],
  "sizeMax": [1.1, 1.5],
  "windScale": [1.0, 0.0]
}

Здесь:

  • Тип 0 — биллбордная трава.
  • Тип 1 — инстансная форма камня.
Наземное покрытие на основе форм обходится дороже, чем биллбордное. Используйте его умеренно — для камней, кустарников, небольших растений или декоративных объектов.

Требования к материалу

Рендер биллбордов GroundCover использует внутри движка функцию материала растительности.

Материал биллборда, как правило, должен быть создан как материал растительности/прозрачности со следующими характеристиками:

  • Атлас диффузной/альбедо-текстуры
  • Альфа-канал или прозрачность для вырезанной растительности
  • Качественные мипмапы
  • Правильная разметка атласа текстур для billboardUVs
  • Необязательные карты нормалей/шероховатости там, где это поддерживается настройкой материала

Если материал не найден, GroundCover переключается на предупреждающий материал.

Пример:

"material": "grass_atlas"

Фильтрация по слою ландшафта

Используйте layer для ограничения типа материалом ландшафта.

"layer": ["grass"],
"probability": [1.0]

Это размещает тип только на участках ландшафта с материалом grass.

Используйте invertLayer для исключения материала ландшафта:

"layer": ["rock"],
"invertLayer": [true]

Это размещает тип везде, кроме участков с материалом ландшафта rock.

Значение layer должно соответствовать внутреннему имени материала ландшафта, а не обязательно отображаемой метке материала.

Фильтрация по высоте

Используйте minElevation и maxElevation для ограничения размещения по высоте в мировом пространстве.

Пример: размещать альпийскую траву только выше 500 метров:

"minElevation": [500],
"maxElevation": [99999]

Пример: размещать пляжную траву только ниже 8 метров:

"minElevation": [-99999],
"maxElevation": [8]

Фильтрация по уклону

Используйте maxSlope для ограничения размещения более пологими участками ландшафта.

"maxSlope": [35]

Это разрешает размещение только там, где уклон ландшафта составляет не более примерно 35 градусов.

Значение 0 отключает фильтрацию по уклону:

"maxSlope": [0]

Настройки размера

Для каждого типа можно задать случайный размер:

"sizeMin": [0.4],
"sizeMax": [1.2],
"sizeExponent": [1.5]
Поле Значение
sizeMin Наименьший возможный размер.
sizeMax Наибольший возможный размер.
sizeExponent Смещение между минимальным и максимальным значением.

Чем выше значение sizeExponent, тем больше размещений смещается к меньшему концу диапазона.

Вероятность

probability управляет относительным распределением типов.

"probability": [0.7, 0.2, 0.1]

Значения нормализуются внутри движка, поэтому они эквивалентны по соотношению следующим:

"probability": [7, 2, 1]

Типы без действительных UV/соотношения сторон биллборда и без действительного shapeFilename игнорируются.

Кластеризация

Кластеризация определяет, появляются ли элементы равномерно рассеянными или сгруппированными в пятна.

Пример:

"minClumpCount": [4],
"maxClumpCount": [12],
"clumpExponent": [1.5],
"clumpRadius": [2.5]
Поле Значение
minClumpCount Минимальное количество элементов в кластере.
maxClumpCount Максимальное количество элементов в кластере.
clumpExponent Смещение для распределения количества элементов в кластере.
clumpRadius Радиус вокруг центра кластера, в котором могут появляться элементы.

Используйте кластеризацию для:

  • Пучков травы
  • Цветов, растущих пятнами
  • Тростника у воды
  • Групп камней
  • Зарослей кустарников

Используйте меньшее количество элементов в кластере и больший радиус для более равномерного распределения.

Биллборды наземного покрытия поддерживают анимацию ветра.

Поля ветра

Поле Тип Описание
windDirection array[2] Двумерное направление ветра.
windGustLength number Расстояние в метрах между пиками порывов.
windGustFrequency number Частота пиков порывов в секунду.
windGustStrength number Максимальное смещение от порыва в метрах.
windTurbulenceFrequency number Частота/интенсивность турбулентности.
windTurbulenceStrength number Максимальное смещение от турбулентности в метрах.

Пример:

"windDirection": [1, 0],
"windGustLength": 25,
"windGustFrequency": 0.12,
"windGustStrength": 0.25,
"windTurbulenceFrequency": 1.2,
"windTurbulenceStrength": 0.05

Поле windScale для каждого типа умножает эффект ветра:

"windScale": [1.0, 0.4, 0.0]

Используйте 0 для объектов, которые не должны качаться, например камней.

Сетка, радиус и плотность

radius 

"radius": 80

radius — это внешняя область вокруг камеры, в которой генерируется и отрисовывается наземное покрытие.

Бо́льшие значения показывают растительность дальше, но требуют больше ячеек и элементов.

dissolveRadius 

"dissolveRadius": 65

dissolveRadius управляет тем, где биллборды начинают исчезать.

Диапазон затухания приблизительно:

dissolveRadius -> radius

Например:

"radius": 80,
"dissolveRadius": 65

означает, что элементы полностью видны примерно до 65 метров и исчезают к 80 метрам.

gridSize 

"gridSize": 7

gridSize — это количество ячеек по каждой оси.

Общее количество активных ячеек:

gridSize * gridSize

Примеры:

gridSize Ячеек
5 25
7 49
9 81

Увеличение gridSize создаёт меньшие ячейки и может снизить ограничения на количество биллбордов в ячейке, однако потребует управления большим числом ячеек.

maxElements 

"maxElements": 12000

maxElements — приблизительное максимальное количество элементов, распределённых по активной сетке.

Приблизительное количество размещений в ячейке:

maxElements / (gridSize * gridSize)

с учётом глобального масштабирования плотности.

Если в ячейке слишком много биллбордов, движок может выдать предупреждение о превышении количества элементов. Уменьшите maxElements или увеличьте gridSize.

Глобальный масштаб плотности

На плотность наземного покрытия также влияет глобальная переменная настроек:

$pref::GroundCover::densityScale

Она масштабирует плотность размещения глобально.

Движок также предоставляет статистику во время выполнения:

$GroundCover::renderedCells
$GroundCover::renderedBillboards
$GroundCover::renderedBatches
$GroundCover::renderedShapes

Это полезно для отладки и настройки производительности.

Отражения и тени

reflectScale 

"reflectScale": 0.5

reflectScale масштабирует радиус рендера/отсечения при рендере проходов отражений, как правило для отражений на воде.

Используйте меньшие значения для снижения стоимости отражений.

Установите 0, чтобы исключить наземное покрытие из отражений:

"reflectScale": 0

shapesCastShadows 

"shapesCastShadows": true

Управляет тем, отрисовывается ли наземное покрытие на основе форм во время проходов теней.

Биллборды не передаются в проходы теней по этому пути кода.

В целях производительности оставьте этот параметр отключённым, если только формы недостаточно важны для отбрасывания видимых теней.

Поля отладки

Поле Тип Описание
lockFrustum bool Замораживает усечённую пирамиду отсечения/генерации для отладки.
renderCells bool Отображает отладочные рамки для активных ячеек рендера.
noBillboards bool Отключает рендер биллбордов.
noShapes bool Отключает рендер форм.

Пример:

"renderCells": true,
"noShapes": true

Используйте эти поля для проверки ячеек размещения и отдельного анализа стоимости биллбордов и форм.

Зависимость от ландшафта

GroundCover требует наличия ландшафта.

Он запрашивает активные объекты TerrainBlock и считывает:

  • Высоту ландшафта
  • Нормаль ландшафта
  • Слой материала ландшафта
  • Цвет лайтмапа ландшафта
  • Данные текстуры/высоты слоя ландшафта для новых путей размещения

При отсутствии ландшафта ячейки наземного покрытия не могут генерироваться.

GroundCover предназначен для поверхностей ландшафта. Он не проецируется на произвольные статичные меши, такие как DecalRoad с overObjects.

Обновления ландшафта

GroundCover прослушивает обновления ландшафта.

При изменении ландшафта:

Обновление ландшафта Поведение
Обновление лайтмапа Освобождает/перегенерирует все ячейки.
Обновление карты высот Освобождает ячейки, перекрывающие изменённую область.
Обновление слоя Освобождает ячейки, перекрывающие изменённую область.
Маркер пустой/обновления Освобождает ячейки, перекрывающие изменённую область там, где это поддерживается.

Это позволяет наземному покрытию перегенерироваться после скульптинга ландшафта, покраски или изменения освещения.

Рекомендации по производительности

Наземное покрытие может отрисовывать большое количество элементов, поэтому настройка имеет значение.

Основные факторы стоимости

Настройка Влияние на производительность
maxElements Бо́льшие значения генерируют/отрисовывают больше инстансов.
radius Бо́льший радиус увеличивает количество видимых ячеек и элементов.
gridSize Больше ячеек для управления; меньшие пакеты на ячейку.
Типы форм Дороже биллбордов.
shapeCullRadius Бо́льшее значение отрисовывает формы дальше.
shapesCastShadows Тени форм добавляют затраты на проходы теней.
Рендер отражений Управляется параметром reflectScale.

Рекомендации по настройке

  • Предпочитайте биллборды для травы и небольших растений.
  • Используйте формы только для редких объектов, таких как камни или кустарники.
  • Держите radius умеренным; используйте dissolveRadius для скрытия внезапного появления.
  • Увеличивайте gridSize при появлении предупреждений о превышении количества элементов в ячейке.
  • Уменьшайте maxElements на менее мощных картах или в плотных сценах.
  • Устанавливайте reflectScale в 0 или малое значение, если отражённая растительность не важна.
  • Отключайте shapesCastShadows, если тени форм визуально не обязательны.
  • Используйте фильтры layer ландшафта, чтобы не тратить размещения на неподходящие зоны.
  • Используйте maxSlope и ограничения по высоте для сокращения нежелательных размещений.

Частые проблемы

GroundCover не отображается

Возможные причины:

  • Отсутствует TerrainBlock.
  • maxElements равно 0.
  • Все значения probability равны 0.
  • Нет действительных billboardUVs и нет действительного shapeFilename.
  • Материал отсутствует или недействителен.
  • layer не соответствует внутреннему имени материала ландшафта.
  • minElevation / maxElevation исключает ландшафт.
  • maxSlope слишком ограничительный.
  • Глобальный масштаб плотности равен 0.

Проверьте:

$pref::GroundCover::densityScale

Растительность появляется на неправильном материале ландшафта

Проверьте, что layer использует внутреннее имя материала ландшафта.

Также проверьте invertLayer:

"invertLayer": [false]

Если invertLayer равно true, выбранный слой исключается вместо включения.

Растительность парит над ландшафтом или уходит в него

Скорректируйте:

"zOffset": 0.02

Используйте малые значения. Слишком большое смещение заставляет траву парить.

Предупреждение о слишком большом количестве элементов

Количество биллбордов в ячейке слишком велико.

Исправьте, уменьшив:

"maxElements": 8000

или увеличив:

"gridSize": 9

Увеличение gridSize распределяет тот же общий бюджет элементов по большему числу ячеек.

Наземное покрытие внезапно появляется или исчезает слишком близко

Увеличьте радиус рендера и диапазон затухания:

"radius": 100,
"dissolveRadius": 80

Учтите, что бо́льшие радиусы требуют больше ресурсов.

Наземное покрытие на основе форм слишком затратно

Попробуйте:

"shapeCullRadius": 35,
"shapesCastShadows": false

Также снизьте probability типа формы или разбейте крупные/детализированные формы на более простые ресурсы.

Трава появляется на крутых склонах

Задайте maxSlope:

"maxSlope": [30]

Значение 0 отключает фильтрацию по уклону, поэтому используйте положительный угол для ограничения размещения.

Ветер слишком сильный

Снизьте глобальные поля ветра:

"windGustStrength": 0.1,
"windTurbulenceStrength": 0.02

или снизьте ветер для отдельного типа:

"windScale": [0.5]

Итоги

GroundCover — основной объект рассеивания растительности и деталей на ландшафте.

Он процедурно генерирует локальные относительно камеры ячейки из биллбордных и/или инстансных элементов форм. Размещение можно контролировать по слою ландшафта, вероятности, диапазону размеров, кластеризации, высоте, уклону и случайному зерну. Поддерживается анимация ветра для биллбордов, масштабирование плотности во время выполнения, масштабирование отражений и отладочная визуализация.

Используйте его для плотных деталей ландшафта — травы и цветов, а типы форм применяйте умеренно — для небольших камней, кустарников и другого редкого декора.

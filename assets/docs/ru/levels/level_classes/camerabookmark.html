CameraBookmark хранит позиции закладок редактора/камеры в уровне.

Эти объекты часто встречаются в поставляемых уровнях в составе групп закладок камеры. Они преимущественно полезны для редакторов, инструментов, скриншотов, синематиков и повторяемых точек обзора.

Базовый пример 

{
  "class": "CameraBookmark",
  "name": "overview_01",
  "__parent": "CameraBookmarks",
  "position": [100, 200, 50],
  "rotationMatrix": [1, 0, 0, 0, 1, 0, 0, 0, 1],
  "scale": [1, 1, 1],
  "observer": true
}

Важные поля

Поле Тип Описание
class string Должно быть "CameraBookmark".
name string Название закладки.
__parent string Родительская группа, часто CameraBookmarks.
position array[3] Позиция закладки камеры.
rotationMatrix array[9] Ориентация закладки камеры.
scale array[3] Стандартный масштаб объекта сцены. Обычно [1, 1, 1].
observer bool Помечает закладку как подходящую для использования камерой наблюдателя.

Примечания для инструментов

Для инструментов, обрабатывающих файлы объектов уровня:

  • Сохраняйте закладки камеры, если только вы намеренно не очищаете данные редактора.
  • Оставляйте названия стабильными, если на них ссылаются скриншоты, инструменты проверки или скрипты.
  • Рассматривайте их как вспомогательные объекты редактора, а не как игровую геометрию.
  • По возможности группируйте их в чётко обозначенную группу закладок камеры.

Распространённые проблемы

  • Закладка открывается с неправильным видом, поскольку rotationMatrix был изменён или пересчитан некорректно.
  • Очистка инструментом удаляет закладки, которые используются в процессах проверки или съёмки скриншотов.
  • Закладки неудобно использовать, потому что их названия слишком общие или дублируются.

Критерии валидации

  • Закладки загружаются в редакторе.
  • Точки обзора направлены на полезные области уровня.
  • Названия достаточно описательны для использования инструментами и редактором.
  • Инструменты сохраняют закладки, если только их удаление не является намеренным.

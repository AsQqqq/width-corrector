BeamNGWaypoint — лёгкий объект сцены, используемый в качестве именованной точки на уровне. Он широко применяется системами ИИ/навигации, миссиями, триггерами, гонками, чекпойнтами и при ручной настройке навигационного графа.

Вейпойнты не отображаются во время обычной игры, если не включена отладочная отрисовка или объект не выбран в World Editor.

Базовый пример

Пример BeamNGWaypoint в items.level.json:

{
  "class": "BeamNGWaypoint",
  "name": "wp_tunnel_01",
  "position": [100, 200, 5],
  "rotationMatrix": [1, 0, 0, 0, 1, 0, 0, 0, 1],
  "scale": [5, 5, 5],
  "drawDebug": false,
  "directionalWaypoint": false,
  "excludeFromMap": false
}

Важные поля

Поле Тип Описание
class string Должно быть "BeamNGWaypoint".
name string Имя вейпойнта. Важно для ссылок из миссий, скриптов и map.json.
position array[3] Мировая позиция.
rotationMatrix array[9] Вращение объекта в виде матрицы 3×3. Актуально преимущественно для направленных вейпойнтов.
scale array[3] Радиус/размер вейпойнта. Внутри движка трактуется как равномерный.
drawDebug bool Рисует отладочную сферу в позиции вейпойнта.
directionalWaypoint bool Делает вейпойнт направленным: вместо простой сферы используется обнаружение через полупространство/плоскость.
excludeFromMap bool Исключает данный вейпойнт из генерации карты ИИ/навигации.

Имя

Поле name очень важно.

Пример:

"name": "tunnel_A_01"

Имена вейпойнтов используются:

  • Ручными навигационными сегментами в map.json
  • Логикой миссий
  • Чекпойнтами
  • Маршрутами ИИ
  • Lua-скриптами
  • Инструментами редактора

Используйте стабильные, понятные имена. Избегайте переименования вейпойнтов после того, как на них уже ссылаются другие файлы.

Хорошие примеры:

wp_city_start
tunnel_A_01
bridge_exit_02
race_checkpoint_05

Избегайте:

Waypoint1
test
new
abc

Позиция

position задаёт центр вейпойнта в мировом пространстве.

"position": [100, 200, 5]

Для инструментов навигации и построения маршрутов размещайте вейпойнт вблизи центра предполагаемого пути или дороги.

Для чекпойнтов размещайте его там, где транспортное средство должно проехать.

Масштаб / радиус

Вейпойнты используют масштаб как фактический радиус/размер.

"scale": [5, 5, 5]

Движок трактует масштабирование вейпойнта как равномерное. Если изменяется одна ось, вейпойнт применяет это значение равномерно.

Например, при задании:

"scale": [8, 1, 1]

результат фактически будет соответствовать равномерному изменению радиуса/масштаба, а не истинному неравномерному объёму.

Используйте равномерные значения масштаба для вейпойнтов. Неравномерное масштабирование не имеет смысла при обычном поведении вейпойнтов.

drawDebug 

"drawDebug": true

При включении вейпойнт рисует отладочную сферу в редакторе/вьюпорте.

Вейпойнты также отображаются, когда:

  • Объект выбран
  • Включена глобальная отладочная отрисовка вейпойнтов
  • Это запрошено настройками визуализации редактора

Это удобно для проверки позиции и размера вейпойнта.

directionalWaypoint 

"directionalWaypoint": true

Направленный вейпойнт ведёт себя иначе, чем обычный сферический.

При включении вейпойнт усекается направляющей плоскостью, образуя направленную/плоскую поверхность обнаружения. Это полезно для чекпойнтов, где важно направление движения.

Направленные вейпойнты наиболее востребованы для:

  • Чекпойнтов гонок
  • Односторонних пропускных ворот
  • Валидации маршрута
  • Триггеров миссий, где важно направление пересечения

Ориентация вейпойнта определяется его трансформом/вращением.

excludeFromMap 

"excludeFromMap": true

При включении вейпойнт игнорируется системой карты ИИ/навигации.

Используйте это для вейпойнтов, предназначенных только для миссий или скриптов и не должных становиться навигационными нодами.

Примеры использования:

  • Маркеры только для миссий
  • Визуальные/отладочные вейпойнты
  • Чекпойнты, не предназначенные для маршрутизации ИИ
  • Вспомогательные точки, используемые скриптами

Если excludeFromMap равно false, вейпойнт может быть включён как ручная навигационная нода.

Связь с map.json

BeamNGWaypoint часто используется совместно с map.json.

Пример вейпойнта:

{
  "class": "BeamNGWaypoint",
  "name": "Bridge1_A",
  "position": [0, 0, 0],
  "scale": [6, 6, 6],
  "excludeFromMap": false
}

Пример сегмента map.json, ссылающегося на него:

{
  "segments": {
    "bridge1": {
      "nodes": ["Bridge1_A", "Bridge1_B"],
      "drivability": 1
    }
  }
}

Если вейпойнт, на который ссылается map.json, не существует, сегмент не может быть корректно связан.

Не устанавливайте excludeFromMap в true для вейпойнтов, на которые ссылается map.json как на навигационные ноды.

Обновления навигационного графа

Когда вейпойнт добавляется, удаляется, перемещается или редактируется, движок уведомляет систему карты Lua.

Это позволяет обновлять навигационный граф при редактировании вейпойнтов.

События включают:

  • Вейпойнт добавлен
  • Вейпойнт удалён
  • Вейпойнт изменён
  • Изменён трансформ

В рабочих процессах редактора перезагрузка карты может быть незначительно отложена, чтобы не перестраивать навигационный граф на каждом кадре во время перемещения объекта.

Связь с миссиями и чекпойнтами

Вейпойнты широко используются системами геймплея и миссий.

Примеры:

  • Чекпойнты гонок
  • Ноды пути для испытаний на время
  • Позиции восстановления/чекпойнтов
  • Точки триггеров миссий
  • Целевые точки ИИ
  • Точки спавна/ориентиры

Для использования в качестве чекпойнтов задавайте подходящий масштаб, чтобы вейпойнт был достаточно большим для надёжного проезда транспортных средств.

Для направленных чекпойнтов используйте directionalWaypoint.

Отладочная визуализация

При отрисовке вейпойнт отображается как полупрозрачная оранжевая сфера.

Если включён directionalWaypoint, дополнительно отрисовывается:

  • Стрелка направления
  • Направляющая плоскость/сетка

Это помогает проверить направление вейпойнта и поведение триггера/чекпойнта.

Минимальный навигационный вейпойнт 

{
  "class": "BeamNGWaypoint",
  "name": "road_node_01",
  "position": [0, 0, 0],
  "scale": [5, 5, 5],
  "excludeFromMap": false
}

Используйте это для нод навигационного графа или ссылок в map.json.

Вейпойнт только для миссий 

{
  "class": "BeamNGWaypoint",
  "name": "mission_marker_01",
  "position": [50, 100, 2],
  "scale": [10, 10, 10],
  "drawDebug": false,
  "excludeFromMap": true
}

Используйте это для ссылок в миссиях/скриптах, которые не должны влиять на навигацию ИИ.

Направленный чекпойнт-вейпойнт 

{
  "class": "BeamNGWaypoint",
  "name": "checkpoint_01",
  "position": [100, 200, 5],
  "rotationMatrix": [0, -1, 0, 1, 0, 0, 0, 0, 1],
  "scale": [12, 12, 12],
  "directionalWaypoint": true,
  "excludeFromMap": true
}

Используйте это для логики в стиле чекпойнтов, где важно направление пересечения.

Рекомендации

  • Используйте чёткие, стабильные имена вейпойнтов.
  • Используйте равномерные значения масштаба.
  • Используйте excludeFromMap: true для вейпойнтов только для миссий.
  • Оставляйте excludeFromMap: false для вейпойнтов, используемых в map.json.
  • Используйте directionalWaypoint для направленных чекпойнтов.
  • Размещайте вейпойнты по центру предполагаемого пути или зоны срабатывания.
  • Используйте drawDebug только при необходимости.
  • Избегайте размещения лишних вейпойнтов в навигационном графе.
  • После редактирования навигационных вейпойнтов проверяйте ИИ/поиск пути.

Распространённые проблемы

Вейпойнт не отображается

Возможные причины:

  • drawDebug равно false
  • Глобальная отладочная отрисовка вейпойнтов отключена
  • Объект не выбран
  • Вейпойнт находится за пределами области видимости/отладочной отрисовки

Сегмент map.json не может найти вейпойнт

Возможные причины:

  • Имя вейпойнта не совпадает
  • Вейпойнт был переименован
  • Вейпойнт отсутствует в items.level.json
  • excludeFromMap равно true
  • Вейпойнт не загрузился

Маршрут ИИ не использует вейпойнт

Проверьте:

"excludeFromMap": false

Также убедитесь, что вейпойнт подключён через map.json или допустимый источник дорожного графа.

Чекпойнт срабатывает с неверного направления

Используйте:

"directionalWaypoint": true

и проверьте вращение вейпойнта.

Радиус вейпойнта кажется неверным

Используйте равномерный масштаб:

"scale": [8, 8, 8]

Избегайте неравномерного масштаба для вейпойнтов.

Итог

BeamNGWaypoint — именованный пространственный маркер, используемый навигацией, миссиями, чекпойнтами, скриптами и инструментами редактора.

Он может выступать в роли:

  • Ручной навигационной ноды
  • Опорной точки миссии
  • Объёма чекпойнта/триггера
  • Ссылки на ноду в map.json

Используйте excludeFromMap, чтобы управлять участием в навигации, и directionalWaypoint, когда требуется обнаружение с учётом направления.

См. также: Навигационная карта (map.json) , Маршруты автобусов , Файлы объектов уровня (items.level.json) .

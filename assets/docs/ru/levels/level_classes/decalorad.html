DecalRoad — это объект-декаль в форме полосы, задаваемый цепочкой дорожных узлов. Он используется главным образом для дорог, путей, окрашенных полос на поверхности, следов шин, разметки и других протяжённых декальных поверхностей, следующих рельефу местности или статическим мешам.

DecalRoad не является дорожной геометрией. Он проецируется/обрезается по существующим поверхностям, как правило по рельефу, и отображается в виде декали.

Кроме того, это один из основных источников данных для ИИ/навигации. Дороги с положительным значением drivability могут вносить вклад в навигационный граф, используемый ИИ, трафиком и системами маршрутизации.

Базовый пример

Простая запись DecalRoad в файле items.level.json:

{
  "class": "DecalRoad",
  "name": "road_main_01",
  "material": "road_asphalt",
  "textureLength": 8,
  "drivability": 1,
  "autoLanes": true,
  "oneWay": false,
  "renderPriority": 10,
  "nodes": [
    [0, 0, 0, 8],
    [50, 0, 0, 8],
    [100, 20, 0, 8]
  ]
}

Каждый узел хранится в виде:

[x, y, z, width]

Важные поля

Поле Тип Описание
class string Должно быть "DecalRoad".
name string Имя объекта сцены.
material string Материал, используемый для отрисовки декали дороги.
nodes array Список дорожных узлов. Каждый узел задаётся как [x, y, z, width].
textureLength number Длина в метрах до повтора текстуры материала.
renderPriority integer Порядок отрисовки. DecalRoad-объекты отрисовываются в порядке убывания приоритета.
zBias number Дополнительное смещение по глубине/рендеру.
decalBias number Высота/смещение над обрезаемой поверхностью.
distanceFade array[2] Параметры затухания по расстоянию.
startEndFade array[2] Длина затухания в начале и конце дороги.
overObjects bool Разрешает обрезку поверх статических мешей, а не только по рельефу.
hiddenInNavi bool Скрывает дорогу с навигационного отображения там, где это поддерживается.
persistentId string Постоянный идентификатор редактора/движка. Обычно генерируется автоматически.

Данные узлов

DecalRoad хранит свою форму в виде упорядоченного списка узлов.

Пример:

"nodes": [
  [0, 0, 0, 8],
  [50, 0, 0, 8],
  [100, 20, 0, 10]
]

Каждый узел содержит:

Индекс Значение
0 Позиция X
1 Позиция Y
2 Позиция Z
3 Ширина дороги

Дорога генерируется между последовательными узлами.

Ширина узла управляет шириной дороги в данной точке. Ширина интерполируется вдоль дороги.

Если overObjects отключён, высота узла проецируется на высоту рельефа при регенерации.

Проекция на рельеф

По умолчанию DecalRoad проецируется на рельеф.

Он считывает высоту рельефа и корректирует сгенерированные края так, чтобы они лежали на поверхности рельефа.

Это означает, что значение Z узла зачастую менее важно для дорог на рельефе, поскольку дорога привязывается к нему при регенерации.

Если уровень не содержит рельефа и параметр overObjects не включён, дорога не сможет сгенерироваться корректно.

overObjects 

"overObjects": true

При включении DecalRoad может проецироваться поверх статических мешей наравне с рельефом.

Это полезно для:

  • Мостов
  • Эстакад
  • Тоннелей
  • Дорог на меш-геометрии
  • Парковочных конструкций
  • Пользовательских мешей дорог

При отключении дорога рассчитана на рельеф.

Если на карте нет рельефа и дорога не использует overObjects, дорога может не сгенерироваться.

Материал

Поле material ссылается на датаблок Material.

"material": "road_asphalt"

Материал должен быть разработан для отрисовки декали дороги и обычно тайлится вдоль её длины.

Важные требования к материалу:

  • Текстура должна тайлиться вертикально/вдоль дороги.
  • Прозрачность может использоваться для мягких краёв или разметки.
  • Карта нормалей/шероховатости может применяться для детализации поверхности дороги.
  • Лаковое покрытие не используется для материалов декальных дорог.

Если материал не найден, дорога использует предупреждающий материал.

Длина текстуры

textureLength управляет частотой повтора материала дороги вдоль неё.

"textureLength": 8

Это означает, что текстура повторяется примерно каждые 8 метров.

Используйте меньшие значения для плотных повторяющихся узоров, большие — для крупной разметки или протяжённых текстур.

Приоритет отрисовки

renderPriority управляет порядком наложения пересекающихся декальных дорог.

"renderPriority": 10

Декальные дороги отрисовываются в порядке убывания приоритета.

Используйте приоритет отрисовки при наложении:

  • Разметки полос поверх асфальта
  • Пешеходных переходов поверх дорожного покрытия
  • Следов шин поверх дорожного покрытия
  • Патч-декалей поверх базовых дорог

zBias и decalBias

zBias и decalBias влияют на то, как декальная дорога смещается/сдвигается по глубине относительно целевой поверхности.

"zBias": 0,
"decalBias": 0

Эти параметры помогают избежать z-fighting, при котором дорога мерцает относительно поверхности под ней.

Используйте небольшие значения. Большие значения могут вызвать видимое парение или артефакты сортировки.

distanceFade

distanceFade управляет затуханием по расстоянию.

"distanceFade": [300, 50]

Первое значение — основное расстояние затухания, второе — диапазон затухания.

Если distanceFade не задан, используется значение по умолчанию:

[300, 50]

Первое значение, как правило, должно быть больше второго.

Некорректные значения затухания могут привести к тому, что дороги исчезают слишком рано или ведут себя непредсказуемо.

startEndFade

startEndFade создаёт затухание дороги в её начале и конце.

"startEndFade": [5, 5]

Это полезно для плавного перехода декалей дорог в другие поверхности или устранения резких обрывов.

Значение Смысл
Первое значение Расстояние затухания в начале дороги.
Второе значение Расстояние затухания в конце дороги.

Настройки сплайна

DecalRoad может генерировать края, используя либо устаревшую, либо улучшенную логику сплайна.

improvedSpline 

"improvedSpline": true

Использует более новую генерацию сплайна на основе Catmull-Rom.

smoothness 

"smoothness": 0.5

Управляет плавностью сплайна:

Значение Описание
0 Более линейный
0.5 Средняя плавность
1 Полностью плавный

detail 

"detail": 0.1

Управляет шагом дискретизации сплайна.

Меньшие значения создают больше подразделений. Например:

0.1 = every 10%
0.2 = every 20%
0.5 = midpoint only

looped 

"looped": true

Соединяет конец дороги с её началом.

Полезно для замкнутых трасс и кольцевых маршрутов.

startTangent / endTangent 

"startTangent": true,
"endTangent": true

Использует первую/последнюю точку в качестве маркеров касательной для более плавных конечных точек.

Режимы генерации сплайна

DecalRoad поддерживает два режима генерации сплайна:

  • Устаревший режим сплайна
  • Улучшенный режим сплайна

Оба режима преобразуют управляющие узлы дороги в сгенерированные края. Эти края затем используются для обрезки по рельефу/статике, генерации UV-развёртки, отрисовки и данных ИИ/навигации.

Устаревший режим сплайна

Устаревший режим используется, когда:

"improvedSpline": false

или когда поле не включено.

Устаревший режим строит сплайн Catmull-Rom через узлы дороги, а затем определяет, где размещать сгенерированные края, основываясь на изменениях направления кривой.

Процесс в общих чертах:

  1. Построить сплайн Catmull-Rom из позиций узлов.
  2. Построить второй сплайн для ширины дороги.
  3. Пройти вдоль сплайна между каждой парой управляющих узлов.
  4. Добавить сгенерированный край, когда:
    • это начало сегмента,
    • это конец дороги,
    • или изменение направления превышает breakAngle.

Таким образом, breakAngle управляет плотностью подразделения дороги на поворотах.

Пример:

"improvedSpline": false,
"breakAngle": 3

Меньшие значения breakAngle создают больше сгенерированных краёв на поворотах, улучшая точность формы, но увеличивая объём сгенерированной геометрии.

Большие значения создают меньше краёв, что может быть дешевле, но выглядеть менее плавно.

Улучшенный режим сплайна

Улучшенный режим используется, когда:

"improvedSpline": true

Этот режим также использует интерполяцию Catmull-Rom, но подразделение управляется более непосредственно через detail и smoothness.

Процесс в общих чертах:

  1. Считать позиции узлов и их ширину.
  2. При необходимости использовать первый/последний узел как маркеры касательной, если включены startTangent или endTangent.
  3. Если включён looped, завернуть индексы узлов так, чтобы дорога образовывала замкнутый контур.
  4. Вычислить кривую Catmull-Rom с равными интервалами.
  5. Сгенерировать край в каждом интервале.
  6. Интерполировать ширину дороги вдоль той же кривой.
  7. Спроецировать сгенерированные точки на высоту поверхности рельефа/статики.

В улучшенном режиме:

  • smoothness управляет натяжением кривой Catmull-Rom.
  • detail управляет плотностью дискретизации.
  • looped позволяет создавать замкнутые дороги/трассы.
  • startTangent и endTangent улучшают управление направлением в конечных точках.

Пример:

"improvedSpline": true,
"smoothness": 0.5,
"detail": 0.1

Здесь detail: 0.1 означает, что кривая дискретизируется примерно каждые 10% каждого сегмента.

Выбор режима

По возможности используйте улучшенный сплайн для новых дорог. Он обеспечивает более предсказуемое управление плавностью, поведением петель и подразделением.

Используйте устаревший режим главным образом для старого контента или когда нужно сохранить точную форму существующей дороги.

Режим Лучшее применение
Устаревший Существующие старые дороги, совместимость.
Улучшенный Новые дороги, более плавные кривые, петли, лучший контроль.
Переключение существующей дороги с устаревшего на улучшенный режим сплайна может незначительно изменить её форму. После смены режима проверьте края дороги, UV-развёртку, декали и поведение ИИ/навигации.

breakAngle

breakAngle используется алгоритмом устаревшего сплайна.

"breakAngle": 3

Дорога подразделяется, когда угол кривой превышает этот порог.

Меньшие значения генерируют больше промежуточных краёв на поворотах.

Сгенерированные края

В процессе регенерации узлы дороги преобразуются в края дороги.

Каждый сгенерированный край содержит:

  • Центральную точку
  • Точки левого/правого краёв
  • Ширину
  • Вектор направления вперёд
  • Вектор направления вправо
  • Вектор направления вверх
  • Индекс родительского узла

Эти края затем используются для захвата/обрезки треугольников лежащего ниже рельефа/статического меша.

Захват геометрии

DecalRoad не создаёт плоскую меш-дорогу напрямую. Вместо этого он:

  1. Строит полосу из краёв дороги.
  2. Создаёт плоскости обрезки для каждого сегмента.
  3. Захватывает треугольники рельефа/статического меша внутри полосы дороги.
  4. Генерирует вершины и индексы декальной дороги.
  5. Назначает UV-координаты на основе ширины дороги и textureLength.
  6. Разбивает дорогу на пакеты для отрисовки.

Это позволяет дороге следовать лежащей в основе поверхности.

Пакетирование и отрисовка

Декальные дороги группируются по материалу и приоритету отрисовки.

Менеджер статических декальных дорог перестраивает и пакетирует геометрию дорог для эффективной отрисовки.

Дороги разбиваются на узлы/пакеты рендера для:

  • Отсечения (culling)
  • Затухания по расстоянию
  • Упорядочивания по приоритету отрисовки
  • Управления большими объёмами данных дороги

Это важно, поскольку уровни могут содержать множество сегментов декальных дорог.

Диапазон упакованных вершин

Рендерер декальных дорог упаковывает позиции вершин относительно начал блоков.

Большие пакеты декальных дорог разбиваются на блоки, чтобы упакованные локальные позиции оставались в пределах поддерживаемого диапазона.

Если дорога формирует пакет слишком большого размера или слишком большого охвата, в журнал могут записываться ошибки о диапазоне упакованных позиций или количестве вершин.

Для наилучших результатов:

  • Избегайте крайне длинных одиночных дорог с огромными непрерывными пролётами.
  • Разбивайте очень длинные дороги на более короткие участки.
  • Избегайте очень широких дорог со сложной обрезкой по многим объектам.

Обновления рельефа и статических объектов

Декальные дороги могут регенерироваться при изменении лежащей в основе поверхности.

Примеры:

  • Изменение высоты рельефа
  • Перемещение статического объекта при включённом overObjects
  • Изменение узлов дороги
  • Изменение материала
  • Изменение настроек сплайна
  • Изменение настроек смещения/затухания

Когда изменения рельефа перекрываются с дорогой, дорога может быть помечена для регенерации.

DecalRoad может участвовать в формировании данных ИИ/навигации.

Важные поля навигации:

Поле Тип Описание
drivability number Определяет, использует ли ИИ эту дорогу и каким образом.
autoLanes bool Автоматически генерирует данные о полосах движения.
lanesLeft integer Количество полос с одной стороны (задаётся вручную).
lanesRight integer Количество полос с другой стороны (задаётся вручную).
oneWay bool Помечает дорогу как одностороннюю.
flipDirection bool Разворачивает логическое направление движения.
gatedRoad bool Помечает дорогу как закрытую/частную.
autoJunction bool Разрешает автоматические соединения перекрёстков.
useSubdivisions bool Генерирует более детальные данные дороги для ИИ.
hiddenInNavi bool Скрывает дорогу с навигационного отображения там, где это поддерживается.

drivability

drivability управляет тем, используется ли дорога ИИ/навигацией и каким образом.

"drivability": 1

Типичные значения:

Значение Смысл
-1 Не используется для ИИ/навигации.
1 Обычная/оптимальная дорога.
0.5 Менее предпочтительная дорога, часто грунтовая или с плохим покрытием.
0 Очень плохой маршрут, как правило, избегается.

Если вы хотите, чтобы ИИ-трафик использовал дорогу, установите drivability выше 0.

Если дорога является только визуальной, используйте:

"drivability": -1

autoLanes 

"autoLanes": true

При включении полосы движения генерируются автоматически на основе ширины дороги и правил дорожного движения.

При отключении используйте:

"lanesLeft": 1,
"lanesRight": 1

Ручные настройки полос интерпретируются совместно с настройкой roadRules.rightHandDrive уровня из файла info.json.

oneWay и flipDirection

oneWay делает дорогу односторонней:

"oneWay": true

По умолчанию направление соответствует порядку узлов.

flipDirection разворачивает логическое направление:

"flipDirection": true

Используйте flipDirection, если ИИ едет в неверном направлении.

gatedRoad 

"gatedRoad": true

Помечает дорогу как закрытую/частную.

Используется для:

  • Частных дорог
  • Дорог с ограниченным доступом
  • Грунтовых трасс
  • Гоночных трасс
  • Непубличных маршрутов

При обработке навигации частные/закрытые дороги обрабатываются иначе, чем обычные общественные дороги.

autoJunction 

"autoJunction": true

При включении навигационная система может автоматически соединять пути в местах их пересечения.

Отключите для дорог, где автоматическое слияние создаёт некорректные соединения:

"autoJunction": false

Полезно для:

  • Эстакад
  • Тоннелей
  • Параллельных дорог
  • Сложных перекрёстков
  • Близко расположенных дорог на разных высотах

useSubdivisions 

"useSubdivisions": true

При включении граф ИИ/навигации может использовать более детальные данные о подразделении краёв дороги вместо только управляющих узлов.

Это может улучшить точность навигационного пути на криволинейных дорогах.

hiddenInNavi 

"hiddenInNavi": true

Скрывает дорогу с навигационного отображения там, где это поддерживается, при этом позволяя ей существовать в графе в зависимости от поведения системы.

Сериализация

В современном формате items.level.json узлы дороги хранятся в массиве nodes:

"nodes": [
  [0, 0, 0, 8],
  [50, 0, 0, 8]
]

В устаревшей сериализации TorqueScript использовались повторяющиеся поля Node = "x y z width";. В современном JSON следует использовать массив nodes.

Минимальный пример дороги 

{
  "class": "DecalRoad",
  "name": "road_asphalt_01",
  "material": "road_asphalt",
  "textureLength": 8,
  "renderPriority": 10,
  "drivability": 1,
  "autoLanes": true,
  "autoJunction": true,
  "nodes": [
    [0, 0, 0, 8],
    [50, 0, 0, 8],
    [100, 20, 0, 8]
  ]
}

Пример односторонней дороги 

{
  "class": "DecalRoad",
  "name": "highway_ramp_01",
  "material": "road_asphalt",
  "textureLength": 12,
  "drivability": 1,
  "oneWay": true,
  "flipDirection": false,

Рекомендации

  • Используйте DecalRoad для декалей дорог и навигационных поверхностей, а не для приподнятой дорожной геометрии.
  • Используйте TSStatic или рельеф для реальной физической геометрии.
  • Используйте overObjects для мостов, эстакад, тоннелей или меш-дорог.
  • Подбирайте textureLength в соответствии с задуманным масштабом материала.
  • Используйте renderPriority для слоистой разметки.
  • Устанавливайте drivability правильно:
    • 1 для обычных дорог
    • 0.5 для грунтовых/бездорожных дорог
    • -1 для только визуальных декалей
  • Отключайте autoJunction для наложенных дорог, эстакад или близко расположенных параллельных путей.
  • Используйте правильные настройки полос для ИИ/трафика.
  • Разбивайте очень длинные или сложные дороги на управляемые участки.
  • Регенерируйте дороги после изменения рельефа или узлов.

Распространённые проблемы

Дорога не отображается

Возможные причины:

  • Менее двух узлов
  • Неверный или отсутствующий материал
  • Нет рельефа и overObjects отключён
  • Дорога не захватила никакой лежащей в основе геометрии
  • Скрыта затуханием по расстоянию
  • Материал — road_invisible

Дорога мерцает или проявляется z-fighting

Отрегулируйте:

"decalBias": 0.001

или проверьте материал/приоритет отрисовки.

Дорога парит над рельефом

Возможно, слишком большое значение decalBias, или дорога проецируется поверх непредвиденного объекта.

Дорога не следует мешу моста

Включите:

"overObjects": true

ИИ не использует дорогу

Проверьте:

"drivability": 1

Также проверьте настройки полос и участвует ли дорога в навигационном графе.

ИИ едет в неверном направлении

Переключите:

"flipDirection": true

или проверьте порядок узлов.

Дороги соединяются некорректно

Установите:

"autoJunction": false

для проблемных дорог.

Текстура повторяется слишком часто или слишком редко

Отрегулируйте:

"textureLength": 8

Меньшие значения повторяют чаще. Большие значения растягивают/повторяют реже.

Дорога исчезает на расстоянии

Проверьте:

"distanceFade": [300, 50]

или увеличьте расстояние затухания.

Итоги

DecalRoad — это объект декали дороги, проецируемой на рельеф/статические поверхности.

Он используется как визуально, так и для ИИ/навигации. Визуально он отображает текстурированную полосу, обрезаемую по рельефу или статическим мешам. Для игрового процесса он может передавать данные о дороге/полосах/проходимости в навигационный граф.

Используйте его для дорог, путей, разметки и поверхностей, подключённых к навигации. Для реальной приподнятой геометрии используйте TSStatic или рельеф, при необходимости проецируя поверх DecalRoad.

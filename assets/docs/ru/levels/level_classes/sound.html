SFXSpace и SFXEmitter — объекты сцены для работы со звуком.

Класс Назначение
SFXSpace Определяет трёхмерную зону/объём с фоновым аудио.
SFXEmitter Размещает позиционный или непозиционный источник звука на уровне.

Используйте SFXSpace для изменения фоновой атмосферы в зависимости от области — например, в тоннелях, помещениях, пещерах, лесах или в зонах реверберации/атмосферы.

Используйте SFXEmitter для воспроизведения конкретного звука в точке мира — например, гул машин, водопады, птицы, сирены, зацикленные или одноразовые звуки, локальные звуки окружения.

SFXSpace — трёхмерный объём, задающий зону с фоновым звуком.

Обычно используется для изменения атмосферы, воспринимаемой слушателем при входе в объём или пересечении с ним. Например:

  • Атмосфера внутреннего помещения
  • Реверберация/атмосфера тоннеля
  • Атмосфера пещеры
  • Атмосфера леса
  • Атмосфера путепровода
  • Атмосфера промышленной зоны
  • Локальные звуковые зоны окружения

SFXSpace — объект сцены с позицией, поворотом и масштабом. Масштаб объекта по умолчанию:

[10, 10, 10]

Локальный бокс объекта представляет собой единичный куб от -0.5 до 0.5, поэтому итоговый размер зоны задаётся через scale.

Basic SFXSpace example 

{
  "class": "SFXSpace",
  "name": "tunnel_audio_space",
  "position": [120, 45, 8],
  "rotationMatrix": [1,0,0,0,1,0,0,0,1],
  "scale": [40, 12, 8],
  "soundAmbience": "TunnelAmbience"
}

Создаёт прямоугольную аудиозону с центром в точке [120, 45, 8].

SFXSpace fields

SFXSpace наследует большинство полезных полей от объекта сцены и от поведения миксина фонового звука.

Поле Тип Описание
class string Должно быть "SFXSpace".
name string Имя объекта сцены.
position array[3] Центральная позиция аудиообъёма.
rotationMatrix array[9] Ориентация аудиообъёма.
scale array[3] Размер объёма SFXSpace.
soundAmbience object/datablock name Атмосфера, назначенная пространству, при наличии поддержки в реализации объекта фонового звука.

Shape and scale

SFXSpace имеет форму параллелепипеда.

Пример:

"scale": [30, 20, 6]

Это даёт объём приблизительно:

30 meters wide
20 meters long
6 meters tall

Используйте масштаб для подгонки зоны под область, в которой должна применяться атмосфера.

Editor visibility

Отрисовка SFXSpace по умолчанию отключена:

getStaticClassRep()->mIsRenderEnabled = false;

Это означает, что SFXSpace обычно не отображаются как рендеримые объекты. Они являются вспомогательными объёмами преимущественно для редактора.

Цвет сплошной заливки в редакторе может браться из назначенной атмосферы:

getSoundAmbience()->getEditorColor()

Если атмосфера не назначена, используется цвет родительского объекта в редакторе по умолчанию.

SFXEmitter

SFXEmitter — невидимый трёхмерный объект сцены, излучающий звук.

Он может воспроизводить:

  • Существующий SFXTrack
  • Напрямую назначенный звуковой файл
  • Трёхмерный позиционный звук
  • Непозиционный звук
  • Зацикленное или незацикленное аудио
  • Потоковое или буферизованное аудио в зависимости от настроек

Для создания атмосферы уровня проще использовать настройку атмосферы через LevelInfo или SFXSpace. Применяйте SFXEmitter, когда звук привязан к конкретному местоположению или требует явного управления воспроизведением.

Basic SFXEmitter example 

{
  "class": "SFXEmitter",
  "name": "waterfall_emitter",
  "position": [100, 250, 15],
  "rotationMatrix": [1,0,0,0,1,0,0,0,1],
  "track": "WaterfallLoop",
  "playOnAdd": true,
  "useTrackDescriptionOnly": true
}

Этот эмиттер воспроизводит трек WaterfallLoop при загрузке уровня.

File-based SFXEmitter example 

{
  "class": "SFXEmitter",
  "name": "generator_hum",
  "position": [20, 15, 2],
  "rotationMatrix": [1,0,0,0,1,0,0,0,1],
  "fileName": "/levels/example/art/sound/generator_loop.ogg",
  "playOnAdd": true,
  "is3D": true,
  "isLooping": true,
  "isStreaming": false,
  "volume": 0.8,
  "pitch": 1,
  "referenceDistance": 5,
  "maxDistance": 60,
  "fadeInTime": 0.5,
  "fadeOutTime": 1.0
}

SFXEmitter media fields

Поле Тип Описание
track SFXTrack name Существующий трек/событие/профиль для воспроизведения. Имеет приоритет над fileName.
fileName string path Звуковой файл для прямого воспроизведения. Используется, если track не назначен.

Используйте либо track, либо fileName.

Если назначены оба:

track wins

SFXEmitter sound fields

Поле Тип Описание
playOnAdd bool Автоматически запускает воспроизведение при добавлении/загрузке эмиттера.
useTrackDescriptionOnly bool Если true, поля эмиттера не переопределяют описание назначенного трека.
isLooping bool Зацикливать ли воспроизведение fileName. Игнорируется при назначенном track.
isStreaming bool Использовать ли потоковое воспроизведение для fileName. Игнорируется при назначенном track.
sourceGroup SFXSource name Необязательный источник/канал/группа, к которой привязывается звук.
volume number Множитель громкости воспроизведения.
pitch number Множитель высоты тона. 1 — нормальная высота.
fadeInTime number Время нарастания громкости в секундах.
fadeOutTime number Время затухания громкости в секундах.

SFXEmitter 3D sound fields

Поле Тип Описание
is3D bool Воспроизводить ли fileName как позиционный 3D-звук. Игнорируется при назначенном track.
referenceDistance number Расстояние, с которого начинается затухание.
maxDistance number Расстояние, на котором затухание достигает максимума. Также используется для границ эмиттера и масштаба в редакторе.
scatterDistance array[3] Границы случайного смещения начальной 3D-позиции звука.
coneInsideAngle integer Угол внутреннего направленного конуса в градусах.
coneOutsideAngle integer Угол внешнего направленного конуса в градусах.
coneOutsideVolume number Множитель громкости за пределами внешнего конуса.
rangeSphereColor color[4] Цвет визуализации сферы дальности в редакторе.

Default SFXEmitter behavior

Конструктор инициализирует эмиттеры со следующими значениями по умолчанию:

Настройка Значение по умолчанию
playOnAdd true
is3D true
isLooping true
isStreaming false
fadeInTime -1
fadeOutTime -1

Бокс объекта по умолчанию:

[-1, -1, -1] to [1, 1, 1]

При добавлении/обновлении масштаб эмиттера устанавливается из maxDistance:

scale = [maxDistance, maxDistance, maxDistance]

Using track vs fileName

Using track

Используйте track, когда аудионастройка уже существует в виде датаблока/события/профиля.

{
  "class": "SFXEmitter",
  "name": "factory_loop",
  "position": [0, 0, 3],
  "track": "FactoryMachineryLoop",
  "playOnAdd": true,
  "useTrackDescriptionOnly": true
}

Преимущества:

  • Централизованная настройка звука
  • Повторно используемые определения звуков
  • Подходит для событий FMOD или настроенных SFX-треков
  • Описание настроек может использоваться совместно несколькими эмиттерами

Using fileName

Используйте fileName для простых локальных звуков уровня без создания отдельного трека.

{
  "class": "SFXEmitter",
  "name": "buzzing_light",
  "position": [5, 10, 3],
  "fileName": "/levels/example/art/sound/light_buzz.ogg",
  "playOnAdd": true,
  "is3D": true,
  "isLooping": true,
  "volume": 0.5,
  "referenceDistance": 1,
  "maxDistance": 12
}

Преимущества:

  • Быстрая настройка
  • Подходит для простых размещённых звуков
  • Не требует отдельного датаблока трека

useTrackDescriptionOnly 

"useTrackDescriptionOnly": true

При включении эмиттер использует звуковое описание трека и игнорирует большинство локальных звуковых полей эмиттера.

Это полезно, когда трек уже настроен с правильными параметрами:

  • Поведение 3D/2D
  • Зацикливание
  • Затухание по расстоянию
  • Настройки конуса
  • Громкость/высота тона/затухание

На эмиттере остаются значимыми только playOnAdd и track.

Если назначенный трек не допускает переопределения описания, эмиттер также использует описание трека.

3D attenuation

Для 3D-звуков поведение в зависимости от расстояния управляется главным образом через:

"referenceDistance": 5,
"maxDistance": 60
Поле Значение
referenceDistance Ближнее расстояние, на котором звук воспроизводится с полной или опорной громкостью.
maxDistance Дальнее расстояние, используемое для затухания и визуализации в редакторе.

Пример:

{
  "referenceDistance": 3,
  "maxDistance": 40
}

Используйте меньшие значения для тихих/локальных звуков и бо́льшие для громких звуков окружения.

Emitter scale and maxDistance

В редакторе масштабирование SFXEmitter изменяет maxDistance.

Реализация вычисляет среднее осей масштаба:

maxDistance = (scale.x + scale.y + scale.z) / 3

Затем значение ограничивается снизу величиной referenceDistance.

Итоговый масштаб объекта принудительно устанавливается в:

[maxDistance, maxDistance, maxDistance]

Это означает, что эмиттеры ведут себя как сферические объекты радиуса действия, а не как произвольные параллелепипедные аудиообъекты.

Directional sound cones

SFXEmitter поддерживает настройки направленного 3D-конуса.

"coneInsideAngle": 90,
"coneOutsideAngle": 180,
"coneOutsideVolume": 0.25
Поле Описание
coneInsideAngle Угол конуса полной громкости.
coneOutsideAngle Угол внешнего переходного конуса.
coneOutsideVolume Громкость за пределами внешнего конуса.

Используйте конусы для направленных звуков, например:

  • Динамики
  • Сирены
  • Вентиляционные отверстия
  • Выхлоп механизмов
  • Водопады, ориентированные в определённую сторону
  • Громкоговорители или системы трансляции

Если оба угла конуса близки к 360, звук фактически является ненаправленным.

Directional emitter example 

{
  "class": "SFXEmitter",
  "name": "speaker_emitter",
  "position": [10, 0, 3],
  "rotationMatrix": [1,0,0,0,1,0,0,0,1],
  "fileName": "/levels/example/art/sound/speaker_loop.ogg",
  "playOnAdd": true,
  "is3D": true,
  "isLooping": true,
  "volume": 1,
  "referenceDistance": 4,
  "maxDistance": 80,
  "coneInsideAngle": 60,
  "coneOutsideAngle": 140,
  "coneOutsideVolume": 0.15
}

Ориентация трансформа эмиттера важна для направления конуса.

Playback lifecycle

При добавлении SFXEmitter:

  1. Эмиттер проверяет своё звуковое описание.
  2. Он создаёт или обновляет внутренний SFXSource.
  3. Применяет трансформ и скорость к источнику.
  4. Устанавливает громкость, высоту тона, времена затухания, расстояние и настройки конуса там, где это допустимо.
  5. Если playOnAdd равно true, воспроизведение начинается.

При удалении эмиттер удаляет свой внутренний источник.

Manual playback

SFXEmitter поддерживает методы ручного управления воспроизведением:

play()
stop()

Существуют устаревшие консольные методы:

emitter.play();
emitter.stop();

Lua-привязки также предоставляют доступ к поведению эмиттера.

Lua API

SFXEmitter доступен из Lua как подкласс SceneObject.

Creating an emitter 

local emitter = SFXEmitter()

Объект переходит под управление C++ после регистрации/добавления через стандартные рабочие процессы объектов движка.

Static variables

Lua name Тип Описание
SFXEmitter.renderEmitters bool Включает отладочную отрисовку эмиттеров.
SFXEmitter.renderFarEmitters bool Принудительно отображает далёкие эмиттеры в отладочном режиме редактора.

Пример:

SFXEmitter.renderEmitters = true
SFXEmitter.renderFarEmitters = true

Lua methods

Метод Описание
setVolumePitchCT(volume, pitch, color, texture) Устанавливает громкость, высоту тона и параметры color/texture на источнике.
setPosition(position) Устанавливает позицию эмиттера.
setVelocity(velocity) Устанавливает скорость эмиттера.

Пример:

emitter:setPosition(Point3F(10, 20, 3))
emitter:setVelocity(Point3F(0, 0, 0))
emitter:setVolumePitchCT(0.8, 1.0, 0.2, 0.5)

setVolumePitchCT влияет на эмиттер только при наличии внутреннего SFXSource.

Editor visualization

SFXEmitter отображает отладочные визуальные элементы только при редактировании миссии.

Может отображать:

  • Сферу максимального радиуса действия
  • Сферу опорного/минимального расстояния
  • Стрелки ориентации вперёд/вверх
  • Визуализацию направленного конуса
  • Изменение цвета в зависимости от состояния (воспроизводится/остановлен, в зоне слышимости/вне зоны)

Цвет сферы дальности задаётся через:

"rangeSphereColor": [0.8, 0, 0, 0.3]

Эмиттер отображает отладочные визуальные элементы только при выполнении следующих условий:

  • Активен мировой редактор, и
  • Эмиттер выбран или SFXEmitter.renderEmitters равно true, и
  • Звук является 3D

Best practices

SFXSpace

  • Используйте пространства для областей, а не для точечных звуков.
  • Подбирайте масштаб объёма под реальный физический размер области.
  • Используйте понятные имена, например tunnel_audio_space или garage_reverb_space.
  • Избегайте большого количества перекрывающихся пространств, если предполагаемое поведение смешивания/приоритетов не изучено досконально.
  • Используйте цвета атмосферы в редакторе, чтобы аудиозоны было проще идентифицировать.

SFXEmitter

  • Предпочитайте track для повторно используемых звуков или звуков, созданных в FMOD.
  • Используйте fileName для простых локальных звуковых файлов.
  • Делайте maxDistance настолько малым, насколько это практически возможно.
  • Используйте isStreaming для длинных аудиофайлов или атмосферных петель.
  • Используйте непотоковое воспроизведение для коротких звуков.
  • Используйте useTrackDescriptionOnly, когда трек уже настроен корректно.
  • Используйте направленные конусы для динамиков, сирен, вентиляций или других направленных источников.
  • Избегайте большого количества зацикленных эмиттеров с большим радиусом действия в одной области.
  • Используйте времена затухания, чтобы избежать резких стартов/остановок.

Common issues

SFXEmitter does not play

Возможные причины:

  • playOnAdd равно false.
  • track указывает на несуществующий трек.
  • Путь fileName недействителен.
  • Оба поля — track и fileName — пусты.
  • Аудиоустройство/система не инициализированы.
  • Звук является 3D и слушатель находится за пределами maxDistance.
  • Громкость равна 0.
  • Громкость группы источников/канала отключена.

fileName is ignored

Если назначен track, он имеет приоритет.

Удалите track, если хотите использовать:

"fileName": "/path/to/sound.ogg"

Emitter settings do not affect the sound

При использовании трека проверьте:

"useTrackDescriptionOnly": true

При включении локальные поля эмиттера, такие как громкость, высота тона, расстояние 3D и настройки конуса, игнорируются в пользу описания трека.

Также некоторые треки могут не допускать переопределения своих описаний.

Sound is not positional

Проверьте:

"is3D": true

При использовании track собственный SFXDescription трека определяет, является ли звук трёхмерным.

Sound fades out too early

Увеличьте:

"maxDistance": 100

или проверьте настройки расстояния трека, если используется track.

Directional cone points the wrong way

Конус следует ориентации трансформа эмиттера.

Поверните эмиттер так, чтобы его направление вперёд указывало туда, куда должен проецироваться звук.

SFXSpace has no audible effect

Возможные причины:

  • Атмосфера не назначена.
  • Слушатель находится за пределами объёма.
  • Масштаб объёма слишком мал или объём расположен неверно.
  • Рейкастинг/реверберация окружения или настройки смешивания атмосферы переопределяют или отключают поведение пространства.
  • Другое аудиопространство или атмосфера имеет более высокий приоритет в зависимости от аудионастроек.

Summary

SFXSpace и SFXEmitter выполняют разные роли в размещении звука.

  • SFXSpace задаёт объём зоны фонового звука.
  • SFXEmitter создаёт и управляет источником звука в сцене.

Используйте SFXSpace для зон с фоновой атмосферой и реверберацией. Используйте SFXEmitter для фактического воспроизведения звука из точки или ориентации в мире. Для повторно используемых звуков или звуков, созданных в FMOD, назначайте track; для простых локальных звуков используйте fileName.

В BeamNG есть несколько типов объектов воды на сцене:

Класс Форма Основное применение
WaterPlane Бесконечная горизонтальная плоскость Океаны, большие озёра, вода у горизонта.
WaterBlock Ограниченный прямоугольный объём воды Бассейны, пруды, водохранилища, замкнутые водные зоны.
River Объём воды в форме сплайна Реки, каналы, ручьи, изогнутые водные пути.

Все три класса наследуют общее поведение воды от WaterObject, включая водные материалы, проверки нахождения под водой, поддержку тумана/подводных эффектов и опциональные планарные отражения.

Какой объект воды выбрать?

Сценарий использования Рекомендуемый класс
Океан вокруг острова WaterPlane
Очень большая плоская водная поверхность, уходящая к горизонту WaterPlane
Бассейн / небольшое озеро / прямоугольный пруд WaterBlock
Ограниченная плоская водная зона с вращением/масштабом WaterBlock
Река, следующая по пути River
Канал с переменной шириной/глубиной River
Извилистый водоём с направлением течения River

WaterPlane представляет бесконечную горизонтальную водную поверхность.

Этот объект предназначен главным образом для воды океанского типа, наблюдаемой с уровня земли. Он простирается бесконечно по осям X/Y, а его положение задаётся только высотой.

WaterPlane игнорирует вращение и масштаб. Значимым является только компонент Z его позиции.

Пример WaterPlane 

{
  "class": "WaterPlane",
  "name": "ocean_water",
  "position": [0, 0, 0],
  "material": "water_ocean",
  "gridSize": 101,
  "gridElementSize": 1
}

Ключевая часть:

"position": [0, 0, 0]

Только position[2] — высота по Z — используется в качестве уровня воды.

Поля WaterPlane

Поле Тип Описание
class string Должно быть "WaterPlane".
name string Имя объекта сцены.
position array[3] Высота воды берётся из position.z. Компоненты X/Y не влияют на охват.
material string Водный материал, используемый при рендеринге.
gridSize integer Размер сетки рендеринга.
gridElementSize number Расстояние между вершинами водной сетки.

Бесконечные границы

WaterPlane использует глобальные границы.

Это означает, что он считается существующим везде по горизонтали. Подходит для океанов, но не для небольших водоёмов.

Используйте WaterBlock, если вода должна иметь конечные границы.

Поведение трансформации

WaterPlane принимает только изменение высоты.

Поля вращения и масштаба исключены из постоянных полей и не используются.

Пример:

{
  "class": "WaterPlane",
  "name": "sea_level",
  "position": [0, 0, -2]
}

Создаёт водную плоскость на высоте Z = -2.

Настройки сетки

gridSize и gridElementSize управляют сеткой рендеринга, используемой водной плоскостью.

"gridSize": 101,
"gridElementSize": 1

WaterPlane рендерит центрированную по камере сетку, а не буквально бесконечный меш. Шейдер воды проецирует эту сетку как большую водную поверхность.

Очень большие или нестандартные настройки сетки могут увеличить стоимость рендеринга или вызвать визуальные артефакты.

Ограничения WaterPlane

WaterPlane предназначен для рендеринга океанского типа на уровне земли.

Поскольку он фактически бесконечен и проецируется вплоть до дальней плоскости отсечения, артефакты могут появляться, когда крупные объекты пересекают дальнюю плоскость отсечения или визуально совпадают с водной плоскостью.

Типичные проблемные случаи:

  • Очень большая высота камеры
  • Малое расстояние дальней плоскости отсечения
  • Крупные объекты вблизи дальней плоскости отсечения
  • Ландшафт, приближающийся к дальней плоскости отсечения или выходящий за неё

Для наилучших результатов:

  • Держите ландшафт и крупные объекты в пределах дальней плоскости отсечения камеры.
  • Не используйте WaterPlane для небольших бассейнов или ограниченных зон.
  • Используйте WaterBlock для локальных водных объёмов.

WaterBlock

WaterBlock — это конечный прямоугольный объём воды.

Поддерживает позицию, вращение и масштаб. Подходит для ограниченных водоёмов: озёр, бассейнов, прудов, водоёмов и других локальных водных зон.

В отличие от WaterPlane, WaterBlock имеет конечные границы и поддерживает проверку вхождения точки в объём.

Пример WaterBlock 

{
  "class": "WaterBlock",
  "name": "lake_water",
  "position": [100, 200, 5],
  "rotationMatrix": [1,0,0,0,1,0,0,0,1],
  "scale": [200, 150, 20],
  "material": "water_lake",
  "gridElementSize": 5
}

Поля WaterBlock

Поле Тип Описание
class string Должно быть "WaterBlock".
name string Имя объекта сцены.
position array[3] Позиция водного блока. Поверхность воды определяется на основе плоскости позиции объекта.
rotationMatrix array[9] Вращение объекта.
scale array[3] Размер водного объёма. X/Y задают площадь; Z задаёт вертикальный объём/поведение глубины.
material string Водный материал, используемый при рендеринге.
gridElementSize number Расстояние между вершинами генерируемого водного меша.
gridSize number Псевдоним для обратной совместимости с gridElementSize.

Масштаб и высота поверхности

WaterBlock использует трансформацию и масштаб объекта для определения прямоугольного водного объёма.

Типичная настройка:

"position": [0, 0, 10],
"scale": [100, 100, 20]

Масштаб X/Y управляет видимой площадью. Масштаб Z управляет вертикальным объёмом воды, используемым при проверках нахождения под водой и глубины.

Высота поверхности воды определяется позицией объекта.

gridElementSize

gridElementSize управляет расстоянием между генерируемыми вершинами.

"gridElementSize": 5

Меньшие значения создают более плотную геометрию:

"gridElementSize": 1

Большие значения создают более дешёвую, менее плотную геометрию:

"gridElementSize": 10

Если gridElementSize превышает масштаб блока по X или Y, значение ограничивается меньшим из этих двух измерений.

Очень маленькие значения gridElementSize для больших WaterBlockов могут создать значительный объём геометрии.

Генерация меша

WaterBlock строит равномерную сетку по своей площади.

Для больших блоков меш может быть разбит на несколько буферов вершин/индексов, чтобы не превышать ограничение 16-битных индексов.

Объект регенерирует водный меш при:

  • изменении gridElementSize
  • изменении масштаба
  • применении изменений редактором
  • изменении статических полей

Повёрнутые WaterBlock-и

WaterBlock поддерживает вращение.

Это может быть полезно для водных объёмов, размещённых на нестандартной геометрии или в зонах, не выровненных по осям.

Поведение отражений может использовать либо направленную вверх нормаль, либо нормаль поверхности объекта — в зависимости от унаследованных настроек отражений.

River

River — это объём воды в форме сплайна.

Задаётся списком узлов. Каждый узел хранит позицию, ширину, глубину и нормаль.

Реки подходят для ручьёв, каналов, дренажных канав, извилистых водных путей или любого водного объёма, который должен следовать по кривой.

Пример River 

{
  "class": "River",
  "name": "main_river",
  "material": "water_river",
  "SegmentLength": 10,
  "SubdivideLength": 2.5,
  "FlowMagnitudePhysics": 3,
  "LowLODDistance": 50,
  "nodes": [
    [0, 0, 5, 20, 4, 0, 0, 1],
    [50, 10, 5, 22, 4, 0, 0, 1],
    [100, 40, 4, 18, 3, 0, 0, 1],
    [150, 60, 4, 16, 3, 0, 0, 1]
  ]
}

Каждый узел реки хранится в виде:

[x, y, z, width, depth, normalX, normalY, normalZ]

Поля River

Поле Тип Описание
class string Должно быть "River".
name string Имя объекта сцены.
material string Водный материал, используемый при рендеринге.
nodes array Список узлов реки. Каждый узел: [x, y, z, width, depth, normalX, normalY, normalZ].
SegmentLength number Приблизительная длина в метрах генерируемых сегментов объёма реки.
SubdivideLength number Максимальный размер подразбиения для рендеринга высокодетализированной геометрии.
FlowMagnitudePhysics number Скорость течения реки в метрах в секунду.
LowLODDistance number Расстояние, на котором сегменты реки переключаются на низкодетализированный рендеринг.

Данные узла реки

Пример узла:

[50, 10, 5, 22, 4, 0, 0, 1]
Индекс Значение
0 Позиция X
1 Позиция Y
2 Позиция Z
3 Ширина в метрах
4 Глубина в метрах
5 Нормаль поверхности X
6 Нормаль поверхности Y
7 Нормаль поверхности Z

Обычно нормали задаются следующим образом:

0, 0, 1

для вертикально ориентированной водной поверхности.

Генерация реки

Объект River использует управляющие узлы для построения сплайна Катмулла–Рома.

Процесс генерации приблизительно таков:

  1. Чтение узлов реки.
  2. Построение сплайна через позиции узлов.
  3. Генерация срезов вдоль сплайна.
  4. Интерполяция ширины, глубины и нормали.
  5. Построение сегментов объёма между срезами.
  6. Генерация геометрии рендеринга.
  7. Генерация границ для запросов вхождения, глубины, течения и покрытия водой.

Для генерации пригодной геометрии воды реке необходимо как минимум два узла.

SegmentLength 

"SegmentLength": 10

SegmentLength управляет длиной каждого генерируемого сегмента объёма реки.

Меньшие значения создают больше сегментов и повышают точность запросов вхождения/покрытия.

Большие значения создают меньше сегментов и могут снизить нагрузку.

Минимально допустимое значение приблизительно равно:

1 meter

SubdivideLength 

"SubdivideLength": 2.5

SubdivideLength управляет подразбиением рендеринга для высокодетализированной геометрии реки.

Меньшие значения создают более плотную геометрию и лучше передают визуальное волнение.

Большие значения создают более дешёвую геометрию.

LowLODDistance 

"LowLODDistance": 50

Рендеринг реки использует два уровня детализации:

LOD Описание
Высокий LOD Подразбитая геометрия с волнением воды.
Низкий LOD Более простая полосовая геометрия для дальних от камеры участков.

Сегменты ближе LowLODDistance используют высокодетализированный рендеринг.

Сегменты дальше используют низкодетализированный рендеринг.

FlowMagnitudePhysics 

"FlowMagnitudePhysics": 3

Управляет скоростью течения реки в метрах в секунду.

Направление течения определяется направлением от одного среза сегмента реки к следующему.

Порядок узлов имеет значение:

node 0 -> node 1 -> node 2

Течение воды следует этому порядку.

Если река течёт в обратную сторону, измените порядок узлов на обратный.

Ширина и глубина реки

Ширина и глубина хранятся для каждого узла и интерполируются вдоль сплайна.

Пример:

"nodes": [
  [0, 0, 5, 10, 2, 0, 0, 1],
  [50, 0, 5, 30, 5, 0, 0, 1]
]

Создаёт реку, расширяющуюся с 10 метров до 30 метров и углубляющуюся с 2 метров до 5 метров.

Ширина и глубина внутренне ограничиваются безопасными диапазонами.

Приблизительные ограничения согласно реализации:

Значение Минимум Максимум
Ширина узла 0.25 1000
Глубина узла 0.25 500

Нормали реки

Каждый узел имеет вектор нормали.

Для обычных горизонтальных рек используйте:

[0, 0, 1]

Наклонные нормали могут применяться в особых случаях, однако они влияют на ориентацию генерируемых срезов, плоскость поверхности, вхождение в объём и поведение отражений.

Для большинства рек нормали следует оставлять вертикальными.

Общее поведение воды

Водные классы разделяют общее поведение рендеринга и водного объёма.

Материалы

Водные объекты используют водные материалы.

Пример:

"material": "water_lake"

Водный материал обычно управляет:

  • Базовым цветом
  • Картами нормалей
  • Внешним видом волн
  • Внешним видом преломления/отражения
  • Поведением пены или ряби (там, где поддерживается)
  • Непрозрачностью поверхности и эффектами типа Френеля

Если материал отсутствует или недействителен, объект может отображаться некорректно или использовать резервный материал.

Отражения

Водные объекты поддерживают динамические планарные отражения через унаследованное поведение воды/отражений.

Обычно это управляется полями базового водного объекта/материала, например fullReflect, где применимо.

Планарные отражения лучше всего работают для плоской воды.

Объект Заметки о качестве отражений
WaterPlane Хорошо подходит для океанов и плоских поверхностей.
WaterBlock Хорошо подходит для плоских ограниченных поверхностей.
River Может быть менее точным для извилистых, наклонных или сильно изогнутых рек.

Для рек плоскость отражения выбирается из ближайшей зоны реки и/или усреднённых нормалей поверхности реки. Сильно изогнутые или скрученные реки могут давать артефакты отражений.

Используйте менее отражающие водные материалы для рек, которые изгибаются, наклоняются или резко поворачивают.

Проверки нахождения под водой

Водные объекты реализуют поведение под водой/глубины по-разному:

Класс Поведение под водой
WaterPlane Под водой, если точка находится ниже высоты плоскости.
WaterBlock Под водой, если точка находится внутри объёма водного блока ниже поверхности.
River Под водой, если точка находится внутри одного из генерируемых сегментов объёма реки.

Это влияет на подводный туман, покрытие водой, запросы плавучести и другие игровые/рендерные системы, запрашивающие у водных объектов глубину или вхождение точки.

Высота поверхности

Водные объекты могут сообщать высоту поверхности:

Класс Поведение высоты поверхности
WaterPlane Возвращает высоту Z плоскости.
WaterBlock Возвращает высоту Z объекта, если запрашиваемая точка XY находится внутри границ блока, иначе -1.
River Выполняет луч вниз к поверхности реки и возвращает высоту столкновения, иначе -1.

Сериализация

WaterPlane JSON 

{
  "class": "WaterPlane",
  "name": "ocean",
  "position": [0, 0, 0],
  "material": "water_ocean",
  "gridSize": 101,
  "gridElementSize": 1
}

WaterBlock JSON 

{
  "class": "WaterBlock",
  "name": "pool_water",
  "position": [0, 0, 2],
  "rotationMatrix": [1,0,0,0,1,0,0,0,1],
  "scale": [20, 10, 4],
  "material": "water_pool",
  "gridElementSize": 1
}

River JSON 

{
  "class": "River",
  "name": "small_stream",
  "material": "water_stream",
  "SegmentLength": 8,
  "SubdivideLength": 2,
  "FlowMagnitudePhysics": 1.5,
  "LowLODDistance": 40,
  "nodes": [
    [0, 0, 3, 6, 1, 0, 0, 1],
    [30, 5, 3, 7, 1.2, 0, 0, 1],
    [60, 20, 2.8, 8, 1.5, 0, 0, 1]
  ]
}

Устаревшие поля узла River

В более старой сериализации в стиле TorqueScript использовались повторяющиеся поля Node:

Node = "0 0 3 6 1 0 0 1";
Node = "30 5 3 7 1.2 0 0 1";
Node = "60 20 2.8 8 1.5 0 0 1";

В современном JSON следует использовать массив nodes.

Практические примеры

Океан вокруг острова 

{
  "class": "WaterPlane",
  "name": "level_ocean",
  "position": [0, 0, 0],
  "material": "water_ocean",
  "gridSize": 101,
  "gridElementSize": 1
}

Используйте WaterPlane для больших открытых водных пространств.

Небольшое озеро 

{
  "class": "WaterBlock",
  "name": "mountain_lake",
  "position": [250, -100, 42],
  "rotationMatrix": [1,0,0,0,1,0,0,0,1],
  "scale": [180, 120, 12],
  "material": "water_lake",
  "gridElementSize": 4
}

Используйте WaterBlock, если озеро должно иметь конечные границы.

Плавательный бассейн 

{
  "class": "WaterBlock",
  "name": "pool_water",
  "position": [10, 20, 1.2],
  "rotationMatrix": [1,0,0,0,1,0,0,0,1],
  "scale": [12, 6, 2],
  "material": "water_pool",
  "gridElementSize": 0.5
}

При необходимости используйте меньший gridElementSize для воды при крупном плане.

Река с течением 

{
  "class": "River",
  "name": "valley_river",
  "material": "water_river",
  "SegmentLength": 10,
  "SubdivideLength": 2.5,
  "FlowMagnitudePhysics": 4,
  "LowLODDistance": 60,
  "nodes": [
    [-100, 0, 15, 18, 3, 0, 0, 1],
    [-50, 20, 14, 20, 3, 0, 0, 1],
    [0, 30, 13, 22, 4, 0, 0, 1],
    [70, 10, 12, 18, 3, 0, 0, 1],
    [130, -20, 11, 16, 3, 0, 0, 1]
  ]
}

Направление течения определяется порядком узлов: от первого к последнему.

Рекомендации

  • Используйте WaterPlane только для очень большой/бесконечной воды.
  • Используйте WaterBlock для ограниченных прудов, озёр, бассейнов и водоёмов.
  • Используйте River для изогнутой воды с шириной, глубиной и течением.
  • Располагайте WaterPlane вблизи ожидаемого уровня моря и избегайте экстремальных настроек дальней плоскости отсечения.
  • Не используйте WaterPlane для небольших локальных водных зон.
  • Подбирайте разумное значение WaterBlock.gridElementSize в соответствии с размером блока.
  • Избегайте чрезмерно плотного подразбиения реки, если оно не требуется.
  • Используйте вертикальные нормали [0, 0, 1] для обычных рек.
  • Измените порядок узлов на обратный, если направление течения неверное.
  • Используйте менее отражающие материалы для извилистых или сильно изогнутых рек.
  • При необходимости разбивайте очень длинные или сложные реки на управляемые участки, если редактирование или рендеринг становятся затратными.
  • Проверьте настройку водного материала, если вода отображается чёрной, невидимой или без отражений.

Типичные проблемы

Вода не отображается

Возможные причины:

  • Отсутствующий или недействительный материал
  • Объект не добавлен на сцену
  • River имеет менее двух узлов
  • Масштаб WaterBlock слишком мал
  • Камера/проход рендеринга не включает диффузный проход воды
  • Инициализация материала завершилась ошибкой

WaterPlane отображается на неправильной высоте

Значимым является только position.z.

Проверьте:

"position": [0, 0, desiredWaterHeight]

Вращение и масштаб не влияют на WaterPlane.

WaterBlock не покрывает ожидаемую область

Проверьте:

"scale": [width, length, depth]

Также проверьте вращение и позицию объекта.

River не отображается

Убедитесь, что у него как минимум два узла:

"nodes": [
  [0, 0, 0, 10, 2, 0, 0, 1],
  [50, 0, 0, 10, 2, 0, 0, 1]
]

Также проверьте настройки материала и подразбиения.

Течение реки направлено в обратную сторону

Течение реки следует порядку узлов.

Измените порядок узлов на обратный или отредактируйте реку так, чтобы первый узел находился выше по течению, а последний — ниже.

Река выглядит слишком угловатой

Уменьшите:

"SegmentLength": 10

или уменьшите:

"SubdivideLength": 2.5

Меньшие значения создают больше генерируемых сегментов/подразбиений.

Отражения воды выглядят неправильно

Возможные причины:

  • Отражения отключены
  • Водный материал не настроен для отражений
  • Река сильно изогнута или наклонена
  • Плоскость отражения не совпадает с видимой поверхностью воды
  • Камера находится под водой или вблизи поверхности воды

Для рек уменьшите отражательность материала, если форма реки сильно скручена.

Вода вызывает артефакты на дальних расстояниях

Это в основном касается WaterPlane.

Возможные решения:

  • Увеличьте расстояние дальней плоскости отсечения
  • Держите крупный ландшафт/объекты в пределах дальней плоскости отсечения
  • Избегайте наблюдения плоскости с очень большой высоты при малом расстоянии дальней плоскости отсечения
  • Используйте WaterBlock для ограниченной воды вместо этого

Итог

BeamNG предоставляет три основных водных объекта сцены:

  • WaterPlane — для бесконечной воды океанского типа.
  • WaterBlock — для конечных прямоугольных водных объёмов.
  • River — для текущей воды на основе сплайна с переменной шириной и глубиной.

Используйте WaterPlane для океанов, WaterBlock для локальных водоёмов и River для воды, следующей по пути. Все водные объекты в значительной мере зависят от правильной настройки материала, разумных параметров геометрии и корректной конфигурации отражений.

Ниже приведены несколько примеров материалов, которые помогут вам создать материалы для собственных проектов.

В любом случае не стесняйтесь использовать стандартный контент в качестве образца, а также применять готовые текстуры и материалы там, где это уместно. В общей папке игры есть набор предопределённых материалов для таких элементов, как решётки радиатора, обычные фонари и т. п.

Имейте в виду, что перечислить все возможные материалы, которые вам могут понадобиться, невозможно, однако эти примеры помогут вам начать работу. Не бойтесь экспериментировать с редактором материалов, доступным через меню F11: это отличный способ в реальном времени увидеть эффект от различных настроек.

При создании материалов рекомендуется использовать файлы png в соответствии со схемой именования, описанной на странице Texture Cooker . После завершения редактирования текстур и проверки того, что всё выглядит так, как нужно, скопируйте файлы dds и удалите файлы png из папки мода. Этот шаг поможет сократить время загрузки до минимума: в противном случае игра будет вынуждена конвертировать текстуры в dds каждый раз при первом запуске вашего мода у пользователя.

Есть 2 способа редактировать материалы: вручную в текстовом редакторе (рекомендуется Visual Studio Code для подсветки синтаксиса json) или в редакторе материалов игры. Редактор доступен через Мировой редактор (клавиша F11 по умолчанию): откройте вкладку Window и нажмите Material Editor. Редактор предлагает систему динамического предпросмотра и более читаемые параметры, которые автоматически преобразуются в код Json при сохранении материала. Результат будет сохранён в пользовательской папке, в папке транспортного средства, которому принадлежит данный материал.

Header

Начнём с параметров, которые используются всегда.

Можно начать с создания нового файла main.materials.json, вставить в него пару фигурных скобок и поместить внутрь следующее:

"vivace_main": {
  "name": "vivace_main",
  "mapTo": "vivace",
  "class": "Material",
  "Stages": [
      {},
      {},
      {},
      {}
  ],
  "dynamicCubemap": true,
  "materialTag0": "beamng",
  "materialTag1": "vehicle",
  "version": 1.5
},

Вот как эти параметры выглядят в редакторе материалов:

Первая строка вместе с именем должна быть уникальной для данного материала. Её редактирование в редакторе материалов потребует перезагрузки игры, поэтому лучше всего сделать это вручную до входа в редактор.

MapTo определяет, какой материал в Blender применяется к данному материалу. Для наглядности обычно это то же самое, что и имя материала, за исключением основного материала транспортного средства, у которого суффикс _main присутствует в имени, но отсутствует в mapTo.

Параметр Class следует оставить без изменений.

Stages — это место, куда вы будете вносить информацию о различных используемых текстурах; в редакторе этот раздел отображается как Layers.

Dynamic cubemap означает, что данный материал будет использовать динамические отражения. Как правило, этот параметр оставляют включённым. В редакторе материалов он управляется настройкой Reflection Mode.

Параметр Material tags можно оставить без изменений.

Версия 1.5 означает, что данный материал является PBR-материалом. Не откатывайте его в Мировом редакторе, иначе потеряете информацию о текстурах, не поддерживавшихся в старых версиях.

Main vehicle material

Основной материал любого автомобиля должен быть в основном перекрашиваемым, чтобы игрок мог изменить цвет краски кузова.

На стандартных транспортных средствах этот материал нередко включает и некоторые механические компоненты, которые не перекрашиваются. Разделение между перекрашиваемыми и неперекрашиваемыми участками выполняется с помощью 2 слоёв. В первом слое находятся все неперекрашиваемые части материала, во втором — все перекрашиваемые.

Параметр "activeLayers" указывает, сколько объявленных слоёв будет использоваться; его следует установить равным количеству слоёв в вашем материале, если их больше 1. В противном случае параметр следует удалить — тогда его значение по умолчанию станет равным 1.

Если вы создаёте скины, имейте в виду, что у большинства стандартных транспортных средств на кузове есть 2 разные UV-карты. Первая зеркальная и используется для большинства компонентов материала — она помогает экономить UV-пространство для небольших деталей. Вторая не зеркальная и включает только поверхности кузова. Именно эту UV-карту вы обычно будете использовать при создании скинов.

Если вы создаёте материалы для собственного автомобиля и все поверхности основного материала перекрашиваемы, можно использовать содержимое второго слоя в качестве первого. Просто убедитесь, что параметр "activeLayers" удалён.

  "vivace_main": {
    "name": "vivace_main",
    "mapTo": "vivace",
    "class": "Material",
    "persistentId": "c149d182-981e-4864-8d0d-6449b0372ece",
    "Stages": [
      {
        "ambientOcclusionMap": "/vehicles/vivace/vivace_main_ao.data.png",
        "baseColorMap": "/vehicles/vivace/vivace_main_b.color.png",
        "metallicMap": "/vehicles/vivace/vivace_main_m.data.png",
        "metallicFactor": 1,
        "normalMap": "vehicles/vivace/vivace_n.dds",

        "useAnisotropic": true
      },

PersistentId

PersistentId — это постоянный идентификатор блока данных в игровом движке; всю эту строку следует удалить при ручном редактировании материала. При использовании редактора материалов новый идентификатор генерируется автоматически и эта строка добавляется за вас.

First layer

Первый слой практически идентичен стандартному материалу. Подробнее см. соответствующий раздел.

Second layer

Второй слой — хороший пример полностью перекрашиваемого материала.

{
  "ambientOcclusionMap": "/vehicles/vivace/vivace_main_ao.data.png",
  "instanceDiffuse": true,
  "useAnisotropic": true,
  "colorPaletteMapUseUV": 1,
  "clearCoatFactor": 1,
  "clearCoatRoughnessFactor": 1,
  "metallicFactor": 1,
  "clearCoatMap": "/vehicles/vivace/vivace_main_cc.data.png",
  "normalMap": "/vehicles/vivace/vivace_n.dds",
  "opacityMap": "/vehicles/vivace/vivace_main_c.data.png",
  "colorPaletteMap": "/vehicles/common/nullcolormaskR.dds",
}

В редакторе материалов этот слой разделён на базовые и расширенные свойства. Обратите внимание: из-за сжатия DDS-файлов у оттенков серого каналы зелёного и синего цвета удаляются и текстуры отображаются красными в целях оптимизации; аналогично у карт нормалей отсутствует синий канал и они выглядят жёлтыми. При работе с png-файлами не следует удалять эти каналы вручную — это будет сделано Texture Cooker.

ambientOcclusionMap : Это обычная чёрно-белая карта окружающего затенения. Она будет такой же, как и в первом слое.

instanceDiffuse : В редакторе материалов также называется «Instance BaseColor» — означает, что данный материал будет реагировать на выбор цвета.

useAnisotropic : При значении true данный слой будет использовать анизотропную фильтрацию. Для большинства материалов это значение обычно оставляют без изменений.

colorPaletteMapUseUV : Значение 1 означает, что эта текстура привязана ко второй, незеркальной UV-карте. Это очень распространённая настройка для скинов; здесь она полезна, если вы хотите, чтобы краска по умолчанию была двухтонной. В редакторе управляется через список выбора.

clearCoatFactor : Умножает значения цвета на clearCoatMap. При значении 1 игрок может использовать ползунок лакового покрытия во вкладке покраски транспортного средства; при значении 0 ползунок недоступен. Промежуточные значения делают ползунок частично эффективным. Значение не должно быть выше 1. В редакторе материалов управляется ползунком Factor рядом с картой лакового покрытия.

clearCoatRoughnessFactor : Работает аналогично clearCoatFactor, но применительно к шероховатости лакового покрытия. Поскольку карта шероховатости лакового покрытия в данном случае не задана, по умолчанию выбирается полностью белая текстура, что делает это значение коэффициентом шероховатости лакового покрытия. В редакторе материалов управляется ползунком Factor Roughness рядом с картой лакового покрытия.

metallicFactor : Работает аналогично вышеописанным параметрам, но для эффекта металлика. Важно установить значение 1 в материале, так как значение по умолчанию равно 0, из-за чего ползунок металлической краски становится недоступным.

clearCoatMap: Карта в оттенках серого, определяющая, какие части текстуры могут получить эффект лакового покрытия. Все внешние окрашиваемые части автомобиля следует обозначить белым, остальные окрашиваемые части — различными оттенками серого в зависимости от желаемого максимального уровня лакового покрытия, а остальное — чёрным. Не используйте этот параметр за пределами области покраски транспортного средства, так как он требователен к производительности.

normalMap : Карта нормалей кузова. Поскольку она не содержит специфической информации, её можно оставить такой же, как и в первом слое.

opacityMap : Карта в оттенках серого, преимущественно чёрно-белая, определяющая, какие части текстур данного слоя будут видимы, а какие нет. В данном случае она фактически определяет, какие части автомобиля будут перекрашиваться. Эти части следует обозначить полностью белым, остальные — полностью чёрным. В редких случаях, когда нужен плавный переход, некоторые части можно обозначить различными оттенками серого.

colorPaletteMap : Определяет, какая цветовая палитра будет использоваться для каждой перекрашиваемой части транспортного средства. Красный цвет — первая палитра, зелёный — вторая, синий — третья. В большинстве случаев вы захотите, чтобы эта карта была полностью красной (оставьте ту, что задана по умолчанию), но если вы хотите, чтобы заводская двухтонная опция была выбрана по умолчанию без необходимости выбирать скин, можно обозначить некоторые части полностью зелёным. Имейте в виду, что здесь используется вторая незеркальная UV-карта.

Other materials

Single color material

Одноцветный материал, хоть и не самый красивый, способен справиться с задачей для начинающих и может быть задан очень простым способом с использованием модификаторов.

Имейте в виду, что такие материалы будут выглядеть несколько плоскими, однако они представляют собой простой способ добавить цвет в первый проект.

Схожую логику можно применить для некоторых конкретных параметров других примеров материалов, когда для данного применения подходит однородный цвет или карта шероховатости.

"example_single_color_material" : {
  "name" : "example_single_color_material",
  "mapTo" : "example_single_color_material",
  "class" : "Material",
  "persistentId" : "7d9f2774-c0f3-4440-ba16-de7086669244",
  "Stages" : [
    {
      "baseColorFactor" : [ 0.11, 0.16, 0.20, 2.0 ],
      "roughnessFactor": 0.4,
      "useAnisotropic": true
    },
    {},
    {},
    {}
  ],
  "activeLayers": 1,
  "dynamicCubemap": true,
  "materialTag0": "beamng",
  "materialTag1": "vehicle",
  "order_simset": 0,
  "version": 1.5
},

baseColorFactor : умножает значение в baseColorMap на набор чисел — по одному для каждого канала RGBA. Управляется значениями Color рядом с картой базового цвета в редакторе материалов. Поскольку в данном случае baseColorMap не задана, по умолчанию используется чисто белый цвет, а числа выступают непосредственно в качестве значений RGBA, каждое из которых определяет коэффициент своего канала.

roughnessFactor : Умножает значения цвета на roughnessMap. Поскольку roughnessMap не задана, по умолчанию используется полностью белый цвет, поэтому это число выступает как коэффициент шероховатости. Аналогичным образом можно использовать metallicMap, clearCoatMap и clearCoatRoughnessMap. Все они управляются ползунками Factor в редакторе.

Standard material

Стандартные материалы включают всё, что является непрозрачным и неперекрашиваемым. Такие материалы создаются с использованием одного слоя и используют текстуры для всех или большинства карт текстур. Первый слой кузова автомобиля выполнен очень схожим образом.

"vivace_mechanical": {
  "name": "vivace_mechanical",
  "mapTo": "vivace_mechanical",
  "class": "Material",
  "Stages": [
    {
      "ambientOcclusionMap": "/vehicles/vivace/vivace_mechanical_ao.data.png",
      "baseColorMap": "/vehicles/vivace/vivace_mechanical_b.color.png",
      "metallicFactor": 1,
      "metallicMap": "/vehicles/vivace/vivace_mechanical_m.data.png",
      "normalMap": "vehicles/vivace/vivace_mechanical_nm.normal.png",
      "roughnessMap": "/vehicles/vivace/vivace_mechanical_r.data.png",
      "useAnisotropic": true
    },
    {},
    {},
    {}
  ],
  "dynamicCubemap": true,
  "materialTag0": "beamng",
  "materialTag1": "vehicle",
  "order_simset": 0,
  "version": 1.5
},

Вот как слой вышеуказанного материала выглядит в редакторе материалов:

ambientOcclusionMap : Как и в перекрашиваемом материале, это текстура в оттенках серого, управляющая окружающим затенением материала.

baseColorMap : В литературе часто называется «Albedo» — эта карта просто задаёт цвета объекта. Имейте в виду, что тени включать не следует, поэтому использовать необработанную фотографию реальной детали транспортного средства здесь невозможно.

metallicFactor : Умножает значения яркости, заданные metallicMap. Важно установить значение 1 при использовании metallicMap, так как значение по умолчанию равно 0, что обнуляет карту.

metallicMap : Чёрно-белая карта, определяющая, какие части объекта будут реагировать на свет как металлическая поверхность. В большинстве случаев не следует использовать оттенки серого, так как поверхность не может быть частично металлической в обычных условиях.

roughnessMap : Карта в оттенках серого, определяющая степень рассеивания света различными частями поверхности.

useAnisotropic : При значении true данный слой будет использовать анизотропную фильтрацию. Для большинства материалов это значение обычно оставляют без изменений.

Transparent materials

Прозрачные материалы в основном используются для таких элементов, как стёкла и линзы фонарей. У них есть уникальные свойства, отсутствующие у обычных материалов.

"etkc_glass": {
  "name": "etkc_glass",
  "mapTo": "etkc_glass",
  "class": "Material",
  "Stages": [
    {
      "ambientOcclusionMap": "/vehicles/etkc/etkc_glass_ao.data.png",
      "baseColorMap": "vehicles/etkc/etkc_glass_b.color.png",
      "metallicFactor": 1,
      "metallicMap": "/vehicles/etkc/etkc_glass_m.data.png",
      "normalMap": "/vehicles/etkc/etkc_glass_nm.normal.png",
      "opacityMap": "/vehicles/etkc/etkc_glass_o.data.png",
      "pixelSpecular": true,
      "roughnessFactor": 1,
      "roughnessMap": "/vehicles/etkc/etkc_glass_r.data.png"
    },
    {},
    {},
    {}
  ],
  "alphaRef": 0,
  "castShadows": false,
  "dynamicCubemap": true,
  "materialTag0": "beamng",
  "materialTag1": "vehicle",
  "translucent": true,
  "translucentBlendOp": "PreMulAlpha",
  "translucentRecvShadows": true,
  "version": 1.5
},

Эти свойства находятся на вкладке "Advanced - All Layers" в редакторе материалов.

alphaRef : Также известен как Alpha Clip Threshold. Определяет, когда отображать объекты, перекрытые другими прозрачными объектами. Для стёкол транспортного средства следует оставить значение 0.

castShadows : Определяет, должен ли объект отбрасывать тени на другие объекты; для прозрачных материалов следует установить значение false.

translucent: Определяет, будет ли материал просвечивающим. При значении true области, не закрытые полностью белым цветом на картах непрозрачности данного материала, будут полностью или частично прозрачными в зависимости от совокупной яркости пикселей карт opacityMap. Проще говоря, чем темнее участок на нашей opacityMap, тем он прозрачнее.

translucentBlendOp : Также известен как «Alpha Blend Mode» — определяет, как несколько просвечивающих объектов будут отображаться поверх друг друга. Лучшее значение для стёкол транспортного средства — "PreMulAlpha".

translucentRecvShadows : При значении true прозрачные объекты будут получать тени от других объектов, несмотря на то что сами тени не отбрасывают.

Параметры pixelSpecular и order_simset следует оставить без изменений.

Многие моддеры забывают о том, что внутренняя сторона стекла должна иметь материал, отличный от внешней, с отключёнными отражениями. Это связано с тем, что транспортное средство не может отражать собственные части, поэтому внутренняя сторона стекла отражала бы дорогу и окружение. Это можно сделать, убрав roughnessMap, что установит её значение по умолчанию в полностью белый цвет и тем самым отключит отражения.

"etkc_glass_int": {
  "name": "etkc_glass_int",
  "mapTo": "etkc_glass_int",
  "class": "Material",
  "persistentId": "00181ae3-4f76-4c7b-947f-5e78687b5f3f",
  "Stages": [
    {
      "ambientOcclusionMap": "/vehicles/etkc/etkc_glass_ao.data.png",
      "baseColorMap": "vehicles/etkc/etkc_glass_b.color.png",
      "metallicFactor": 1,
      "metallicMap": "/vehicles/etkc/etkc_glass_m.data.png",

Emissive materials

Светящиеся материалы используются для таких элементов, как фонари и приборные панели, чтобы они выглядели включёнными при работающих огнях. Варианты в выключенном и повреждённом состоянии вместо этого должны использовать стандартные материалы.

  "name": "bastion_lights_on",
  "mapTo": "bastion_lights_on",
  "class": "Material",
  "persistentId":"5e194b74-4312-436d-8815-0fb83c22bd20",
  "Stages": [
    {
    "ambientOcclusionMap": "/vehicles/bastion/bastion_lights_ao.data.png",

      "emissiveFactor": [
        5.0980401,
        5.0980401,
        5.0980401
      ],
      "emissiveMap": "/vehicles/bastion/bastion_lights_g.color.png",
      "metallicFactor": 1,
      "metallicMap": "/vehicles/bastion/bastion_lights_m.data.png",
      "normalMap": "vehicles/bastion/bastion_lights_nm.normal.png",
      "pixelSpecular": true,
      "roughnessMap": "/vehicles/bastion/bastion_lights_r.data.png"

В редакторе материалов эти настройки находятся в расширенных свойствах того слоя, с которым вы работаете.

emissiveFactor : Таблица коэффициентов RGB, умножающих значения в emissiveMap. Значения выше 1 добавляют дополнительную энергию светящемуся материалу, что в некоторых случаях создаёт ослепляющий эффект, известный как Bloom. Bloom появляется, когда энергия превышает пороговое значение по умолчанию; использование больших чисел не усилит Bloom, но облегчит преодоление порога, что в некоторых условиях освещения может сделать материал нереалистичным. В редакторе материалов это значения Factor рядом с картой эмиссии в формате RGB.

emissiveMap : Управляет цветом света, излучаемого частями материала. Полностью чёрный цвет означает отсутствие эмиссии. Может использовать текстуры BC6H, позволяя задавать значения RGB выше 255. Текстура тогда будет содержать значения энергии для отдельных пикселей, что может дать лучший эффект, чем использование emissiveFactor.

Фонари транспортного средства нередко имеют 2 активных состояния: on и on_intense. У последнего значение emissiveFactor должно быть немного выше, чтобы оно светилось ярче.

Decals

Декали (decals) — это очень простые меши с частично прозрачным фоном. Обычно они используются для таких элементов, как эмблемы, что позволяет легко добавлять и убирать их без необходимости сложной геометрии.

Они создаются с использованием одного слоя. Имейте в виду, что предпочтительно размещать все варианты декали на одной текстуре с UV-привязкой каждого к нужной области — это упрощает определение материала.

Dummy materials

Dummy-материалы — это «пустые» определения материалов, используемые для материалов, которым назначены glowMaps . Поскольку glowMap задаёт материалы, которые всегда будут заменять данный, основному материалу не нужно назначать никаких текстур, и обычно он остаётся максимально простым для уменьшения размера файла материалов (это может существенно снизить размер, если в разделе glowMap задано множество материалов).

"bastion_chmsl": {
  "name": "bastion_chmsl",
  "mapTo": "bastion_chmsl",
  "class": "Material",
  "Stages": [{}, {}, {}, {}]
},

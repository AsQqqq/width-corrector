Для создания собственных скинов вам понадобится несколько вещей:

  • Шаблон скина. Все стандартные автомобили должны содержать в своей папке файл PNG с именем вроде «skin_UVs». Этот файл используется как основа для рисования скина.
  • Графический редактор. Хотя скины можно создавать в MS Paint, настоятельно рекомендуется освоить программу с поддержкой слоёв. GIMP и Paint.Net — отличные бесплатные варианты.
  • Шаблонные файлы материалов и jbeam. В качестве основы для материалов следует взять файл main.materials.json автомобиля из его zip-архива. Для файла jbeam можно воспользоваться этим шаблоном

Данное руководство включает примеры для 3 наиболее распространённых сценариев работы со скинами транспортных средств:

  • Полностью перекрашиваемые скины, цвет которых полностью регулируется с помощью палитры цветов.
  • Частично перекрашиваемые скины, у которых большая часть скина настраивается через палитру, но некоторые элементы — например, нашивки и логотипы — имеют фиксированный цвет.
  • Неперекрашиваемые скины, на которые палитра цветов не влияет.

Введение в UV-развёртки и систему слоёв.

Одна из особенностей большинства стандартных автомобилей заключается в том, что они имеют 2 различные UV-карты и 2 отдельных слоя в основном материале.

2 слоя состоят из базового слоя, содержащего все элементы транспортного средства, и второго слоя только с окрашиваемыми областями кузова вместе с картой прозрачности.

При работе в редакторе материалов следует редактировать только второй слой, оставляя первый без изменений.

Что касается UV-карт, каждый стандартный автомобиль настроен с 2-мя картами. Первая UV-карта, называемая UV map 0, является зеркальной и предназначена для оптимального использования UV-пространства для механических компонентов, деталей нормалей и прочего.

Именно поэтому у всех стандартных автомобилей есть вторая UV-карта, известная как UV map 1, которая не является зеркальной и специально предназначена для создания схем окраски. При создании скинов всегда следует использовать именно эту UV-карту.

На этой UV-карте все различные дополнительные компоненты наложены друг на друга, чтобы все опциональные детали были совместимы с различными скинами.

Шаблон окраски для большинства стандартных автомобилей можно найти внутри их zip-файла. Это будет файл PNG с именем, обычно содержащим «skin UVs».

Если вы хотите создавать скины для модифицированного контента, проверьте, включил ли автор мода аналогичный шаблон в свои файлы. Имейте в виду, что некоторые моды не имеют UV-развёртки, допускающей создание скинов. Если мод включает скины, но не содержит шаблона, вы можете попросить автора мода добавить его.

Настройка файла jbeam

После копирования файла jbeam обязательно переименуйте его в уникальное имя, чтобы он не перезаписал существующие файлы.

В файле jbeam необходимо выполнить следующие изменения:

  • Присвойте компоненту уникальное имя.
  • Измените информационные параметры по своему усмотрению. Имя будет отображаться в селекторе деталей.
  • Измените значение globalSkin на другое уникальное значение, поскольку оно также будет использоваться в файле материалов.
{
"tutorialSkin": {
    "information":{
        "authors":"BeamNG",
        "name":"Tutorial Skin",
        "value":1000
    },
    "slotType" : "paint_design",
    "globalSkin" : "TutorialSkin"
  },
}

Если вам нужно создать несколько скинов, можно скопировать деталь несколько раз, убедившись, что значения name и globalSkin всегда уникальны. Как и в любом jbeam-файле, проверьте корректность скобок. Файл всегда должен начинаться и заканчиваться скобкой. Вот пример с несколькими скинами.

{

        "authors":"BeamNG",

    "globalSkin" : "TutorialSkin"
  },

        "name":"Tutorial Skin 2",
        "value":1000
    },
    "slotType" : "paint_design",
    "globalSkin" : "TutorialSkin2"
  },
}

Редактор материалов

Редактор материалов используется для всех остальных изменений, необходимых для работы скинов, поскольку он значительно удобнее ручного редактирования JSON.

Чтобы редактировать скины в редакторе материалов, скин сначала должен быть применён к автомобилю — это значит, что нужно сначала выбрать деталь скина в селекторе деталей.

Затем можно открыть редактор материалов внутри редактора, доступного по клавише F11.

Перейдите в «Windows»->«Material Editor». Появится новое окно со списком материалов. Нажмите на автомобиль в мировом редакторе, чтобы получить список применяемых к нему материалов.

Затем можно выбрать материал скина в списке, чтобы увидеть его свойства. Если материал не отображается, проверьте синтаксис файла материалов и перезапустите игру.

После сохранения файла он должен появиться в папке /vehicles/VEHICLENAME/.

Конвертер текстур (Texture Cooker)

Конвертер текстур — это инструмент, преобразующий файлы .png в соответствующие файлы .dds.

Поскольку не всё программное обеспечение (особенно бесплатное) совместимо с некоторыми новыми типами сжатия .dds, данный рабочий процесс настоятельно рекомендуется и именно на нём основано это руководство.

Для получения дополнительной информации ознакомьтесь со страницей конвертера текстур .

Важнейший аспект при использовании конвертера текстур — правильное именование текстур.

В нашем случае мы будем создавать только полноцветные изображения, которые должны оканчиваться на .color.png.

Например: «CustomRacingSkin.color.png».


Полностью перекрашиваемый скин

Полностью перекрашиваемые скины — наиболее распространённые и наиболее простые. Они работают за счёт определения новой colorPalette, в которой различные цветовые каналы используются для создания ливреи.

Каждый ползунок в палитре цветов соответствует RGB-каналу на карте цветовой палитры: красный канал соответствует первому ползунку, зелёный — второму, синий — последнему. Это означает, что каждый пиксель будет перекрашиваться в зависимости от своего цвета.

Неперекрашиваемые скины

В эту категорию входят, например, машины экстренных служб, у которых все цвета ливреи фиксированы.

Такие скины используют только baseColorMap и не будут иметь регулируемых параметров roughness, metallicity или clearCoat в палитре цветов, поэтому их, возможно, потребуется настроить в файле материалов.

Подготовка мода к публикации

После завершения всех изменений в материалах и проверки корректности работы пришло время выполнить финальные шаги для подготовки скин-мода к публикации.

Первый шаг — если вы ещё не сделали этого — скопировать изменённый файл материалов.

Затем необходимо скопировать файлы .dds, созданные конвертером текстур. Их можно найти в папке temp. Просто скопируйте их в папку вашего мода.

После копирования файлов .dds необходимо удалить файлы png из папки мода.

Для этого проверьте, у каких файлов png есть соответствующий файл .dds с совпадающим именем.

Также следует удалить файлы, не нужные для работы мода, — например, файлы проектов графического редактора и справочные изображения.

Хорошей практикой является хранение копии исходных файлов png в отдельном месте на случай, если вы захотите продолжить работу над модом в будущем.

Последний шаг — упаковать мод в zip-файл. Для этого в игре перейдите в «Repository» -> «Mod Manager». Выберите мод в списке и нажмите «Pack».

Теперь в папке «mods» должен появиться zip-файл вашего скин-мода, которым можно поделиться с друзьями или с сообществом через репозиторий.

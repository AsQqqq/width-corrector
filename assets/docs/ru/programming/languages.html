При программировании модов для BeamNG вы, скорее всего, будете использовать следующие языки:

  • Lua: основной язык программирования, как правило применяется для вычисления максимально возможного количества логики. Например, игровая логика, различное поведение транспортных средств, части физики и т.д.
  • Javascript/Html: используется для отображения информации в пользовательском интерфейсе. Как правило, мы рекомендуем применять его только для отображения в UI, оставляя как можно больше логики на стороне Lua. Сторона Lua затем передаёт результаты всех вычислений на сторону UI, где Javascript/Html лишь отображает их. Это обычно помогает сделать ваш код более надёжным.
  • LUA Extensions : это модульная система BeamNG. Она включает сериализацию, события и т.д.
  • Virtual Machines : BeamNG запускает несколько независимых систем Lua параллельно, а также несколько экземпляров UI, которые мы называем VM.
  • Virtual Machines Queues : основная система BeamNG для обмена данными между виртуальными машинами.
  • Virtual Machines Mailboxes `: вторичная система BeamNG для обмена данными между виртуальными машинами VLUA.

Соглашения о написании кода в BeamNG

Стиль

  • Пример именования папок: ge/someFolderHere/lowerCamelCase/
  • Пример именования файлов: lowerCamelCase.lua
  • Отступы: 2 пробела (не табуляция)
  • Удаление пробелов в конце строк: включено
  • Имена функций и переменных: camelCase
  • Имена «классов»: верхний camelCase (PascalCase), пример: MyCoolClass

Основы языка Lua

Хотя вам следует самостоятельно изучить программирование и язык Lua, здесь приведены самые основные сведения о языке:

Базовая терминология

  • Table: базовый и единственный тип контейнера в Lua. В зависимости от способа использования таблицы, она будет напоминать массив (список) или словарь (отображение).
  • Array: таблица Lua, содержащая только целочисленные ключи от 1 до бесконечности. Можно корректно подсчитать с помощью #tbl
  • Dictionary (dict): таблица Lua, содержащая ключи любых типов. Нельзя подсчитать с помощью #tbl.
  • Module: традиционный module Lua является устаревшим. Не используйте его
  • Package: стандартный пакет Lua . Используется, но при возможности отдавайте предпочтение расширениям (Extensions). Смотрите здесь и также здесь .

Синтаксический сахар

Для удобства язык Lua иногда предлагает различные способы написания идентично работающего кода. Например:

  • myTable.ident эквивалентно myTable['ident']
  • myObject:name(args) является синтаксическим сахаром для myObject.name(myObject, args)
  • myFunction{fields} является синтаксическим сахаром для myFunction({fields})
  • myFunction'string' (или myFunction"string", или myFunction[[string]]) является синтаксическим сахаром для myFunction('string')
  • function t.a.b.c:foobar(params) body end является синтаксическим сахаром для t.a.b.c.foobar = function(self, params) body end

Существует несколько способов отладки написанного вами кода.

Отладка кода Lua через консоль:

Как и в большинстве языков, это наиболее базовый способ отладки: а) добавить вывод в консоль, б) перезагрузить код, в) просмотреть результаты в консоли.

а) Добавьте запись в лог

Для вывода в консоль можно использовать:

  • log("E", "context", "My message: "..dumps(myContext)): создаёт сообщение в логе с уровнем серьёзности «Ошибка» (используйте E для ошибки, W для предупреждения или I для информации).
    • Примечание: контекст можно оставить пустым, если он не нужен, например: log("I", "", "My message")
  • dump("My message", myContext): создаёт запись в логе с наивысшим уровнем серьёзности. Преобразует все переданные объекты в строку с помощью вызовов dumps().
  • print("My message: "..dumps(myContext)): создаёт сообщение в логе с наивысшим уровнем серьёзности.

Примечание: как правило, все сообщения лога, которые вы планируете публиковать вместе с модом, должны содержать хотя бы одну переменную для обеспечения надлежащего контекста. Если переменной нет, а лог представляет собой просто жёстко заданную строку, существует высокая вероятность того, что одного лога окажется недостаточно при решении проблем в будущем.

б) Перезагрузите код

Для перезагрузки Lua VM см. перезагрузка виртуальных машин .

В редких случаях может потребоваться полностью завершить работу программы и снова запустить её с нуля. Это может произойти, если ранее вы запускали ошибочный код, который привёл всю систему перезагрузки в нефункциональное, неисправимое состояние. Наша система перезагрузки делает всё возможное, чтобы пережить возможные ошибки программирования — однако её возможности не безграничны.

в) Проверьте консоль

Для открытия консоли нажмите клавишу ~ (тильда) на клавиатурах с раскладкой US.

Примечание: для других раскладок клавиатуры проверьте, какая клавиша назначена действию Toggle System Console в меню Options > Controls > Bindings, в разделе General Debug

Описание

Расширения BeamNG на Lua — это простой и масштабируемый способ расширить функциональность. Расширения BeamNG позволяют объединять несколько фрагментов кода без необходимости изменять официальные файлы (что, скорее всего, создавало бы проблемы после каждого официального обновления и делало бы ваш мод неприемлемым в официальном репозитории модов BeamNG ).

Например: вы можете написать мод, который реагирует на использование системы замедленного воспроизведения игроком, не редактируя официальные исходные файлы этой системы и не вставляя явные вызовы вашего мода.

Это достигается с помощью функциональности «хуков», при которой расширения могут подписываться на различные события и сами вызывать события.

Покажи мне код

Расширение BeamNG на Lua — это по сути файл Lua, возвращающий таблицу. Например:

  local M = {}
  M.myData = 10
  M.myFunction = function() print("hello world") end
  return M

После упаковки мода в zip-файл файлы расширений должны располагаться где-либо в:

  (someMod.zip) /lua/common/extensions/
  (someMod.zip) /lua/ge/extensions/
  (someMod.zip) /lua/vehicle/extensions/

Ваш выбор определит, в какой виртуальной машине Lua можно использовать расширение BeamNG. Дополнительный контекст см. в разделе расположение исходного кода .

Использование расширения

Созданное вами расширение не будет загружено до тех пор, пока вы явно не напишете код для его загрузки. Например:

  extensions.load("myMod_myExtension")       -- this loads extensions/myMod/myExtension.lua

После загрузки расширения BeamNG вы можете получить доступ к его таблице:

  myMod_myExtension.myFunction()
  myMod_myExtension.myData = 20

Вы также можете выполнить функцию во всех расширениях, загруженных в данный момент в этой виртуальной машине Lua:

  extensions.hook("myCustomEvent")        -- this will call myMod_myExtension.myCustomEvent(), and it will myCustomEvent() in all other loaded extensions

Когда расширение больше не нужно, его необходимо выгрузить:

  extensions.unload(...)

Держать расширения загруженными без необходимости — значит расходовать ресурсы компьютера и снижать частоту кадров, поэтому избегайте этого.

Примечание 1: Существует синтаксис вида extensions.myMod_myExtension.foo(). Избегайте его, потому что а) он снижает частоту кадров по сравнению с myMod_myExtension.foo(), и… б) он автоматически загружает расширение, если оно не было загружено. Как правило, вам не следует загружать расширение автоматически — делайте это явно.

Примечание 2: Ваше расширение может явно объявить, от каких других расширений оно зависит. Это делается следующим образом:

  M.dependencies = { "foo_bar", "baz_qux", ...}

Это сообщит системе расширений BeamNG, что она должна загружать/выгружать эти зависимости за вас. Система также упорядочит вызовы хук-функций так, чтобы они следовали дереву зависимостей в соответствующем порядке. Иными словами, хук сначала выполнится в зависимостях, затем в вашем собственном расширении, и так далее.

Общие функции/данные расширений

Помимо ваших собственных специфических членов «myFunction», «myData», ваша таблица может содержать более общие члены.

Например:

  M.onExtensionLoaded = function() ... end   -- called when the extension is loaded
  M.onUpdate = function() ... end            -- called once per GFX frame
  M.onGuiUpdate = function() ... end         -- called once per UI frame
  M.state = { foo=1, bar=2, ... }            -- may be used to save+load extension state during a reload of the Lua virtual machine
  M.onSerialize = function() ... end         -- may be used to save+load extension state during a reload of the Lua virtual machine using a custom serializing function
  M.dependencies = { "core_camera", ... }    -- may be used to automatically load other extensions
  etc

Централизованного списка с этими специальными функциями и переменными не существует; исходный код И ЕСТЬ документация (tm). Например, вы можете изучить код внутри lua/ge/main.lua, lua/common/extensions.lua и т. д. Посмотрите, кто вызывает хуки расширений, или обратите внимание на то, как написаны другие расширения, и т. д.

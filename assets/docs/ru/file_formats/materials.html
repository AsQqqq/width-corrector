Этот документ описывает JSON-файлы материалов BeamNG.

JSON-файлы материалов определяют объекты Material, используемые уровнями, транспортными средствами, общими ресурсами, декалями, дорогами, водой, скайбоксами, пропсами и другим отрисовываемым контентом.

Концепции создания материалов описаны в разделе Materials . Описание окна World Editor см. в разделе Material Editor .

Имена файлов

Типичные файлы материалов включают:

main.materials.json
skin.materials.json
*.materials.json

Примеры:

levels/<levelName>/main.materials.json
levels/<levelName>/art/shapes/buildings/main.materials.json
vehicles/<vehicleName>/main.materials.json
vehicles/<vehicleName>/skin.materials.json
art/shapes/objects/main.materials.json

JSON-файлы материалов загружаются из путей игры, уровня, транспортного средства и модов в зависимости от загружаемого контента.

Базовая структура

Файл материала — это JSON-объект. Каждый ключ верхнего уровня определяет один материал.

{
  "m_concrete_wall": {
    "name": "m_concrete_wall",
    "class": "Material",
    "mapTo": "m_concrete_wall",
    "baseColorMap": ["/levels/example/art/textures/concrete_b.color.png"],
    "normalMap": ["/levels/example/art/textures/concrete_n.normal.png"],
    "roughnessMap": ["/levels/example/art/textures/concrete_r.roughness.png"],
    "metallicFactor": [0],
    "roughnessFactor": [1]
  }
}

Ключ верхнего уровня и поле name обычно совпадают.

Важные поля

Поле Тип Описание
class string Обычно "Material".
name string Имя объекта материала. Обычно совпадает с ключом верхнего уровня.
mapTo string Имя слота меша/материала, к которому привязан данный материал.
baseColorMap array[path] Путь к текстуре базового цвета для каждого слоя материала.
normalMap array[path] Путь к карте нормалей для каждого слоя материала.
roughnessMap array[path] Путь к карте шероховатости для каждого слоя материала.
metallicMap array[path] Путь к карте металличности для каждого слоя материала.
baseColorFactor array/color Множитель базового цвета там, где он задан.
roughnessFactor array[number] Скалярный коэффициент шероховатости для каждого слоя.
metallicFactor array[number] Скалярный коэффициент металличности для каждого слоя.
emissiveMap array[path] Путь к эмиссионной текстуре там, где она используется.
alphaRef number Пороговое значение альфа-отсечения там, где оно используется.
translucent bool Включает полупрозрачную отрисовку там, где используется.
useAnisotropic bool Включает анизотропную фильтрацию там, где используется.
annotation string Необязательная аннотация/классификация для отладки.

Определения материалов могут содержать множество полей, специфичных для рендерера. При разработке инструментов сохраняйте неизвестные поля.

Не удаляйте поля только потому, что они здесь не перечислены. JSON-файл материала нередко содержит поля рендерера, редактора, совместимости или сгенерированные поля.

mapTo

mapTo связывает определение материала с именем слота материала, используемым мешем или процедурной геометрией.

Пример:

"mapTo": "road_asphalt"

Для импортированных мешей значение mapTo должно совпадать с именем материала, заданным в исходном файле DAE/DTS.

Для объектов, ссылающихся на материалы напрямую, например DecalRoad.material или MeshRoad.topMaterial, поле объекта обычно ссылается на имя объекта материала.

Массивы текстур

Многие поля текстур сохраняются в виде массивов, поскольку материалы могут поддерживать несколько слоёв.

Пример:

"baseColorMap": [
  "/levels/example/art/textures/concrete_b.color.png"
]

Материал с одним слоем по-прежнему обычно использует массив с одним элементом.

Пути к материалам

Используйте пути, работающие как для обычного игрового контента, так и для запакованных модов.

Типичные пути:

/levels/<levelName>/art/textures/...
/levels/<levelName>/art/shapes/...
/vehicles/<vehicleName>/...
/art/shapes/objects/...

Соблюдайте единообразие регистра символов.

Связь с материалами террейна

Обычные определения Material отличаются от определений TerrainMaterial.

Тип Основное применение
Material Меши, декали, вода, скайбоксы, дороги, пропсы, транспортные средства и общая отрисовка.
TerrainMaterial Окраска террейна, слои текстур террейна и привязки groundmodel террейна.

Данные материалов, специфичные для террейна, описаны в разделе Terrain files .

Замечания для инструментов

Для инструментов, генерирующих или редактирующих файлы материалов:

  • Сохраняйте неизвестные поля.
  • Сохраняйте ключи материалов верхнего уровня и значения name, если только вы намеренно не переименовываете их.
  • Обновляйте mapTo только при изменении имён слотов материалов меша.
  • Оставляйте поля текстур в виде массивов, если в исходном файле они заданы массивами.
  • Проверяйте корректность указанных путей к текстурам.
  • Сохраняйте карты нормалей в ориентации, ожидаемой рендерером BeamNG.
  • Избегайте генерации дублирующихся имён материалов в загруженных файлах материалов.

Типичные проблемы

  • Меш отображает предупреждающий материал, потому что mapTo не совпадает со слотом материала меша.
  • Материал не найден, потому что файл материала сохранён вне загружаемого пути.
  • Текстура отсутствует из-за неверного пути или потому что она не включена в мод.
  • Материал выглядит слишком блестящим из-за неверных значений шероховатости или карты шероховатости.
  • Импортированный меш использует имя материала, отличающееся от определения материала в BeamNG.

Критерии валидации

  • JSON-файл материала загружается без ошибок разбора.
  • Имена материалов уникальны в пределах области загружаемого контента.
  • Значения mapTo совпадают со слотами материалов источника меша там, где это необходимо.
  • Пути к текстурам существуют и используют ожидаемые форматы.
  • Ссылающиеся объекты, такие как TSStatic, DecalRoad, MeshRoad, WaterObject, SkyBox или детали транспортного средства, отрисовываются без предупреждающих материалов.

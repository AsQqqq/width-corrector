На этой странице описаны файлы групп транспортных средств, используемые трафиком, парковкой, миссиями, инструментами редактора и другими игровыми системами.

Группа транспортных средств — это именованный пул записей модель/конфигурация. Она определяет, что может быть создано при спауне, но не то, где транспортные средства появляются и как они ведут себя на дороге. Размещение и поведение по-прежнему определяются трафиком, парковкой, миссиями, локациями и данными навигационного графа.

Структура файла

Большинство авторских групп транспортных средств используют следующую структуру:

{
  "type": "custom",
  "name": "Example Traffic Group",
  "tags": ["example", "traffic"],
  "data": [
    {
      "model": "covet",
      "config": "DXi_A"
    },
    {
      "model": "pickup",
      "config": "d15_4wd_A_facelift"
    }
  ]
}

Поля верхнего уровня

Поле Обязательно Примечания
type Рекомендуется Используйте custom для групп, созданных вручную.
name Рекомендуется Отображаемое/отладочное имя, используемое инструментами.
tags Опционально Помогает организовать группы по уровню, миссии, назначению или роли.
data Да Массив записей транспортных средств. Пустые группы являются корректным JSON, но бесполезны.

Поля записи транспортного средства

Поле Обязательно Примечания
model Да Папка/идентификатор модели транспортного средства.
config Рекомендуется Идентификатор конфигурации транспортного средства. Если отсутствует или некорректен, спаун может завершиться неудачей или использовать нежелательное значение по умолчанию в зависимости от вызывающей системы.
paintName Опционально Может использоваться некоторыми рабочими процессами редактора/трафика для задания варианта окраски.

Некоторые инструменты могут сохранять дополнительные поля для каждого транспортного средства, например, данные об окраске или метаданные спауна. Не удаляйте неизвестные поля из существующей группы, если вы не уверены, что потребляющая система в них не нуждается.

Минимальная группа из одного автомобиля:

{
  "type": "custom",
  "name": "One Car Test",
  "data": [
    {
      "model": "pickup",
      "config": "d15_4wd_A_facelift"
    }
  ]
}

Расположение файлов

Общие группы транспортных средств могут храниться по пути:

vehicleGroups/**/*.vehGroup.json

Группы, специфичные для уровня, могут храниться в корне уровня:

levels/<levelName>/*.vehGroup.json

Распространённые имена файлов для уровня:

levels/<levelName>/traffic.vehGroup.json
levels/<levelName>/police.vehGroup.json

Группы, специфичные для миссии, как правило, хранятся вместе с миссией:

gameplay/missions/<levelName>/<missionType>/<missionName>/custom.vehGroup.json

Поведение при поиске

Инструменты трафика и миссий могут загружать группы транспортных средств из разных мест в зависимости от контекста:

Контекст Типичный поиск
Трафик уровня Ищет в директории текущего уровня соответствующие файлы *.vehGroup.json, например, с именами, содержащими traffic или police.
Пользовательский трафик миссии Использует локальный файл миссии, как правило custom.vehGroup.json.
Общие/пользовательские группы инструментов При наличии поддержки обращается к файлам в директории /vehicleGroups/.

Если существует несколько совпадающих файлов, инструменты могут выбрать один из подходящего набора. Используйте понятные имена файлов и теги, если уровень или миссия содержат несколько групп.

Распространённые варианты использования

Имя файла Типичное использование
traffic.vehGroup.json Общий пул транспортных средств для фонового трафика уровня.
police.vehGroup.json Пул полицейских транспортных средств для уровня.
custom.vehGroup.json Пользовательская группа транспортных средств для конкретной миссии.

Создание групп с помощью инструментов

Простейший способ создать группу транспортных средств — воспользоваться встроенными инструментами редактора:

  1. Откройте World Editor.
  2. Откройте Traffic Manager или инструмент Vehicle Groups из редактора.
  3. Выберите модели/конфигурации транспортных средств, которые вы хотите включить в группу.
  4. Сохраните группу в файл *.vehGroup.json.
  5. Поместите файл туда, где его будет искать нужная потребляющая система.
  6. Протестируйте группу в Traffic Manager, Mission Editor или в игровом функционале, который её использует.

Traffic Manager также может загружать файл группы транспортных средств напрямую для тестирования. Если группа загружается там, но не в миссии или уровне, проблема, как правило, связана с расположением файла, именованием или настройкой потребляющей системы, а не со структурой файла.

Создание групп вручную

Файлы групп транспортных средств достаточно просты, чтобы писать их вручную:

  1. Выберите место хранения файла.
  2. Добавьте type, name и опционально tags.
  3. Добавьте массив data.
  4. Добавьте по одному объекту на каждую конфигурацию транспортного средства.
  5. Протестируйте группу в системе, которая её использует.

При редактировании вручную проверяйте идентификаторы модели/конфигурации по заведомо рабочим конфигурациям транспортных средств. Опечатка может привести к тому, что группа загрузится, но нужное транспортное средство так и не появится.

Группы трафика уровня

Для уровней наиболее распространённое соглашение таково:

traffic.vehGroup.json
police.vehGroup.json

Используйте traffic.vehGroup.json для обычного фонового трафика и police.vehGroup.json для спауна полиции. Эти файлы должны соответствовать региону, эпохе и дорожной сети уровня. Например, избегайте крупногабаритных транспортных средств на уровне с узкими дорогами, если только это не предусмотрено намеренно.

Чтобы создать группу для уровня:

  1. Создайте временную группу в Traffic Manager или инструменте Vehicle Groups.
  2. Сохраните её в корень уровня как traffic.vehGroup.json или police.vehGroup.json.
  3. Загрузите уровень и запустите трафик.
  4. Проверьте разнообразие, размеры транспортных средств и производительность.
  5. Корректируйте группу до тех пор, пока трафик не будет соответствовать уровню.

Пользовательские группы для миссий

Миссии могут использовать локальную группу, когда требуется детерминированный или ограниченный трафик ИИ. Типичный путь:

gameplay/missions/<levelName>/<missionType>/<missionName>/custom.vehGroup.json

Локальные группы миссии полезны, когда:

  • Для выполнения задания должны появляться только конкретные транспортные средства.
  • Миссии нужен тематический трафик, независимый от настроек уровня по умолчанию.
  • В тесте или обучении необходимо исключить случайный выбор транспортных средств.

Держите пользовательские группы миссий небольшими, если только миссия намеренно не предполагает разнообразия.

Чтобы создать пользовательскую группу для миссии:

  1. Откройте миссию в Mission Editor.
  2. Включите опцию пользовательской группы транспортных средств для соответствующего модуля трафика/настройки.
  3. Сохраните или выберите custom.vehGroup.json в папке миссии.
  4. Протестируйте миссию с нуля, а не только из предварительного просмотра в редакторе.

Выбор транспортных средств

Рекомендации для опытных авторов:

  • Отдавайте предпочтение конфигурациям, которые полны и надёжно появляются при спауне.
  • Учитывайте габариты, массу, скорость и тип привода транспортного средства применительно к целевым дорогам.
  • В группах трафика обеспечивайте достаточное разнообразие, чтобы избежать однообразия, но не настолько большое, чтобы редкие транспортные средства начали преобладать.
  • В группах полиции используйте конфигурации с ожидаемым полицейским оснащением и поведением.
  • Избегайте конфигураций, зависящих от модов, если только группа транспортных средств не является частью этого мода и явно не требует их наличия.
  • Если конфигурация переименована или удалена, обновите все группы, ссылающиеся на неё.

Устранение неполадок

Группа не найдена

Проверьте:

  • Файл сохранён в расположении, которое просматривает потребляющая система.
  • Имя файла соответствует ожидаемому соглашению, например traffic, police или custom.
  • Расширение файла точно соответствует .vehGroup.json.
  • Файл является корректным JSON.

Группа загружается, но транспортные средства не появляются

Проверьте:

  • Каждая модель model существует.
  • Каждая конфигурация config существует для данной модели.
  • Потребляющая система трафика или миссии действительно использует эту группу.
  • На уровне есть корректные дороги для спауна, места парковки или данные спауна миссии.

Появляются не те транспортные средства

Проверьте:

  • Выбирается другой подходящий файл группы.
  • Группа конкретной миссии переопределяет группу уровня.
  • Игровая система использует генерируемый трафик вместо пользовательской группы.

Контрольный список проверки

  • JSON-файл загружается без ошибок.
  • Каждая модель model существует.
  • Каждая конфигурация config существует для своей модели.
  • Файл группы сохранён там, где его ожидает потребляющий инструмент или система.
  • Имя группы и теги описывают предполагаемое назначение.
  • Миссия, уровень или инструмент, которому нужна группа, могут её найти.
  • Выбранные транспортные средства соответствуют дорогам, миссии и бюджету производительности.

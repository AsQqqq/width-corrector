Файл-ссылка — это json-файл с расширением .link, предназначенный для перенаправления к другому реальному пути ресурса.

Такие файлы также используются для дедупликации (устранения копий) одного и того же ресурса, а также для перенаправления устаревшего пути к новому пути ресурса — это решает проблему с модами, которые ссылаются на ресурсы по пути, который необходимо сохранить неизменным.

По поведению они примерно соответствуют симлинкам в Windows.

Внутри файла-ссылки хранится информация о пути, на который он перенаправляет, а также дополнительные сведения о ресурсе и о том, что именно эта ссылка должна представлять.

Рассмотрим пример файла-ссылки:

levels/Gridmap/art/brick.dds.link

В игре (в скриптах или редакторе) ссылка используется как обычный путь, но без расширения .link:

levels/Gridmap/art/brick.dds

VFS сначала проверяет, существует ли реальный файл .dds; если нет — проверяет, существует ли по этому пути файл с добавленным расширением .link, и если да — загружает файл-ссылку и выполняет перенаправление на новый путь.

Обычные ссылки

Диаграмма обычных файлов-ссылок

Разберём подробнее:

path - путь перенаправления для ссылки; именно этот путь/файл будет фактически загружен игрой

assetUuid - UUID ресурса, используется как запасной способ найти ресурс во всей папке data, если указанный путь не существует (не реализовано, TBD)

assetVersion - (необязательно) версия ресурса для использования (не реализовано, TBD)

type - (необязательно) тип ссылки, возможные значения:

  • normal - обычная ссылка на файл (используется для дедупликации); это тип по умолчанию, если тип не указан
  • migrated - ссылка на перемещённый файл; так разработчики модов узнают, что файл был перемещён, но могут по-прежнему использовать старый путь, хотя рекомендуется переходить на новый
  • removed - ссылка с пустым путём, означающая, что ресурс был удалён; служит заглушкой, чтобы пользователи знали об окончательном удалении ресурса

Как правило, файлы-ссылки следует применять для папок /levels; все ресурсы, которые можно разместить в общей папке /assets и которые не относятся к конкретному уровню, нужно перенести туда, а на прежнем месте оставить ссылку со старым именем файла.

В идеале уровни должны содержать только файлы-ссылки на ресурсы и специфичные для уровня файлы (папки и файлы дерева сцены, ландшафт и т. д.).

Файлы-ссылки также решают проблему, когда несколько модов переопределяют один и тот же (глобальный) ресурс. Если поместить общий ресурс, используемый, например, в 3 уровнях, в папку /assets и создать ссылки на него в каждой из папок уровней, то при переопределении этого ресурса модом для конкретного уровня ресурс на остальных уровнях останется неизменным.

Тогда на каждом уровне для загрузки ресурса будут использоваться его специфичные пути:

Например, на уровне Industrial в редакторе или lua-скриптах используется: /levels/Industrial/art/red_brick.dds

Если мод переопределит /levels/Industrial/art/red_brick.dds, он фактически заменит ссылку реальным ресурсом или даже тем же файлом-ссылкой, но с другим путём перенаправления.

При этом остальные уровни по-прежнему будут использовать неизменённый ресурс red_brick.dds из папки /assets.

Ссылки удаления ресурсов

Когда требуется удалить ресурс из официальных файлов, на его месте можно разместить файл-ссылку с тем же именем и расширением .link, задав ему тип removed; так инструменты и разработчики модов будут знать, что произошло с ресурсом, и смогут действовать соответствующим образом:

Следует учитывать, что материалы, модели и другие json-ресурсы, использующие путь /levels/Industrial/art/red_brick.dds, должны находиться внутри папки уровня, чтобы всё было локальным для этого уровня. Это необходимо потому, что если, например, материал находится в общей папке /assets и используется на нескольких уровнях, а этот материал ссылается на путь конкретного уровня, то при переопределении материала модом он изменится на всех уровнях — чего мы, возможно, не хотим.

Это потребует дублирования содержимого в папках уровней для json-ресурсов; по сути, общая папка материалов невозможна.

В идеале файлы-ссылки должны указывать преимущественно на ресурсы, не ссылающиеся на другие ресурсы, — например, текстуры, звуки, данные сырых мешей и т. д.

Моды, переопределяющие файлы, будут корректно работать с файлами-ссылками: при переопределении файла-ссылки добавляется реальный ресурс без расширения .link, который имеет приоритет в VFS над файлами-ссылками.

Файлы-ссылки поддерживают уведомления об изменении файлов: при изменении содержимого (json) файла-ссылки генерируется соответствующее уведомление. При изменении ресурса, на который указывает файл-ссылка, он добавляется в список уведомлений об изменении файлов вместе со всеми указывающими на него файлами-ссылками — это должно корректно работать при редактировании данных игры в редакторе и в самой игре.

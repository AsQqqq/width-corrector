В этой статье описаны шаги по созданию автобусных маршрутов — пути для автобуса, состоящего из нескольких автобусных остановок.

Настройка файлов и папок

  1. Создайте две новые папки в уровне, который хотите изменить:

    • scenarios
    • bus lines

    Например, если вы изменяете уровень "smallgrid", создайте папки в ..\levels\smallgrid\

Если эти папки уже существуют в изменяемом уровне, пропустите этот шаг.

  1. Внутри папки scenarios создайте новую папку и назовите её "bus".

    Внутри папки bus необходимо создать файл bus_level.json. Содержимое файла должно быть следующим:

  • Измените первые две переменные, "name" и "description", заменив часть, относящуюся к уровню, на название вашего уровня. Например, используйте bus_utah.json при изменении уровня "Utah". Помните, что это правило именования не является строгим. В данном примере для уровня smallgrid используется "bus\_grid.json".
  • Объект "busdriver" должен содержать маршрут по умолчанию, который будет воспроизводиться. Измените "routeID" и "variance" соответствующим образом.

Рекомендации по JSON

На этом этапе полезно ознакомиться с несколькими рекомендациями по созданию документа levelName.buslines.json.

Файл .json представляет собой json-объект, содержащий 2 переменные: "version" — число, обозначающее версию файла автобусных маршрутов. "routes" — массив json-объектов, каждый из которых определяет автобусный маршрут. Каждый объект маршрута содержит:

  • routesID должен быть строкового типа, хотя фактически является числом, например "routeID":"101"
  • direction — в идеале каждый маршрут имеет два направления: одно из точки A в точку B и другое из B в A. Эта переменная используется в целях отображения. Поэтому рекомендуется использовать соглашения об именовании, которые легко распознать.
  • variance — строка, описывающая конкретный вариант маршрута. Это необходимо и используется внутри системы, когда идентификатор маршрута имеет более одного варианта.
  • routeColor — строка, представляющая цвет маршрута в шестнадцатеричном формате html
  • vehicle — транспортное средство для использования. Оставьте model равным citybus и config равным base
  • tasklist — json-массив строк (без пробелов между ними), представляющий триггеры для ваших автобусных остановок.
  • requiredWaypoints — необязательный json-объект, указывающий обязательные точки, через которые должно пройти транспортное средство, и сообщение об ошибке, если транспортное средство через них не проехало. Каждый элемент объекта обязательных путевых точек также является объектом, содержащим 2 переменные: массив со строковым идентификатором путевой точки, через которую необходимо проехать, и сообщение об ошибке, отображаемое при её пропуске.
  • spawnLocation — json-объект, содержащий 2 объекта: "pos" и rotAngAxisF, представляющих начальное положение и ориентацию транспортного средства. Объект вращения может быть одного из двух типов:
    • rot: кватернионное представление вращения (задаётся в LUA)
    • rotAngAxisF: представление вращения в виде "угол и ось", используемое в редакторе. Значения x, y, z обозначают ось вращения, а w — угол в градусах. Этот тип рекомендуется, так как его удобно использовать.
  • navhelp — необязательный json-объект, содержащий массивы, названные по идентификаторам остановок, требующих навигационной помощи. Эти массивы содержат путевые точки (строки), через которые будет проходить навигационная стрелка.

  1. Переименуйте и измените переменные и значения в документе levelName.buslines.json по своему усмотрению и в соответствии с инструкциями из предыдущего шага.

  2. Сохраните файл.

Точки появления

  1. Загрузите игру и перейдите в Play > Bus Routes > Select Map. Затем выберите карту для изменения (на которой были созданы новые папки) и выберите направление (например, "Rexoni Diner"). Не беспокойтесь, если транспортное средство появится в воздухе или под землёй — следующие шаги позволят скорректировать точку появления.

  2. Нажмите Shift+C, чтобы переключить Свободную камеру. Перемещайтесь по карте (с помощью клавиш навигации W A S D), пока не выберете и не достигнете желаемого места начала поездки. Постарайтесь переместить камеру как можно ближе к земле.

  3. Нажмите F11, чтобы открыть Редактор мира.

  4. Нажмите на вкладку Camera и выберите Place Player at Camera.

  1. Нажмите на транспортное средство, чтобы отобразить его маркер. Если маркер не отображается, перейдите в инструмент Scene Tree редактора (если его нет, добавьте его через вкладку Window).

    Прокрутите вниз до конца к папке ScenarioObjectsGroup. В этой папке выберите scenario\_player0 — это должно выделить ваше транспортное средство и отобразить его маркер.

  1. Вы можете задать положение и поворот, вводя значения непосредственно во вкладке Transform инструмента Inspector редактора, или используя соответственно инструменты редактора Translate и Rotate.

  1. Когда вы будете удовлетворены желаемым начальным положением и ориентацией, скопируйте эти значения из вкладки transform инструмента Inspector в spawnLocation вашего файла levelName.buslines.json.

  2. Сохраните файл.

Автобусные остановки

  1. В Scene Tree нажмите на папку ScenarioObjectsGroup, затем на значок Новая папка в верхнем левом углу окна Scene Tree.

Назовите новую папку busstops и откройте её.

  1. В игре нажмите Shift+C, чтобы переключить Свободную камеру и вернуться к камере транспортного средства. Затем перемещайтесь по карте, пока не достигнете желаемого места для вашей автобусной остановки.

  2. На панели инструментов Редактора мира:

  • Нажмите кнопку Add Object
  • Нажмите на значок BeamNG на панели инструментов, появившейся под основной
  • Нажмите на значок LUA Trigger
  • Нажмите в любом месте в 3D-виде

Это создаёт LUA-триггер в месте нажатия в 3D-виде, и объект также появится в списке Scene Tree.

LUA-триггеры видны только при выборе, если только в меню Visualization не включена опция Triggers (доступ к этому меню осуществляется через кнопку вкладки Window

  1. Измените положение, поворот и масштаб триггера-куба, как описано в step 11.

    В масштабе триггера задайте значение "x" с учётом ширины транспортного средства, а значение "y" — с учётом длины транспортного средства.

Так как этот триггер будет активирован, когда транспортное средство окажется внутри блока триггера, задайте его размеры достаточными для размещения транспортного средства. Также рекомендуется опустить ограничивающий блок немного ниже уровня земли.

  1. Когда достигнуты нужные размеры и расположение, откройте окно Inspector Редактора мира. Скопируйте идентификатор триггера из вашего tasklist в файле levelName.buslines.json (например, tmpl_bs_07) и вставьте его в поле имени на панели Inspector.

Помните, что этот идентификатор должен быть единой строкой без пробелов внутри, как указано в step 3. Рекомендуется делать имя коротким.

  1. Выделив триггер, откройте Inspector и измените следующие параметры в разделе Trigger:
    • TriggerType > Box
    • TriggerMode > Contains
    • TriggerTestType > Bounding box
    • TickPeriod > 1000

  1. Выполнив это, прокрутите окно Inspector до раздела Dynamic Fields

  1. Введите type и нажмите Add, затем введите busstop

  1. Добавьте ещё одну запись в Dynamic Fields с именем stopName и введите фактическое название автобусной остановки, которое будет отображаться в игре (например, Fleet Street)

  2. Повторите steps 15 to 22 для каждой автобусной остановки, которую хотите создать.

Альтернативой использованию предопределённых шаблонов миссий является создание новой миссии с помощью системы флоуграфа (flowgraph).

Flowgraph — это динамическая нодовая система скриптинга, позволяющая создавать более сложные миссии из взаимосвязанных элементов.

Перед созданием миссии на основе флоуграфа рекомендуется ознакомиться со вводной страницей , в особенности с разделами об элементах и потоке , поскольку они используются практически в любой миссии на флоуграфе.

Чтобы создать миссию на флоуграфе, выберите "flowgraph" в качестве типа миссии, затем нажмите "Create New and Open Flowgraph Project" -> и выберите один из доступных шаблонов.

Доступно 3 шаблона:

  • Empty Project — пустой шаблон без каких-либо элементов. Рекомендуется только для опытных пользователей.
  • Barebones Mission Template — базовый предопределённый шаблон миссии с тремя состояниями: вступление, игровой процесс и завершение.
  • Simple Scenario-like Template — шаблон, похожий на сценарии предыдущих версий, включающий цикл сброса и больше предопределённых элементов по сравнению с минималистичным шаблоном.

Если вы никогда раньше не создавали миссии на флоуграфе, рекомендуем ознакомиться с примером миссии на флоуграфе , чтобы понять, какие элементы можно использовать в миссии и для чего они нужны.

Базовая структура

Большинство миссий, создаваемых с нуля, следует делать на основе минималистичного шаблона миссии, который включает 3 основных состояния:

  • Intro (вступление)
  • Gameplay (игровой процесс)
  • Outro (завершение)

Intro

Состояние вступления — это этап, на котором выполняется всё необходимое для запуска миссии: позиционирование транспортного средства игрока, загрузка префабов, показ заставки и т. д.

При входе в любую миссию игра выполняет затемнение экрана, позволяя производить скрытые от игрока действия, например расстановку префабов.

Нода "Fade from black" ОБЯЗАТЕЛЬНА на каком-либо этапе состояния вступления — она восстанавливает изображение на экране игрока.

После завершения вступления управление должно переходить к состоянию игрового процесса.

Если в миссии предусмотрен цикл перезапуска, позволяющий игроку сбросить миссию нажатием клавиши R в случае застревания транспортного средства, этот цикл должен повторно запускать состояние вступления.

Gameplay

Состояние игрового процесса — это основная часть миссии, в которой управление передаётся игроку.

В этом состоянии должны быть определены все задачи и условия, которые игрок должен выполнить для успешного завершения сценария.

Outro

Завершение используется для подведения итогов миссии, отображения финального экрана с результатами игрока и выгрузки префабов.

Самой последней нодой должно быть затемнение экрана, поскольку по завершении миссия автоматически выгружает префабы и снимает затемнение, после чего управление возвращается игроку.

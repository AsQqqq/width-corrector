DirectDraw Surface (DDS)

DirectDraw Surface (DDS) — это контейнерный формат для растровых изображений, широко используемый для несжатых или сжатых текстур во многих играх. BeamNG использует .dds для эффективной потоковой передачи и рендеринга текстур (например, карт нормалей, диффузных текстур).

Portable Network Graphics (PNG)

PNG (Portable Network Graphics) — это формат растровых изображений без потерь, подходящий для иконок интерфейса, наложений и текстур, где важны детализация и прозрачность. BeamNG часто использует .png для элементов UI или небольших текстурных ассетов.

Исходные файлы JavaScript (JS)

Скрипты JavaScript (.js) используются в различных веб-контекстах и контекстах пользовательского интерфейса, например во внутриигровом UI, построенном на веб-технологиях. Они также могут встречаться в инструментах сборки, модах или других вспомогательных скриптах.

Файлы данных JSON

JSON (JavaScript Object Notation) — это лёгкий текстовый формат для структурированных данных. BeamNG может использовать .json для хранения конфигураций транспортных средств, настроек UI, показателей производительности или данных модов.

3D-модели Collada (.dae)

Collada (COLLAborative Design Activity) — открытый стандарт файлового формата на основе XML для 3D-ассетов, разработанный группой Khronos. Подробнее на Wikipedia: COLLADA .

В BeamNG файлы .dae обычно содержат геометрию, материалы и данные сцены. Они могут быть преобразованы в более быстро загружаемые файлы .cdae (кэшированная Collada) для повышения производительности.

Код Torque3D

Это устаревший формат скриптов. Пожалуйста, не используйте и не изменяйте эти файлы. Используйте вместо этого Lua.

Скрипты Lua

BeamNG активно использует Lua для внутриигровых скриптов, логики и поведения сценариев. Это основной встроенный язык сценариев движка для игровых функций и поддержки модов.

HTML-документы

HTML-файлы составляют основу внутриигрового UI-фреймворка BeamNG, который использует веб-технологии для меню и наложений. Они также могут встречаться в документации или пользовательских интерфейсах модов.

Аудио Ogg Vorbis

Ogg Vorbis — популярный аудиоформат с открытым исходным кодом, используемый для фоновой музыки, звуков двигателя или атмосферных треков. Его сжатие обеспечивает приемлемое качество при меньшем размере файла по сравнению с несжатыми форматами.

WAV-аудиофайлы

WAV — стандартный несжатый аудиоконтейнер, часто используемый при редактировании или в банках звуков двигателя благодаря высокой точности воспроизведения. BeamNG может хранить короткие звуковые эффекты или высококачественные исходные записи в формате .wav.

Файлы карт

В контексте BeamNG файлы .map могут содержать данные миссий или уровней, включая компоновку, размещение объектов или конфигурацию окружения. Они также могут встречаться в инструментах картографирования, хранящих различные ссылки для дизайна трасс или сценариев.

Файлы изображений JPEG

Изображения JPEG (.jpg или .jpeg) используют сжатие с потерями для уменьшения размера файла, что делает их популярными для текстур, фонов UI или изображений в документации. По сравнению с .png они эффективнее для больших фотографий, но не поддерживают прозрачность без потерь.

Векторная графика SVG

Scalable Vector Graphics (.svg) — векторные изображения на основе XML. Они масштабируются без потери качества, что делает их идеальными для иконок, диаграмм или масштабируемых элементов UI. В BeamNG файлы .svg могут использоваться в веб-интерфейсе или документации.

Документы Markdown

Файлы Markdown (.md) — это лёгкие текстовые документы с простым синтаксисом форматирования (заголовки, списки и т. д.). Они могут использоваться для файлов README, внутриигровых руководств или документации для разработчиков.

Файлы конфигурации деталей

Файлы конфигурации деталей (.pc) определяют конфигурации транспортных средств и содержат такую информацию, как цвет окраски, детали и параметры настройки.

Flow-файлы

Файлы флоуграфов (.flow.json) содержат динамические скрипты на основе нод, определяющие игровую логику.

Файлы префабов

Файлы префабов (.prefab) объединяют группу ассетов (транспортные средства, статические объекты, триггеры и т. д.), чтобы их можно было загружать и выгружать в любое время во время работы уровня. Их также можно использовать как шаблон, позволяющий легко копировать и вставлять группы объектов. Префабы можно сохранять и загружать в двух форматах: .prefab и .prefab.json. В настоящее время рекомендуется использовать .prefab.

Каскадные таблицы стилей (CSS)

Каскадные таблицы стилей (.css) определяют визуальные стили для HTML-интерфейсов. Система UI в BeamNG, построенная на веб-технологиях, может использовать .css для оформления меню, наложений сценариев или интерфейсов модов.

Аудиофайлы FLAC

FLAC (Free Lossless Audio Codec) — высококачественный формат сжатия аудио без потери деталей. Хотя .ogg более распространён для внутриигровых ассетов, в .flac могут храниться мастер-записи или высококачественные звуковые файлы.

Однофайловые компоненты Vue

Однофайловые компоненты .vue используются во фреймворке Vue.js для веб-интерфейсов. Если UI BeamNG использует Vue, файлы .vue могут содержать HTML, CSS и JS в одном файле компонента.

Высокоуровневый язык шейдеров (HLSL)

High-Level Shading Language (.hlsl) — язык шейдеров Microsoft для Direct3D. Такие файлы могут встречаться, если BeamNG использует пользовательские шейдеры для улучшенных графических эффектов.

Файлы простого текста

.txt — универсальный формат простого текста для журналов, заметок или описаний. BeamNG или моды могут включать простые текстовые данные в файлах .txt.

Таблицы стилей SASS/SCSS

SASS/SCSS (.scss) — надмножество CSS, предоставляющее переменные, вложенность и другие возможности. Компилируется в стандартный .css и может встречаться в расширенных стилях UI BeamNG или инструментах разработчика.

Шрифты TrueType

Файлы шрифтов TrueType содержат глифы на основе кривых. .ttf широко используется для отображения текста в UI игры, обеспечивая единообразие гарнитуры в меню и наложениях.

Файлы пакетов/архивов

Файлы пакетов или архивов (.pak) объединяют несколько ресурсов в один контейнер. Они могут использоваться для сжатия и распространения игровых ассетов или содержимого модов в виде единого пакета.

Файлы YAML

Файлы YAML (.yml или .yaml) — удобочитаемые форматы структурированных данных. Они могут хранить параметры конфигурации, переменные окружения или другие структурированные данные в BeamNG.

Динамические библиотеки Windows

Динамически подключаемые библиотеки (.dll) содержат скомпилированный код для Windows. Они могут использоваться для внешних нативных модулей, плагинов или пользовательских функций движка в BeamNG.

Скрипты Python

Файлы .py — это скрипты Python, используемые для автоматизации, обработки данных или создания пользовательских инструментов. В рабочем процессе BeamNG они могут генерировать данные, управлять сборками или предоставлять скрипты для управления модами.

Шрифты OpenType

Шрифты OpenType (.otf) схожи с .ttf, но могут содержать расширенные типографские возможности. Они используются для отображения пользовательских или дополнительных шрифтов в UI BeamNG или в текстах модов.

Файлы аудиобанков

Файлы аудиобанков (.bank) могут хранить несколько звуковых ассетов или сигналов в одном контейнере. Иногда они генерируются аудиомидлвером (например, FMOD, Wwise) для использования в движке.

Исполняемые файлы Windows

Исполняемые файлы Windows (.exe) могут быть инструментами, установщиками или внешними программами, связанными с BeamNG. Моды редко распространяют файлы .exe напрямую, однако инструменты разработчика могут присутствовать в репозитории.

2D-атласы спрайтов

Файлы 2D-атласов спрайтов (.atlas) определяют подизображения (спрайты) внутри одной текстуры. Они могут повышать производительность, сокращая переключение текстур в UI или 2D-элементах игры.

Файлы журналов

Файлы журналов (.log) записывают диагностическую информацию, сообщения об ошибках или трассировки выполнения. Они помогают отлаживать проблемы в BeamNG или отслеживать поведение игры во время выполнения.

Данные рельефа

Файлы рельефа (.ter) могут хранить карты высот, текстуры поверхности и определения рельефа. В BeamNG они могут использоваться системой рельефа для крупномасштабных карт или внедорожных окружений.

Заголовочные файлы C/C++

Заголовочные файлы (.h) объявляют структуры данных, функции или константы C/C++. Они могут встречаться в нативном коде BeamNG или во внешних библиотеках, используемых игрой.

Функциональные модели (FMU)

Функциональные модели (.fmu) — это контейнеры для моделей динамических систем (например, из Modelica). Они могут применяться в сложных физических симуляциях или внешних инструментальных цепочках, интегрированных с BeamNG.

Значения, разделённые запятыми (CSV)

CSV (Comma-Separated Values) (.csv) — текстовый формат для хранения табличных данных. Журналы данных, показатели производительности или справочники можно легко обрабатывать в файлах .csv с помощью Excel или скриптов.

Общие объекты Linux

Общие объекты (.so) — это динамические библиотеки для Linux/Unix. Они могут предоставлять платформозависимый код или модули движка, хотя в BeamNG более распространены файлы .dll для Windows.

Модели в формате GL Transmission Format

GL Transmission Format (.gltf) — современный стандарт на основе JSON для 3D-моделей, сцен и анимаций. Он может использоваться в рабочих процессах, требующих открытого эффективного формата времени выполнения, хотя в BeamNG более распространён .dae.

Статические библиотеки

Статические библиотеки (.lib) содержат скомпилированный код, компонуемый в исполняемые файлы. Они могут встречаться в проектах нативного кода, взаимодействующих с движком BeamNG или его инструментами.

Zip-архивы

Zip-архивы (.zip) сжимают несколько файлов в один. Это распространённый способ распространения модов, данных или упакованных сборок. BeamNG может загружать запакованные архивы модов из папки «mods».

Растровые изображения

Растровые изображения (.bmp) хранят несжатые пиксельные данные. Они менее эффективны по пропускной способности, но могут встречаться в старых ассетах модов или специализированных инструментах.

PDF-документы

Portable Document Format (.pdf) используется для документации, руководств или справочных материалов. Разработчики модов могут включать PDF-файлы с описанием расширенных настроек или пользовательскими инструкциями.

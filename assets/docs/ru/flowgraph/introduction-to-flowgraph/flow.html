Понимание потока

Поток — это энергия, пульсирующая по связям вашего флоуграфа и оживляющая его.

Как уже было объяснено в разделе «Нода» страницы Элементы , большинству нод необходим поток для какого-либо функционирования. Только базовые ноды, такие как нода «Number», дают выходной сигнал даже без потока.
Флоуграф, не имеющий никакого входящего потока, не будет иметь функционирующих нод, и поэтому при его запуске ничего не произойдёт.
Вот ключевая информация о потоке:

  • Поток существует только во флоуграфах, но не в стейтграфах.
  • Отправной точкой для потока является нода «on Update». Она передаёт поток только в том случае, если флоуграф находится в активном состоянии.
  • Большинство нод имеют входной пин потока для активации и выходной пин потока для передачи потока к следующей ноде.
  • Некоторые ноды имеют дополнительные выходные пины потока, которые срабатывают только тогда, когда нода завершила свою задачу.

Постоянный поток vs. Импульсный поток

Основным источником потока в вашем флоуграфе является выходной пин потока ноды «onUpdate». Она активируется каждый кадр и передаёт поток всем нодам в вашем графе.
Но, разумеется, есть вещи, которые не должны происходить постоянно, например, отображение сообщения игроку или сброс ноды.
Простейшее решение для этого — нода «Once», которая пропускает постоянный поток только один раз, или, говоря более технически, только в течение одного кадра:

Вы можете заметить, что выходной пин потока и пин сброса ноды «Once» выглядят иначе, чем обычный пин потока. Это то, что мы называем «импульсным» пином.
Существует два способа понять импульсные пины во Flowgraph:

  • Выходной:
    Идея выходного импульсного пина проста. Он выдаёт поток только один раз, а не постоянно.
  • Входной:
    Идея входного импульсного пина немного более абстрактна.
    Он должен получать поток только в течение одного кадра. Если пин получает постоянный поток, нода не будет работать должным образом.

Ещё один хороший пример импульсного поведения — нода «Countdown»:


  • Выходной пин «finished»
    Выходной импульсный пин «finished» передаёт поток только один раз — по завершении работы ноды. Это удобно для запуска событий после окончания обратного отсчёта.
  • Входной пин «reset»
    Входной импульсный пин «reset» теоретически мог бы получать постоянный поток, но постоянный сброс обратного отсчёта сделал бы ноду бессмысленной. Поэтому он должен получать поток только один раз — для сброса ноды при необходимости.

Работа с потоком

Общая идея флоуграфа — строить его слева направо и передавать поток от ноды «on Update» через весь граф.
Эта концепция естественным образом приводит к идее цепочечного соединения нод слева направо, что обеспечивает линейную и хорошо читаемую структуру вашего флоуграфа:

Также возможно разветвлять граф, поскольку количество связей, подключённых к пину, не ограничено.
Это особенно уместно для вещей, которые логически происходят одновременно.

Обратите внимание, что при построении подобных разветвлённых структур порядок выполнения различных нод не гарантирован.
Также следует учитывать, что при установке переменной новое значение не будет применено до следующего кадра.
